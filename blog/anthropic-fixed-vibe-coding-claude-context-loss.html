The Context Loss Problem

AI coding assistants like Claude have revolutionized development speed for simple tasks, but hit a wall with complex projects. The core issue stems from context window limitations - when working on large features, the AI would compact its memory multiple times, gradually losing sight of the original requirements.

Imagine assigning Claude to build a complete feature. After several context compactions, it might forget the authentication system was supposed to integrate with your existing user database, or that the admin panel needed specific permissions. The result? Half-implemented features marked as complete but missing critical components.

84% context utilization: The new workflow maintains high context efficiency compared to traditional methods that would hit compaction limits at just 60-70% utilization for similar tasks.

Anthropic's Engineering-Inspired Solution

Anthropic's breakthrough came from observing how human engineering teams successfully manage complex projects. Real developers don't try to build entire features in one sitting - they break work into manageable pieces, document progress, and verify each component works before moving forward.

The solution implements this through a dual-agent system: an initializing agent that sets up proper project structure and documentation, and a coding agent that works incrementally with enforced testing. This creates continuity between sessions that pure "vibe coding" could never achieve.

Setting Up the Environment

The initialization phase is crucial for long-term success. At 2:15 in the video, you'll see the exact commands used to create three key files:

  1. claw.md - Project documentation at the root containing overview and key information
  2. features.json - Structured list of all features with corresponding test steps (initially marked failing)
  3. progress.md - Tracking file updated after each implementation

Additional setup includes Puppeteer for browser testing (since Claude can't visually verify interfaces) and git initialization with clear commit guidelines. This structure forces Claude to work like a professional developer rather than an overeager intern.

The Incremental Coding Process

With the environment ready, coding proceeds feature-by-feature from the JSON list. For each item:

  1. Claude implements just that feature's requirements
  2. It runs all defined tests (including Puppeteer browser tests)
  3. Only after passing tests does it update the JSON (false → true)
  4. It makes a descriptive git commit with clear progress notes
  5. Updates progress.md with current status

Session recovery: If interrupted, you can resume exactly where you left off. Claude reconstructs context from git logs and progress files rather than trying to remember everything.

Why This Beats Traditional AI Coding

Traditional "vibe coding" approaches fail because they treat AI coding like a single conversation. Each compaction loses context, like playing telephone with your own project requirements. The new workflow provides:

  • Continuity through git commits and progress tracking
  • Accountability via enforced testing before feature completion
  • Efficiency by focusing on one feature at a time
  • Recoverability when sessions end unexpectedly

Most importantly, it prevents Claude's tendency to mark features complete based on code appearance rather than actual functionality - a critical flaw in previous methods.

Comparison to BMAD Method

At first glance, this resembles the BMAD (Build-Measure-Act-Debug) approach, but key differences make Claude's workflow superior for many use cases:

Factor BMAD Claude Workflow
Context Usage Hits compaction faster Maintains 84% utilization
Testing Optional Enforced per feature
Progress Tracking Story-based Structured JSON + git

While BMAD remains useful for quick prototypes, Claude's method shines for production-grade development where continuity and testing matter most.

Make.com Basics for Beginners

Make.com (formerly Integromat) is a visual automation platform that connects apps and services without coding. Think of it as digital duct tape - it sticks different systems together to create seamless workflows.

The platform works through "scenarios" - visual workflows where you connect modules that represent different actions. In our case, we'll create a scenario that:

  1. Receives a request from an AI agent (trigger)
  2. Processes the natural language query with OpenAI
  3. Checks your Google Calendar availability
  4. Returns available time slots in natural language

At 2:15 in the video tutorial, you'll see how simple it is to create your first module by double-clicking on the scenario canvas.

Step 1: Setting Up the Webhook Trigger

Every automation needs a starting point. For our calendar checker, we'll use a webhook - a URL that other systems (like your AI agent) can call to trigger the workflow.

In Make.com:

  1. Create a new scenario
  2. Add a "Webhook" module as your first step
  3. Configure it as a "Custom Webhook"
  4. Save to generate your unique webhook URL

Pro Tip: Set the webhook to trigger "immediately as data arrives" (shown at 1:32 in the video) rather than "run once." This ensures real-time responses instead of manual triggering.

This webhook URL is what your AI agent will call whenever someone asks about your availability. The beauty? It works with any platform - Vapi, Voiceflow, even custom solutions.

Step 2: Adding OpenAI for Natural Language Processing

At 4:50 in the tutorial, we add the OpenAI module to interpret the incoming request. This converts natural language questions ("Is John available this afternoon?") into structured data we can use to check your calendar.

The key components:

  • System Message: Instructions telling OpenAI how to format the output (we use JSON for easy parsing)
  • User Message: The actual query from the caller combined with current date/time context
  • Model Selection: We use gpt-3.5-turbo for fast, cost-effective processing

The output will be cleanly formatted date/time ranges that our next modules can understand, like: {"start":"2023-11-15T14:00:00","end":"2023-11-15T14:30:00"}

Step 3: Connecting to Google Calendar's API

The magic happens when we connect to Google Calendar's Free/Busy API (shown at 7:20 in the video). This checks your actual availability without exposing full calendar details.

You'll need to:

  1. Enable the Google Calendar API in your Google Cloud Console
  2. Create API credentials (OAuth 2.0 Client ID)
  3. Connect Make.com to your Google account
  4. Configure the HTTP module to call the freeBusy endpoint

The API returns all your busy slots during the requested period. Our automation then calculates the available windows between those busy times.

Security Note: The freeBusy API only shares availability, not event details. Your actual appointments remain private while still preventing double bookings.

Smart Weekend and Holiday Handling

At 9:45 in the tutorial, you'll see how we automatically handle weekends and holidays using Make.com's router module. This creates separate paths for:

  • Weekday requests (check calendar availability)
  • Weekend requests (immediate "unavailable" response)
  • Error cases (fallback responses)

The router checks the day of week (formatted as "Sa" or "Su") and routes accordingly. You could expand this to check against a holiday calendar as well.

This automatic filtering prevents wasted API calls and gives callers immediate feedback when requesting unavailable times.

Building the Response System

The final piece (shown at 12:30) formats the availability data into natural language responses your AI agent can speak aloud. We use:

  1. An iterator to process each available time slot
  2. OpenAI to convert the slots into friendly responses ("I'm available between 2-3pm or after 4:30pm")
  3. A webhook response module to send the answer back to the calling AI agent

The complete workflow takes about 2-3 seconds to run end-to-end - faster than you could manually check your calendar while trying to sound professional on a call.

Customization Tip: Adjust the OpenAI instructions to match your brand voice. For example, lawyers might want more formal responses while creatives could use casual language.

How the Automation Works in Make.com

This workflow solves the cancellation chaos with four automated steps that trigger instantly when a student cancels:

  1. Calendly detects the cancellation and sends the event details to Make.com
  2. A filter confirms it's a cancellation (not a reschedule or new booking)
  3. Airtable updates automatically with the cancellation status and timestamp
  4. The tutor receives an instant notification via email with student details

The entire process completes in under 60 seconds — faster than most staff could even open their email client. At 1:15 in the video tutorial, you'll see the exact moment the test cancellation triggers all downstream actions simultaneously.

Step 1: Setting Up the Calendly Trigger

The workflow begins with Calendly's "Event Cancellation" webhook. When integrated with Make.com, this trigger fires instantly whenever any scheduled session changes to "canceled" status — no polling or manual checks required.

Configuration requires three key pieces of information from Calendly:

  • Your organization's unique Calendly API key
  • The specific event type(s) to monitor (e.g., "30-min tutoring session")
  • Webhook security settings to validate incoming requests

Pro tip: Create separate workflows for different cancellation types. A "student-initiated" cancellation might trigger different follow-ups than a "tutor-rescheduled" event.

Step 2: Filtering Only Cancellation Events

Not all Calendly webhook events are cancellations. The workflow uses Make.com's filter module to ensure only true cancellations proceed, ignoring:

  • New bookings
  • Rescheduled sessions
  • Calendar sync updates
  • Reminder notifications

The filter checks the event object's status field for the exact value "canceled" (case-sensitive). At 2:30 in the video, you'll see how testing with different status values proves the filter's accuracy before deployment.

Step 3: Updating Airtable in Real-Time

With a confirmed cancellation, the workflow updates your Airtable base with:

  • Cancellation timestamp
  • Student name and contact info
  • Original session details
  • Reason for cancellation (if provided)

This happens through Airtable's API using the "Update Record" action. The tutorial shows how to map Calendly's event data to your specific Airtable fields — critical for maintaining data consistency across platforms.

Implementation note: While we demonstrate Airtable, this step works identically with Google Sheets, Notion, or any database with API access. The data structure remains consistent regardless of platform.

Step 4: Automatic Tutor Notifications

The final step sends an immediate notification to the affected tutor via SendGrid email (though you could substitute SMS, Slack, or any communication channel). The message includes:

  • Student name and subject
  • Original session time
  • Cancellation reason (if available)
  • Direct link to reschedule

At 4:10 in the video, you'll see the test email arrive with all details pre-populated from the Calendly event — zero manual data entry required.

The WhatsApp Automation Solution

The solution handles three specific client response types: "Shortlist me" (acceptance), "Not available" (rejection), and "I have a question" (inquiry). Each triggers different actions while maintaining a professional, timely response.

At 5:20 in the tutorial, we see how the automation routes different responses to appropriate actions. This intelligent routing is the core of the system, ensuring each client gets the right follow-up while your records stay perfectly updated.

Setting Up the WhatsApp Trigger

The automation begins by watching for new WhatsApp messages. At 7:15, we configure the WhatsApp module in Make.com to trigger whenever a new message arrives from a client.

Key configuration points include establishing the WhatsApp connection and setting up message filters. The phone number format is particularly important - at 12:30 we see how inconsistent number formatting can break the automation until corrected.

Creating Response Routes

The router module at 9:45 splits incoming messages into three paths based on their content. Each route has a specific filter checking for exact match phrases that clients will use.

For strict control, you can configure WhatsApp buttons that only allow these three responses. At 11:20, we discuss how this prevents invalid responses that would otherwise require manual handling.

Updating Airtable Records

Each response route updates the client's Airtable record with their status. At 14:00, we configure the Airtable "Update Record" module to change the status field based on which response was received.

The system searches Airtable using the client's WhatsApp number (15:30), then updates the appropriate record. This requires consistent number formatting between WhatsApp and your database - a crucial detail highlighted when troubleshooting at 17:45.

Sending Confirmation Messages

Immediately after updating records, the system sends an appropriate WhatsApp confirmation. At 19:00, we craft three distinct messages that acknowledge each response type professionally.

These templates are fully customizable - you can add personalization using data from Airtable. The messages send within seconds of receiving the original response, creating an impressively quick turnaround that delights clients.

Testing the Automation

At 21:30, we test the complete workflow by sending each response type. The video shows real-time updates to Airtable and immediate confirmation messages - proving the system works exactly as designed.

Testing revealed one crucial adjustment needed: phone number formatting. This demonstrates why thorough testing is essential before relying on any automation for business-critical communications.

Setting Up the n8n Workflow

Start with a manual trigger in n8n, then add an HTTP Request node. This will communicate with APIFY's API. The key configuration happens in three places:

Step 1: APIFY Account Setup

Create a free APIFY account and navigate to the Instagram scraper. Get your API token from Settings → API and Integrations.

Step 2: HTTP Request Configuration

In n8n, set the HTTP method to POST and use APIFY's asynchronous endpoint URL. This ensures the workflow waits for scraping to complete.

Step 3: Authentication

Create an APIFY credential in n8n using your API token. This authenticates all requests automatically.

Pro Tip: Always use the "Run Actor Asynchronously and Get Dataset Items" endpoint - it handles Instagram's variable scrape times automatically.

Configuring the HTTP Request Node

The magic happens in the HTTP Request node's configuration. At 2:15 in the video tutorial, you'll see the exact settings:

  • Method: POST
  • URL: https://api.apify.com/v2/acts/apify~instagram-scraper/run-sync-get-dataset-items
  • Authentication: APIFY Credential
  • Headers: Content-Type → application/json

This configuration tells APIFY to: 1) Start scraping, 2) Wait for completion, and 3) Return the formatted dataset directly to n8n.

Customizing JSON Parameters

The request body contains JSON parameters that control what data gets scraped. At 3:40 in the video, you'll see how to customize these:

{   "username": "target_profile",   "resultsLimit": 5,   "scrapeReels": true,   "scrapeStories": false,   "scrapeComments": true }

You can modify this to scrape different content types, set result limits, or add filters. The JSON structure matches APIFY's web interface, making configuration intuitive.

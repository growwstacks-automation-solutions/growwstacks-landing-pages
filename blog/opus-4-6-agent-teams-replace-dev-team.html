Agent Teams Explained

Opus 4.6 introduces a revolutionary approach to AI-assisted development with agent teams. Unlike traditional AI coding assistants that work linearly, agent teams simulate an entire development team with specialized roles. When given a project like building the Claude Fit fitness tracker app, the main agent first creates a task list then spawns specialized team members.

These agents include UX/UI designers, backend developers, technical architects, database experts, and crucially - a devil's advocate that questions all decisions. Each agent operates with its own Claude Code instance, allowing true parallel processing while maintaining awareness of other team members' work through a shared task list.

Key advantage: Agent teams enable peer-to-peer communication between specialists. When the database expert designs a schema, the API developer immediately knows the structure and can adapt endpoints accordingly - just like human teams.

How Agent Teams Differ From Sub Agents

Traditional sub agents operate in isolation - they perform tasks independently and report back to the main agent. This creates potential compatibility issues where one sub agent's changes might conflict with another's work without either being aware.

Agent teams fundamentally change this dynamic by enabling direct communication between team members. The technical architect can discuss infrastructure needs with the backend developer, while the UX designer receives real-time feedback from the devil's advocate. This creates a collaborative environment that more closely mirrors human team dynamics.

Critical difference: Sub agents report upwards only, while agent team members communicate horizontally with full context of each other's work.

Team Dynamics In Action

One of the most powerful features is the ability to interact with individual team members mid-process. At 3:12 in the video, the creator interrupts just the UI designer to request use of front-end design skills, without affecting other team members.

This granular control allows for targeted human guidance where most needed. The team lead automatically shuts down members as their tasks complete, though manual shutdown commands can be issued if needed. All team members have full access to Claude Code features including skills and MCP servers.

Human-In-The-Loop Advantages

Agent teams support sophisticated human oversight through delegation mode (activated with Shift+Tab). In this mode, the main agent becomes a pure coordinator while team members handle all implementation.

Developers can cycle through team members using arrow keys to review progress or provide targeted instructions. This creates a development environment where AI handles implementation details while humans focus on high-level direction and quality control.

Setup Guide For Your Projects

Enabling agent teams requires adding an experimental flag to your settings.json file. Create a .claude folder in your project directory if none exists, then add a settings.json file with the property "enableAgentTeams": true.

To create your first team, structure your prompt with: "Create an agent team to explore this from different angles." Then specify your desired team composition - including specialized roles and optionally different models for different members (Opus for architects, Haiku for documentation, etc).

Pro tip: Including a devil's advocate role significantly improves output quality by forcing the team to justify decisions.

When To Use Agent Teams vs Sub Agents

Agent teams excel at complex, multi-faceted projects requiring coordinated specialization - like setting up an entire application from scratch. The collaborative nature ensures compatibility between components.

Sub agents remain better for simple, isolated tasks like file exploration or single-feature implementation. They're lighter weight for jobs where cross-component coordination isn't required.

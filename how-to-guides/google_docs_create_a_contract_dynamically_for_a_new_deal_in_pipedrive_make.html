Google Docs - Create a Contract Dynamically for a New Deal in Pipedrive

Learn how to automate contract creation, email the contract to the client, and update the deal stage in Pipedrive using Google Docs and Make.

This guide will walk you through setting up an automation scenario using Make (formerly Integromat) to streamline your sales process. The goal is to automatically generate a contract using Google Docs when a deal reaches a specific stage in Pipedrive, then email that contract to the client, and finally update the deal's status in Pipedrive.

This automation is designed for an intermediate user and can typically be implemented in about 30 minutes. You'll need active accounts for Pipedrive, Google Drive, Google Docs, and Gmail, with all necessary connections already established within your Make account.

Configuring the Scenario

The automation scenario involves several key modules working in sequence to achieve the desired outcome. Each module performs a specific task, from retrieving deal information to sending the final contract.

The modules used in this scenario are:

  • Pipedrive CRM > List deals
  • Pipedrive CRM > Update a deal
  • Google Drive > Create a folder
  • Google Drive > Search for files/folders
  • Gmail > Send an email
  • Google Docs > Create a document from a template
  • Google Docs > Download a document
An overview of the Make.com scenario, illustrating the sequence of modules.

This visual representation helps understand the flow of data and actions within the automation. We will now delve into the configuration of each module, starting with the Pipedrive trigger.

Setting Up the Pipedrive Trigger

The first step is to configure the "Pipedrive CRM > List deals" module. This module acts as the trigger for our scenario, initiating the automation whenever a deal meets specific criteria in Pipedrive.

Within this module, you'll need to define the conditions that will cause the scenario to run. First, select "Issue contract" from the Stage menu. This ensures that the automation only activates for deals that have reached the stage where a contract needs to be generated. Next, choose "All open deals" in the Filter menu to process all active deals within the selected stage.

Configuring the Pipedrive CRM 'List deals' module to trigger on deals in the 'Issue contract' stage.

You can also specify a particular user if you only want to process deals assigned to them. For this tutorial, we'll leave the user field blank to include deals from all users. Finally, set the Status to "Open" to focus on active deals. The maximum number of results is set to 1 by default, which is suitable for this demonstration, but can be adjusted based on your needs.

Pro tip: Always test your Pipedrive trigger with a single deal first to ensure it's correctly identifying and pulling the desired data before running it on multiple deals.

Further settings for the Pipedrive 'List deals' module, ensuring only open deals are processed.

Creating a Google Drive Folder

Once the Pipedrive deal is identified, the next step is to create a dedicated folder in Google Drive for the new contract. This ensures that all documents related to a specific deal are neatly organized and easily accessible.

Add the "Google Drive > Create a folder" module to your scenario. In its configuration, specify the Destination and the New folder location within your Google Drive. For the folder's name, dynamically map the "Deal ID" element outputted by the preceding Pipedrive module. This creates a unique folder for each deal, making document management much more efficient.

Setting up the Google Drive 'Create a folder' module, using the Pipedrive Deal ID for dynamic naming.

Finding the Contract Template

Before a new contract can be generated, the automation needs to locate your pre-existing contract template in Google Drive. This template will serve as the base for all new contracts, ensuring consistency and accuracy.

Integrate the "Google Drive > Search for files/folders" module into your scenario. Configure it by selecting the Destination and the specific folder where your contract template is stored. Under "Retrieve", select "Files", and for "Search", choose "Search within files/folders". In the "Query" field, simply type the name of your contract template, for example, "Contract". Under "Search options", select "Search for names containing the search term" and keep the "Maximum number of returned results" at 10 (or 1 if you're certain of a unique template name).

Configuring the Google Drive 'Search for files/folders' module to locate the contract template.

This module will efficiently find the correct template, allowing the subsequent steps to proceed with generating the personalized contract.

Specifying search options to accurately find the contract template by name.

Generating the Contract from Template

This is the core step where the dynamic contract is created. The "Google Docs > Create a document from a template" module will take the template found in the previous step and populate it with specific deal and client information from Pipedrive.

In this module, select "By Mapping" under the "Copy a document" menu. Then, map the "File ID" outputted by the "Google Drive > Search for files/folders" module to link to your template. The crucial part is creating "tags" within your Google Docs template. These tags, enclosed in double curly braces (e.g., {{client_name}}), act as placeholders that the automation will replace with actual data.

Configuring the Google Docs module to create a document from the template using mapped File ID.

Click "Add item" in the "Values" field to define each tag. For example, for the current date, set "tag1" as the tag and use the formatDate function (found under the Date and Time tab) to display the "Now" variable in your desired format, such as DD-MM-YYYY.

Mapping the current date with a specific format to a tag in the Google Docs module.

Repeat this process for other dynamic fields. For the client's name, add another item and map the "Person:Name" element from the Pipedrive CRM module to the "Replaced Value" field. Similarly, for the deal's value, map the "currency" and "value" elements from Pipedrive to display the monetary amount.

Adding a tag for the client's name, mapped from Pipedrive deal data.

Continue by creating tags for the client's email address and phone number, mapping them to their respective fields from the Pipedrive data. It's crucial that these tags (e.g., {{tag1}}, {{client_name}}) are manually inserted into your Google Docs template in the exact locations where you want the dynamic data to appear.

Configuring tags for deal currency and value, pulling data from Pipedrive.

After all tags are defined, scroll down to the "Title" field in the Google Docs module. Map the "Title" element from the "Google Drive > Search for files/folders" module and then the "Deal ID" from the Pipedrive module to create a descriptive and unique title for the new contract. Finally, map the "Folder ID" from the "Google Drive > Create a folder" module to ensure the new document is saved in the correct, dynamically created folder.

Further tag configurations for client contact details.

Pro tip: Double-check your Google Docs template to ensure all tags are correctly formatted with double curly braces (e.g., {{tag_name}}) and match the tags defined in your Make scenario. Any mismatch will prevent the data from populating correctly.

Downloading the Contract as PDF

After the contract document has been dynamically created in Google Docs, the next logical step is to convert it into a shareable and uneditable format, typically a PDF. This is handled by the "Google Docs > Download a document" module.

This module has a straightforward configuration. Simply map the "Document ID" element outputted by the preceding "Google Docs > Create a document from a template" module to the "Document ID" field. Then, select "PDF Document" under the "Type" option. This will ensure that the newly generated contract is downloaded as a PDF file, ready for emailing.

Configuring the Google Docs 'Download a document' module to convert the contract to PDF.

Emailing the Contract to the Client

With the contract now generated and converted to a PDF, the automation proceeds to send it to the client via email. This is achieved using the "Gmail > Send an email" module, a familiar tool for most users.

In this module, you will specify the recipient, which should be dynamically mapped from the client's email address available in the Pipedrive deal data. Compose a clear and professional subject line for the email. In the email content, you can include relevant deal information, again mapped from Pipedrive, such as the "Deal ID". Crucially, under the attachments section, click the edit icon and select the output from the "Google Docs > Download a document" module as the "Source file". This ensures the newly created PDF contract is attached to the email before it is sent to the client.

Setting up the Gmail 'Send an email' module to attach the generated PDF contract.

Updating the Pipedrive Deal Stage

The final step in this automation scenario is to update the deal's status in Pipedrive, reflecting that the contract has been sent. This keeps your sales pipeline accurate and ensures your team has the most up-to-date information.

Add the "Pipedrive CRM > Update a deal" module. Map the relevant elements outputted by the initial "Pipedrive CRM > List deals" module to their corresponding fields in this update module. This ensures you are updating the correct deal. Scroll down and select the next appropriate stage in your sales process, such as "Awaiting signed contract", and keep the status as "Open". This action automatically progresses the deal in Pipedrive, completing the automation cycle.

Configuring the Pipedrive 'Update a deal' module to reflect the contract being sent.

Pro tip: Regularly review your Pipedrive stages to ensure they align with your automated workflows. Clear and well-defined stages prevent confusion and maximize the efficiency of your automation.

By automatically moving the deal to the next stage, you maintain a clean and current Pipedrive pipeline, allowing your sales team to focus on follow-ups rather than manual administrative tasks.

Selecting the next stage for the Pipedrive deal after the contract has been sent.

Scenario Execution

Once all modules are configured, you can run the scenario to see the automation in action. The trigger module will fetch a deal that meets the specified criteria, and the data bundle will flow through each subsequent module, executing the defined actions.

You will observe that the contract is dynamically generated, emailed to the client, and the Pipedrive deal is automatically moved to the "Awaiting signed contract" stage. This seamless process demonstrates the power of automation in streamlining your sales operations.

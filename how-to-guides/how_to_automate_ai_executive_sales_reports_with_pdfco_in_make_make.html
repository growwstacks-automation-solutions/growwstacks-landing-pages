How the Workflow Comes Together

The automated sales report generation process within Make.com follows a straightforward, five-step structure. Each step is designed to efficiently transform raw data into a polished, executive-ready document.

This modular approach ensures clarity and ease of setup, allowing users to understand and implement the workflow with minimal effort. By breaking down the complex task of report generation into manageable stages, Make.com makes advanced automation accessible.

  1. Google Sheets: This serves as the central repository where all sales data is collected and stored.

Step 1. Prepare Your Google Sheet

The foundation of this automation lies in a well-structured Google Sheet. This sheet will serve as your primary data source, so ensuring its accuracy and proper configuration is crucial for the entire workflow.

You'll need to set up specific columns to capture all relevant sales data and include a formula that automatically flags recent entries for inclusion in your weekly reports.

  • Sheet name: Raw Sales Data
  • Columns: Date | Rep | Client | Amount | Status | Region | Plan | Notes | IncludeInReport
  • Formula for IncludeInReport:

=IF(AND(A2>=TODAY()-7, A2<TODAY()), TRUE, FALSE)

This formula is essential as it automatically flags only the sales data from the last 7 days for inclusion in the report, ensuring that your executive summaries are always current and relevant.

Step 2. Pull Sales Data

Once your Google Sheet is prepared, the next step is to integrate it with Make.com to extract the raw sales data. This is achieved using a specific Google Sheets module within your Make scenario.

This module acts as the initial trigger, pulling all necessary information into your automation workflow for subsequent processing and analysis.

  • Module: Google Sheets → Get Range Values
  • Range: A1:I
  • Includes headers: false
  • Sheet: Raw Sales Data

This configuration instructs Make.com to retrieve all rows and columns from your specified Google Sheet, bringing the raw data into the scenario for the next stages of filtering and formatting.

Step 3. Filter & Format Rows

After pulling the raw data, the next critical step is to refine it by filtering for relevant entries and formatting them appropriately. This ensures that only the most pertinent information is passed on to the AI for analysis, maintaining the report's focus and accuracy.

The "Set Variable" module in Make's Tools is used here to apply a filter and structure the data cleanly, preparing it for the AI Toolkit.

  • Module: Tools → Set Variable
  • Filter: Only rows where IncludeInReport = TRUE (column I).
Screenshot showing the filter settings in Make.com to include only relevant rows.
  • Variable name: Record
  • Value: Date: {{1.`0`}}, Rep: {{1.`1`}}, Client: {{1.`2`}}, Amount: {{replace(1.`3`; "$"; )}}, Status: {{1.`4`}}, Region: {{1.`5`}}, Plan: {{1.`6`}}
Configuration of the 'Set Variable' module, defining how data fields are mapped and formatted.

This crucial step ensures that only the sales rows from the previous week are processed, and they are formatted cleanly, making them easily digestible for the subsequent AI analysis.

Step 4. Aggregate the Records

With the filtered and formatted data, the next logical step is to consolidate it into a single, coherent block of text. This aggregation is essential before feeding the information to the AI Toolkit, as it provides the AI with a unified context for generating insights.

The "Text Aggregator" module from Make's Tools is specifically designed for this purpose, efficiently combining multiple data points into a single, manageable output.

  • Module: Tools → Text Aggregator
  • Source Module: Google Sheets - Get Range Values
  • Row separator: New row
  • Advanced Settings → Text: {{2.Record}}

By configuring these settings, you now have one consolidated text block containing all the relevant sales records, perfectly prepared for the AI to process and analyze.

Step 5. Generate Report with AI

This is where the intelligence of the automation truly shines. The Make AI Toolkit is employed to transform the aggregated sales data into a comprehensive and professional executive report. This module leverages advanced AI capabilities to generate human-like text, tables, and even charts based on your specific instructions.

Crafting a detailed and precise prompt is key to ensuring the AI produces the desired output, making the report insightful and ready for C-level executives.

  • Module: Make AI Toolkit → Ask anything
  • Model: Small (GPT OSS 20B) (or compatible AI provider)
  • Input:

Generate a professional executive sales report in HTML format from the following sales data:

Period: start date to end date

Sales data:

{{3.text}}

The report should include the following sections:

1. Executive Summary — total deals, closed-won count, win rate, pipeline summary.

2. Deal Performance by Region — HTML table with region, deal count, closed-won, win rate.

3. Deal Performance by Plan — same structure as above.

4. Top Performers — HTML table with Rep, Closed-Won, Other Status, and Notes.

5. Pipeline & Risks — bullet points highlighting low close rates.

6. Recommendations — exactly FIVE bullet points with strategic actions based on data patterns (e.g., coaching, focus regions, plan upsell).

Also:

- Use HTML tags only (e.g., <h2>, <table>, <ul>, <li>, <img>).

- Format all tables professionally with borders.

- Include QuickChart bar charts using <img> tags where helpful.

- Output must be complete and self-contained HTML ready to convert to PDF.

When embedding QuickChart images, use URL-encoded JSON inside the <img src=""> tag. The chart URL must be fully valid and renderable in PDF. Use encodeURIComponent formatting to ensure no characters break the link.

Example of correctly formatted chart image tag:

https://www.google.com/search?q=%3Cimg+src%3D%22https%3A...

IMPORTANT:

- Do not use raw curly braces {} in the image src.

- Always encode the chart config for reliable rendering in PDF.

- Continue formatting all other content using standard HTML tags and inline CSS.

Make the output clean, professional, and suitable for C-level executives.

The AI produces a fully styled HTML report with QuickCharts included, ready for the next conversion step.

Step 6. Convert HTML to PDF

Once the AI has generated the comprehensive sales report in HTML format, the next step is to transform this digital content into a polished, printable PDF document. This is where PDF.co comes into play, providing a seamless conversion process within the Make.com scenario.

This module ensures that the final report is not only professional in appearance but also easily shareable and consistent across different viewing platforms.

  • Module: PDF.co → Convert HTML to PDF
  • Convert Type: HTML to PDF
  • Input HTML Code: {{4.answer}}
  • File name:

Executive_Sales_Report_{{formatDate(setDay(addDays(now; -7); "monday"); "YYYY-MM-DD")}}_to_{{formatDate(addDays(setDay(addDays(now; -7); "monday"); 6); "YYYY-MM-DD")}}.pdf

  • Orientation: Portrait
  • Paper size: Letter
  • Print background: Yes

This configuration produces a polished, multi-page PDF report, dynamically named to reflect the reporting period, ensuring easy identification and organization.

Step 7. Send the Email

The final stage of the automation involves distributing the newly generated PDF report to the intended recipients. This is handled by the Gmail module, ensuring that the executive sales report reaches the relevant stakeholders automatically and on schedule.

This step completes the end-to-end automation, delivering valuable insights directly to decision-makers without any manual intervention.

  • Module: Gmail → Send an email
  • To: Your execs (or distribution list)
  • Subject:

Executive Sales Report {{formatDate(setDay(addDays(now; -7); "monday"); "YYYY-MM-DD")}} to {{formatDate(addDays(setDay(addDays(now; -7); "monday"); 6); "YYYY-MM-DD")}}

  • Attachments: PDF.co Convert HTML to PDF

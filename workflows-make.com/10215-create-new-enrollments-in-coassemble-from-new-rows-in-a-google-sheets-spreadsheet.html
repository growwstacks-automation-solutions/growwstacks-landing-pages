What This Workflow Does

This automation solves the tedious and error-prone process of manually enrolling students or employees into Coassemble training courses. When working with spreadsheets of learners (common in HR onboarding or corporate training scenarios), administrators typically copy-paste data between systems - a time-consuming task that often leads to mistakes.

The workflow automatically detects new rows added to your designated Google Sheet, extracts the relevant student and course information, and creates the corresponding enrollments in Coassemble. It runs on a schedule you define (hourly, daily, etc.), ensuring your training roster stays perfectly synchronized without manual intervention.

How It Works

1. Scheduled Google Sheets Check

The workflow begins by checking your specified Google Sheet at regular intervals. It looks for new rows added since the last run, using a timestamp or row ID system to track what's been processed.

2. Data Validation

Each new row is validated to ensure it contains the required information: at minimum a valid email address and course identifier. The system can be configured to skip incomplete entries or flag them for review.

3. Coassemble API Integration

Validated data is sent to Coassemble via API, creating enrollments with all specified parameters. The integration handles user creation if needed, or simply adds courses to existing learner profiles.

4. Confirmation & Logging

Successful enrollments are logged back to your sheet or a separate tracking system. The workflow can also trigger email confirmations to students or notifications to training administrators.

Pro tip: Add a "Training Due Date" column in your sheet to automatically set completion deadlines in Coassemble. The workflow can map this field to create time-sensitive enrollments.

Quick Setup Guide

  1. Connect your Google Sheet: In Make.com, authorize access to the specific spreadsheet containing your enrollment data.
  2. Map your fields: Identify which columns contain email addresses, course IDs, and any additional enrollment parameters.
  3. Configure Coassemble connection: Enter your API credentials and test the connection.
  4. Set your schedule: Choose how often the workflow should check for new rows (hourly recommended for most use cases).
  5. Test with sample data: Add a few test rows to verify enrollments are created correctly before going live.

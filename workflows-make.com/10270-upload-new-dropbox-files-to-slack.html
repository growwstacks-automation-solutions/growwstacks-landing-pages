What This Workflow Does

This automation solves the common problem of team members missing important files because they weren't manually shared in Slack. When files get added to specific Dropbox folders, they often need to be separately uploaded to Slack channels - a repetitive task that wastes time and creates version control issues.

The workflow automatically monitors designated Dropbox folders and instantly uploads any new files to specified Slack channels. This ensures all team members have immediate access to the latest versions of documents, images, or other shared files without manual intervention.

How It Works

1. Dropbox folder monitoring

The workflow continuously watches your specified Dropbox folders for new file additions using Make.com's Dropbox integration.

2. File detection trigger

When a new file appears in the monitored folder, the workflow immediately captures the file details including name, type, size, and direct download link.

3. Slack channel upload

The system then uploads the file directly to your chosen Slack channel while preserving the original filename and format.

4. Optional notifications

You can configure custom messages to accompany each file upload, informing team members about the new shared resource.

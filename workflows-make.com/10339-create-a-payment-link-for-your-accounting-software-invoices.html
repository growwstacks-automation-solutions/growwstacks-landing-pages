What This Workflow Does

This automation solves a common pain point for European businesses using QuickBooks - the manual process of creating and attaching payment links to invoices. Without automation, accounting teams waste hours generating Mollie links, copying them into invoices, and tracking which payments have been made.

The workflow automatically creates a unique Mollie payment link whenever a new invoice is generated in QuickBooks Online. The link is then added to the invoice before it's sent to the customer, enabling one-click payments. This reduces payment delays from an average of 5-7 days (with bank transfers) to instant processing while eliminating manual errors.

How It Works

1. New invoice detection

The workflow monitors QuickBooks for new invoices via Make.com's QuickBooks module. It triggers whenever an invoice status changes to "sent" or "published".

2. Payment link generation

The system extracts invoice details (amount, description, customer email) and uses Mollie's API to create a payment link with matching parameters. Dynamic fields ensure each link references the correct invoice number.

3. Link attachment

The generated Mollie link is automatically inserted into the invoice's payment instructions field or added as a clickable button in the email body template.

4. Payment reconciliation

When customers pay via the link, Mollie marks the payment as completed and the workflow can optionally update QuickBooks payment status or send payment confirmation emails.

Pro tip: For EU businesses, configure the links to default to SEPA Direct Debit to save 1-2% in payment processing fees compared to credit cards.

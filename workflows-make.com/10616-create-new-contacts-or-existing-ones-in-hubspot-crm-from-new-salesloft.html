What This Workflow Does

This automation bridges the gap between your sales and marketing teams by automatically syncing contacts from Salesloft (your sales engagement platform) to HubSpot CRM (your marketing and sales hub). When sales reps add or update prospects in Salesloft, those changes flow seamlessly into HubSpot where marketing can nurture those leads with targeted campaigns.

The workflow intelligently checks whether each Salesloft contact already exists in HubSpot. For new contacts, it creates complete CRM records with all relevant fields. For existing contacts, it updates their information with the latest sales activity data while preserving marketing engagement history. This two-way visibility helps both teams work from the same customer data.

How It Works

1. Trigger: New or updated Salesloft contact

The workflow activates whenever a sales rep creates or modifies a person record in Salesloft. This could be from manual entry, imported lists, or Salesloft's own automation rules.

2. Contact lookup in HubSpot

The system searches HubSpot's contact database using the Salesloft record's email address as the unique identifier. This prevents duplicate records while ensuring all activity gets associated with the correct contact.

3. Conditional path: Create or update

If no matching HubSpot contact exists, the workflow creates a new record with all mapped Salesloft fields. If a match is found, it updates the existing record with new information from Salesloft while preserving HubSpot's marketing data.

4. Field mapping and data transformation

The template includes pre-configured field mappings between Salesloft and HubSpot, which you can customize. It handles data format conversions (like date fields) and can apply conditional logic to certain fields.

5. Error handling and notifications

If any step fails (like API rate limits), the workflow retries with exponential backoff and can notify your team via email or Slack if manual intervention is needed.

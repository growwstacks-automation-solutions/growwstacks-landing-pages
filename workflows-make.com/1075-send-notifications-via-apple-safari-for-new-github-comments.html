What This Workflow Does

This automation solves the problem of missing important GitHub comments during active development work. Developers often need to constantly check repositories for new feedback, which disrupts workflow and reduces productivity. The workflow automatically detects new GitHub comments and delivers them as native Safari browser notifications on your Mac.

Unlike email notifications that get buried or Slack alerts that require switching contexts, Safari notifications appear unobtrusively while you work. They provide just enough information to decide whether immediate attention is needed, without forcing you away from your current task.

How It Works

1. GitHub Comment Detection

The workflow monitors your specified GitHub repositories through Make.com's GitHub integration. It checks for new comments at regular intervals using GitHub's API.

2. Notification Processing

When a new comment is detected, the workflow extracts key details like the repository name, comment author, and a preview of the comment content. It formats this information for optimal display in Safari notifications.

3. Safari Notification Delivery

Make.com triggers the notification through Safari's web push notification system. The notification appears in your Mac's notification center and optionally plays a sound, depending on your system settings.

Pro tip: Configure the workflow to only notify about specific repositories or when certain team members comment to reduce notification fatigue.

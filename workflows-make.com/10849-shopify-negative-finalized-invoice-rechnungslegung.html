What This Workflow Does

This automation solves a critical accounting challenge for Shopify merchants using FreeFinance. When customers receive refunds, accounting systems need corresponding negative invoices to maintain accurate financial records. Manually creating these is time-consuming and prone to errors.

The workflow automatically detects new refunds in Shopify and generates matching negative finalized invoices in FreeFinance. It runs every 60 minutes (configurable) to ensure your accounting stays current without manual intervention. All refund details - including amounts, products, taxes, and customer information - transfer accurately between systems.

How It Works

1. Scheduled Shopify Refund Check

The workflow begins by checking Shopify's API for new refunds at your configured interval. It identifies both full and partial refunds, capturing all relevant transaction details.

2. Data Transformation

Refund data gets formatted for FreeFinance's invoice requirements. This includes converting currency values, mapping product SKUs, and structuring line items to match your accounting preferences.

3. Negative Invoice Creation

The system creates a finalized negative invoice in FreeFinance with all refund details. The invoice references the original order number and clearly marks it as a refund for accounting clarity.

4. Error Handling

If any step fails (like API timeouts or data mismatches), the workflow logs the error and retries according to your configured policies, ensuring no refunds get missed.

Pro tip: Set up email notifications for failed refund syncs to quickly address any issues before month-end closing.

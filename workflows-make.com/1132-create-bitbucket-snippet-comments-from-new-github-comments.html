What This Workflow Does

This automation solves a common pain point for development teams working across GitHub and Bitbucket. When code snippets move between platforms or teams use both services simultaneously, valuable feedback and discussions often get fragmented. The workflow automatically mirrors new GitHub comments to specified Bitbucket snippets, ensuring all stakeholders see relevant discussions regardless of which platform they prefer.

By eliminating manual comment copying, this solution saves developers significant time while preventing information loss during code migrations or cross-platform collaborations. It maintains discussion context with original timestamps and attribution, creating a seamless experience for distributed teams.

How It Works

1. GitHub Comment Trigger

The workflow monitors your specified GitHub repository for new comments. When detected, it captures the comment content, author details, timestamp, and any threaded replies.

2. Bitbucket Snippet Identification

The system matches the GitHub discussion to the corresponding Bitbucket snippet using your predefined mapping rules. This can be based on snippet IDs, file paths, or other identifiers.

3. Comment Formatting

The workflow adapts GitHub's markdown formatting to Bitbucket's syntax, preserves @mentions, and maintains code block styling to ensure readability in both platforms.

4. Bitbucket Comment Creation

Using Bitbucket's API, the workflow creates a new comment on the target snippet with all original content and metadata. It handles authentication and rate limiting automatically.

Pro tip: Configure the workflow to add a "Synced from GitHub" label to mirrored comments for easy tracking and filtering in Bitbucket.

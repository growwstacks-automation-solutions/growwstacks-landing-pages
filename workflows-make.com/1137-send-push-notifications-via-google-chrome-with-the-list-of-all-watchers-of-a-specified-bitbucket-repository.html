What This Workflow Does

This automation solves the visibility gap in code repository engagement by automatically sending Chrome push notifications whenever users watch your Bitbucket repositories. Development teams often miss important signals when stakeholders start monitoring their code, which can lead to delayed responses to feedback or missed collaboration opportunities.

The workflow connects Bitbucket's API with Chrome's notification system, creating a real-time alert mechanism that shows exactly who's watching your repositories. This is particularly valuable for engineering managers, open-source maintainers, and teams working on critical projects where code visibility matters.

How It Works

1. Repository Monitoring

The workflow periodically checks your specified Bitbucket repositories using API calls to identify all current watchers. You can configure the monitoring frequency based on your team's needs.

2. Watcher List Processing

Make.com processes the API response to extract relevant watcher information, including usernames and timestamps. The system compares this against previous checks to identify new watchers.

3. Notification Generation

When changes in watchers are detected, the workflow formats a clear notification message showing the repository name and list of watchers. The message is optimized for Chrome's notification system.

4. Push Delivery

The notification is sent through Chrome's push notification service, appearing instantly on authorized browsers. Team members can click the notification to jump directly to the repository.

Pro tip: Combine this with other Bitbucket automations to create a comprehensive code activity monitoring system that includes pull requests, commits, and branch updates.

What This Workflow Does

This automation solves the tedious manual process of downloading email attachments and re-uploading them to secure cloud storage. Professionals in legal, accounting, and client services often waste hours each week managing document transfers between Gmail and Citrix ShareFile.

The workflow monitors your Gmail inbox for new messages with attachments, then automatically uploads those files to your designated ShareFile folder structure. It maintains security by using OAuth authentication and preserves original file names and formats during transfer.

How It Works

1. Trigger on new Gmail attachments

The workflow activates whenever a new email arrives in your Gmail account with one or more attachments. It can be configured to watch specific labels or your entire inbox.

2. Extract attachment details

Make.com retrieves the file name, size, type, and content from each attachment while preserving the original email's metadata like sender and subject line.

3. Connect to Citrix ShareFile

The system authenticates with your ShareFile account using secure OAuth 2.0 protocols, ensuring enterprise-grade security for your document transfers.

4. Upload to designated folder

Attachments are automatically uploaded to your specified ShareFile location. Advanced configurations can route files based on sender, date, or keywords in the subject line.

Pro tip: Set up folder rules in ShareFile first to automatically apply retention policies or access permissions to uploaded files.

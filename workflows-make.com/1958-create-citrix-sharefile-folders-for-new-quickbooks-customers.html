What This Workflow Does

This automation solves a common pain point for businesses using both QuickBooks for accounting and Citrix ShareFile for secure document sharing. Every time you add a new customer in QuickBooks, the system automatically creates a corresponding client folder in ShareFile with the appropriate naming convention and permissions.

Manual folder creation is time-consuming and prone to errors - staff might forget to create folders, use inconsistent naming, or set incorrect permissions. This workflow eliminates those issues while saving 5-15 minutes per new client that would otherwise be spent on repetitive administrative tasks.

How It Works

1. New Customer Detection

The workflow monitors your QuickBooks account for new customer additions through Make.com's instant trigger system. No polling or scheduled checks needed - it reacts immediately to changes.

2. Data Extraction

Key customer information like company name, contact person, and QuickBooks customer ID are extracted to use in folder naming and metadata.

3. ShareFile Folder Creation

The system connects to your Citrix ShareFile account and creates a new folder in your specified location (typically a "Clients" parent folder). Folder names can incorporate any combination of customer data fields.

4. Permission Assignment

Optionally, the workflow can set folder permissions - giving the client access to their folder while restricting access to others, or setting internal team permissions according to your policies.

Pro tip: Add a second action to send a welcome email with the folder link when setting up new clients. This creates a seamless onboarding experience.

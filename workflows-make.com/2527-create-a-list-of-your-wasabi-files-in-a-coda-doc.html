What This Workflow Does

This automation solves the common challenge of maintaining accurate documentation for files stored in Wasabi cloud storage. Instead of manually updating spreadsheets whenever files are added, moved, or deleted, this workflow automatically maintains a complete inventory in your Coda document.

The system creates a structured table in Coda containing all your Wasabi file metadata - names, paths, sizes, modification dates, and any custom properties. This becomes a living document that stays perfectly synchronized with your actual cloud storage, saving hours of manual tracking while providing powerful search and organization capabilities.

How It Works

1. Connect to your Wasabi account

The workflow authenticates with your Wasabi storage using secure API credentials, giving it read-only access to your file metadata without exposing actual file contents.

2. Scan your storage buckets

It recursively scans designated Wasabi buckets and folders, building a complete list of files including their metadata properties.

3. Structure the data for Coda

The raw file information gets transformed into a clean table format with consistent column types that work optimally in Coda.

4. Update your Coda document

The workflow either creates a new table or updates an existing one in your specified Coda doc, adding new files, updating changed ones, and removing deleted entries.

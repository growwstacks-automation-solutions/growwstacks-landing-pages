What This Workflow Does

This automation solves the common problem of losing track of files added to shared Google Drive folders. Instead of manually updating documentation every time someone uploads a new file, this workflow automatically adds the file details to a specified table in your Coda doc.

The moment any team member adds a file to your monitored Google Drive folder, Make.com detects the change and populates your Coda doc with key file information including the file name, creation date, owner, and direct link. This creates a searchable, organized record of all project files alongside your other documentation.

How It Works

1. Monitoring Google Drive for new files

The workflow continuously watches your specified Google Drive folder for new file additions using Make.com's Google Drive trigger. It detects uploads instantly without polling delays.

2. Extracting file metadata

For each new file, the workflow collects comprehensive metadata including the file name, type, size, creation date, last modified date, owner email, and sharing permissions.

3. Formatting data for Coda

The raw file data is transformed into a structured format that matches your Coda table columns, with proper date formatting and hyperlinks created for easy access.

4. Updating your Coda doc

Make.com authenticates with your Coda account and adds a new row to your specified table with all the file details. The connection uses Coda's API for reliable updates.

What This Workflow Does

This automation solves the common problem of lost or inaccurate call records by automatically logging every completed and missed call from Yodel.io directly to your Google Sheets spreadsheet. Manual call logging is time-consuming and prone to errors, especially for teams handling high call volumes.

The workflow triggers whenever a call ends in Yodel.io, capturing all relevant call details like timestamp, duration, caller information, and call status. It then formats this data and adds it as a new row in your designated Google Sheet, creating a complete historical record of all call activity.

How It Works

1. Yodel.io Trigger

The workflow monitors Yodel.io's API for call completion events. Whenever a call ends (whether completed or missed), it immediately captures the call data payload.

2. Data Processing

Make.com extracts key call details like call duration, start/end times, caller ID, and call direction. You can customize which fields to capture based on your reporting needs.

3. Google Sheets Update

The processed call data gets formatted into a spreadsheet row and appended to your designated Google Sheet. The template maintains consistent column structure while adding new records.

Pro tip: Add a timestamp column in Sheets to track when each record was added, helping you monitor the automation's performance.

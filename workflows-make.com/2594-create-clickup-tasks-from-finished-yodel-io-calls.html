What This Workflow Does

This automation bridges the gap between phone conversations and task management by automatically creating ClickUp tasks whenever you complete calls through Yodel.io. Instead of manually noting call details and creating follow-up tasks, the workflow handles everything instantly.

The system captures all relevant call information - including caller details, duration, notes, and timestamps - then formats this data into a properly categorized ClickUp task. This ensures no important call falls through the cracks while eliminating repetitive administrative work.

How It Works

1. Yodel.io detects call completion

The workflow triggers whenever a call ends in your specified Yodel.io phone number. Make.com receives the call data through Yodel.io's API, including all available metadata about the conversation.

2. Data is processed and formatted

The workflow extracts key information like caller number, call duration, and any notes added during the call. You can customize which data points to include and how they're formatted for ClickUp.

3. Task is created in ClickUp

Using ClickUp's API, the workflow creates a new task in your specified list or space. The task includes all relevant call details as custom fields, description content, or comments based on your configuration.

Pro tip: Add conditional logic to create different task types based on call characteristics. For example, sales calls could generate 'Follow Up' tasks while support calls create 'Ticket' items.

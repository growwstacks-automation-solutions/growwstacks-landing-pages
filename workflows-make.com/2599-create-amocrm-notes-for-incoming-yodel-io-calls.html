What This Workflow Does

This automation solves the common problem of sales teams forgetting call details or spending excessive time manually logging interactions. Every time you receive an incoming call through Yodel.io, the workflow automatically creates a detailed note in Kommo (formerly AmoCRM) with the call timestamp, duration, and caller information.

By eliminating manual data entry, this integration ensures your CRM always reflects the most current customer interactions. Sales managers gain visibility into team call activity, while reps can focus on conversations rather than note-taking. The automated system reduces human error and creates consistent documentation for every customer touchpoint.

How It Works

1. Call Received in Yodel.io

The workflow triggers immediately when Yodel.io detects an incoming call to your designated phone number. It captures the call metadata including the caller's number, call start time, and duration.

2. Data Processing in Make.com

Make.com formats the call data and checks against any filters you've set (e.g., minimum call duration or specific caller groups). This step ensures only relevant calls generate Kommo notes.

3. Kommo Note Creation

The system locates or creates the contact in Kommo, then adds a new note with the call details. The note template can be customized to include specific fields like call purpose or follow-up actions.

What This Workflow Does

This automation solves the common problem of lost or disorganized call recordings by automatically saving every Yodel.io recording to your Google Drive. Customer service teams, sales departments, and compliance officers often struggle with manually downloading and organizing call recordings - this workflow eliminates that tedious process.

Whenever Yodel.io generates a new call recording, Make.com instantly detects it and uploads the file to your specified Google Drive folder. The system maintains all recording metadata including timestamps, caller information, and duration while providing enterprise-grade backup security.

How It Works

1. Trigger on New Recording

The workflow monitors your Yodel.io account for new call recordings via webhook or API polling. As soon as a recording is available, the automation begins processing.

2. Retrieve Recording File

Make.com securely accesses the recording file from Yodel.io's servers using authenticated API calls, ensuring no recordings are missed even during high call volumes.

3. Upload to Google Drive

The system then transfers the recording file to your designated Google Drive folder, maintaining the original file format and adding relevant metadata to the filename.

Pro tip: Configure your Google Drive folder structure to automatically organize recordings by date, department, or agent for easier retrieval.

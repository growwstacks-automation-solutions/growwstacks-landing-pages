What This Workflow Does

This automation solves a critical data protection challenge for businesses using Yodel.io for call recording. Every time a call is recorded in Yodel.io, the workflow automatically uploads that recording to your Wasabi cloud storage account. This creates a secure backup independent of Yodel.io's retention policies.

Call recordings often contain sensitive customer information, sales agreements, and training materials. Relying solely on Yodel.io for storage risks permanent data loss if accounts expire, recordings get accidentally deleted, or if you need historical records beyond Yodel.io's retention period. This workflow ensures your recordings remain accessible in Wasabi's affordable, enterprise-grade cloud storage.

How It Works

1. Trigger: New recording in Yodel.io

The workflow monitors your Yodel.io account for new call recordings via webhook or API polling. As soon as a recording is available, the process begins.

2. Retrieve recording file

Make.com securely accesses the recording file from Yodel.io's servers using authenticated API calls. The original audio quality and metadata are preserved.

3. Upload to Wasabi

The recording is transferred directly to your designated Wasabi bucket. You can configure folder structures by date, client ID, or other parameters for easy organization.

4. Confirmation & logging

The workflow logs each successful transfer and can optionally notify your team via email or Slack. Failed transfers automatically retry with error reporting.

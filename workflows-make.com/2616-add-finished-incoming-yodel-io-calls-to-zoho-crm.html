What This Workflow Does

This automation solves the tedious manual process of logging call details from Yodel.io into Zoho CRM. Every time a sales or support call completes in Yodel.io, the workflow automatically captures all relevant call data and creates or updates the corresponding contact record in Zoho CRM.

The system ensures no call goes unlogged, providing your team with complete customer interaction histories without the administrative burden. This eliminates the common problem of missed or incomplete call notes that can lead to poor customer experiences and lost sales opportunities.

How It Works

1. Trigger on completed calls

The workflow monitors Yodel.io for call completion events via webhook. Each time an incoming call finishes in Yodel.io, it sends the call details to Make.com.

2. Process call data

Make.com extracts key information like caller number, duration, timestamp, and call status. The system can optionally enrich this data with additional context from Yodel.io's call analytics.

3. Match or create CRM record

The workflow searches Zoho CRM for existing contacts matching the caller's phone number. If found, it updates their record. If not, it creates a new contact with the call details.

4. Log call activity

A detailed call note gets added to the contact's activity timeline in Zoho CRM, including all relevant metadata for future reference by your sales team.

Pro tip: Configure the workflow to also create follow-up tasks in Zoho CRM based on call outcomes (e.g., "Call back in 3 days" for interested prospects).

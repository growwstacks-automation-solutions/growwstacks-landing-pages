What This Workflow Does

This automation solves a critical data synchronization challenge for sales teams using both Pipedrive (CRM) and Yodel.io (customer communication platform). Every time a new contact is added or updated in Pipedrive, the workflow automatically creates or updates the corresponding contact in Yodel.io.

Manual contact management between systems leads to errors, duplicates, and outdated information. This workflow eliminates those issues by maintaining perfect contact record parity between platforms, ensuring your sales and communication teams always work with the same accurate customer data.

How It Works

1. Trigger: New or updated Pipedrive contact

The workflow activates whenever a contact is created or modified in your Pipedrive account. Make.com monitors your Pipedrive instance in real-time through API connections.

2. Contact data extraction

The system retrieves all relevant contact details including name, email, phone number, company information, and any custom fields you've configured to sync.

3. Yodel.io contact lookup

Before creating a new record, the workflow checks Yodel.io for an existing contact with the same email address to prevent duplicates.

4. Create or update decision

If the contact exists in Yodel.io, their record is updated with the latest Pipedrive information. If not found, a new Yodel.io contact is created with all transferred data.

5. Error handling and logging

The workflow includes robust error handling to manage connection issues or data mismatches, with detailed logging of all synchronization attempts.

Pro tip: Configure custom field mappings to ensure specific Pipedrive data populates the correct Yodel.io fields, especially if you use custom properties in either system.

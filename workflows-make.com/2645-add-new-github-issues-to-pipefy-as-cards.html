What This Workflow Does

This automation bridges the gap between GitHub's issue tracking and Pipefy's workflow management by automatically creating Pipefy cards whenever new issues are opened in GitHub. It eliminates the manual process of recreating GitHub issues in your project management system, saving development teams hours of repetitive data entry each week.

The workflow captures all relevant issue details including title, description, labels, assignees, and status. These are mapped to corresponding fields in Pipefy, ensuring your project managers have complete context without needing to switch between tools. For teams using both GitHub and Pipefy, this creates a seamless handoff between development and project tracking.

How It Works

1. GitHub Trigger

The workflow monitors your specified GitHub repositories for new issues using Make.com's GitHub module. You can configure it to watch specific repos, organizations, or filter by labels.

2. Data Transformation

Make.com extracts key information from the GitHub issue and formats it for Pipefy. This includes converting GitHub labels to Pipefy tags, mapping assignees, and structuring the issue description.

3. Pipefy Card Creation

The transformed data creates a new card in your designated Pipefy board. You can specify which pipe (column) the card should appear in and set initial status based on GitHub issue labels.

Pro tip: Add a GitHub issue template that includes Pipefy-specific fields to ensure consistent data quality in your automated cards.

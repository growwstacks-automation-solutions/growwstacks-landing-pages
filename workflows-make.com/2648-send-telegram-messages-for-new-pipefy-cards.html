What This Workflow Does

This automation solves the common problem of delayed response times when new tasks are created in Pipefy. Instead of requiring team members to constantly check the platform or rely on email notifications, it pushes instant alerts directly to Telegram where teams are already communicating.

The workflow triggers whenever a new card is added to your specified Pipefy board, capturing all relevant task details. It then formats this information into a clear Telegram message that includes the card title, assignee, due date (if set), and a direct link back to the Pipefy card for quick access.

How It Works

1. Pipefy Trigger Setup

The workflow begins by monitoring your chosen Pipefy board for new card creations. You can specify which phases or lanes should trigger notifications, filtering out irrelevant card movements.

2. Data Extraction

Make.com captures the complete card payload including custom fields, labels, attachments, and comments. This data is temporarily stored for processing and mapping to your notification template.

3. Telegram Message Formatting

The system formats the extracted data into a clean Telegram message using markdown for readability. You can customize which fields appear and how they're organized to match your team's preferences.

4. Secure Delivery

The formatted message is sent to your designated Telegram channel or group using a bot you control. The workflow includes error handling to retry failed deliveries and logs all notification attempts.

Quick Setup Guide

  1. Copy the template to your Make.com account
  2. Connect your Pipefy account and select the target board
  3. Create a Telegram bot via @BotFather and note its API token
  4. Specify which Telegram channel or group should receive alerts
  5. Test with sample cards to verify the notification format

Pro tip: Create separate Telegram channels for different project types or priority levels, then use Pipefy filters to route alerts accordingly. This keeps notifications organized and prevents alert fatigue.

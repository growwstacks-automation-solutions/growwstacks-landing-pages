What This Workflow Does

This automation bridges the gap between project management and development teams by automatically creating Bitbucket issues whenever new cards are added to your Pipefy boards. Instead of developers manually recreating tasks or product managers chasing status updates, the workflow maintains a seamless flow of work items from planning to execution.

Teams using this integration typically save 5-10 hours per week previously spent on manual ticket creation and status synchronization. The automation captures all relevant card details - titles, descriptions, assignees, due dates, and custom fields - ensuring nothing gets lost in translation between platforms.

How It Works

1. Pipefy Triggers the Workflow

The automation monitors your specified Pipefy board for new card creations. You can filter by phase, label, or custom fields to control which cards generate Bitbucket issues.

2. Data is Transformed

Make.com maps Pipefy card properties to Bitbucket issue fields according to your configuration. This includes converting Pipefy assignees to Bitbucket reporters, calculating due dates, and applying appropriate issue types.

3. Bitbucket Issue is Created

The workflow creates a new Bitbucket issue with all mapped data, then optionally updates the Pipefy card with a link to the new issue for full traceability.

Pro tip: Configure your workflow to add a Pipefy custom field with the Bitbucket issue URL. This creates two-way visibility between platforms.

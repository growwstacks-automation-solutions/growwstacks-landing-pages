What This Workflow Does

This automation solves the problem of missing important refunds in your Square account. Without constant dashboard checking, large refunds can go unnoticed for days - impacting cash flow and potentially hiding fraudulent activity.

The workflow monitors your Square account in real-time, triggering an instant Android push notification whenever a refund exceeds your predefined amount threshold. You'll know immediately when significant refunds occur, allowing for prompt investigation and action.

How It Works

1. Square refund detection

The workflow continuously checks Square's API for new refund events using webhooks. When Square processes a refund, it sends real-time data to Make.com.

2. Amount threshold check

Each refund amount is compared against your configured threshold (e.g. $100). Only refunds exceeding this amount proceed to the next step.

3. Notification generation

The system formats a clear notification message containing the refund amount, customer details, and transaction timestamp.

4. Android push delivery

Using Firebase Cloud Messaging, the alert is instantly pushed to your registered Android device, appearing like any other app notification.

Pro tip: Set different thresholds for weekdays vs weekends when fraud patterns often differ.

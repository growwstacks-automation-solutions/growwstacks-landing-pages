What This Workflow Does

This automation bridges the gap between Slack conversations and mobile SMS delivery, ensuring critical messages reach recipients instantly regardless of whether they're actively monitoring Slack. When configured properly, it can reduce response times for urgent matters by 60-80% compared to relying solely on Slack notifications.

The workflow solves the common problem of important Slack messages being missed by team members who aren't constantly checking the app. By converting selected Slack messages into SMS texts, you create a failsafe communication channel that works even when someone's phone has poor data connectivity or the Slack app isn't running in the foreground.

How It Works

1. Slack Message Trigger

The workflow monitors your specified Slack channel for new messages. You can configure it to watch for specific keywords, mentions, or message types.

2. Message Processing

Make.com processes the Slack message content, extracting relevant details like the sender, channel, and message text. You can customize which elements to include in the SMS.

3. SMS Delivery via ClickSend

The processed message gets formatted and sent through ClickSend's SMS API to your designated recipients. Delivery confirmations are logged for tracking.

Pro tip: Set up message filtering to only forward messages containing specific trigger words like "URGENT" or "ALERT" to avoid SMS spam.

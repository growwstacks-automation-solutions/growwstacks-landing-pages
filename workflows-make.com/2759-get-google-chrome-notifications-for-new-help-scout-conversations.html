What This Workflow Does

This automation solves the common problem of delayed response times in customer support by eliminating the need to constantly monitor Help Scout inboxes. Whenever a new conversation starts—whether through email, chat, or phone—your team receives instant Chrome notifications with key details.

The system monitors all designated Help Scout mailboxes and triggers real-time desktop alerts showing the customer's name, subject line, and message preview. Support agents can click notifications to jump directly into the conversation, reducing first response times by 30-60% according to industry benchmarks.

How It Works

1. New Conversation Detection

The workflow continuously monitors your Help Scout account for newly created conversations across all connected mailboxes. It captures the full conversation details including customer information, subject, and initial message.

2. Notification Processing

Make formats the conversation data into a standardized notification structure, extracting the most relevant details for immediate visibility while maintaining all context for when agents open the full conversation.

3. Chrome Notification Delivery

Using Chrome's notification API, the system pushes alerts directly to authorized browsers. Notifications remain visible until dismissed and include quick-action buttons for common responses when supported by your Help Scout configuration.

Pro tip: Combine this with automated tagging in Help Scout to prioritize notifications—critical issues can use urgent notification styles while general inquiries use standard alerts.

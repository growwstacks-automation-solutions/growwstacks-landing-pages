What This Workflow Does

This automation solves a common pain point for customer support teams using Help Scout alongside task management tools like Meistertask. When conversations get assigned to specific team members in Help Scout, those assignments often represent work that needs to be tracked and completed - yet manually creating corresponding tasks in Meistertask wastes valuable time and introduces the risk of tasks being forgotten.

The workflow automatically creates a new task in Meistertask whenever a conversation is assigned to a specified user in Help Scout. This ensures every assigned support item becomes a trackable task with all relevant context, while eliminating repetitive administrative work for your team. The automation maintains a clear connection between the original conversation and the resulting task, helping agents stay organized.

How It Works

1. Trigger: New assignment in Help Scout

The workflow monitors Help Scout for conversations being assigned to your specified team member. You can configure which mailbox(es) to watch and which assignment events should trigger task creation.

2. Data collection

The automation gathers all relevant conversation details from Help Scout, including customer information, conversation subject, tags, and any custom fields you want to include in the task.

3. Task creation in Meistertask

Using the collected data, the workflow creates a new task in your specified Meistertask project. You can customize how information maps from Help Scout to Meistertask fields, setting priorities, due dates, and assigning additional team members as needed.

4. Optional notifications

The template can be extended to notify team members about new tasks via Slack, email, or other communication channels, ensuring prompt attention to newly assigned work.

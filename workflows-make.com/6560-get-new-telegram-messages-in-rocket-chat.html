What This Workflow Does

This automation solves the common problem of team members missing important messages that arrive on external platforms like Telegram while they're working in internal collaboration tools like Rocket.Chat. It eliminates the need for manual message forwarding, ensuring seamless communication flow between platforms.

The workflow monitors specified Telegram channels or groups for new messages and automatically posts them to designated Rocket.Chat channels. This keeps all team communications centralized while still allowing you to engage with external contacts on their preferred platform.

How It Works

1. Telegram Trigger Setup

The workflow starts by connecting to your Telegram account via Make.com's Telegram bot. You specify which groups or channels to monitor for new messages.

2. Message Processing

Each new Telegram message triggers the workflow. The system extracts the message content, sender information, and any attachments.

3. Rocket.Chat Integration

Make.com authenticates with your Rocket.Chat instance using API credentials, then posts the formatted message to your specified channel with appropriate metadata.

4. Error Handling

The template includes basic error handling to retry failed deliveries and notify administrators if messages can't be forwarded after multiple attempts.

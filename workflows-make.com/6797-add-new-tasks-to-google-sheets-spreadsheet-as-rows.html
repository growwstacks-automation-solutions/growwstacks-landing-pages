What This Workflow Does

For e-commerce businesses using Shopify, manually creating invoices, receipts, or packing slips for each order is a repetitive, time-consuming task. This process often leads to delays in order fulfillment, inconsistencies in document formatting, and potential errors in customer details or pricing.

This automation workflow solves that problem entirely. It connects your Shopify store to Plumsail Documents, a powerful document generation platform. Every time a new order is placed, the workflow instantly triggers, pulls the complete order data, and uses it to populate a pre-designed document template in Plumsail. Within seconds, a professional, accurate, and branded document is created automatically, ready for delivery or storage.

The value is clear: you eliminate manual data entry, ensure 100% accuracy by using direct system data, and provide a faster, more professional customer experience. Your team can focus on growth and customer service instead of administrative paperwork.

How It Works

Step 1: Shopify Triggers the Workflow

The workflow is set to watch for the "Order Created" event in your Shopify store. As soon as a customer completes a purchase, Make.com receives a notification with the full order payload—customer information, items, quantities, prices, shipping address, and more.

Step 2: Data is Prepared and Sent

The workflow maps the relevant Shopify data fields (like customer name, order ID, line items, total amount) into a structured format. This data package is then sent securely to the Plumsail Documents module within the automation.

Pro tip: You can filter which orders trigger document creation. For example, you might only generate invoices for orders over a certain value or exclude test orders.

Step 3: Plumsail Generates the Document

Plumsail Documents uses the incoming data to fill a template you've previously designed. The template can be an invoice, receipt, packing slip, or any other document format. Plumsail merges the live data into the template, applying your branding (logo, colors, fonts) and layout, resulting in a finished PDF or document file.

Step 4: The Document is Output

The newly generated document is now available in your Plumsail account. The workflow can be extended to automatically email this document to the customer, attach it to the order in Shopify, or save it to a cloud storage service like Google Drive for your records.

Quick Setup Guide

  1. Get the Template: Use the "Get This Workflow" button above to copy the scenario into your Make.com account.
  2. Connect Your Apps: In Make.com, add your Shopify and Plumsail Documents connections. Authorize the modules using your account credentials.
  3. Select Your Template: In the Plumsail Documents module, select the specific document template you want to use for order generation.
  4. Map the Data: Configure the data mapping between Shopify's "Order Created" output and the input fields required by your Plumsail template. Ensure key fields like customer name, address, and order total are correctly linked.
  5. Test and Activate: Run a test with a recent Shopify order to verify the document is generated correctly. Once confirmed, activate the scenario. It will now run automatically for all new orders.

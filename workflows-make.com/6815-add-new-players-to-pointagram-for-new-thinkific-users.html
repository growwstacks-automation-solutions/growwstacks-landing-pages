What This Workflow Does

For online course creators using Thinkific, student engagement doesn't stop at enrollment. Gamification platforms like Pointagram are powerful tools to boost completion rates and foster community through points, badges, and leaderboards. However, manually adding every new student from Thinkific into Pointagram is a tedious, error-prone task that wastes valuable time you could spend creating content.

This automation solves that problem entirely. Every time a new user signs up or enrolls in a course on your Thinkific site, this Make.com workflow springs into action. It automatically checks if that student already exists in your Pointagram community. If they're not found, it instantly creates a new player profile for them. This ensures your gamified learning environment is always current, and every student can start their engaging journey immediately, without any delay or administrative overhead.

How It Works

The workflow is a smart, efficient sequence that acts as a bridge between your learning management system and your engagement platform.

Step 1: Trigger on New Thinkific User

The automation is initiated by the "Watch Users" module in Make.com, which monitors your Thinkific account for new student sign-ups. The moment a new user profile is created, the workflow captures key details like their name, email address, and user ID.

Step 2: Search for Existing Pointagram Player

Before creating a duplicate, the workflow takes the student's email and searches your Pointagram account. This check is crucial for maintaining data integrity, especially if a student uses the same email for multiple courses or if you run manual imports occasionally.

Step 3: Create Player if Not Found

If the search comes up empty, the workflow proceeds to the "Create a Player" module in Pointagram. It passes the student's information from Thinkific to create a new, fully-fledged player profile. If the student already exists, the workflow simply ends, preventing any duplicate entries.

Pro tip: You can enhance this workflow by adding a step to assign the new player to a specific "group" or "cohort" in Pointagram based on the Thinkific course they enrolled in, enabling targeted challenges and quests from day one.

What You'll Need

  1. A Thinkific account with active courses and student sign-ups.
  2. A Pointagram account where you manage your gamified community and players.
  3. A Make.com account (free tier available) to host and run the automation.
  4. API connections for both Thinkific and Pointagram, which you'll configure within Make.com using your admin credentials.

Quick Setup Guide

Getting this automation live takes less than 30 minutes. Follow these steps:

  1. Get the Template: Click the "Get This Workflow" button above to clone the template into your Make.com account.
  2. Connect Thinkific: In the first module, click to add your Thinkific connection. You'll need your Thinkific domain and an API key from your Thinkific admin settings.
  3. Connect Pointagram: In the search and create modules, add your Pointagram connection using your Pointagram API key (found in your account settings).
  4. Test the Flow: Use the "Run once" feature in Make.com to test with a recent student. Check your Pointagram account to confirm the player was created successfully.
  5. Activate & Forget: Turn the scenario "ON" in Make.com. It will now run automatically in the background, keeping your platforms in perfect sync.

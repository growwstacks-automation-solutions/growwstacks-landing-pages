What This Workflow Does

Incomplete contact data is a silent revenue killer for sales and marketing teams. Manually searching for phone numbers for every new lead is tedious, error-prone, and slows down your response time dramatically. This automation solves that by creating a seamless bridge between your Airtable database and Data24's extensive contact directory.

Every time a new person is added to your specified Airtable base—whether from a web form, imported list, or sales tool—this Make.com scenario instantly triggers. It takes the available contact details (like name, company, email, or location) and queries the Data24 API to find and retrieve the associated phone number. The number is then written back into the original Airtable record, completing the contact profile automatically. This turns your Airtable from a static list into a dynamically enriched powerhouse, ready for immediate outreach.

How It Works

The workflow operates on a simple but powerful trigger-action model, ensuring your data is enriched in real-time without any manual intervention.

Step 1: Airtable Watches for New Records

The scenario starts with the Airtable module set to "Watch Records." It monitors a specific table in your base, waiting for a new row to appear. This is the trigger that kicks off the entire enrichment process.

Step 2: Data Preparation and API Call

Make takes the data from the new record, such as first name, last name, email, and company name. It structures this information into a precise query format required by the Data24 "Find Person" API module. The API call is sent to Data24's servers to search for a matching profile.

Step 3: Phone Number Extraction and Validation

Data24 returns a response containing the found person's details, including phone numbers. The Make scenario includes a router to handle the outcome: if a match is found, it proceeds to parse the response and extract the most relevant phone number (e.g., a mobile or direct line).

Step 4: Update the Original Airtable Record

Finally, the workflow uses another Airtable module to "Update a Record." It targets the original record that triggered the scenario and writes the newly acquired phone number into a designated column. Your Airtable is now updated with enriched data, ready for your team to use.

Pro tip: Use a "Last Enriched Date" column in Airtable. Update it each time this workflow runs. This creates a clear audit trail and helps you avoid wasting API credits on re-enriching the same records repeatedly.

Quick Setup Guide

Follow these steps to get your phone number enrichment live in under 30 minutes.

  1. Clone the Template: Click "Get This Workflow" and duplicate the scenario into your own Make.com workspace.
  2. Connect Airtable: In the first module, authorize Make to access your Airtable account and select the specific base and table you want to monitor.
  3. Configure Data24: In the "Find Person" module, paste your Data24 API key. Map the incoming fields from Airtable (name, email, company) to the corresponding search parameters in Data24.
  4. Map the Output: In the final Airtable "Update Record" module, ensure the phone number extracted from Data24's response is correctly mapped to your target column in Airtable.

What This Workflow Does

Modern teams often use multiple project management tools. Marketing might live in ClickUp, while the sales or operations team prefers the native integration of Microsoft 365 Planner within their daily workflow. This creates silos, manual double-entry of tasks, and constant risk of miscommunication.

This automation solves that by creating a seamless, one-way bridge from ClickUp to Microsoft Planner. Every time a project manager or team lead creates a new task in a designated ClickUp list, this Make.com workflow instantly mirrors it as a new task in a specified Microsoft Planner bucket. It transfers key details like the task name, description, due date, and assignee, ensuring both teams are looking at the same information without ever leaving their preferred tool.

The business value is clear: it eliminates repetitive administrative work, reduces human error in data transfer, and accelerates project kick-offs by ensuring visibility is immediate across all departments. It turns two separate systems into one cohesive operating environment.

How It Works

The workflow is an elegant, three-step automation built on Make.com's visual platform. It acts as a intelligent middleware, listening for events in one app and triggering actions in another.

Step 1: Watch for New ClickUp Tasks

The scenario starts with a ClickUp module set to "Watch Tasks." It continuously monitors a specific ClickUp list or space for the creation of new tasks. You can configure filters to only trigger for tasks with certain tags, priorities, or assigned to specific team members, ensuring the automation is precise and relevant.

Pro tip: Use ClickUp's custom fields to add important metadata (like client name or budget code). The workflow can map these fields to the Planner task's description or checklist, creating a richer, context-aware task in Microsoft 365.

Step 2: Map and Transform the Data

Once a new task is detected, Make.com processes the incoming data. This is where you define the mapping: the ClickUp task name becomes the Planner task title, the description is carried over, the due date is formatted correctly, and the assignee's email is matched to a Microsoft 365 user. This step ensures data integrity and compatibility between the two different platforms.

Step 3: Create the Task in Microsoft 365 Planner

The final module uses the Microsoft 365 Planner app to "Create a Task." It takes the transformed data from the previous step and posts it directly into the chosen Planner plan and bucket. The task appears in Microsoft Planner within seconds, complete with all the mapped details, ready for the assigned team member to act upon.

Quick Setup Guide

Getting started is straightforward. Follow these steps to deploy the template in under 15 minutes.

  1. Clone the Template: Click the "Get This Workflow" button to open the template on Make.com. Press the "Copy" button to create your own editable version of the scenario.
  2. Connect Your Apps: In your new scenario, click the ClickUp and Microsoft 365 Planner modules. Authenticate each by logging into your accounts when prompted. This grants Make.com the necessary permissions to read and write data.
  3. Configure the Trigger: In the ClickUp "Watch Tasks" module, select your Workspace, Space, and the specific List you want to monitor for new tasks.
  4. Set the Destination: In the Microsoft Planner "Create a Task" module, select your Group, Plan, and the Bucket where new tasks should land. You can also set a default assignee here if needed.
  5. Map Your Fields (Optional): Review the router and mapping steps. Adjust them if you want to transfer additional custom fields from ClickUp or apply conditional logic (e.g., only sync high-priority tasks).
  6. Test and Activate: Turn on the scenario and create a test task in your chosen ClickUp list. Within moments, check your Microsoft Planner board to confirm the task was created correctly. Once verified, your automation is live.

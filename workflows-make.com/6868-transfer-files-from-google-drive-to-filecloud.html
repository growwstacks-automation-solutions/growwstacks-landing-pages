What This Workflow Does

Businesses today use multiple cloud storage platforms for different purposes. Google Drive is excellent for real-time collaboration and ease of use, while FileCloud often serves as a secure, internal file server or client portal. The problem arises when files need to exist in both places. Manually downloading from Drive and re-uploading to FileCloud is a repetitive, time-consuming task that is prone to errors, version mix-ups, and delays.

This automated workflow solves that problem completely. It acts as a silent, reliable bridge between the two platforms. Whenever a new file is added to a specific "watch" folder in Google Drive, the automation instantly detects it, securely transfers the file, and places it in the correct destination folder within FileCloud. This ensures your teams can collaborate in the tool they prefer, while critical files are always backed up and available in your centralized, governed storage system without any manual intervention.

How It Works

The workflow is built on Make.com and operates on a simple trigger-action principle, with built-in logic for reliability.

Step 1: Monitoring Google Drive

The scenario starts by watching a specific folder you designate in Google Drive. It uses Make.com's Google Drive module to poll for new files or updates at regular intervals. You can set it to watch for any new file, or filter for specific types like documents, images, or spreadsheets.

Step 2: Downloading & Processing the File

Once a new file is detected, Make.com temporarily downloads it to its secure, ephemeral server environment. During this step, you can add logic—like renaming the file with a timestamp, checking its size, or parsing metadata—to prepare it for its new home in FileCloud.

Step 3: Uploading to FileCloud

The processed file is then uploaded to your specified FileCloud account. The automation authenticates securely with FileCloud's API and places the file in the target folder you've configured. It can also create subfolders based on date or file type automatically.

Pro tip: Use a dedicated "To-FileCloud" folder in Google Drive as the trigger. This gives users clear control over what gets synced and prevents accidental transfers of draft or temporary files.

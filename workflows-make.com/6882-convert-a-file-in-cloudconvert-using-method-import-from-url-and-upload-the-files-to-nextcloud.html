What This Workflow Does

Manually downloading files from a list, converting their format, and then uploading them to a secure storage location is a tedious, error-prone, and time-consuming task. Whether you're processing client documents, converting media for marketing campaigns, or archiving reports, this manual cycle eats into productive hours and creates bottlenecks.

This Make.com workflow solves that by creating a fully automated pipeline. It connects three powerful tools: Google Sheets as your source list, CloudConvert as your conversion engine, and Nextcloud as your secure destination. You simply maintain a spreadsheet with file URLs and desired output formats. The automation handles the rest—downloading, converting, and organizing the finished files—without any manual intervention.

The business value is clear: it transforms a multi-step, clerical process into a reliable background operation. This eliminates waiting time, ensures consistency in file handling, and allows your team to focus on analysis, creativity, or client service instead of file management.

How It Works

The workflow operates on a trigger-based schedule, executing a precise sequence of actions for each file in your list.

Step 1: Read the Source List from Google Sheets

The scenario starts by connecting to your specified Google Sheet. It reads rows that contain essential data: the public URL of the source file and the target format you want it converted to (e.g., PDF, JPG, MP3). This sheet is your single point of control for the entire batch process.

Step 2: Import and Convert the File via CloudConvert

For each row, the workflow sends the file URL to CloudConvert's API using the "import from URL" method. CloudConvert fetches the file, performs the conversion to your specified format with high fidelity, and makes the converted file available for download. This step leverages CloudConvert's vast support for over 200 file formats.

Pro tip: Use this workflow to create standardized archives. For instance, automatically convert all incoming .DOCX contracts to searchable PDFs and store them in a dated Nextcloud folder for easy compliance and retrieval.

Step 3: Upload the Converted File to Nextcloud

Once conversion is complete, the workflow retrieves the converted file from CloudConvert and uploads it directly to a designated folder in your Nextcloud instance. You can configure the folder structure dynamically—using data from your Google Sheet like client name or date—to keep your cloud storage perfectly organized from the start.

Step 4: Log Completion and Handle Errors

The final module can update your Google Sheet to mark the row as processed, log the download link from Nextcloud, or send a notification. Robust error handling ensures that if a conversion fails (e.g., due to an invalid URL), the workflow can skip that item, log the error, and continue with the next, ensuring the entire batch isn't compromised.

What You'll Need

To implement this workflow, you'll need active accounts and basic configuration for the following services:

  1. A Make.com account (free tier available).
  2. A Google Sheets document with at least two columns: one for source file URLs and one for the target output format (e.g., 'pdf', 'jpg').
  3. An API key from a CloudConvert account. Their paid plans offer higher conversion limits and faster processing.
  4. A Nextcloud instance (self-hosted or through a provider) with login credentials. You'll need the server URL, username, and password/app password for API access.

Quick Setup Guide

Follow these steps to get your automated file pipeline running in under 30 minutes.

  1. Clone the Template: Click "Get This Workflow" and duplicate the scenario into your Make.com account.
  2. Connect Google Sheets: In the first module, authorize Make.com to access your Google account and select the specific Sheet and worksheet containing your file list.
  3. Configure CloudConvert: Add your CloudConvert API key to the respective module. Test the connection with a sample URL and format.
  4. Set Up Nextcloud: In the upload module, enter your Nextcloud server base URL, folder path, and login credentials. Use an "App Password" for more secure API access than your main password.
  5. Test and Schedule: Run the scenario once with a single test row to ensure the file downloads, converts, and uploads correctly. Then, set the schedule module to run hourly, daily, or on-demand.

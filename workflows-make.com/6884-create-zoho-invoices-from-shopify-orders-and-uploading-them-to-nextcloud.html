What This Workflow Does

For e-commerce businesses, the gap between receiving an order and issuing an invoice is filled with manual, error-prone work. Someone must log into Shopify, copy customer details, line items, and totals, then manually create a duplicate invoice in Zoho Books or Invoice. This process delays billing, risks data entry mistakes, and leaves invoice PDFs scattered across emails or local drives.

This automation solves that by creating a seamless bridge between your sales platform and your accounting system. Every time a new order is placed on Shopify, this Make.com workflow instantly triggers. It captures all order data, transforms it into a properly formatted invoice within Zoho Invoice, and then takes the final generated PDF and uploads it to a designated folder in your Nextcloud storage. The result is a fully automated, accurate, and secure invoice lifecycle from sale to record-keeping.

How It Works

The workflow operates in a clear, sequential chain, ensuring data integrity at each step.

Step 1: Shopify Order Trigger

The automation starts the moment a new order is created in your Shopify store. Make.com watches for this event using a webhook or scheduled check, capturing the complete order payload including customer information, shipping address, products, quantities, prices, taxes, and discounts.

Step 2: Data Preparation & Mapping

The raw Shopify data is parsed and formatted to match the expected fields in Zoho Invoice. This step involves extracting the customer's email for billing, calculating line totals, applying the correct tax settings, and ensuring the invoice date and due date are set appropriately.

Pro tip: Use this stage to add custom logic, like applying specific payment terms based on customer tags or excluding test orders from invoicing.

Step 3: Invoice Creation in Zoho

The formatted data is sent to Zoho Invoice via its API to create a new draft or final invoice. The workflow maps the Shopify items to existing items in your Zoho product list, applies taxes, and adds the customer to Zoho if they're a new contact. A professional invoice is generated within your Zoho account.

Step 4: PDF Generation & Upload to Nextcloud

Once the invoice is created in Zoho, the workflow retrieves the official PDF version. It then connects to your Nextcloud instance and uploads the PDF file to a pre-defined folder structure (e.g., /Invoices/2024/05/). The file can be named intelligently using the order number and date for easy retrieval.

Quick Setup Guide

Follow these steps to configure the template for your business.

  1. Clone the Template: Click "Get This Workflow" and duplicate the scenario into your Make.com account.
  2. Authorize Connections: In each module (Shopify, Zoho Invoice, Nextcloud), click to add a new connection and authenticate your accounts with the required permissions.
  3. Map Your Data: In the Zoho Invoice creation module, map the Shopify fields (like product title, price, quantity) to the corresponding Zoho Invoice fields. Ensure your Zoho product list is synchronized.
  4. Configure Nextcloud Path: Set the destination folder path in your Nextcloud where invoice PDFs should be saved. You can use dynamic values like [Order Date] to create organized subfolders.
  5. Test with a Live Order: Run the scenario once manually with a recent Shopify order to verify an invoice is created and uploaded correctly. Check the execution history for errors.
  6. Activate the Schedule: Turn on the scheduler or webhook to run the automation in real-time. Start with a low frequency for testing before moving to instant execution.

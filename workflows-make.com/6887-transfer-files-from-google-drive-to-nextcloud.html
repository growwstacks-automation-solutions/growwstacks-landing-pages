What This Workflow Does

Businesses and individuals often use multiple cloud storage solutions for different purposes—Google Drive for collaboration and Nextcloud for privacy-focused, self-hosted storage. Manually moving files between these platforms is a repetitive, time-consuming task that's prone to human error. Files get forgotten, versions become mismatched, and team members waste valuable time on administrative busywork instead of productive work.

This automated workflow solves that problem completely. It acts as a digital bridge between Google Drive and Nextcloud. Whenever a new file is added to a designated folder in your Google Drive, the system instantly detects it, securely downloads it, and uploads a copy to a specified folder in your Nextcloud instance. The process is fully automated, running 24/7, ensuring your files are always synchronized across both platforms without any manual intervention.

The value extends beyond simple time savings. It creates a reliable backup system, enhances data governance by keeping sensitive files on your controlled Nextcloud server, and streamlines workflows for teams that operate across different technological preferences. You maintain the convenience of Google Drive's ecosystem while leveraging the security and control of Nextcloud, all seamlessly connected.

How It Works

The automation follows a clean, event-driven sequence. You set it up once, and it runs indefinitely, reacting to new files in real-time.

Step 1: Monitoring Google Drive

The workflow starts by watching a specific folder you choose within Google Drive. Using Make.com's Google Drive module, it polls for changes at regular intervals or uses webhooks for instant notification. When a new file is detected—whether it's a document, spreadsheet, image, or PDF—the automation captures all its details, including the file name, type, size, and URL.

Step 2: Downloading the File

Once a new file is identified, Make.com securely accesses Google Drive using OAuth authentication and downloads the file to its temporary memory. This step happens in the background without requiring you to open any browser or manually initiate a download. The system handles various file formats and sizes, processing them efficiently.

Step 3: Uploading to Nextcloud

The downloaded file is then immediately transferred to your Nextcloud instance. The workflow authenticates with your Nextcloud server via WebDAV or API, navigates to the target destination folder you've predefined, and uploads the file. The original folder structure from Google Drive can be preserved, or files can be organized according to new rules you set.

Pro tip: Use this workflow to create an automatic backup channel. Designate a "Backup" folder in Google Drive, and any file placed there will be duplicated to your secure Nextcloud server, giving you an instant off-Google copy for disaster recovery.

Quick Setup Guide

Configuring this template is straightforward. Follow these steps to activate your automated file bridge.

  1. Clone the Template: Click the "Get This Workflow" button above to open the template in your Make.com account and create a copy (scenario).
  2. Connect Google Drive: In the first module, authorize Make.com to access your Google Drive. Select the specific folder you want to monitor for new files.
  3. Configure Nextcloud: In the upload module, enter your Nextcloud server's base URL and authentication details (usually WebDAV credentials or an App password). Specify the target folder path within Nextcloud.
  4. Set the Schedule: Configure how often the workflow should check Google Drive for changes (e.g., every 15 minutes, hourly, or instantly via webhook if supported).
  5. Test and Activate: Run a single test cycle by placing a sample file in your watched Google Drive folder. Verify it appears correctly in your Nextcloud. Once confirmed, turn the scenario "ON."

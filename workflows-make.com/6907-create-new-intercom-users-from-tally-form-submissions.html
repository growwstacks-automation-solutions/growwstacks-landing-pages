What This Workflow Does

Every business using forms to capture leads faces a critical bottleneck: manual data entry. When a potential customer fills out your Tally form for a demo, contact, or download, that information typically sits in a spreadsheet until someone has time to copy it into your customer communication platform, Intercom. This delay can cost you deals, as response time is a key factor in conversion.

This automation solves that by creating a seamless bridge between Tally and Intercom. It acts as a diligent assistant that checks for new form submissions every 15 minutes (or on your chosen schedule), extracts the respondent's details, and instantly creates a corresponding user profile in Intercom. The moment a lead comes in, they become a known entity in your system, ready for personalized engagement.

How It Works

The workflow is built on Make.com and follows a simple, reliable sequence to ensure no lead is missed.

Step 1: Scheduled Check for New Responses

The scenario is triggered on a schedule. By default, it polls your Tally account every 15 minutes to look for new form submissions. This interval can be easily adjusted to hourly, daily, or in real-time using webhooks for ultra-fast sync.

Step 2: Retrieve and Parse Form Data

For each new submission found, the automation retrieves all the data from the Tally response. This includes the respondent's email, name, and any other custom fields you've set up in your form, such as company, job title, or phone number.

Pro tip: Structure your Tally form fields with clear, simple names (e.g., "Full Name", "Work Email"). This makes mapping the data to Intercom's user attributes much easier and more accurate during setup.

Step 3: Create or Update User in Intercom

The workflow then sends this data to Intercom's API to create a new user. It uses the email as the unique identifier. If a user with that email already exists in Intercom, the automation can be configured to update their profile with the new information instead, preventing duplicates.

Step 4: Log and Route for Action

Finally, the scenario logs the action—whether a user was created or updated—and can be extended to trigger further actions. For example, it could notify a salesperson in Slack, add the user to a specific Intercom segment, or start an automated welcome message sequence.

Quick Setup Guide

You can have this automation running in under 15 minutes.

  1. Clone the Template: Click "Get This Workflow" to copy the template into your Make.com account.
  2. Connect Your Apps: In the Make scenario, create connections to your Tally and Intercom accounts by authorizing with your API keys.
  3. Configure the Trigger: Select the specific Tally form you want to monitor from the dropdown menu in the first module.
  4. Map the Data Fields: In the Intercom "Create a User" module, map the Tally response fields (like `Respondent Email`) to the corresponding Intercom user attributes (like `email`).
  5. Test and Activate: Run the scenario once manually with a test submission to ensure data flows correctly. Then, turn on the schedule and let it run automatically.

What This Workflow Does

Running viral contests or giveaways with UpViral is a fantastic way to grow your audience, but the real challenge begins after the entry form is submitted. Manually exporting lists from UpViral and importing them into your email marketing platform like Sendlane is a tedious, error-prone process that can take hours each week. Worse, delays mean you miss the critical window to engage a warm lead.

This automation solves that problem entirely. It acts as a silent bridge between UpViral and Sendlane, working in the background to check for new contacts every 15 minutes (or on your custom schedule). When it finds a new lead, it instantly transfers their information—name, email, and any custom contest data—and creates a corresponding contact in your designated Sendlane list. This ensures your nurturing sequences start immediately, turning contest participants into subscribed leads without any manual intervention.

Pro tip: Use this automation to tag contacts in Sendlane based on the specific UpViral campaign they entered. This allows for hyper-targeted follow-up emails, dramatically increasing conversion rates.

How It Works

The workflow is built on Make.com and follows a simple, reliable "trigger-action" logic.

Step 1: Scheduled Check in UpViral

A scheduler module acts as the trigger, set to run at regular intervals (default: every 15 minutes). When it activates, the scenario queries the UpViral API to retrieve a list of all contacts created since the last check.

Step 2: Data Processing & Routing

Each new contact record is processed individually. The workflow maps the relevant fields from UpViral (like email, first name, last name, and custom fields) into a standardized format that Sendlane expects. This step ensures data integrity during the transfer.

Step 3: Create Contact in Sendlane

For each processed lead, the workflow sends a request to the Sendlane API to create a new contact. It adds the contact to a specified list and can also apply tags based on the source campaign. The workflow logs the result of each operation for easy monitoring.

Step 4: Error Handling & Completion

The template includes basic error handling. If the Sendlane API call fails for a contact (e.g., due to a duplicate email), the error is caught and logged, allowing the workflow to continue processing the remaining contacts without stopping. A final module marks the cycle as complete, ready for the next scheduled run.

Quick Setup Guide

You can have this automation running in under 10 minutes.

  1. Clone the Template: Click "Get This Workflow" to open the template on Make.com. Press the "Copy" button to create your own scenario.
  2. Connect Your Apps: In your new scenario, click on the UpViral module and add your UpViral API key to create a connection. Repeat the process for the Sendlane module using your Sendlane API key and Account Hash.
  3. Configure the Trigger: In the Schedule module, set your preferred interval (e.g., every 15 minutes, hourly, or daily).
  4. Map Your Data: In the Sendlane "Create a Contact" module, specify the target List ID. Ensure the field mapping (like `email` and `first_name`) correctly pulls from the UpViral data.
  5. Test & Activate: Run the scenario once manually to test. Check your Sendlane account to confirm a test contact was added successfully. Once verified, toggle the scenario to "ON" to let it run automatically.

Pro tip: For a bulletproof setup, first add a "Search Contact" step in Sendlane before the "Create Contact" step. This prevents duplicates by checking if the email already exists in your list.

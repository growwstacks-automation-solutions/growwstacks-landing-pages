Automate Phone Number Lookup from SeaTable with Data24

What This Workflow Does

Incomplete contact data is a major bottleneck for sales and outreach teams. When a new lead or contact is added to your SeaTable database, you often have a name and address but lack a direct phone number. Manually searching for this information is tedious, time-consuming, and prone to errors, delaying follow-up and potentially losing business opportunities.

This automated workflow solves that problem by connecting SeaTable to Data24-7, a professional data enrichment service. Every time a new record with the required fields (First Name, Last Name, Address, City, State, Zip Code) appears in your designated SeaTable base, Make.com automatically triggers a search. It sends this information to Data24's API, retrieves the associated phone number, and updates the original SeaTable record—all within seconds, without any human intervention.

The result is a continuously enriched database where your team has actionable phone numbers ready for calls, SMS campaigns, or support outreach immediately after a contact is added. It turns a passive data storage system into an active, intelligence-gathering tool.

How It Works

The automation follows a clear, reliable sequence to ensure data flows seamlessly from detection to enrichment.

Step 1: SeaTable Trigger

The workflow is initiated by the "Watch Records" module in Make.com, which monitors your specified SeaTable base for new entries. The moment a row is added that contains data in the critical columns, the automation captures the entire record and passes it to the next step.

Pro tip: Ensure your SeaTable column names exactly match what the workflow expects (First Name, Last Name, etc.). Use SeaTable's "Single Line Text" column type for best compatibility.

Step 2: Data Preparation & API Call

Make.com formats the captured data—concatenating the address fields into a standard format—and prepares the API request for Data24-7. It sends the person's full name and complete address to Data24's "Find Person" or similar endpoint, requesting a phone number in return.

Step 3: Phone Number Retrieval & Processing

Data24's service searches its extensive databases and returns a response. The workflow includes a router to handle different outcomes: a successful phone number find, a "no result" response, or an API error. This ensures robust error handling so the workflow doesn't fail unexpectedly.

Step 4: Update SeaTable Record

If a phone number is successfully retrieved, Make.com uses the "Update a Record" module to write that number back into a new column (e.g., "Phone Number" or "Data24 Phone") in the original SeaTable row. The record is now complete and ready for use.

What You'll Need

To implement this template successfully, you'll need to have the following accounts and configurations ready:

  1. A Make.com account (free or paid plan).
  2. A SeaTable account with a base containing your contact data.
  3. An active Data24-7 API subscription or account with available credits for lookups.
  4. Your SeaTable base must have columns named: First Name, Last Name, Address, City, State, Zip Code. You can add an empty column like "Phone Number" for the results.
  5. Your API keys for both SeaTable (for Make.com to connect) and Data24-7 securely stored and ready for input during setup.

Quick Setup Guide

Follow these steps to get your automation live in under 15 minutes.

  1. Clone the Template: Click "Get This Workflow" and duplicate the template into your own Make.com scenario.
  2. Connect Your Apps: In the scenario, update the connections for the SeaTable and Data24 modules with your respective API keys or login credentials.
  3. Configure the Trigger: Point the "Watch Records" module to your specific SeaTable base and table. Test it by adding a dummy record to ensure it triggers.
  4. Map Data Fields: Verify that the data mapping between SeaTable columns and the Data24 API request is correct. The address should be formatted as a single string.
  5. Set Up the Update: Specify which column in your SeaTable should receive the found phone number. You may want to create a new column for this purpose.
  6. Test with a Single Record: Run the scenario once manually with a known contact to verify the phone number is fetched and written back correctly.
  7. Activate and Monitor: Turn on the scenario's scheduler and monitor the operations for the first few hours to ensure it runs smoothly.

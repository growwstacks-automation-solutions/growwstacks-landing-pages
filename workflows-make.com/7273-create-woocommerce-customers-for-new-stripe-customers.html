What This Workflow Does

If you run an online store with WooCommerce and accept payments via Stripe, you likely face a common operational headache: customer data living in two separate systems. When someone makes their first purchase, Stripe captures their payment details, but WooCommerce might not have a complete customer profile unless they created a store account. This disconnect leads to manual work, data inconsistencies, and missed marketing opportunities.

This free Make.com workflow solves that by creating a seamless, automatic bridge between Stripe and WooCommerce. Every time a new customer is created in your Stripe account—whether from a subscription sign-up, a one-time payment, or a saved payment method—this automation instantly creates a corresponding customer profile in your WooCommerce store. It maps key details like name, email, and address, ensuring your e-commerce platform has a complete record of everyone who has ever paid you.

The result is a unified customer database. Your marketing teams can segment and email customers from WooCommerce with confidence. Your support team can see a full history. And you save countless hours previously spent copying and pasting data between tabs, all while eliminating typos and oversights.

How It Works

The automation is triggered by a specific event in Stripe and executes a series of actions in Make.com to ensure data flows accurately to WooCommerce.

Step 1: Trigger on New Stripe Customer

The workflow starts by monitoring your Stripe account via a webhook. The moment Stripe's "customer.created" event fires—signaling a new customer—the webhook captures the entire customer object and sends it to Make.com as a trigger.

Step 2: Parse and Map Customer Data

Make.com receives the webhook payload containing the new customer's details. The scenario parses this data, extracting the essential fields needed for WooCommerce: first name, last name, email address, and billing information (address, city, country, etc.).

Pro tip: Before running the scenario for the first time, trigger the "New customer created" webhook in Make.com. This allows the platform to learn the data structure and lets you map the correct parameters from Stripe to WooCommerce fields.

Step 3: Create Customer in WooCommerce

Using the WooCommerce module in Make.com, the workflow sends a "Create a customer" API call to your store. It passes the mapped data from Stripe, automatically generating a new user account in WooCommerce with the same email, name, and billing address. The workflow assumes billing and shipping addresses match for simplicity, which covers the majority of e-commerce transactions.

Step 4: Error Handling & Logging (Optional)

Advanced setups can include error handling. If the WooCommerce API call fails (e.g., duplicate email), the workflow can route to an alternative path, perhaps updating an existing customer or sending an alert to your team via Slack or email, ensuring no data is lost.

Quick Setup Guide

You can have this automation running in under 30 minutes by following these steps.

  1. Get the Template: Click the "Get This Workflow" button above to clone the template into your Make.com account.
  2. Connect Stripe: In the first module, click to add your Stripe connection. You'll be prompted to enter your Stripe Secret Key, which you can find in the Stripe Developer Dashboard under API keys.
  3. Set Up the Stripe Webhook: Make.com will provide a webhook URL. Copy it, go to your Stripe Dashboard → Developers → Webhooks, and add a new endpoint. Select the customer.created event and paste the URL.
  4. Connect WooCommerce: In the WooCommerce module, add a new connection using your store URL, Consumer Key, and Consumer Secret.
  5. Map the Data Fields: Run the scenario once manually to trigger the data mapping interface. Map Stripe fields (like customer.email) to the corresponding WooCommerce customer fields.
  6. Test and Activate: Create a test customer in Stripe or use the "Test webhook" feature. Verify the customer appears in your WooCommerce Users list. Once confirmed, activate the scenario to run automatically.

Create a MantisBT Issue from Gmail

Automatically convert customer support emails into structured bug reports in your development tracker, eliminating manual entry and ensuring every issue is logged.

What This Workflow Does

For software teams, customer feedback and bug reports often arrive via email. Manually copying these emails into your bug tracking system like MantisBT is a tedious, error-prone task that slows down your development cycle. This workflow solves that by creating a direct, automated bridge between Gmail and MantisBT.

Every time a new email arrives in a designated folder (like "Support" or "Bug Reports"), this automation instantly creates a corresponding issue in MantisBT. It captures the email subject, body, sender information, and even attachments, structuring them into a proper bug ticket. This ensures no customer issue is missed, your support team can focus on communication, and your developers receive actionable, tracked tasks without delay.

The automation acts as a silent, efficient coordinator between your customer-facing channels and your internal development process. It transforms unstructured email conversations into structured, prioritized work items, dramatically improving response times and the overall quality of your software maintenance.

How It Works

The workflow is built on Make.com and operates on a simple trigger-action principle. It listens for new emails and then performs a series of actions to create a detailed bug report.

Step 1: Monitor a Gmail Folder

The scenario starts by watching a specific folder or label in your Gmail account. You can configure it to trigger for all new emails or filter based on sender, subject keywords, or labels to capture only relevant bug reports.

Step 2: Extract Email Content

When a new email arrives, the workflow extracts all relevant data: the subject line, the full body text, the sender's name and email address, the timestamp, and any file attachments. This data is parsed and prepared for the next step.

Step 3: Create a MantisBT Issue

The core action: the workflow uses the MantisBT API module in Make.com to create a new issue. It maps the extracted email data to MantisBT fields. The subject becomes the issue title, the body becomes the description, and the sender's email is added for reference. You can also set default values for project, category, priority, and assignee.

Step 4: Handle Attachments & Notifications

If the email contained attachments (like screenshots or log files), the workflow can upload them directly to the newly created MantisBT issue. Optionally, you can add further steps to notify your team via Slack or email that a new bug has been logged, closing the loop instantly.

Pro tip: Use Gmail filters to pre-sort incoming emails into a dedicated "Bug Reports" label. This makes the automation more reliable and keeps your general inbox clean.

What You'll Need

To set up this workflow, you'll need the following accounts and access:

  1. A Make.com account (free or paid plan).
  2. A Gmail account with access to the inbox or specific folder you want to monitor.
  3. A MantisBT instance with API access enabled. You'll need the base URL of your MantisBT installation and an API token for authentication.
  4. Basic understanding of your MantisBT project structure (project ID, categories, etc.) to map the incoming emails correctly.

Quick Setup Guide

Getting this automation running is straightforward. Follow these steps after importing the template into your Make.com account:

  1. Connect Your Gmail: In the first module, authenticate your Gmail account and select the specific folder or label you want the workflow to watch.
  2. Configure MantisBT Connection: In the MantisBT module, enter your instance's base URL and generate/input your API token. Test the connection to ensure it works.
  3. Map the Data Fields: Define how email data translates to MantisBT fields. Set the default project, category, and priority. You can use the email subject for the title and the body for the description.
  4. Set Up Filtering (Optional): Add a filter module between Gmail and MantisBT if you only want to process emails from certain senders or containing specific keywords.
  5. Test the Flow: Send a test email to the monitored folder and run the scenario once to verify a MantisBT issue is created correctly with all the details.
  6. Activate & Schedule: Turn on the scenario and set its schedule (e.g., run every 5 minutes) to process emails continuously.

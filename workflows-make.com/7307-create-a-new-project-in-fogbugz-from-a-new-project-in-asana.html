What This Workflow Does

For teams using Asana for high-level project planning and FogBugz for detailed development tracking, a significant bottleneck exists: the manual handoff. When a new project is greenlit in Asana, someone must manually recreate it in FogBugz—copying names, descriptions, dates, and owners. This process is not only time-consuming but also prone to errors and delays, leaving development teams waiting for setup while project managers juggle administrative tasks.

This automation solves that disconnect. It acts as a seamless bridge between your planning and execution platforms. Whenever a new project is created in a specified Asana workspace, this workflow automatically detects it and creates a corresponding project in FogBugz. It transfers all essential information, ensuring the development team can immediately begin breaking down tasks, logging bugs, and tracking time against the correct project framework. The result is a synchronized workflow where project initiation triggers instant readiness for execution.

The template is designed to run on a schedule (every 15 minutes by default), providing near-real-time syncing without overloading the APIs. This "set-and-forget" system eliminates a recurring manual task, freeing up several hours per month for project managers and ensuring developers always have the latest project structures to work within.

How It Works

The workflow follows a simple, reliable trigger-action sequence to keep data flowing from Asana to FogBugz.

Step 1: Scheduled Check for New Asana Projects

The automation begins with a scheduled trigger from Make.com. At your chosen interval, it queries your Asana workspace, specifically looking for projects created since the last check. It intelligently filters and retrieves only the new projects, capturing key details like the project name, description, notes, owner, and custom fields.

Pro tip: Configure the schedule based on your team's velocity. For fast-moving teams, a 15-minute check is ideal. For less frequent project creation, a daily check might suffice and reduce API calls.

Step 2: Data Mapping and Preparation

Once a new project is detected, the workflow maps the Asana data fields to the required format for FogBugz. This step is crucial for ensuring information is placed correctly. You can customize this mapping to include specific Asana custom fields or notes that your development team needs to see in FogBugz.

Step 3: Create the Project in FogBugz

The prepared data is then sent to FogBugz via its API. The workflow uses your FogBugz connection to authenticate and execute the "Create Project" command, populating the new project with the transferred information. A successful creation is confirmed, and the project becomes immediately active for your development team.

Step 4: Error Handling & Logging (Optional)

The template includes basic error handling. If the FogBugz creation fails—perhaps due to a duplicate name or API issue—the workflow can be extended to send an alert (e.g., via email or Slack) to an administrator. This ensures no project falls through the cracks due to a technical glitch.

What You'll Need

To implement this workflow, ensure you have the following prerequisites:

  1. A Make.com account (free or paid tier).
  2. An Asana account with administrative or member access to the workspace and projects you wish to sync.
  3. A FogBugz account with API access and permissions to create new projects.
  4. The API keys or OAuth credentials for both Asana and FogBugz to connect them to Make.com.
  5. Basic knowledge of the project field structure in both tools to configure the data mapping correctly.

Quick Setup Guide

You can have this automation running in under 20 minutes by following these steps:

  1. Get the Template: Click the "Get This Workflow" button to copy the template to your Make.com account.
  2. Connect Your Apps: In the Make.com scenario editor, establish connections to your Asana and FogBugz accounts by authorizing Make.com to access them.
  3. Configure the Trigger: In the first module (the Asana "Watch Projects" trigger), select the specific Asana workspace you want to monitor for new projects.
  4. Map the Data: In the FogBugz "Create a Project" module, map the fields from Asana to the corresponding FogBugz project properties (e.g., map Asana project name to FogBugz project title).
  5. Set the Schedule: Adjust the schedule icon on the first module to run as frequently as needed (default is every 15 minutes).
  6. Test and Activate: Run a single test cycle by creating a dummy project in Asana. Verify it appears correctly in FogBugz. Once confirmed, turn the scenario on.

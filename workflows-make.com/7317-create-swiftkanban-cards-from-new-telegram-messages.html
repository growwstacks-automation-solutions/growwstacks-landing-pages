What This Workflow Does

In today's fast-paced work environment, critical tasks and ideas are often born in instant messaging platforms like Telegram. The problem is that these action items get buried in chat history, leading to missed deadlines, forgotten requests, and frustrated teams. Manually copying information from chats to your project management board is a tedious, error-prone process that wastes valuable time.

This automation solves that by creating a seamless bridge between communication and execution. Every time a new message arrives in a designated Telegram chat or channel, this workflow instantly captures it and creates a corresponding card in your SwiftKanban board. It transforms informal conversations into structured, trackable tasks with a clear owner and status, right where your team manages work.

The result is a dramatic reduction in administrative overhead. Project managers, team leads, and support staff no longer need to play the middleman. The moment a request is made or an idea is shared, it becomes a visible item on the Kanban board, ready to be actioned. This ensures accountability, improves response times, and keeps your project's momentum going.

How It Works

The workflow is a simple, reliable sequence that operates 24/7, monitoring your chats and updating your Kanban board without any manual intervention.

Step 1: Monitor Telegram for New Messages

The automation starts by watching a specific Telegram group, channel, or private chat. Using the Telegram Bot API, it listens for new messages in real-time. You can configure it to watch multiple sources or filter messages based on keywords, senders, or message types.

Step 2: Extract and Format Task Data

When a new message is detected, the workflow extracts key information: the message text, the sender's name, the timestamp, and any attached files. It then formats this data to fit SwiftKanban's card structure, preparing the title, description, and other custom fields.

Step 3: Create a Card in SwiftKanban

The formatted data is sent to SwiftKanban via its API. The automation creates a new card on your specified board and lane. You can pre-set rules to assign the card to a particular team member, apply labels, set a due date, or link it to a parent task, making it ready for immediate work.

Pro tip: Use Telegram hashtags (e.g., #bug, #feature) in your messages. Configure the automation to read these hashtags and automatically assign the new SwiftKanban card to the appropriate lane (like "Bugs" or "Backlog"), adding an extra layer of intelligence.

What You'll Need

To set up this automation, you will need the following accounts and access:

  1. A Make.com account (free tier available).
  2. A Telegram account and access to the chat/channel you want to monitor.
  3. A Telegram Bot Token, which you can get by creating a new bot via @BotFather on Telegram.
  4. A SwiftKanban account with an existing board where you want cards to be created.
  5. Your SwiftKanban API Key, found in your account settings.

Quick Setup Guide

Follow these steps to get your automation running in under 15 minutes:

  1. Get the Template: Click "Get This Workflow" to copy the template into your Make.com account.
  2. Connect Telegram: In the first module, add your Telegram Bot Token. Test the connection and select the chat or channel to watch.
  3. Connect SwiftKanban: In the second module, add your SwiftKanban API Key. Select the target Board and Lane where new cards should appear.
  4. Map the Data: Configure the workflow to map the Telegram message text to the SwiftKanban card title/description. You can also map the sender's name or add static values like labels.
  5. Activate & Test: Turn the scenario on. Send a test message to your Telegram chat. Within seconds, check your SwiftKanban board to see the new card created automatically.

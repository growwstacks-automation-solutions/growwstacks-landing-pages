What This Workflow Does

For businesses using SignalWire for SMS communications—be it for customer support, appointment reminders, or system alerts—a major challenge is ensuring these messages are seen and acted upon promptly. Relying on a single person to monitor an SMS inbox leads to delays, missed messages, and frustrated customers. This workflow solves that by creating a seamless bridge between SignalWire and Telegram.

Every time a new SMS arrives in your designated SignalWire number, this automation instantly forwards the full message content—including the sender's number and timestamp—to a specified Telegram chat, group, or channel. This transforms individual SMS into collaborative, real-time notifications that your entire team can see and respond to from their mobile devices, dramatically improving response times and operational visibility.

How It Works

The automation is a simple yet powerful chain reaction that happens in the background, requiring zero manual intervention after setup.

Step 1: Trigger on New SignalWire Message

The Make.com scenario uses the SignalWire module to watch for new incoming messages to your specified phone number. It acts as a vigilant listener, checking at regular intervals for any new activity.

Step 2: Parse and Format the Message Data

Once a message is detected, the workflow extracts key details: the sender's phone number, the message body, and the time it was received. It can also optionally look up a contact name if the number exists in a connected database.

Step 3: Send to Telegram

The formatted information is then sent via the Telegram Bot API to your pre-configured Telegram destination. The notification appears as a clear message in the chat, often with the sender's number highlighted for easy identification.

Pro tip: You can customize the message format in Telegram. Add prefixes like "[Support]" or "[Alert]" to categorize messages at a glance, making it easier for your team to prioritize responses.

What You'll Need

To set up this workflow, you'll need active accounts and a few pieces of configuration data:

  1. A Make.com account (free tier is sufficient to start).
  2. A SignalWire account with a phone number capable of receiving SMS.
  3. Your SignalWire Space URL, Project ID, and API Token (found in your SignalWire dashboard).
  4. A Telegram account and a Telegram Bot created via @BotFather.
  5. The Bot Token from @BotFather and the Chat ID of the Telegram group or channel where messages should be sent.

Quick Setup Guide

Follow these steps to deploy the template in under 10 minutes:

  1. Clone the Template: Click "Get This Workflow" to copy the scenario to your Make.com account.
  2. Connect SignalWire: In the first module, add your SignalWire connection using your Space URL and API Token. Select the phone number you want to monitor.
  3. Configure Telegram: In the Telegram module, add your Bot Token. Paste the numeric Chat ID of your target group or channel. You can get this ID using tools like @getidsbot.
  4. Test the Flow: Send a test SMS to your SignalWire number. You should see it appear in your Telegram chat within seconds.
  5. Activate & Schedule: Turn the scenario ON. Set the schedule (e.g., check for new messages every minute) based on your expected volume.

What This Workflow Does

In today's hybrid work environment, teams often use Slack for internal coordination while managing communities, clients, or contractors on Telegram. This fragmentation creates a communication silo, forcing someone to manually copy-paste messages, updates, or support requests between platforms. This is inefficient, error-prone, and causes delays in response times.

This automation template solves that by creating a seamless, two-way bridge between Slack and Telegram. It listens for new messages in specified channels or groups on one platform and automatically posts them to the other. Whether it's a customer question in a Telegram group that needs internal team attention on Slack, or an important company announcement in Slack that must reach a public Telegram community, the workflow handles it instantly and reliably.

The business value is clear: it eliminates manual relay work, ensures consistent information flow across all stakeholders, and accelerates decision-making. Your team stays in sync without ever having to switch contexts or manage multiple inboxes manually.

How It Works

The workflow uses Make.com to act as an intelligent router between Slack and Telegram's APIs.

Step 1: Trigger on a New Message

The scenario starts by watching a specific Slack channel or a Telegram group for new messages. You configure the trigger to monitor the exact source where communication originates.

Step 2: Process the Message Content

When a new message arrives, the automation captures key details: the sender's name, the message text, any attachments, and timestamp. It can apply simple filters at this stage, such as checking for specific keywords or ignoring messages from certain users.

Pro tip: Use filters to prevent spam or internal-only messages from being broadcast externally. For example, only forward messages containing #announce or #alert.

Step 3: Format and Send to the Destination

The workflow then formats the message appropriately for the destination platform. It might add a prefix like "[From Slack]" or include the original sender's name for context. Finally, it uses the respective bot API to post the formatted message into the designated destination channel or group on the other platform.

This process runs 24/7, creating a real-time, automated link between your team's internal and external communication hubs.

Quick Setup Guide

Follow these steps to deploy your Slack-Telegram bridge in under 30 minutes:

  1. Clone the Template: Click "Get This Workflow" to copy the template into your Make.com account.
  2. Create Your Bots: In Slack, create a new app via api.slack.com, install it to your workspace, and note the "Bot User OAuth Token". In Telegram, message @BotFather to create a new bot and save the HTTP API token.
  3. Configure Connections: In the Make.com scenario, add your Slack and Telegram accounts using the tokens you obtained.
  4. Set Your Channels: In the trigger and action modules, select the exact Slack channel and Telegram chat ID you want to connect. You can set up multiple routes for different channel pairs.
  5. Add Filters (Optional): Use the router or filter module to define rules, like only syncing messages with specific keywords or from certain users.
  6. Test and Activate: Turn on the scenario, send a test message in Slack, and verify it appears in Telegram (and vice-versa). Once confirmed, your automation is live.

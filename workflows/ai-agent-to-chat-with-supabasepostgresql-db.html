What This Workflow Does

This workflow solves a critical business problem: data trapped in databases because team members lack SQL expertise. It creates a conversational AI agent that connects OpenAI with your Supabase or PostgreSQL database, allowing anyone in your organization to ask questions in plain English and get instant answers.

The agent dynamically generates SQL queries based on user requests, retrieves records, analyzes JSON data, and provides summaries or specific data points. It eliminates the bottleneck of waiting for technical staff to write queries, turning days of delay into seconds of response time.

Imagine your marketing team asking "Which campaign generated the most leads last quarter?" or operations asking "What's our current inventory status for top-selling products?"—and getting immediate, accurate answers without writing a single line of SQL.

How It Works

1. User Query Input

The workflow starts when a user submits a natural language question through a chat interface, webhook, or integrated platform like Slack. The question is passed to the AI agent for processing.

2. Database Schema Analysis

The agent first examines your database structure, understanding tables, columns, data types, and relationships. This context allows it to map user questions to the correct database elements.

3. SQL Query Generation

Using OpenAI's capabilities, the agent translates the natural language question into an optimized SQL query. It considers joins, filters, aggregations, and sorting to produce accurate results.

4. Query Execution & Validation

The generated SQL is executed against your database with safety checks in place. The workflow includes read-only permissions, query timeouts, and result limits to protect database performance.

5. Response Formatting

Raw database results are transformed into human-readable formats—tables, summaries, charts, or plain text explanations—and delivered back to the user through the original interface.

Quick Setup Guide

Follow these steps to implement this conversational database agent in under 30 minutes:

  1. Download and Import – Click the download button above and import the JSON file into your n8n instance.
  2. Configure Database Connection – Update the PostgreSQL/Supabase node with your database credentials (host, database, username, password).
  3. Add OpenAI API Key – Enter your OpenAI API key in the AI agent node settings.
  4. Set Up Trigger – Configure how users will submit queries (webhook URL for API calls, or connect to Slack/Teams).
  5. Test with Simple Queries – Start with basic questions like "How many users signed up yesterday?" to verify the connection.
  6. Deploy and Share – Activate the workflow and share the interface with your team members.

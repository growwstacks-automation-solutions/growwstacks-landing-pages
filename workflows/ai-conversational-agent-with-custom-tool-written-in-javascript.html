What This Workflow Does

This template demonstrates how to build an AI-powered conversational agent that isn't limited to pre‑defined actions. It includes a custom JavaScript tool that executes specific business logic—in this case, generating a random color while excluding user‑specified colors. This pattern is powerful for creating assistants that can interact with your unique data, perform calculations, or trigger internal processes.

Instead of relying solely on the AI's knowledge, you equip it with tools that bridge the gap to your systems. The agent decides when to use the tool based on the conversation, runs the JavaScript code, and returns a contextual result. This turns a generic chatbot into a capable business assistant that can handle specialized tasks.

For example, you could modify the JavaScript tool to check inventory levels, calculate shipping estimates, validate coupon codes, or fetch customer data from your database—all within a natural conversation flow.

How It Works

The workflow combines n8n's AI Agent node with a Code node to create a dynamic, tool‑enhanced conversation.

Step 1: Conversation Trigger

The workflow starts with a manual trigger (or you can connect it to a webhook, Slack, etc.). The user's message is passed to the AI Agent node, which is configured with a system prompt defining the agent's role and available tools.

Step 2: AI Decision Making

The AI model (like GPT‑4) analyzes the user's request. If the user asks for something that matches the custom tool's description—like "give me a random color, but not blue or red"—the agent decides to invoke the JavaScript tool.

Step 3: Custom Tool Execution

The agent calls the Code node, which contains the JavaScript function. The function receives parameters from the conversation (e.g., excluded colors), runs its logic, and returns a result (e.g., "Here is a random color: Green").

Step 4: Response Delivery

The tool's output is fed back to the AI Agent, which formulates a natural‑language reply incorporating the result. The final response is sent back to the user, completing the interaction.

Pro tip: The real power lies in replacing the example JavaScript with your own business logic. Think of it as giving your AI agent a Swiss Army knife of custom functions.

Quick Setup Guide

Import and configure this template in under 10 minutes:

  1. Download the template using the button above and import it into your n8n workspace.
  2. Configure the AI Agent node with your OpenAI API credentials and adjust the system prompt to match your use case.
  3. Edit the Code node to replace the random‑color function with your own JavaScript logic. This is where you add your business‑specific tool.
  4. Test the conversation via the Manual Trigger node. Ask your agent a question that should trigger the custom tool.
  5. Connect a real trigger like a webhook, Slack bot, or website chat widget to make the agent live.

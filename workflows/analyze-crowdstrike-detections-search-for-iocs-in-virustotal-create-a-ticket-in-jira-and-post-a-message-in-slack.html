What This Workflow Does

This security automation workflow transforms how your organization handles security incidents by creating a seamless pipeline from detection to resolution. When CrowdStrike identifies a potential threat, this n8n workflow automatically springs into action, eliminating manual investigation steps that typically delay response times.

The system fetches recent security detections, enriches each finding with threat intelligence from VirusTotal, creates detailed Jira tickets with all relevant context, and notifies your security team in Slack—all without human intervention. This ensures that every detection is properly documented, investigated according to consistent procedures, and assigned to the right personnel immediately.

By automating this critical security process, you reduce mean time to detection (MTTD) and mean time to response (MTTR), improve compliance with documented procedures, and free your security analysts to focus on complex threats rather than routine data gathering and ticket creation.

How It Works

1. Scheduled Detection Collection

The workflow triggers daily (configurable) to fetch recent security detections from CrowdStrike Falcon via API. It retrieves all new alerts within your specified timeframe, ensuring no potential incident goes unnoticed.

2. IOC Enrichment with VirusTotal

Each detection is automatically checked against VirusTotal's global threat intelligence database. The workflow queries both SHA256 hashes and other Indicators of Compromise (IOCs), gathering reputation scores, detection ratios, and behavioral analysis to provide context for the security team.

3. Intelligent Batching & Processing

Using n8n's Split in Batches node, the workflow processes detections efficiently while respecting API rate limits. A built-in wait node prevents overwhelming external services, ensuring reliable operation even during high-volume security events.

4. Comprehensive Ticket Creation

For each enriched detection, the workflow creates a detailed Jira ticket with severity classification, host information, user context, threat intelligence findings, and direct links to both CrowdStrike and VirusTotal analyses. This creates a complete audit trail from the moment of detection.

5. Team Notification & Alerting

Finally, the workflow posts a formatted message to your designated Slack channel, alerting the security team with actionable information including severity level, affected assets, and direct links to the newly created Jira ticket for immediate investigation.

What You'll Need

  1. CrowdStrike Falcon instance with API access credentials
  2. VirusTotal API key (free tier available with rate limits)
  3. Jira instance with project access and API credentials
  4. Slack workspace with a webhook URL for your security channel
  5. n8n instance (self-hosted or cloud) with HTTP Request, Set, Split in Batches, Wait, and Item Lists nodes available

Pro tip: Start with the free VirusTotal API tier to test the workflow. For production use with high-volume environments, consider upgrading to a paid tier to avoid rate limiting during security incidents.

Auto n8n updater (Docker)

Automate updates for your self-hosted n8n instance. Checks for new versions, summarizes release notes with AI, and requests approval via Telegram.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Keeping a self-hosted n8n instance up-to-date is a critical but often overlooked DevOps task. Manual checks are time-consuming, and missing updates can lead to security vulnerabilities or missed features. This workflow solves that by fully automating the update process for n8n running in Docker.

It acts as a smart update manager: it periodically checks if a new Docker image is available, fetches and summarizes the official release notes using AI, and then sends you a clear approval request via Telegram. You get a concise overview of what’s changing and can approve or ignore the update with a single tap. If approved, the workflow securely executes the Docker commands on your server, pulling the new image and restarting the service, ensuring your automation platform is always current without any manual intervention.

How It Works

Step 1: Scheduled Check & Environment Confirmation

A Schedule Trigger node runs the workflow at a regular interval (e.g., hourly). The workflow first confirms it's operating within a Docker environment to ensure compatibility.

Step 2: Version Discovery & Comparison

Using an SSH connection to your host server, the workflow inspects the running n8n container to identify its current version tag (like 'latest'). It then queries the Docker Hub API to fetch the latest digest (a unique image ID) for that same tag. A comparison determines if an update is available.

Step 3: AI-Powered Release Note Summarization

If an update is detected, the workflow fetches the detailed release notes from GitHub. An AI model (like Google Gemini) processes these technical notes, extracting and summarizing the key features, fixes, and changes into a brief, easy-to-read paragraph.

Step 4: Human Approval via Telegram

The summarized update information is sent to a designated Telegram chat via a bot. The message includes two inline buttons: "✅ Update" and "❌ Ignore". The workflow pauses, waiting for your response. This creates a secure approval gate.

Step 5: Secure Update Execution

Upon receiving an "Update" response, the workflow uses SSH again to run predefined Docker Compose commands on your server (e.g., docker compose pull and docker compose up -d). This pulls the new image and restarts the containers, completing the update cycle.

What You'll Need

  1. A self-hosted n8n instance running via Docker Compose.
  2. SSH access to the host server where n8n is deployed.
  3. A Telegram Bot created via BotFather, with its API token.
  4. Credentials for an AI model provider (Google AI, OpenAI, etc.) integrated into your n8n instance.
  5. Your personal Telegram Chat ID for receiving approval messages.

Pro tip: For fully automated, zero-touch updates, you can modify the workflow to skip the Telegram approval step. Connect the update detection directly to the SSH execution node. Use this only in environments where you fully trust the update process.

Quick Setup Guide

  1. Import the Template: Download the JSON file and import it into your n8n instance.
  2. Configure SSH Credentials: In n8n Credentials, create an SSH credential for your server. Apply it to the "Get n8n Current Version" and "Update Docker" nodes.
  3. Set Up Telegram: Create a Telegram bot, get its token, and add a Telegram credential in n8n. Enter your Chat ID in the "Approve Update" node.
  4. Configure AI Model: Ensure your AI provider credentials are set up. Select them in the "Google Gemini Chat Model" node (or swap it for your preferred LLM node).
  5. Adjust Paths & Commands: In the "Docker Path" node, set the docker_path to your server's Docker Compose file location (e.g., /root/n8n). Adjust the worker_command if you use n8n workers.
  6. Activate & Test: Activate the workflow. It will run on its schedule. Test the approval flow by simulating an update check.

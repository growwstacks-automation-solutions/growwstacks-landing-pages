What This Workflow Does

This automation solves the challenge of being constantly available on Slack while maintaining accurate, on-brand responses. It combines GPT-5's natural language capabilities with your company's Google Docs knowledge base to create responses that sound like you.

The system monitors specified Slack channels for messages directed at you or containing keywords you define. When triggered, it analyzes the message context, retrieves relevant information from your connected Google Docs, and crafts a response that maintains your unique communication style.

How It Works

1. Message Monitoring

The workflow continuously watches designated Slack channels for mentions or keyword triggers, operating 24/7 without fatigue.

2. Context Analysis

When activated, the system evaluates the message content, sender, and conversation history to determine appropriate response parameters.

3. Knowledge Retrieval

The RAG system searches connected Google Docs for relevant information, prioritizing the most current project documentation.

4. Response Generation

GPT-5 synthesizes the retrieved information into a response that matches your typical Slack communication style, including tone, length, and formatting preferences.

5. Quality Control

Optional approval steps can be added to review responses before sending or flag uncertain answers for human follow-up.

What You'll Need

  1. Slack workspace admin access to install the bot
  2. Google Workspace account with document access
  3. GPT-5 API access (or GPT-4 as fallback)
  4. n8n instance (cloud or self-hosted)
  5. 15-30 minutes for initial configuration

Quick Setup Guide

  1. Import the JSON template into your n8n instance
  2. Connect your Slack and Google Docs accounts
  3. Configure trigger keywords and response parameters
  4. Test with non-critical channels first
  5. Gradually expand to more conversations as confidence grows

Pro tip: Start with a "training mode" that logs what the AI would say without actually sending messages, allowing you to refine the system before going live.

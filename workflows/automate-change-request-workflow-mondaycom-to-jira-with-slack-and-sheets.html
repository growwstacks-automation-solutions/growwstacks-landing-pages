Automate Change Request Workflow: Monday.com to Jira with Slack & Sheets

Free n8n template to streamline IT change management with automated approvals, Jira ticket creation, audit logging, and real-time alerts.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manual change request processes create bottlenecks in IT and operations teams. Requests get lost in email, approvals take days, and there's no clear audit trail between request submission and implementation. This disconnect between project management (Monday.com) and development tracking (Jira) leads to miscommunication, delays, and compliance risks.

This automation solves that by creating a seamless pipeline from request to resolution. It automatically monitors Monday.com for new change requests, routes them based on status, creates corresponding Jira tickets for approved items, logs everything to Google Sheets for compliance, and keeps everyone informed via Slack. The result is a transparent, efficient change management process that reduces turnaround time from days to hours.

How It Works

Step 1: Scheduled Trigger & Data Collection

The workflow runs automatically every weekday at 3:00 AM (configurable) to fetch new or updated change requests from your Monday.com board. It extracts key fields including request name, component, risk level, assigned approvers, and description for structured processing.

Step 2: Conditional Routing by Status

Each request is intelligently routed based on its current status. Pending requests trigger Slack alerts to approvers with full context. Approved requests proceed to Jira ticket creation. Rejected requests automatically create resubmission items in Monday.com with feedback.

Step 3: Multi-Platform Execution

For approved requests: Jira tickets are created with all relevant details, Google Sheets logs the approval for audit purposes, and confirmation emails are sent to requesters via Gmail with ticket references. The entire process happens in seconds without manual intervention.

Pro tip: Customize the cron expression (0 3 * * 1-5) to match your team's working hours. Consider running it multiple times daily for time-sensitive changes, or add webhook triggers from Monday.com for instant processing.

Companies with compliance requirements (SOX, HIPAA, ISO) benefit from the automated audit logging to Google Sheets.

What You'll Need

  1. Monday.com account with API access and a board configured for change requests
  2. Jira instance with project permissions to create issues
  3. Google Sheets document for audit logging (OAuth2 credentials)
  4. Slack workspace with a bot token having chat:write permissions
  5. Gmail account or SMTP service for sending confirmation emails
  6. n8n instance (cloud or self-hosted) to run the automation

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Download and import the JSON template into your n8n instance.
  2. Connect your Monday.com account and update the board ID and group ID in the first node.
  3. Configure Jira credentials and set the target project key for new tickets.
  4. Link your Google Sheets document and specify the sheet ID and tab name for logging.
  5. Add Slack credentials and update the channel ID for notifications.
  6. Set up email credentials (Gmail OAuth2 or SMTP) for confirmation messages.
  7. Test with a sample request in Monday.com to verify end-to-end execution.
  8. Activate the workflow and enjoy automated change management.

Pro tip: Start with a test Monday.com board and Jira project to validate the workflow before connecting to production systems. Use n8n's "Execute Workflow" button to test each step individually during setup.

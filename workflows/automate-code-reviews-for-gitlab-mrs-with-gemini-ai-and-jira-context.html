What This Workflow Does

Manual code reviews create bottlenecks in development cycles, especially when reviewers are busy or in different time zones. This automation solves that by providing instant, AI-powered feedback on GitLab merge requests. When a developer comments "ai-review" on an MR, the workflow automatically analyzes the code diff, checks for logic errors, security vulnerabilities, and performance issues, then posts constructive feedback as inline comments.

The system becomes even smarter by integrating with JIRA. If the MR description contains a JIRA ticket ID (like PROJ-123), the workflow fetches the ticket summary and uses it to provide context-aware feedback. This means the AI understands what feature or bug fix is being implemented and can suggest improvements aligned with the actual requirements.

How It Works

1. Trigger & Filter

A webhook listens for GitLab note events (comments on merge requests). When a comment containing the trigger phrase "ai-review" is detected, the workflow activates and captures all relevant MR data including project ID, merge request IID, and the comment details.

2. Fetch & Parse Code Changes

The workflow retrieves the complete merge request details from GitLab's API, including the full diff. It then parses this diff into manageable chunks, separating each file change into "original" and "new" code blocks for clear analysis.

3. Enrich with JIRA Context

If a JIRA ticket key is found in the MR description, the workflow fetches the ticket details from JIRA's API. For subtasks, it also retrieves the parent ticket. This context is summarized into a brief paragraph that helps the AI understand the business requirements behind the code changes.

4. AI Analysis & Review

The parsed code diff and optional JIRA context are sent to an LLM (Google Gemini in this template) with specific instructions to identify logic flaws, security issues, performance bottlenecks, and suggest improvements. The AI focuses on high-value findings—if nothing critical is detected, it returns an "all clear" message instead of nitpicking minor style issues.

5. Post Results Back to GitLab

The AI's feedback is formatted into inline comments and posted back to the GitLab merge request at the appropriate file and line positions. The workflow also posts a status update comment showing when the review was initiated and completed.

What You'll Need

  1. GitLab instance (cloud or self-hosted) with API access
  2. GitLab Personal Access Token with API permissions stored as an environment variable
  3. LLM provider credentials (Google Gemini API key, or alternatively OpenAI, Anthropic, etc.)
  4. Optional: JIRA credentials if you want context-aware reviews
  5. n8n instance (self-hosted or cloud) to run the workflow
  6. Webhook endpoint accessible from your GitLab instance

Pro tip: Start with a small pilot project or specific repository to test the AI feedback quality before rolling out to your entire codebase. Adjust the LLM prompt to match your team's coding standards and priorities.

Quick Setup Guide

  5. Test the workflow by commenting "ai-review" on any merge request in your GitLab project

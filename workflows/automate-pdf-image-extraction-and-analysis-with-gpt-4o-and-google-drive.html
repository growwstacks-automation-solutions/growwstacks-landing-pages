What This Workflow Does

Manually extracting images from PDF files for analysis is a slow, tedious, and error-prone task. Teams often resort to taking screenshots of each page, uploading them individually to an AI tool, and then manually copying the insights into a separate document. This process can take hours for a single multi-page report and doesn't scale.

This n8n workflow completely automates that pipeline. It connects Google Drive, a PDF processing service, and OpenAI's GPT-4o to create a seamless, end-to-end automation. Simply upload a PDF to a designated Google Drive folder, and the workflow will extract every embedded image, send each one to GPT-4o for detailed analysis, and compile all the insights—along with the image URLs—into a neatly organized text file.

The result is a fully automated system that turns visual data trapped in documents into actionable, searchable text insights in minutes, eliminating manual effort and human error.

How It Works

The workflow orchestrates several services to transform a static PDF into analyzed data.

Step 1: Trigger & File Retrieval

The automation can be triggered manually or configured to watch a specific Google Drive folder. When a new PDF is detected, the workflow fetches the file and prepares it for processing.

Step 2: PDF Image Extraction

Using a dedicated service (like ConvertAPI), the workflow parses the PDF document page-by-page, identifying and extracting all embedded images. It handles various image formats and preserves quality.

Step 3: AI-Powered Image Analysis

Each extracted image is sent to the OpenAI GPT-4o model. You can configure the AI prompt to perform specific tasks: describe the image, extract text (OCR), identify objects, analyze charts, or summarize visual content based on your business needs.

Step 4: Data Compilation & Output

The workflow collects the AI-generated analysis for each image, pairs it with the corresponding image URL (often hosted temporarily), and structures the data. Finally, it creates a comprehensive .txt file containing all insights and saves it back to Google Drive or another destination of your choice.

  2. Google Drive account with API access enabled (for the trigger and file storage).

  4. ConvertAPI account & key (or a similar PDF processing service) to handle the image extraction from PDFs.

Pro tip: For sensitive documents, consider using a self-hosted n8n instance and review OpenAI's data usage policies. You can also swap the AI model for a local, on-premises vision model if required by your compliance standards.

Quick Setup Guide

Get this workflow running in your n8n instance in under 10 minutes.

  1. Download the template using the button above and save the JSON file.
  2. In your n8n dashboard, go to Workflows and click Import from file. Select the downloaded JSON.
  3. Configure credentials: Click the credential icons next to the Google Drive, OpenAI, and ConvertAPI nodes. Add your respective API keys and accounts.
  4. Set the trigger: Configure the Google Drive trigger node to watch a specific folder for new PDF files.
  5. Customize the AI prompt: In the OpenAI node, adjust the system and user prompts to tailor the analysis to your specific needs (e.g., "Extract all numerical data from charts" or "Describe the scene in detail").
  6. Test the workflow: Upload a sample PDF to your watched Google Drive folder and click "Execute Workflow" on the trigger node. Check the output in the subsequent nodes to ensure images are extracted and analyzed correctly.
  7. Activate: Once tested, toggle the workflow to "Active". It will now run automatically for every new PDF.

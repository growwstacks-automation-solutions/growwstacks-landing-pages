Automate Security Incident Triage with GPT‑4o‑mini & Gmail

Free n8n workflow template: Automatically triage AWS Security Hub findings, classify severity with AI, generate remediation plans, and send Gmail alerts.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Security teams are overwhelmed with alerts. AWS Security Hub, EventBridge, SNS, and other monitoring tools generate hundreds of findings daily. Manually reviewing each one is slow, error‑prone, and drains engineering time.

This n8n workflow automates the entire triage process. It receives security findings via a webhook, normalizes the data, uses GPT‑4o‑mini to classify the incident type and severity, generates a concise 3‑step remediation plan, and sends a structured email alert via Gmail. The result is a prioritized, actionable incident brief delivered to the right team within seconds.

The pipeline is modular: Webhook → Clean Finding → Classify (LLM) → Plan (LLM) → Gmail. You can swap Gmail for Slack, Teams, or any other notification channel.

How It Works

Step 1: Webhook Trigger

AWS Security Hub (or any security tool) sends a JSON finding via EventBridge → SNS → Webhook to your n8n instance. The webhook node captures the raw alert payload.

Step 2: Data Normalization

A “Set” node extracts and standardizes key fields: title, description, AWS account ID, resource ID/type, and timestamp. This ensures consistent input for the AI model.

Step 3: AI Classification

The GPT‑4o‑mini node analyzes the normalized finding. It assigns an incident_type (e.g., “Unauthorized Access,” “Misconfigured Resource”), a severity level (P0‑P3), urgency, a short_title, and a why rationale—all based on the context of the alert.

Step 4: Remediation Plan Generation

A second LLM call creates a practical 3‑step remediation plan. Each step includes clear actions, an owner_hint (which team should handle it), and success_criteria to verify resolution.

Step 5: Gmail Notification

The workflow compiles everything into a clean HTML email. The subject line includes the short title, resource ID, and account. The body presents the type, account, urgency, rationale, next actions, owner hint, and success criteria—ready for immediate action.

What You'll Need

  1. An n8n instance (cloud or self‑hosted) reachable via HTTP for the webhook.
  2. OpenAI API credentials (or a compatible LLM service) configured in n8n.
  3. Gmail OAuth2 credentials (or SMTP settings for another email provider).
  4. A source that can POST security findings in JSON format (AWS Security Hub via EventBridge/SNS, a custom SIEM, or even a curl script for testing).
  5. Basic understanding of n8n node configuration—no deep coding required.

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n workspace.
  2. Configure the Webhook node: set a unique path and ensure your security tool can send POST requests to it.
  3. Add your OpenAI API key in the LLM nodes (GPT‑4o‑mini). Adjust the prompt if you want different classification criteria.
  4. Set up Gmail OAuth2 credentials (or replace with Slack/Teams if preferred).
  5. Test the flow by sending a sample AWS Security Hub finding (or a mock JSON payload) to the webhook URL.
  6. Activate the workflow and monitor the email output for real alerts.

Pro tip: Use a “Switch” node after the AI classification to route P0‑P1 alerts to a high‑priority channel (like Slack) and P2‑P3 alerts to a log or low‑priority email. This further automates escalation.

What This Workflow Does

This automation solution solves the challenge of keeping self-hosted UniFi Network Controller instances up-to-date without manual intervention. Network administrators often struggle with remembering to check for updates, scheduling maintenance windows, and documenting changes.

The workflow automatically checks Ubiquiti's official Debian repository for new UniFi Controller releases daily. When it detects an update published within the last 24 hours, it securely connects via SSH to your server and performs the upgrade while sending real-time notifications through Telegram.

How It Works

1. Scheduled Trigger

The workflow runs daily at a configurable time (default 13:13) to check for updates during low-traffic periods.

2. Repository Check

An HTTP request fetches the latest InRelease file from Ubiquiti's repo, parsing the Codename and Date fields to determine freshness.

3. Update Logic

An IF gate ensures the workflow only continues if the repository shows activity within the last 24 hours, preventing unnecessary operations.

4. Secure Upgrade

The SSH node executes two commands: updating package lists and upgrading only the UniFi package to maintain stability.

5. Notification

Optional Telegram notifications provide update confirmation and can include LLM-generated summaries of release notes.

Pro tip: For production environments, consider adding a backup step before upgrades and testing new versions on a staging controller first.

What You'll Need

  1. SSH access to your UniFi controller server
  2. Sudo privileges for package management commands
  3. Telegram bot token and chat ID (optional for notifications)
  4. OpenAI API key if using release note summarization

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure SSH credentials for your UniFi server
  3. Set the TELEGRAM_CHAT_ID environment variable if using notifications
  4. Adjust the schedule trigger to your preferred maintenance window
  5. Test in inactive mode before activating the workflow

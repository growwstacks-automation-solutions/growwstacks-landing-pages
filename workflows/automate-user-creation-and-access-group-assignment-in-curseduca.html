What This Workflow Does

Manual user creation in learning management systems like Curseduca creates bottlenecks, delays student access, and increases administrative overhead. Every time someone signs up for your course, purchases a bundle, or registers through a landing page, someone on your team must manually create their account and assign them to the correct access group.

This n8n workflow automates the entire user onboarding process in Curseduca. It captures user details from any source (web forms, payment processors, chatbots), creates the user account via Curseduca's API, assigns them to the appropriate access group based on their purchase or registration, and sends confirmation notifications—all within seconds of the initial trigger.

Pro tip: This automation is particularly valuable during product launches or promotional periods when you might receive hundreds of sign-ups simultaneously. It ensures every student gets immediate access while your team focuses on delivering value.

How It Works

The workflow follows a streamlined two-step process that connects your customer touchpoints directly to Curseduca's user management system.

Step 1: Capture User Data

A webhook node receives user information from your registration forms, payment processors, or marketing automation tools. It captures essential details including name, email address, and the specific access group ID they should be assigned to based on what they purchased or registered for.

Step 2: Create User & Assign Access

An HTTP Request node sends the collected data to Curseduca's API with proper authentication headers. The system creates the user account, assigns them to the specified access group, and triggers Curseduca's built-in email notification to welcome the new user and provide login instructions.

Pro tip: You can extend this workflow to include additional steps like adding users to your email marketing platform, creating calendar invites for course sessions, or logging the transaction in your CRM—all from the same initial trigger.

What You'll Need

  1. Curseduca Account: An active Curseduca subscription with administrative access to generate API credentials.
  2. API Credentials: Your Curseduca API Key and Bearer Token for authenticating API requests.
  3. Access Group IDs: The specific group IDs from your Curseduca account that correspond to different courses, membership levels, or training programs.
  4. Trigger Source: A way to send user data to the workflow—this could be a web form, payment processor webhook, chatbot integration, or any system that can make HTTP POST requests.
  5. n8n Instance: Either n8n.cloud account or self-hosted n8n installation to run the workflow.

Quick Setup Guide

Getting this automation running in your environment takes about 15-20 minutes with these simple steps:

  1. Download and Import: Download the template file and import it into your n8n instance using the "Import from File" option in the workflow editor.
  2. Configure Webhook: Activate the Webhook node and copy the unique URL it generates. This URL will be your endpoint for sending user data.
  3. Set API Credentials: In the HTTP Request node, replace the placeholder values with your actual Curseduca API Key and Bearer Token in the headers section.
  4. Map Group IDs: Update the workflow to use your specific Curseduca access group IDs. You can hardcode these or map them dynamically based on the incoming data.
  5. Test the Flow: Send a test POST request to your webhook URL with sample user data (name, email, groupId) and verify the user appears in your Curseduca account with correct access.
  6. Connect Your Systems: Point your registration forms, payment processors, or other systems to send data to your webhook URL whenever a new user should be created.

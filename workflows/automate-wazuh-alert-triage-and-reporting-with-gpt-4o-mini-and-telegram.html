What This Workflow Does

Security Operations Centers (SOCs) are overwhelmed by alert fatigue, with analysts spending up to 25% of their time on false positives. This workflow transforms your security operations by automating the entire alert triage process from Wazuh to Telegram with AI-powered analysis.

The system intelligently filters alerts by severity, uses GPT-4o-mini to generate contextual threat reports, and delivers actionable notifications to your security team via Telegram. It reduces manual triage work by 70% while ensuring critical alerts get immediate attention.

How It Works

1. Alert Ingestion

The workflow begins by receiving Wazuh alerts through a webhook trigger, capturing all security events in real-time.

2. Severity Filtering

An If condition node filters alerts based on severity levels (1-15), allowing you to customize which alerts require immediate attention.

3. AI Analysis

Critical alerts are processed by GPT-4o-mini which generates a detailed threat report including MITRE ATT&CK mapping, impacted assets, and recommended actions.

4. Notification Delivery

The analyzed alert is formatted and delivered to your security team via Telegram, with options to extend to Slack or email.

5. Noise Reduction

Low-priority alerts are automatically logged without notification, reducing alert fatigue while maintaining an audit trail.

  1. Download and import the JSON template into your n8n instance
  2. Configure Wazuh webhook credentials
  3. Add your OpenAI API key

  6. Activate the workflow

What This Workflow Does

Manual timesheet reporting is a weekly administrative burden that consumes valuable managerial time and often leads to errors, missed submissions, and delayed project insights. This automation solves that problem by creating a complete, hands-off reporting system that runs every Friday afternoon.

The workflow connects to your Salesforce instance (using the DBT Timesheet App), retrieves all employee timesheets for the previous week, separates submitted from pending entries, uses OpenAI to generate intelligent summaries of work completed, and delivers a polished email report to managers. It transforms raw data into actionable intelligence while highlighting compliance gaps.

Beyond simple reporting, this system creates accountability through automatic visibility. Managers no longer need to chase employees for submissions—the workflow identifies exactly who hasn't submitted and includes that information prominently in the weekly report.

How It Works

The automation follows a logical pipeline that processes data, applies intelligence, and delivers insights.

1. Data Collection from Salesforce

Each Monday morning, the workflow queries Salesforce for all timesheet records from the previous week. It retrieves not just submission status but detailed line items including projects, tasks, hours (billable/non-billable), and work descriptions. This forms the complete dataset for analysis.

2. Intelligent Processing with OpenAI

For submitted timesheets, the workflow sends anonymized line-item descriptions to OpenAI's API. The AI analyzes the work completed and generates four concise summary points that highlight key accomplishments, project progress, and notable work—transforming raw data into manager-ready insights.

3. Report Assembly & Delivery

The system merges AI summaries with employee details, formats everything into a professional HTML email template, and sends it via Gmail to designated managers. The report clearly separates submitted summaries from pending submissions, creating immediate visibility without manual effort.

What You'll Need

  1. Salesforce with DBT Timesheet App – The managed package must be installed from Salesforce AppExchange
  2. OpenAI API access – Any GPT model (GPT-3.5-turbo or GPT-4) with API credits
  3. Gmail account – For sending the weekly reports (OAuth authentication)
  4. n8n instance – Self-hosted or n8n.cloud account
  5. Manager email list – Recipients who should receive the weekly summary

Pro tip: Before full deployment, test with a single manager's email to verify formatting and data accuracy. Adjust the AI prompt to emphasize the type of insights most valuable to your leadership team.

Quick Setup Guide

Implement this automation in under 30 minutes with these straightforward steps:

  1. Download and import the JSON template into your n8n instance
  2. Configure credentials for Salesforce, OpenAI, and Gmail in n8n's credentials management
  3. Update the recipient list in the Gmail node with your actual manager emails
  4. Adjust the schedule trigger to match your reporting cycle (typically Monday mornings)
  5. Test with a single employee to verify data flow and email formatting
  6. Activate the workflow and let it run automatically each week

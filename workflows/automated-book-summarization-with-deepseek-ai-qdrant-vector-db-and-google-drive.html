What This Workflow Does

Manually reading and summarizing books, research papers, or lengthy reports consumes valuable hours that could be spent on analysis and decision-making. This automation solves that problem by creating an intelligent pipeline that processes documents as they're uploaded to Google Drive.

The workflow automatically extracts key insights using AI, makes content searchable through vector embeddings, and delivers concise summaries back to your cloud storage. It transforms passive document storage into an active knowledge management system that works while your team focuses on higher-value tasks.

How It Works

1. Document Upload & Detection

When a new book or document is uploaded to a designated Google Drive folder, the workflow triggers automatically. It downloads the file and prepares it for processing, supporting various formats including PDFs, Word documents, and text files.

2. Intelligent Chunking & Embedding

The system splits the document into logical chunks using advanced text segmentation. Each chunk is converted into vector embeddings using Cohere's AI, capturing the semantic meaning of the content for intelligent retrieval later.

3. Vector Storage & Organization

Embeddings are stored in Qdrant Vector Database, creating a searchable knowledge base of your document's content. This enables semantic search capabilities that understand concepts rather than just keywords.

4. AI-Powered Analysis & Summarization

DeepSeek AI (or your preferred LLM) analyzes the document structure and content, extracting key themes, arguments, and insights. The AI generates comprehensive summaries tailored to different use cases—executive overviews, detailed chapter breakdowns, or thematic analyses.

5. Delivery & Cleanup

The final summary is saved back to Google Drive in your designated output folder. The workflow optionally cleans up temporary vector collections to manage storage efficiently, though you can configure it to maintain long-term memory if needed.

What You'll Need

  1. Google Drive account with API access for input and output folders
  2. n8n instance (cloud or self-hosted) to run the workflow
  3. API keys for DeepSeek AI (or alternative LLM provider)
  4. Cohere API key for generating text embeddings
  5. Qdrant instance (cloud or self-hosted) for vector storage
  6. Basic understanding of folder structure organization in Google Drive

Pro tip: Start with a dedicated "book_uploads" folder in Google Drive and a corresponding "summaries_output" folder. This keeps your automation organized and makes it easy to track processed versus unprocessed documents.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance
  2. Configure Google Drive credentials in n8n's credential manager
  3. Update folder IDs in the Google Drive trigger and create nodes to match your directory structure
  4. Add your API keys for DeepSeek AI, Cohere, and Qdrant in their respective nodes
  5. Test with a sample document by uploading a PDF or text file to your input folder
  6. Review and customize the summarization prompts to match your preferred output format
  7. Activate the workflow and let it run automatically as new documents arrive

Automated Expense Tracking from Emails & Telegram with Gemini AI

Free n8n workflow template that automatically parses bank transaction emails and Telegram messages, categorizes expenses using AI, and logs them into Google Sheets with budget alerts.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manual expense tracking is tedious, error-prone, and time-consuming. Bank emails, UPI notifications, and Telegram messages flood your inbox, making it impossible to maintain an accurate financial log. This automation solves that by automatically extracting transaction details from Gmail and Telegram, using Gemini AI to categorize each expense, and logging everything into a structured Google Sheets budget tracker.

The workflow compares expenses against your monthly budget allocations and sends alerts when limits are exceeded. It consolidates spending across multiple sources into a single, real-time dashboard, eliminating manual data entry and providing instant financial visibility.

Whether you're managing personal finances, household spending, or small business expenses, this template transforms chaotic transaction data into organized, actionable insights.

How It Works

Step 1: Capture Transaction Data

The workflow monitors your Gmail for bank transaction emails (HDFC, Indian Bank, Indian Overseas Bank, UPI apps like Google Pay, Paytm) and listens for expense entries in a Telegram bot. It filters only financial messages and extracts the raw text content.

Step 2: AI-Powered Parsing & Categorization

Gemini AI analyzes each transaction message, identifying key details: date, amount, account, transaction type (credit/debit), description, currency, and category. The AI intelligently categorizes expenses (e.g., Food, Travel, Utilities) based on transaction context.

Step 3: Structured Logging to Google Sheets

Parsed data is appended to a Google Sheets "Expenses" tab with columns for Timestamp, Date, Account, From, To, Type, Category, Description, Amount, Currency, Source, MessageId, and Status. The system automatically distinguishes between credit and debit transactions.

Step 4: Budget Validation & Alerts

The workflow references a "Budgets" tab with monthly category allocations. If an expense exceeds the budget, it triggers an alert email. Monthly and yearly summary tabs auto-calculate totals and variances, providing continuous budget monitoring.

Step 5: Monthly Reset & Reporting

At month-end, a Google Apps Script function calculates totals, updates yearly summaries, clears monthly data, and resets counters. This creates a rolling financial record without manual intervention.

Pro tip: Customize the bank detection logic in the Code node to support additional banks or UPI apps by modifying the regex patterns. You can also add Slack or WhatsApp alert channels for team notifications.

What You'll Need

  1. n8n instance (self-hosted or cloud) – this template uses community nodes requiring self-hosted n8n.
  2. Google account with Gmail and Google Sheets enabled.
  3. Pre-created Google Sheets file with three tabs: Expenses, Budgets, and Yearly Summary (template columns provided).
  4. Gemini API connection configured in n8n for AI categorization.
  5. Telegram bot token if using Telegram input (optional, Gmail-only works too).

Quick Setup Guide

1. Download the JSON template file and import it into your n8n workspace.

2. Create your Google Sheets file with the required column structure: Expenses tab (Timestamp, Date, Account, From, To, Type, Category, Description, Amount, Currency, Source, MessageId, Status), Budgets tab (Month, Category, Budget Amount, Notes, UpdatedAt), and Yearly Summary tab (Year, Month, Category, Total Expense, Budget, Variance, Alert).

3. Configure the Gmail trigger node with your email account and set filters for bank/UPI sender addresses.

4. Set up the Gemini AI node with your API key and adjust the parsing schema if needed.

5. Connect the Google Sheets nodes to your spreadsheet ID and sheet names.

6. For Telegram input, create a bot via @BotFather, get the token, and configure the Telegram trigger node.

7. Test with a sample transaction email or Telegram message to verify parsing and logging.

8. Activate the workflow and let it run automatically on new transactions.

Pro tip: Start with just Gmail parsing to validate the core workflow. Add Telegram integration later for manual expense entries or team contributions.

What This Workflow Does

This automation solves a critical data pipeline problem for businesses using Google Analytics 4 (GA4). Instead of manually exporting reports or building complex scripts, this workflow automatically fetches 13 essential GA4 reports daily and pushes them into Google BigQuery.

It transforms raw GA4 data into structured, queryable tables in BigQuery, enabling advanced analytics, historical trend analysis, and integration with other business data. The Telegram alert provides instant operational visibility, confirming successful runs or highlighting issues.

By automating this process, you eliminate hours of weekly manual work, ensure data consistency, and create a reliable foundation for data-driven decision-making.

How It Works

Step 1: Central Configuration

A single "Config" node holds your GA4 Property ID, Google Cloud Project ID, and BigQuery Dataset ID. This centralizes settings and makes the workflow reusable across different projects or properties.

Step 2: Parallel Report Fetching

The workflow triggers 13 parallel API calls to the GA4 Data API. Each call retrieves a specific pre-processed report—such as User Acquisition, Traffic Sources, E-commerce Items, and Engagement Metrics—for the configured date range.

Step 3: Dynamic Table Creation

For each report, the automation checks if a corresponding table already exists in BigQuery. If not, it automatically creates the table with the correct schema derived from the GA4 report structure, ensuring perfect data alignment.

Step 4: Data Insertion & Alerting

The fetched data is inserted into the newly created or existing BigQuery tables. Upon completion, a Telegram bot sends a concise summary message to your designated chat, listing successful reports, row counts, and any failures.

What You'll Need

  1. A Google Analytics 4 property with data.
  2. A Google Cloud Project with BigQuery enabled.
  3. Google OAuth2 credentials configured in n8n (scopes: Google Analytics API & BigQuery API).
  4. A Telegram Bot Token and Chat ID (optional, for alerts).
  5. An n8n instance (cloud or self-hosted) to run the workflow.

Pro tip: Schedule the workflow to run daily with a date expression like "Today - 2 Days" to allow GA4 time to fully process data before fetching, ensuring completeness.

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n workspace.
  2. Open the "Backfill Config" node and enter your GA4 Property ID, Google Cloud Project ID, and BigQuery Dataset ID.
  3. Set up Google OAuth2 credentials in n8n, ensuring both Google Analytics API and BigQuery API scopes are granted.
  4. If using Telegram alerts, configure the Telegram node with your Bot Token and Chat ID.
  5. Test the workflow manually once to verify connections and table creation.
  6. Set a schedule trigger (recommended: daily) and activate the workflow.

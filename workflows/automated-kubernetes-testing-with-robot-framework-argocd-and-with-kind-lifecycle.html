Automated Kubernetes CI/CD Testing with Robot Framework & ArgoCD

What This Workflow Does

This n8n workflow solves the critical problem of unreliable Kubernetes deployments by providing fully automated CI/CD testing. Manual testing of Kubernetes applications is time-consuming, error-prone, and often skipped due to complexity. This creates deployment risks, configuration drift, and production outages that cost businesses thousands in downtime and troubleshooting.

The workflow automates the entire testing lifecycle: it creates temporary Kubernetes clusters using KinD (Kubernetes in Docker), deploys your applications with ArgoCD, runs comprehensive browser and API tests with Robot Framework, and then completely cleans up all resources. This ensures every deployment is validated in an environment identical to production, catching issues before they reach users.

How It Works

Phase 1: Infrastructure Setup (INIT)

The workflow first establishes a clean testing environment by installing all necessary dependencies on a remote host. It creates a KinD cluster with control-plane and worker nodes, installs Helm and Nginx Ingress, sets up HAProxy for port forwarding, deploys ArgoCD for GitOps-based deployment, and applies your ApplicationSet manifests.

Phase 2: Automated Testing (TEST)

Once infrastructure is ready, the workflow downloads your Robot Framework test scripts from GitLab, installs the Robot Framework and Browser library, executes automated browser tests against your deployed application, packages test results with screenshots and logs, and sends comprehensive reports via Telegram for immediate review.

Phase 3: Complete Cleanup (DESTROY)

After testing completes, the workflow ensures no resources are left behind. It removes HAProxy configurations, deletes the KinD cluster, uninstalls KinD and Docker dependencies, and sends a completion notification. This prevents resource leaks and keeps your testing environment costs at zero.

Pro tip: Run this workflow in "single phase mode" during development to test specific components, then switch to "full pipeline mode" for production CI/CD integration.

What You'll Need

  1. n8n instance (version 1.0+) with the Robot Framework community node installed
  2. Linux target host with SSH access, sudo privileges, 8GB+ RAM, and 20GB+ disk space
  3. GitLab account with OAuth2 application configured and repository containing test files
  4. Telegram Bot with API token and chat ID for notifications
  5. Network access between n8n, target host, GitLab, and Telegram APIs

Quick Setup Guide

  1. Install the n8n-nodes-robotframework community node in your n8n settings
  2. Configure GitLab OAuth2 credentials in n8n with read_api, read_repository, and read_user scopes
  3. Prepare your GitLab repository with test.robot, config.yaml, and demo-applicationSet.yaml files
  4. Create a Telegram bot via @BotFather and obtain the API token and chat ID
  5. Import this workflow JSON into n8n and update the Set Parameters node with your target host details
  6. Configure the three GitLab nodes with your repository owner, name, and file paths
  7. Test with progress_only = true to validate each phase separately before full automation

What This Workflow Does

Manual server monitoring is time-consuming, error-prone, and often misses critical issues until users complain. This automated workflow solves that by continuously checking your servers' health via SSH, analyzing the results with Azure GPT-4 for intelligent insights, and sending immediate email alerts when problems are detected.

It's designed for IT teams managing multiple servers who need reliable, 24/7 monitoring without investing in expensive enterprise tools. The workflow runs periodic checks, gathers system metrics (CPU, memory, disk, services), uses AI to interpret the data, and notifies the right people with actionable information—all automatically.

Beyond simple uptime monitoring, this template provides context-aware alerts. Instead of just "server down," you get "Disk usage at 95% on web-server-01, likely causing slow response times; recommend cleaning /var/log." This transforms raw data into operational intelligence.

How It Works

The workflow follows a logical sequence of checks, analysis, and notification. Here's the step-by-step process:

1. Scheduled Trigger

A Cron node initiates the workflow at regular intervals you configure—every 5 minutes, hourly, or whatever schedule fits your needs. This ensures continuous monitoring without manual intervention.

2. SSH Connection & Command Execution

The workflow establishes secure SSH connections to your target servers using key-based authentication. It executes standard Linux commands to check vital statistics: uptime for load averages, df -h for disk space, free -m for memory usage, and systemctl status for critical services.

3. Data Collection & Parsing

Raw command outputs are parsed into structured data. The workflow extracts numeric values and status indicators, normalizing them for consistent analysis. It handles different Linux distributions and shell variations.

4. AI-Powered Analysis with Azure GPT-4

This is where intelligence kicks in. The collected metrics are sent to Azure OpenAI's GPT-4 model with a prompt designed for system analysis. The AI evaluates the data, identifies anomalies, correlates multiple metrics, and generates plain-English summaries of server health and potential issues.

Pro tip: The AI analysis can spot subtle patterns humans might miss, like gradual memory leaks or correlated disk and network issues that indicate deeper problems.

5. Conditional Alerting

The workflow evaluates the AI's analysis and your predefined thresholds. If any server shows critical issues (high CPU, low disk, service down) or the AI flags a concern, it triggers the email notification path. Otherwise, it logs the check as "healthy" and waits for the next cycle.

6. Email Notification with Context

When problems are detected, the workflow composes a detailed email using your SMTP credentials. The email includes the server name, timestamp, specific issues found, the AI's analysis, and suggested next steps. This gives your team everything needed to respond immediately.

Pro tip: Start with monitoring just your most critical servers first. Once configured, you can easily add more servers by duplicating the SSH nodes in the workflow.

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure SSH credentials in n8n for each server you want to monitor. Use key-based auth for security.
  3. Set up Azure OpenAI credentials in n8n with your API key and endpoint details.
  4. Add your SMTP email credentials (Gmail, Outlook, etc.) for alert delivery.
  5. Customize the Cron node to set your preferred check frequency (default: every 30 minutes).
  6. Test the workflow manually first, then activate it to run automatically.

The template includes comments and example configurations to guide you. Most users have it running in production within an hour.

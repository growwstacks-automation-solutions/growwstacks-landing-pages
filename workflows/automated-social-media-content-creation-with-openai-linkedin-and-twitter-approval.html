What This Workflow Does

Managing social media across multiple platforms is time-consuming and repetitive. This automation solves that by creating a seamless pipeline from content ideation to published post. It uses AI to generate platform-specific content, sends it for human approval, and then publishes it automatically to LinkedIn and Twitter (X).

The workflow eliminates manual copying, formatting, and scheduling. It ensures brand consistency, maintains a regular posting schedule, and provides a clear audit trail of all content. By connecting OpenAI, Gmail, Google Sheets, Slack, and your social accounts, it creates a cohesive system that operates with minimal intervention.

Beyond just posting, it logs every action, tracks approval status, and can be extended to include performance analytics. This turns social media management from a daily chore into a strategic, automated function.

How It Works

The workflow orchestrates data between several key applications to create a hands-off content pipeline.

1. Trigger & Content Brief

The process starts with a manual trigger or a scheduled event. You provide a content brief or topic. This could be a new blog post URL, a product update, or a simple keyword. The workflow captures this input and prepares it for AI processing.

2. AI Content Generation

The core topic is sent to OpenAI (like GPT-4). The AI generates two unique versions of content: one optimized for LinkedIn's professional audience and another for Twitter's concise, engaging format. It can also generate relevant hashtags and calls-to-action.

3. Approval Workflow

Generated posts are sent via Gmail to designated approvers (e.g., marketing manager, founder). The email contains the content and simple "Approve" or "Reject" links. This step ensures brand safety and strategic alignment before anything goes live.

4. Image Generation (Optional)

If the content brief requests an image, the workflow can call DALL-E or another AI image service to create a custom graphic based on the post topic. This image is attached to the social post, significantly boosting engagement.

5. Platform Publishing

Once approved, the workflow uses the official LinkedIn and Twitter (X) APIs to publish the content natively. It handles formatting, character limits, image uploads, and scheduling according to the specified timetable.

6. Notification & Logging

A confirmation message is sent to a Slack channel, alerting the team. All post details—content, platform, time, approval status—are logged in a Google Sheet, creating a master content calendar and audit log.

What You'll Need

  1. An n8n instance: Either n8n.cloud (cloud) or a self-hosted setup.
  2. OpenAI API key: For generating text and optional images. Sign up at platform.openai.com.
  3. LinkedIn Developer Access: Create an app in the Linkedin Developers Portal to get API credentials.
  4. Twitter (X) API Access: Apply for access via the Twitter Developer Portal.
  5. Gmail Account: Used for sending approval requests. Ensure "Less secure app access" is enabled or use OAuth 2.0.
  6. Google Sheets: A spreadsheet to log activity. The workflow will need edit permissions.
  7. Slack Workspace: To receive notifications. You'll need to create a Slack app and get a webhook URL for your channel.

Quick Setup Guide

Follow these steps to get this automation running in your n8n environment in under 30 minutes.

  1. Download & Import: Click the "Download Template" button above. In your n8n dashboard, go to "Workflows" > "Import from file" and select the downloaded JSON.
  2. Configure Credentials: For each colored node (Google Sheets, OpenAI, etc.), click the node and set up the required credentials. n8n will guide you through the OAuth or API key process for each service.
  3. Set Approval Email: In the "Gmail" node, update the "To" field with the email address of the person who should approve posts.
  4. Connect Social Accounts: Authenticate the LinkedIn and Twitter nodes with your business accounts. Ensure the apps have "posting" permissions.
  5. Test the Trigger: Use the "Form Trigger" node or set up a schedule. Run the workflow once manually to ensure all connections work and the approval email is received.
  6. Activate & Schedule: Turn the workflow "Active". You can set the "Schedule Trigger" node to run daily, weekly, or at any interval that fits your content calendar.

Automated Task Generation from Baserow Blueprints

Automatically create tasks from template tables with weekend-aware scheduling. Perfect for project management, HR onboarding, and operational procedures.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves the tedious manual process of copying checklist steps from templates into actual tasks. Instead of someone manually transferring each step from a Standard Operating Procedure template to assignees with deadlines, this workflow automatically generates tasks in Baserow based on blueprint tables.

It connects a master template table (containing procedure information) with a details table (containing individual steps) to create tasks with calculated deadlines that avoid weekends. This eliminates human error, ensures consistency across projects, and saves significant setup time for recurring processes.

The business value is clear: teams can launch new projects, onboard employees, or execute standard procedures in minutes instead of hours, with guaranteed adherence to established templates.

How It Works

Trigger Task Creation

A webhook receives a POST request containing the assignee ID, template ID, start date, and optional note. This trigger can come from a Baserow application button, another automation, or any system needing task generation.

Configure Settings and IDs

The workflow stores Baserow database and table IDs along with API credentials. It validates the incoming data and prepares for the next steps.

Get All Template Steps

It queries the details table to retrieve all steps associated with the selected master template. For example, a "Customer Complaint Handling" template might have 15 specific steps.

Calculate Deadlines for Each Step

Each step's deadline is calculated by adding its "Days to complete" value to the start date. Property names are mapped to match your Baserow task table fields.

Avoid Scheduling During the Weekend

A code node checks if any calculated deadline falls on Saturday or Sunday and shifts it to the next Monday. This maintains realistic business timelines.

Aggregate Tasks for Insert

All prepared tasks are aggregated into an "items" array matching the Baserow batch API structure, enabling efficient bulk insertion.

Generate Tasks in Batch

The workflow calls Baserow's batch API endpoint to insert all tasks simultaneously. This is far faster than creating records individually.

Success / Error Response

A clear response is sent back indicating success or failure, providing feedback to the triggering system.

Pro tip: Use Baserow's Standard Operating Procedures template as your starting schema. It's perfectly structured for this automation with master, detail, assignee, and task tables.

What You'll Need

  1. A Baserow account (cloud or self-hosted)
  2. A Baserow database with four specific tables: Assignee (employee table), Master (template table), Details (steps table with "Days to complete" field), and Tasks (destination table)
  3. A Baserow database token for API authentication
  4. Basic understanding of your table IDs and field names
  5. An n8n instance (cloud or self-hosted) to run the workflow

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n workspace.
  2. In the "Configure settings and ids" node, enter your Baserow database ID, table IDs, and API token.
  3. Review the "Calculate deadlines for each step" node to ensure property names match your Tasks table fields.
  4. Test the workflow by sending a POST request to the webhook with sample data (assignee_id, template_id, schedule_date, note).
  5. Monitor the Baserow Tasks table to verify tasks were created correctly with proper deadlines.
  6. Integrate the trigger into your Baserow application or other systems as needed.

Pro tip: Start with a simple template like "New Employee Onboarding" to test the workflow. Once confirmed, expand to more complex procedures.

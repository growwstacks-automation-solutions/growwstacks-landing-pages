What This Workflow Does

Manual security reviews of your automation platform are time-consuming, inconsistent, and often overlooked until a problem occurs. This workflow solves that by automatically conducting a comprehensive security audit of your n8n instance every week and delivering the findings directly to your inbox.

The system identifies potential security risks across four key areas: exposed or insecure credentials, sensitive nodes (like Code, HTTP Request, SSH, and FTP), global instance security configuration, and third-party community nodes that may pose risks. It doesn't just list problems—it provides direct links to affected workflows, shows their last execution status, and offers actionable recommendations.

By automating this critical monitoring function, you maintain continuous visibility into your security posture without manual effort. The weekly cadence ensures you catch new risks quickly while tracking improvements over time, transforming security from a periodic chore into an ongoing business process.

How It Works

1. Scheduled Trigger & Data Collection

The workflow activates on a weekly schedule (configurable to daily or monthly). It first connects to your n8n instance via API to collect comprehensive data about all workflows, credentials, nodes, and execution history.

2. Risk Analysis & Scoring

Using intelligent algorithms, the system analyzes the collected data to identify security risks. It calculates an overall risk level (Low/Moderate/High), tracks unique credential exposures (not just total counts), and provides detailed breakdowns by node type and severity.

3. Report Generation & Formatting

The findings are compiled into a professional HTML report with color-coded risk levels, emoji icons for easy scanning, and direct clickable links to each affected workflow. The system supports both English and French language outputs.

4. Email Delivery & Notification

The completed report is sent via Gmail (or your configured email service) to designated recipients. The email includes a clear subject line indicating the risk level and provides actionable insights for immediate follow-up.

What You'll Need

  1. An n8n instance with API access enabled
  2. An n8n API key with audit permissions
  3. A Gmail account (or other email service) configured for sending
  4. Basic understanding of your n8n workflow structure
  5. 5-10 minutes for initial configuration

Pro tip: Before deploying, test with a small subset of workflows to understand the report format and risk scoring. Adjust the risk thresholds in the JavaScript nodes if your organization has different risk tolerance levels.

Quick Setup Guide

Follow these steps to implement this security audit system in your environment:

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure credentials for n8n API access and Gmail OAuth2 sending.
  3. Set configuration variables including recipient email, project name, server URL (no trailing slash), and preferred language (EN/FR).
  4. Test execution by manually triggering the workflow and verifying the email delivery.
  5. Activate scheduling to run weekly (default) or adjust to your preferred cadence.
  6. Review first report with your team and establish response procedures for different risk levels.

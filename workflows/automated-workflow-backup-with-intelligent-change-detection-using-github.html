What This Workflow Does

Manual backup of automation workflows is risky and inefficient. Teams often lose critical logic, struggle with version tracking, and waste hours recreating lost automations. This template solves that by creating a fully automated, intelligent backup system that syncs your n8n workflows to GitHub.

It provides version-controlled storage, intelligent change detection (ignoring metadata noise), and rename tracking—so your repository always reflects your current workflow state. Whether you're a solo developer or an enterprise IT team, this ensures your automation investments are protected and collaborative.

The workflow runs on a schedule, compares workflows against their GitHub versions, and commits only when real changes occur. It generates clean commit messages (create, update, rename) and can notify your team via Telegram, Slack, or email. This turns backup from a manual chore into a reliable, automated process.

How It Works

Step 1: Fetch All Current Workflows

The workflow starts by calling the n8n API to retrieve every active workflow in your instance. It captures the full JSON structure, name, and unique ID.

Step 2: List Existing GitHub Files

Simultaneously, it queries your designated GitHub repository to list all previously backed‑up workflow files. This establishes a baseline for comparison.

Step 3: Compare & Decide Action

Each n8n workflow is matched against its GitHub counterpart using the unique ID. The system performs a deep comparison of the actual logic and structure, ignoring metadata like timestamps or temporary IDs. It determines if the workflow is new, updated, renamed, or unchanged.

Step 4: Execute GitHub Operations

For new workflows, a file is created with a human‑readable filename (e.g., “Lead‑Enrichment‑Workflow.json”). For updates, the existing file is overwritten only if the logic changed. For renamed workflows, the old file is deleted and a new one created—maintaining a clean repository.

Step 5: Send Summary Report

Finally, a formatted summary is sent to your chosen notification channel (Telegram by default, easily adaptable to Slack, email, etc.). The report details what was created, updated, renamed, or skipped.

Quick Setup Guide

Import and configure this template in under 10 minutes:

  1. Download the JSON file and import it into your n8n instance.
  2. Create credentials for n8n API (using your instance URL and API key) and GitHub (using your personal access token).
  3. Open the Configuration node and set your GitHub repository owner, name, and folder path (e.g., workflows/).
  4. Adjust the Schedule Trigger to your preferred backup frequency (daily, hourly, etc.).
  5. Optionally, set up Telegram credentials and chat ID for notifications, or replace the notification node with Slack/email.
  6. Activate the workflow. It will now run automatically, backing up changes and sending reports.

Pro tip: Use a dedicated GitHub repository for backups—not your main codebase. This keeps automation files separate and simplifies permission management.

What This Workflow Does

This automation solves the critical business problem of maintaining accurate, up-to-date product information across different systems. When you manage inventory in Zoho Inventory but need that data in Supabase for a custom eCommerce site, mobile app, or internal dashboard, manual copying becomes a time-consuming and error-prone task.

The workflow automatically fetches product data from Zoho Inventory—including SKUs, variants, stock levels, pricing, and custom attributes—then transforms and inserts it into your Supabase database. It runs on a schedule you define (hourly, daily, etc.), ensuring your product catalog is always current without any manual intervention.

Beyond basic sync, this template handles API authentication, data mapping, error recovery, and batch processing. It's production-ready for businesses that need reliable, automated data flow between their inventory management system and application backend.

How It Works

Step 1: Scheduled Trigger & Authentication

The workflow starts on a schedule (cron trigger) and securely authenticates with Zoho Inventory's API using OAuth refresh tokens. This ensures ongoing access without manual reauthorization.

Step 2: Fetch Product Data from Zoho

It retrieves all products and variants from your Zoho Inventory account, including detailed metadata like descriptions, images, pricing, stock quantities, dimensions, and up to three custom attributes per item.

Step 3: Data Transformation & Mapping

Each product record is processed to match your Supabase table structure. The workflow handles data type conversions, field renaming, and conditional logic (like filtering out discontinued items).

Step 4: Batch Insert to Supabase

Transformed records are inserted into your Supabase database table. The template includes error handling for duplicate keys, missing fields, and API rate limits, with retry logic for failed operations.

Step 5: Logging & Notification

Finally, the workflow logs sync statistics (products processed, errors encountered) and can send notifications via email or Slack if critical issues arise during the sync process.

What You'll Need

  1. Zoho Inventory account with API access enabled and OAuth credentials (Client ID, Client Secret, Refresh Token).
  2. Supabase project with a database table ready to receive product data (columns for SKU, name, price, stock, etc.).
  3. Supabase API keys (Project URL and Service Role Key) for secure database access.
  4. n8n instance (self-hosted or cloud) to run the workflow.
  5. Basic understanding of your data structure to map Zoho fields to Supabase columns during setup.

Pro tip: Test the sync with a small subset of products first. Create a separate "test" table in Supabase and run the workflow to verify field mapping before syncing to your production database.

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance.
  2. Configure Zoho credentials in the HTTP Request node—add your Client ID, Client Secret, and Refresh Token.
  3. Set up Supabase connection by entering your Project URL and Service Role Key in the Supabase node.
  4. Map your fields by adjusting the data transformation step to match your Supabase table column names.
  5. Adjust the schedule to match your business needs (hourly for real-time sync, daily for less frequent updates).
  6. Test with active workflow and monitor the execution to ensure data flows correctly between systems.

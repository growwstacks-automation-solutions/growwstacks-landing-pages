What This Workflow Does

This automation solves the time-consuming challenge of creating regular blog content for ecommerce stores. It connects your WooCommerce product catalog with OpenAI's language models to generate engaging, product-focused articles, then publishes them directly to your WordPress site.

The workflow runs on a schedule (default every 6 hours), selecting different products each time to ensure content variety. It transforms product data into reader-friendly blog posts complete with titles, meta descriptions, and proper formatting - all without manual intervention.

How It Works

1. Scheduled Trigger

The workflow activates automatically based on your configured schedule, ensuring consistent content production without manual starts.

2. Product Selection

Pulls your latest WooCommerce products through the REST API and randomly selects one to feature, preventing duplicate posts.

3. AI Content Generation

Sends product details to OpenAI with a carefully crafted prompt that ensures engaging, on-brand content tailored to your products.

4. Content Formatting

Structures the AI output into proper blog post format with title, body, excerpt, and SEO-friendly URL slug.

5. WordPress Publishing

Pushes the completed post to your WordPress site via REST API, including category assignment and proper formatting.

Pro tip: Customize the OpenAI prompt in the workflow to match your brand voice - try adding instructions like "write in a friendly, conversational tone" or "include 3 bullet points highlighting key features."

What You'll Need

  1. WooCommerce store with products
  2. WordPress site with REST API enabled
  3. OpenAI API key (GPT-3.5 or higher)
  4. n8n instance (cloud or self-hosted)
  5. Basic Auth credentials for WooCommerce and WordPress

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure WooCommerce API credentials in the "Pull Products" node
  3. Add your OpenAI API key in the "Generate Content" node
  4. Set up WordPress authentication in the "Publish Post" node
  5. Adjust the schedule trigger frequency if needed
  6. Test with 1-2 products before enabling the full automation

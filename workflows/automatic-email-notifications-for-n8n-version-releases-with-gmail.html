What This Workflow Does

Staying current with n8n updates is crucial for security, performance, and accessing new features, but manually checking for releases is inefficient and easy to forget. This automation solves that problem by continuously monitoring n8n's official release channels and sending you instant email notifications whenever new versions become available.

The workflow checks both stable and beta release streams, identifies only genuinely new versions (avoiding duplicate notifications), fetches detailed release notes from GitHub, and delivers beautifully formatted email alerts directly to your inbox. It can also detect your current n8n instance version, giving you context about whether an update is immediately relevant to your setup.

Beyond simple notifications, this system provides actionable intelligence. You'll receive structured information about what changed, security implications, breaking changes, and upgrade recommendations—all formatted for easy reading and decision-making.

How It Works

This intelligent monitoring system operates through a carefully orchestrated sequence of steps:

1. Scheduled Trigger & Version Checking

The workflow begins with a schedule trigger (configurable from hourly to weekly) that initiates the monitoring process. It queries the npm registry API to fetch the latest version information for both stable and beta release channels, comparing these against previously recorded versions to detect genuinely new releases.

2. Release Data Processing

When a new version is detected, the workflow extracts comprehensive metadata including version numbers, publication dates, and release tags. It then retrieves detailed release notes from GitHub, parsing markdown content to understand what's changed, what's fixed, and any breaking changes that might affect your workflows.

3. Current Instance Version Detection

Optionally, the workflow can query your n8n instance's API to determine your current running version. This context is invaluable—it tells you whether an update is immediately applicable to your environment or if you're already running the latest version.

4. Email Composition & Delivery

The system transforms raw release data into professional HTML email notifications. Using Gmail's integration, it sends formatted alerts that include version comparisons, change summaries, upgrade recommendations, and direct links to official documentation. The emails are designed for quick scanning while providing all necessary details.

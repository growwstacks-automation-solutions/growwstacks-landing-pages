What This Workflow Does

For retail stores, restaurants, and e-commerce businesses using Square, manually compiling weekly sales reports is a repetitive, time-consuming task. This automation eliminates that chore by connecting directly to the Square API, fetching your sales data, and delivering a formatted report directly to your inbox or your team's inbox every Monday morning.

The workflow pulls the exact sales summary figures that match your Square Dashboard, ensuring accuracy and consistency. It processes completed orders from the previous week (Monday to Sunday), aggregates totals per location, and generates a clean CSV file. This file is then attached to an email and sent via Gmail (or Outlook) to designated recipients like managers, accountants, or business partners.

Beyond saving time, this automation reduces human error, ensures reports are never forgotten, and gives stakeholders timely access to performance data for quicker decision-making. It's particularly valuable for multi-location businesses, franchise owners, or anyone who needs to share sales performance with external parties like landlords or commission-based agents.

How It Works

The workflow is triggered on a schedule and executes a series of steps to transform raw Square data into a actionable report.

Step 1: Schedule Trigger & Location Fetch

A Schedule Trigger node is set to run every Monday at 8:00 AM. The workflow first authenticates with the Square API using a Header Auth credential and retrieves a list of all your business locations. This ensures the report covers every store, pop-up, or online sales channel you operate.

Step 2: Retrieve Previous Week's Orders

For each active location, the workflow makes an API call to Square's Orders endpoint, filtering for completed orders from the previous seven days. It handles pagination automatically to capture all transactions, no matter the volume.

Step 3: Data Processing & Aggregation

A Code node processes the raw order data. It calculates key metrics: gross sales, net sales, taxes collected, tips, discounts applied, and total transaction count. This mirrors the logic of Square's own Sales Summary report, creating a familiar format for recipients.

Step 4: CSV Generation & Email Dispatch

The aggregated data is passed to a node that converts it into a CSV file. The workflow then uses your connected Gmail (or Outlook) account to compose an email, attach the CSV, and send it to your predefined recipient list. You can customize the subject line and email body to include key insights or instructions.

Pro tip: Use the "Split Out" node in n8n to create separate report files for each location if your recipients only need to see their specific data. This adds a layer of data privacy and relevance.

What You'll Need

  1. A Square account with access to the Square API. You'll need to generate an access token from your Square Developer Dashboard.
  2. An n8n instance (cloud or self-hosted). The workflow is built for n8n but demonstrates patterns applicable to other platforms.
  3. A Gmail or Microsoft Outlook account for sending the email. The template uses the respective n8n node, which requires OAuth2 setup.
  4. Basic familiarity with n8n to configure the API credentials and set the recipient email address.

Quick Setup Guide

Get this automation running in your n8n instance in under 15 minutes.

  1. Download the template using the button above and import the JSON file into your n8n workspace.
  2. Configure Square credentials: In n8n, create a new "Header Auth" credential. Name it "Authorization" and set the value to Bearer YOUR_SQUARE_ACCESS_TOKEN.
  3. Set up your email credentials: Add and authorize a Gmail or Outlook node in n8n using OAuth2.
  4. Update the workflow variables: Enter the recipient email address and customize the email subject/body in the final node.
  5. Test and activate: Run the workflow once manually to verify data flows correctly. Then activate it to run automatically every Monday.

Important: Ensure your Square access token has the necessary permissions (typically ORDERS_READ and MERCHANT_PROFILE_READ) to retrieve orders and location data.

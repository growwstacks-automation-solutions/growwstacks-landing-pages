What This Workflow Does

Manually downloading files from an FTP server and uploading them to Google Drive is a repetitive, time-consuming task that's prone to human error. This automation solves that by creating a hands-off pipeline between your legacy FTP storage and modern cloud storage.

The workflow connects to your FTP server on a schedule you define, lists all files in a specified directory, processes them in manageable batches to avoid system strain, downloads each file, and uploads it to a designated Google Drive folder—all while preserving the original filenames and folder structure. It's perfect for nightly backups, syncing client deliverables, or migrating archives to the cloud.

By automating this transfer, you eliminate the risk of forgetting to run the manual process, ensure data is backed up consistently, and free up valuable employee time for more strategic work. It turns a multi-step, manual chore into a reliable, silent background process.

How It Works

The automation follows a logical, step-by-step process to ensure reliability and efficiency.

1. Scheduled Trigger

The workflow is initiated automatically by a Schedule Trigger node. You can set it to run hourly, daily, weekly, or at any custom interval that fits your business needs, ensuring your files are synced without any manual intervention.

2. List Files from FTP

The FTP node connects to your server using your credentials and lists all files within a predefined directory path. It retrieves metadata like filenames, sizes, and modification dates, creating a queue of items to process.

3. Split into Batches

A Split in Batches node takes the file list and divides it into smaller groups. This is a critical step for handling large volumes of files without overwhelming system memory or FTP connection limits, making the transfer process stable and efficient.

4. Download Each File

For each file in a batch, the workflow uses the FTP node again in "Download" mode. It securely retrieves the file binary data from the server, preparing it for the next step.

5. Upload to Google Drive

Finally, the Google Drive node takes each downloaded file and uploads it to a specified folder in your Drive. The original filename is maintained, and you can even add timestamps or other metadata to the filename during the transfer for better organization.

Pro tip: Use the "Split in Batches" node's settings to control batch size. For many small files, a larger batch (e.g., 50) is fine. For larger files, use a smaller batch size (e.g., 5-10) to prevent timeouts.

What You'll Need

  1. An n8n Instance: A self-hosted n8n setup or a n8n.cloud account.
  2. FTP Server Credentials: Hostname, port, username, password, and the specific directory path containing the files.
  3. Google Drive Access: A Google Cloud project with the Drive API enabled, and OAuth 2.0 credentials set up in n8n to connect to your target Drive folder.
  4. A Schedule: Decide how often the transfer should run (e.g., every 6 hours, daily at 2 AM).

Quick Setup Guide

Follow these steps to get this automation running in your n8n environment in under 15 minutes.

  1. Download & Import: Click the "Download Template" button above and save the JSON file. In your n8n interface, go to Workflows > Import from File and select the downloaded JSON.
  2. Configure FTP Credentials: Click on the "FTP" node. Enter your FTP server's host, port, username, and password. Set the path to the folder you want to monitor.
  3. Set Up Google Drive Connection: Click on the "Google Drive" node. Use the OAuth2 method to authenticate n8n with your Google account. Select the destination folder ID in Drive.
  4. Adjust the Schedule: Click the "Schedule Trigger" node. Set your desired interval (Cron expression or simple schedule). For daily backups at 3 AM, you might use 0 3 * * *.
  5. Test & Activate: Click "Execute Workflow" to test with a single file. Check the execution logs to confirm the file appears in your Drive. Once verified, toggle the workflow to "Active".

Pro tip: For your first test, create a small test file in your FTP folder and run the workflow manually. This confirms all connections work before letting it run on a schedule with real data.

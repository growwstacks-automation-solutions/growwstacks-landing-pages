What This Workflow Does

Manually responding to every WhatsApp and SMS inquiry is a massive time sink for service-based businesses, agencies, and e-commerce stores. This free n8n automation template solves that by connecting GoHighLevel (GHL) with an AI agent powered by Anthropic Claude, using Redis for intelligent message buffering.

The system listens for incoming customer messages via GHL's automation triggers. When a text message arrives, it's sanitized, grouped with any rapid follow-ups using Redis, and then sent to an AI agent that generates a accurate, context-aware reply based solely on your predefined ClientInfo dataset. The final response is pushed back into GHL via a custom field, triggering an outbound SMS or WhatsApp reply automatically.

This turns your customer support from a reactive, manual task into a proactive, scalable system that works 24/7, ensuring no lead goes unanswered and freeing your team to focus on high-value conversations.

How It Works

The automation follows a logical, fault-tolerant sequence to ensure reliable, accurate customer interactions.

1. Message Reception & Filtering

A webhook in n8n receives the incoming message payload from a GoHighLevel "Customer Replied (SMS)" automation. The workflow first checks the message type. If it's an audio file, it sends a polite fallback asking the customer to send a text. For text messages, it performs a cleanup process: fixing escaped quotes, normalizing line breaks, and removing any invalid fields to create a clean input for the AI.

2. Context Buffering with Redis

Customers often send multiple messages in quick succession. To avoid sending fragmented, confusing replies, the workflow uses Redis to temporarily store and group messages that arrive within a short time window. This creates a single, coherent context for the AI, leading to more accurate and helpful responses.

3. AI-Powered Response Generation

The sanitized and buffered message is passed to an AI Agent (configured for Anthropic Claude, but compatible with OpenAI). Crucially, the agent is instructed to answer only using the provided ClientInfo tool—a dataset you maintain with your service details, branch info, pricing, and FAQs. This prevents the AI from hallucinating and ensures every reply is factual and specific to your business.

4. Response Delivery & Trigger

The AI's generated reply is sanitized once more and then written to a custom contact field in GoHighLevel (named IA_answer). Updating this field triggers a separate GHL outbound automation, which sends the final response back to the customer via SMS or WhatsApp using the Wazzap plugin. This two-automation setup within GHL keeps costs low and control high.

What You'll Need

  1. GoHighLevel Sub-Account API Key: For connecting n8n to your GHL instance to read messages and update contact fields.
  2. Anthropic API Key (or alternative LLM): To power the AI agent. The template is configured for Claude but can be adapted for OpenAI GPT or others.
  3. Redis Database Instance: A Redis server (cloud or self-hosted) for temporary message storage and context management.
  4. Configured GHL Automations: One inbound automation (trigger: Customer Replied) to send messages to n8n, and one outbound automation (trigger: Custom Field Updated) to send replies back.
  5. GHL Custom Field: Create a custom contact field named IA_answer which the workflow will update and the outbound automation will monitor.
  6. Wazzap Plugin for GHL (Optional but Recommended): To enable full WhatsApp message sending and receiving within the GHL environment.

Pro tip: Start by testing with SMS in GHL before connecting WhatsApp via Wazzap. The webhook and automation logic is identical, so you can validate the entire flow with lower-cost SMS first.

Quick Setup Guide

Follow these steps to import and configure the automation in your n8n instance.

  1. Download & Import: Click the "Download Template" button above to get the JSON file. In your n8n dashboard, go to Workflows > Import from File and select the downloaded file.
  2. Configure Credentials: In the imported workflow, set up credentials for the GoHighLevel, Anthropic (or your chosen LLM), and Redis nodes. Use the n8n credential system for secure API key storage.
  3. Set Up GoHighLevel Automations:
    • Create an Inbound Automation triggered by "Customer Replied (SMS)". Its action should be "Webhook" pointing to your n8n webhook URL (found in the Webhook node).
    • Create an Outbound Automation triggered when the custom field IA_answer is updated. Its action should send an SMS/WhatsApp message to the contact, using the field's value as the message body.
    • Create the custom field IA_answer in your GHL settings.
  4. Populate ClientInfo: Prepare a structured dataset (in a tool like Google Sheets or a database) containing all the information the AI should use—service details, branch info, FAQs. Connect this data source to the "ClientInfo" tool within the AI Agent node.
  5. Test Thoroughly: Activate the workflow and send a test SMS to your GHL number. Monitor the n8n execution to see the message flow through sanitization, Redis, AI processing, and back into GHL.

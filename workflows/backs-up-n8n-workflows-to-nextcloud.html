Automated n8n Workflow Backup to Nextcloud

What This Workflow Does

This automation solves a critical risk for businesses relying on n8n: the loss of valuable workflow logic. Manual backups are unreliable and easily forgotten. This template automatically exports all your n8n workflows via its REST API, packages them into timestamped JSON files, and securely uploads them to your Nextcloud instance.

It creates a versioned backup history, so you can track changes and revert to any previous state. This is essential for disaster recovery, team collaboration, and maintaining business continuity when your automation server encounters issues.

By automating this process, you ensure that months of workflow development and complex business logic are never lost due to server failure, accidental deletion, or corruption. It turns a high-risk manual task into a silent, reliable background process.

How It Works

The workflow acts as a bridge between n8n's internal API and your Nextcloud file storage, executing a secure, scheduled backup routine.

Step 1: Scheduled Trigger

A Cron node initiates the backup process on a schedule you define (e.g., daily at 2 AM). This ensures backups happen consistently without any manual intervention.

Step 2: Fetch All Workflows

The workflow uses the HTTP Request node to call n8n's internal API endpoint (/rest/workflows), authenticating with your instance's API key to retrieve a complete list of all workflows, including their names, IDs, and current JSON definitions.

Step 3: Process and Prepare Data

Function nodes process the API response, extracting the essential workflow data and metadata. It creates a clean, timestamped filename for each workflow (e.g., workflow-backup-2025-03-25.json) and structures the payload for upload.

Step 4: Upload to Nextcloud

The core action: the HTTP Request node sends each workflow's JSON file to your Nextcloud WebDAV or REST API endpoint. It handles authentication, creates the necessary folder structure, and ensures the file is saved with proper versioning if Nextcloud's versioning is enabled.

Step 5: Confirmation & Logging

Finally, the workflow can send a confirmation notification (e.g., via Email or Slack) upon successful backup and log the result for auditing. This gives you visibility into the backup process's success or failure.

Pro tip: Set the Cron trigger to run during low-activity hours to avoid impacting n8n's performance. Always test the restore process by downloading a backup file and importing it into a test n8n instance.

What You'll Need

  1. A running n8n instance with administrator access to generate an API key.
  2. A Nextcloud instance (self-hosted or cloud) with WebDAV or REST API access enabled.
  3. Authentication credentials for both systems (n8n API key, Nextcloud username/password or app token).
  4. Basic understanding of how to find your n8n API endpoint (https://your-n8n-domain.com/rest/) and your Nextcloud file upload endpoint.
  5. n8n or Zapier to import and run this template.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n or Zapier account.
  2. Configure the Cron trigger to set your desired backup frequency (e.g., daily, weekly).
  3. Add your n8n credentials: In the HTTP Request node, update the URL to point to your n8n instance and add your API key to the headers.
  4. Configure Nextcloud connection: Update the second HTTP Request node with your Nextcloud base URL, target folder path, and authentication details.
  5. Test the workflow manually once to ensure it can fetch workflows and upload a file to Nextcloud successfully.
  6. Activate the workflow. Turn it on and let it run on its schedule. Check your Nextcloud folder after the first run to confirm the backup files appear.

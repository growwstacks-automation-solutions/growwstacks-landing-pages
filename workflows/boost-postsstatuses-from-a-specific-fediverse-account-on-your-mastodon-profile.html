What This Workflow Does

Managing multiple social media accounts across the decentralized Fediverse—like Mastodon, Threads, and others—can be a time-consuming manual task. This automation solves that by automatically boosting (resharing) posts from a specific Fediverse account you follow directly to your Mastodon profile.

Imagine you want to share content from your company's Threads account to your professional Mastodon network without logging in and copying posts every day. This workflow runs on a schedule, checks for new posts, and boosts them for you—ensuring consistent cross-platform presence while you focus on creating content.

It's perfect for businesses, creators, and community managers who want to maintain an active Fediverse presence without the manual overhead. The workflow uses the Mastodon API and includes filtering to prevent duplicates, making it a reliable, set-and-forget solution.

How It Works

The automation follows a simple but powerful sequence to fetch and reshare content.

Step 1: Scheduled Trigger

The workflow starts with a Schedule Trigger node set to run at your preferred interval—daily, hourly, or whatever fits your posting strategy. This ensures your Mastodon profile stays active without manual intervention.

Step 2: Fetch Recent Posts

An HTTP Request node calls the Mastodon API to retrieve the latest statuses from the source account you specify. You configure it with the account's ID and your Mastodon instance URL.

Step 3: Filter for New Content

A JavaScript node filters the returned posts to only include those from the current day (or within a custom time window). This prevents reposting old content and ensures only fresh, relevant posts are boosted.

Step 4: Boost to Your Profile

For each filtered post, another HTTP Request node sends a "boost" (reblog) action to your Mastodon account using your authenticated credentials. The post appears on your profile as a reshare, maintaining attribution to the original author.

Pro tip: You can easily modify the filter logic to only boost posts containing specific hashtags, keywords, or media types, making the automation even more targeted.

What You'll Need

  1. A Mastodon account on any instance (mastodon.social, fosstodon.org, etc.) with developer access enabled.
  2. The account ID of the Fediverse profile you want to boost from (the template includes instructions on how to find this).
  3. A Mastodon access token with read:statuses and write:statuses permissions.
  4. An n8n instance (cloud or self-hosted) to run the workflow.
  5. Basic familiarity with copying API credentials—no coding required.

Quick Setup Guide

Get this automation running in under 15 minutes with these steps:

  1. Download the template using the button above and import it into your n8n workspace.
  2. Create a Mastodon application in your account settings under Development → New Application. Grant it read:statuses and write:statuses scopes.
  3. Copy your access token from the newly created application.
  4. Find the source account ID using the provided API lookup command or a simple GET request in n8n.
  5. Update the workflow nodes with your credentials, account ID, and preferred schedule.
  6. Test and activate the workflow. It will now automatically boost new posts according to your schedule.

Note: For security, always store your access token as an n8n credential rather than pasting it directly into the workflow. The template instructions show you how.

What This Workflow Does

Stop scrolling through endless GitHub issue lists. This automation turns your repository into an intelligent assistant you can have a conversation with. Instead of manually searching for "that one bug from three months ago," you can simply ask: "What authentication issues have we fixed recently?" or "Show me all open issues related to the payment gateway."

The workflow connects GitHub, OpenAI, and Redis to create a Retrieval-Augmented Generation (RAG) system specifically for your development backlog. It indexes your issues—titles, descriptions, comments, and labels—converts them into semantic vectors, and stores them for lightning-fast, context-aware search. The AI then uses this enriched context to provide precise, helpful answers about your project's history, status, and recurring problems.

This isn't just a search tool; it's a proactive analysis engine. It helps new team members get up to speed, enables product managers to spot trends without manual review, and ensures critical knowledge isn't lost when team members move on. You're building an institutional memory for your codebase.

How It Works

The automation follows a clear, modular pipeline that ingests, processes, and queries your GitHub data intelligently.

1. Data Ingestion & Preparation

The workflow first connects to the GitHub API (using a personal access token) and fetches issues from your specified repository. It can pull all issues or filter by labels, date ranges, or status (open/closed). Each issue's text content is cleaned and prepared for processing.

2. Vector Embedding Generation

Each issue's text is sent to OpenAI's embedding model (like text-embedding-ada-002). This model converts the natural language into a high-dimensional numerical vector—a mathematical representation of its semantic meaning. Issues about similar topics will have vectors that are mathematically close together in this vector space.

3. Vector Storage in Redis

These generated vectors, along with their source metadata (issue number, URL, creator, labels), are stored in a Redis database with the Redis Stack module enabled. Redis acts as a high-performance vector database, creating indexes that allow for extremely fast similarity searches across millions of vectors.

4. Natural Language Query Processing

When you ask a question like "What's blocking the next release?", that query is also converted into a vector using the same OpenAI embedding model. Redis then performs a similarity search to find the most semantically relevant issues from its stored vectors.

5. Context-Aware AI Response Generation

The top matching issues (typically 3-5) are retrieved and their full text is passed as context to an OpenAI chat model (like GPT-4). The AI synthesizes this information, referencing specific issue numbers and details, to generate a coherent, helpful answer to your original question.

6. Conversation Memory & Follow-ups

The workflow maintains conversation history, allowing for follow-up questions. The system remembers previous context, enabling natural dialogue about your codebase as if you were talking to a senior developer who's read every ticket.

What You'll Need

  1. GitHub Repository Access: A repository with issues you want to query. A GitHub Personal Access Token (with 'repo' scope) is recommended for private repos.
  2. OpenAI API Key: An active OpenAI account with API access for both embeddings (text-embedding-ada-002) and chat completions (gpt-3.5-turbo or gpt-4).
  3. Redis with Redis Stack: A Redis instance (version 7.2+) with the Redis Stack module enabled for vector search capabilities. This can be self-hosted or a cloud service like Redis Cloud.
  4. n8n Instance: A running n8n installation (self-hosted or n8n.cloud) where you can import and execute the workflow.
  5. Basic Configuration Details: Your repository owner/name, and optionally, label filters or date ranges for which issues to index.

Pro tip: For testing, you can use the free tier of Redis Cloud and OpenAI's API. The workflow includes batching to handle rate limits efficiently when indexing large repositories.

Quick Setup Guide

Get your AI GitHub assistant running in under 15 minutes with these steps:

  1. Download & Import: Click the download button above to get the JSON file. In your n8n instance, go to Workflows → Import from File and select the downloaded template.
  2. Configure Credentials: Set up the required credentials in n8n: GitHub OAuth2 (or Personal Access Token), OpenAI API Key, and Redis connection details (host, port, password).
  3. Set Repository Parameters: In the 'GitHub: Get All Issues' node, update the 'Owner' and 'Repository' fields to point to your target repo. Adjust filters (state, labels) if needed.
  4. Initialize Vector Index: Run the 'Create Redis Index' node once to set up the vector search schema in your Redis database.
  5. Run Initial Ingestion: Execute the workflow manually to fetch and vectorize your existing issues. This may take a few minutes depending on issue count.
  6. Test Your Queries: Use the 'Manual Trigger' node to ask your first question. The workflow will process it and return AI-generated answers based on your indexed issues.
  7. Schedule Updates: Set up a schedule trigger (daily/weekly) to automatically index new issues, keeping your knowledge base current.

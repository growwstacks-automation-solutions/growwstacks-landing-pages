Check for preview for a link

Automate URL validation and preview generation to ensure every shared link has proper metadata and works correctly

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves a common problem in content marketing and social media management: ensuring shared links have proper previews before distribution. Manual link checking is time-consuming and error-prone, especially when sharing multiple URLs across different platforms.

The workflow automatically validates whether a URL has a preview available, extracts rich metadata (title, description, image), and provides conditional routing based on the result. This eliminates the embarrassment of sharing broken links and ensures consistent, attractive preview cards across all communication channels.

By automating this process, businesses maintain professional presentation standards while saving significant time previously spent on manual verification. The system works silently in the background, integrating seamlessly with content creation and distribution workflows.

How It Works

Step 1: URL Input and Initial Check

The workflow begins by receiving a URL through various triggers—manual entry, webhook, or integration with other systems. The Peekalink node immediately checks if the link has preview metadata available, returning a simple true/false result.

Step 2: Conditional Logic Routing

An IF node evaluates the initial check result. If preview metadata exists (true), the workflow proceeds to extract rich information. If no preview is available (false), the system can trigger alternative actions like notification alerts or database logging.

Step 3: Metadata Extraction

For links with available previews, a second Peekalink node extracts comprehensive metadata including page titles, descriptions, featured images, and publication information. This data forms the basis for attractive social media cards and link previews.

Step 4: Output and Integration

The extracted metadata can be routed to various destinations: social media schedulers, content management systems, internal notifications, or databases. Optional No Operation nodes provide visual clarity in the workflow design without affecting functionality.

Pro tip: Connect this workflow to your content calendar or social media scheduler to automatically validate all links before publication, preventing embarrassing broken links in scheduled posts.

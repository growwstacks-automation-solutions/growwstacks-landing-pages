What This Workflow Does

Managing a busy Facebook Messenger inbox manually is time-consuming and error-prone. Important leads get buried in generic questions, and response delays can cost you sales. This automation solves that by creating a smart, AI-powered triage system for your Facebook Page messages.

The workflow automatically fetches new messages, uses GPT-4 to intelligently classify them as Leads, Customer Queries, or Spam, then routes them through a streamlined approval process in Slack. Once approved, it sends personalized responses and logs everything in Google Sheets for tracking and analysis. It turns your Messenger inbox from a chaotic stream into a structured, actionable pipeline.

How It Works

This end-to-end system connects four powerful platforms to create a seamless customer interaction workflow.

1. Trigger & Data Collection

A scheduled trigger runs hourly to check your Facebook Page for new messages via the Graph API. It extracts the sender's name, profile, message content, and timestamp, preparing the data for AI analysis.

2. AI-Powered Classification

Each message is sent to OpenAI's GPT-4 with specific instructions to categorize it and draft an appropriate reply. The AI determines if it's a sales lead requiring follow-up, a support question needing an answer, or irrelevant spam.

3. Team Approval via Slack

Classified leads and queries are sent to a designated Slack channel with buttons for "Approve" or "Edit." Your team can review the AI's suggestion in seconds and either send it as-is or modify it before responding.

4. Automated Response & Logging

Once approved, the workflow posts the response back to the customer on Facebook Messenger. Simultaneously, it logs the entire interaction—message, classification, response, and status—into a Google Sheet for reporting and future reference.

5. Error Handling

If any step fails, the workflow immediately alerts your team in Slack with details about the error, ensuring no message gets lost in a broken automation.

Pro tip: Customize the GPT-4 prompt with examples of your specific products or services to improve classification accuracy. The more context you give the AI, the better it understands your business.

  3. OpenAI API Key: GPT-4 access for intelligent message classification and response drafting.
  4. Slack Workspace: With permissions to create a bot and post to channels.

Quick Setup Guide

Get this automation running in under 30 minutes with these steps:

  1. Download & Import: Click the download button above to get the JSON file. In your n8n instance, go to Workflows → Import from File and select the downloaded template.
  2. Configure Facebook: Create a Facebook App in the Developer Portal, add the "Pages" product, and generate a Page Access Token with pages_messaging permission. Add this token to the Facebook node.
  3. Set Up OpenAI: Add your OpenAI API key to the GPT-4 node. You may want to adjust the system prompt to better match your business tone and categories.
  4. Connect Slack: Create a Slack app in your workspace, install it, and generate a Bot Token with chat:write scope. Add the token and your preferred channel ID to the Slack nodes.
  5. Link Google Sheets: Create a new Sheet and share it with your service account email. Copy the Sheet ID and tab name into the Google Sheets node.
  6. Test & Activate: Run the workflow once manually to ensure all connections work. Then activate the schedule trigger to run hourly automatically.

Important: Before going live, test with real messages in a controlled environment. Set up a separate test Facebook Page or use a sandbox Slack channel to verify the classification accuracy and response quality.

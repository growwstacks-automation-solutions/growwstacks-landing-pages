Convert Google Tasks to Google Calendar Events via Webhook

Automatically transform tasks with due dates into scheduled calendar events. Perfect for teams and individuals managing tasks from multiple sources.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves a common productivity problem: tasks with deadlines often live separately from your actual calendar, leading to missed appointments and poor time management. The workflow automatically converts Google Tasks into scheduled Google Calendar events whenever a new task with a due date is created or received via webhook.

Instead of manually copying tasks into your calendar—a tedious and error-prone process—this system creates time-blocked events automatically. It checks for duplicate events, formats dates correctly, and can even clean up the original task after scheduling, keeping your task list organized and your calendar accurately reflecting your commitments.

The webhook integration makes this particularly powerful because it allows tasks from ANY source—custom forms, chatbots, project management tools, or other automation systems—to become calendar events. This creates a seamless bridge between task management and time scheduling across your entire workflow ecosystem.

How It Works

1. Webhook Receives Task Data

The workflow starts with a webhook endpoint that listens for incoming task data. When a POST request arrives containing task title and due date (in Unix seconds), the automation triggers. This design allows integration with virtually any application that can send HTTP requests.

2. Date Processing and Formatting

The system converts the Unix timestamp into a proper date-time format for Google Calendar. It calculates the event duration (default 30 minutes, configurable) and prepares all necessary event parameters including optional descriptions, colors, and calendar selection.

3. Duplicate Checking

Before creating a new event, the workflow searches your calendar for recently updated events with similar titles and times. This prevents creating multiple calendar entries for the same task, ensuring a clean calendar without duplicates.

4. Calendar Event Creation

The system creates a new Google Calendar event with the processed task data. You can configure event colors, durations, descriptions, and which calendar to use (primary or specific calendars). All event details are properly formatted for Google's API.

5. Optional Task Cleanup

After successfully creating the calendar event, the workflow can optionally delete the original Google Task. This keeps your task list clean and prevents double-handling, though you can disable this feature if you prefer to keep tasks for tracking.

What You'll Need

  1. Google Account with access to Google Tasks and Google Calendar
  2. n8n instance (cloud or self-hosted) with webhook capabilities
  3. Google API credentials configured in n8n with appropriate scopes for Tasks and Calendar
  4. Source application that can send HTTP POST requests (forms, bots, other automation tools)
  5. Basic configuration including your preferred calendar ID, default event duration, and color settings

Pro tip: Use different calendar colors for different types of tasks (client work, internal meetings, personal tasks) to create visual categorization directly in your calendar. The workflow supports this through configuration.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance
  2. Configure Google credentials for both Tasks and Calendar in n8n's credential management
  3. Set your parameters in the Configuration node: tasklistId, calendarId (use "primary" for main calendar), defaultMinutes, and eventColor
  4. Test with sample data by sending a POST request to your webhook URL with {"TaskName": "Sample Task", "DueDateTimeSeconds": 1743454800}
  5. Activate the workflow once testing confirms events create correctly in your calendar
  6. Connect your task sources by updating your forms, bots, or other tools to send data to the webhook URL

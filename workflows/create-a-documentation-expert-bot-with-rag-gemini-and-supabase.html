How It Works

The workflow operates in two distinct phases: knowledge indexing and intelligent querying.

Phase 1: Knowledge Base Construction

The system begins by ingesting your documentation—whether from websites, PDFs, or internal wikis. It intelligently chunks the content into manageable segments, converts each chunk into numerical embeddings using Google's Gemini AI, and stores these embeddings in a Supabase vector database. This creates a searchable "memory" of your entire knowledge base.

Phase 2: Intelligent Query Processing

When a user asks a question, the workflow converts the query into embeddings, searches the vector database for the most relevant document chunks, and feeds only those relevant sections to Gemini for answer generation. This ensures responses are grounded in your actual documentation rather than the AI's general knowledge.

Phase 3: Continuous Improvement

The system includes monitoring and feedback loops. You can track which questions are asked most frequently, identify knowledge gaps in your documentation, and continuously improve both the bot's performance and your underlying knowledge base.

Quick Setup Guide

Follow these steps to deploy your documentation expert bot in under 30 minutes:

  1. Create Supabase Project: Sign up for a free Supabase account, create a new project, and run the provided SQL script to set up the vector store tables.
  2. Configure API Credentials: Add your Supabase project URL and service role key, plus your Google Gemini API key, as credentials in n8n.
  3. Import the Workflow: Download the template JSON file and import it into your n8n instance using the import functionality.
  4. Configure Document Sources: Update the HTTP Request nodes to point to your documentation URLs or connect to your internal knowledge base.
  5. Test with Sample Queries: Activate the workflow and ask test questions through the chat interface to verify accurate retrieval and responses.
  6. Deploy and Share: Once validated, share the chat interface URL with your team and monitor initial usage patterns.

What This Workflow Does

This advanced automation solves a critical bottleneck in deploying AI-powered knowledge assistants. Instead of manually uploading documents one by one to create a Lookio RAG (Retrieval-Augmented Generation) assistant, this workflow processes an entire CSV file containing your text corpus in minutes.

It transforms spreadsheet data—like FAQ entries, policy documents, or product information—into a fully functional AI assistant that can answer questions based solely on your provided content. The automation handles file conversion, API calls to Lookio, resource aggregation, and final assistant configuration, ensuring consistent, error-free deployment every time.

For businesses managing knowledge bases, this represents a shift from days of manual work to minutes of automated processing, enabling rapid creation of specialized bots for different departments, products, or use cases.

How It Works

1. CSV Upload and Parsing

The workflow begins when a user uploads a CSV file through a form interface. The CSV should contain at least two columns: one for titles and one for content. The system parses this data and prepares each row for processing.

2. Text-to-File Conversion

Each content entry from the CSV is converted into a separate text file. This transformation is crucial because Lookio's API expects file uploads, not raw text. The workflow maintains metadata and ensures proper formatting during this conversion.

3. Bulk Resource Upload to Lookio

The system loops through all generated text files and uploads them to your Lookio workspace via API calls. Each successful upload returns a unique Resource ID, which the workflow collects for the next step.

4. Assistant Creation with Limited Access

Once all resources are uploaded, the workflow aggregates the Resource IDs and creates a new Lookio Assistant. Crucially, it configures this assistant with "Limited selection" access, meaning it can only retrieve information from the uploaded resources—preventing hallucinations and ensuring accurate, relevant responses.

5. Completion and Notification

The workflow returns the new Assistant ID and a success message, completing the process. The assistant is immediately ready for use in chats, integrations, or further automation.

What You'll Need

  1. A Lookio Account: Sign up for free at lookio.app to get your API key and workspace ID.
  2. n8n Instance: Either self-hosted n8n or n8n.cloud account to run the workflow.
  3. CSV File: Structured with columns for titles and content (additional columns can be mapped as metadata).
  4. Basic API Knowledge: Understanding of how to configure API credentials in n8n nodes.
  5. Clear Use Case: Specific knowledge domain you want the assistant to specialize in.

Pro tip: Start with a small CSV (5-10 rows) to test the workflow before processing large knowledge bases. This helps you verify formatting, column mapping, and assistant behavior without consuming excessive API credits.

Quick Setup Guide

  1. Download and Import: Download the template JSON file and import it into your n8n instance.
  2. Configure Lookio Credentials: Add your Lookio API key and workspace ID to the HTTP Request nodes in the workflow.
  3. Test CSV Structure: Prepare a CSV with "Title" and "Content" columns (or adjust the column mapping in the form trigger).
  4. Activate Workflow: Turn on the workflow and use the production URL from the Form Trigger node.
  5. Upload and Verify: Upload your CSV through the form interface and verify the new assistant appears in your Lookio dashboard.

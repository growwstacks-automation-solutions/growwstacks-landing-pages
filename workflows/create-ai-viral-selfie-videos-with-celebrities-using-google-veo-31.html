What This Workflow Does

This workflow solves the time-consuming and expensive challenge of creating engaging, celebrity-style video content for social media and marketing. Instead of relying on manual video editing or closed, proprietary platforms, it builds a fully automated pipeline that generates realistic "selfie" videos featuring famous characters using Google's advanced Veo 3.1 AI model.

The process is controlled via a simple Google Sheet. You specify prompts, starting and ending images, and duration for each clip. The workflow then handles the entire lifecycle: calling the AI video API, waiting for generation, merging multiple clips into a final video, and optionally distributing it to platforms like YouTube, Google Drive, or social media schedulers. It turns a creative idea into a published video asset without manual intervention.

How It Works

The automation follows a logical, step-by-step process to ensure reliability and scalability.

1. Trigger & Data Fetching

The workflow starts manually or on a schedule. It first reads all video production instructions from a designated Google Sheet. Each row represents a video clip, containing the prompt, URLs for the first and last frame images, desired duration, and a flag for whether to include it in the final merged video.

2. AI Video Clip Generation

For each row, the workflow sends a request to the Google Veo 3.1 API (via fal.ai) to generate the video clip. This is an asynchronous process, so the workflow doesn't wait idly. It receives a job ID and moves on, setting up a tracking mechanism for each clip.

3. Status Polling & Result Collection

The workflow periodically checks the status of each video generation job. Once a job is marked "COMPLETED," it retrieves the URL of the finished video clip and writes it back into the Google Sheet, creating a live log of all assets.

4. Video Merging & Final Processing

After all individual clips are ready, the workflow collects the URLs of the clips flagged for merging. It sends this list to a video processing API (FFmpeg via fal.ai) to concatenate them into a single, seamless final video. It again polls until this merging job is complete and then downloads the final video file.

5. Distribution & Upload

The final step involves uploading the video to chosen destinations. The template includes steps for Google Drive, YouTube (via a third-party service), and social media platforms (via Postiz). These steps are modular and can be enabled or configured based on your needs.

What You'll Need

  1. An n8n instance (cloud or self-hosted).
  2. A Google Sheets document structured with columns for prompts, image URLs, duration, and merge flags.
  3. API credentials for Google Veo 3.1 (via a service like fal.ai).
  4. Google Drive access (if using the upload step) via OAuth2.
  5. Destination platform accounts (e.g., YouTube, Postiz) if you plan to use the distribution steps.

Pro tip: Start with a small batch of 2-3 video clips to test your prompt quality and API response times before scaling up to large production runs. This helps you optimize costs and outputs.

Quick Setup Guide

  1. Import the Template: Download the JSON file and import it into your n8n workspace.
  2. Prepare Your Google Sheet: Duplicate the sheet structure shown in the workflow documentation. Add your prompts, image URLs, and settings.
  3. Configure API Credentials: In n8n, set up the HTTP Request nodes with your fal.ai API key for authentication.
  4. Set Up Google Sheets & Drive: Use n8n's OAuth2 for Google to grant access to your specific sheet and a target Drive folder.
  5. Configure Destination Nodes (Optional): Enable and fill in the credentials for the YouTube upload or Postiz nodes if you wish to use them.
  6. Test Execution: Run the workflow manually with a single row in your sheet to verify the entire pipeline works from generation to final output.
  7. Schedule or Trigger: Once tested, set the workflow to run on a schedule or trigger it via a webhook to fit your content calendar.

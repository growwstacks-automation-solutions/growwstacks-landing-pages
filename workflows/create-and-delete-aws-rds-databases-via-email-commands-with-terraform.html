Automate AWS RDS Management via Email Commands

Create and delete cloud databases on-demand using simple email commands. This n8n workflow integrates Gmail, AWS RDS, Terraform, and Google Sheets for complete infrastructure automation.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Managing cloud databases shouldn't require logging into consoles or running complex CLI commands. This automation turns email into a powerful command center for your AWS RDS instances. Team members can simply send an email with commands like "Create RDS mysql-db-prod us-east-1 db.t3.small" or "Delete staging-database," and the workflow handles everything—provisioning, configuration, logging, and notifications.

Built for DevOps teams, IT departments, and platform engineers, this template eliminates manual database operations while maintaining full audit trails. It combines the convenience of email with the power of Infrastructure-as-Code (Terraform) and real-time tracking in Google Sheets.

Beyond basic creation and deletion, the system validates commands, handles errors gracefully, and sends confirmation emails—reducing provisioning time from hours to seconds while ensuring compliance and visibility.

How It Works

1. Email Command Reception

The workflow starts when a designated Gmail inbox receives an email. It triggers on new messages containing specific keywords like "Create RDS" or "Delete RDS" in the subject or body.

2. Command Parsing & Validation

n8n extracts the command type and parameters (database identifier, engine, instance class, region, credentials). It validates required fields and formats them for Terraform consumption.

3. Terraform Execution

The workflow executes Terraform plans or applies using the parsed parameters. For creation, it provisions a new RDS instance with specified configurations. For deletion, it destroys the specified database while preserving snapshots if configured.

4. Status Monitoring & Logging

While Terraform runs, the workflow waits for completion, then logs all details—database endpoint, status, creation time—to a Google Sheet. This creates an immutable audit trail.

5. Confirmation & Notification

Finally, the system sends a reply email confirming the action, including any relevant connection details or error messages. The entire team stays informed without manual follow-up.

What You'll Need

  1. AWS Account with RDS permissions and Terraform configured
  2. Gmail account or Google Workspace for receiving commands
  3. Google Sheets for logging database changes
  4. n8n instance (cloud or self-hosted) with node credentials for Gmail, AWS, and Google Sheets
  5. Terraform modules for RDS provisioning (included in template)
  6. Email security rules to restrict command senders to authorized personnel

Quick Setup Guide

Get this automation running in under 30 minutes:

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure credentials for Gmail (OAuth2), AWS (Access Key/Secret), and Google Sheets (Service Account).
  3. Set up your Google Sheet with columns for Database Identifier, Engine, Instance Class, Region, Status, and Timestamp.
  4. Test with a simple email like "Create RDS test-db us-east-1 mysql db.t3.micro" sent to your configured inbox.
  5. Monitor the Google Sheet for new entries and check your email for confirmation.
  6. Customize Terraform variables like VPC settings, security groups, and backup retention as needed.

Pro tip: Start with a non-production AWS account and small instance types for testing. Use separate Gmail labels/filters to isolate automation emails from regular correspondence.

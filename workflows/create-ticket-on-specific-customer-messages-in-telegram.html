What This Workflow Does

Customer feedback often comes through informal channels like Telegram groups, where messages can get lost in conversation. This automation solves that problem by turning unstructured Telegram messages into structured support tickets automatically.

When customers report issues, ask questions, or provide feedback in your Telegram support channel, this workflow monitors messages in real-time. It scans for specific keywords (like "bug," "error," "help," or "urgent") and automatically creates corresponding tickets in Freshdesk with appropriate tags, priority levels, and customer context.

The system doesn't just create tickets—it also sends acknowledgment messages back to customers in Telegram and creates tracking items in Monday.com for internal follow-up. This creates a complete feedback loop that ensures no customer concern goes unaddressed while giving your team full visibility.

How It Works

Step 1: Telegram Message Monitoring

The workflow continuously monitors your designated Telegram support channel or group. Using the Telegram Trigger node, it captures new messages as they arrive, including message content, sender information, and timestamps.

Step 2: Keyword Analysis & Filtering

Each message passes through a conditional logic filter that checks for predefined keywords. You can set up multiple keyword categories—for example, "urgent" keywords create high-priority tickets, while "feedback" keywords create normal-priority tickets with different tags.

Step 3: Ticket Creation in Freshdesk

When a keyword match occurs, the workflow automatically creates a ticket in your Freshdesk instance. It populates the ticket with the customer's message, sender details, and applies relevant tags based on the detected keywords. The ticket is assigned to the appropriate support queue.

Step 4: Customer Acknowledgment

Immediately after ticket creation, the system sends a confirmation message back to the customer in Telegram, letting them know their issue has been logged and providing a ticket reference number. This immediate acknowledgment improves customer experience significantly.

Step 5: Internal Tracking Setup

Finally, the workflow creates a corresponding item in Monday.com (or your preferred project management tool) for internal tracking. This ensures the support team can monitor resolution progress and maintain accountability.

Pro tip: Start with a small set of 5-7 critical keywords and expand gradually. Monitor false positives for the first week and adjust your keyword logic accordingly. Consider adding negative keywords (like "just kidding" or "not really") to filter out casual conversation.

What You'll Need

  1. A Telegram bot token and channel/group ID where customer messages arrive
  2. Freshdesk account (or alternative like Zendesk, Jira Service Desk)
  3. Monday.com account (optional, for internal tracking)
  4. n8n instance (cloud or self-hosted) with API access to the above services
  5. A list of keywords you want to monitor (start with common support terms)

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance.
  2. Configure Telegram credentials in the Telegram Trigger node with your bot token and channel ID.
  3. Set up Freshdesk connection with your API key and subdomain in the Freshdesk node.
  4. Customize keyword logic in the IF conditions to match your support terminology.
  5. Test with sample messages to ensure tickets are created with correct tags and priorities.
  6. Activate the workflow and monitor the first few days for adjustments.

Implementation note: For production use, consider adding a delay or manual approval step for ambiguous messages. You can also integrate with AI services like OpenAI to perform sentiment analysis for more sophisticated prioritization beyond simple keyword matching.

What This Workflow Does

As your n8n instance grows with dozens or even hundreds of workflows, keeping track of them all becomes a significant challenge. Which workflows use the HubSpot node? Which ones are inactive? When was this workflow last updated? Answering these questions manually is time-consuming and error-prone.

This automation solves that problem by creating a self-updating inventory system. It connects to your n8n instance via its API, fetches details about every workflow, processes the key information, and populates a Google Sheet. The result is a centralized, always-current dashboard that gives you complete visibility into your automation ecosystem.

Beyond simple listing, this system enables better governance, auditing, and optimization of your automation investments. It turns what was previously tribal knowledge into structured, searchable data that your entire team can access and understand.

How It Works

Step 1: Trigger Execution

The workflow can be initiated either on a scheduled basis (daily, weekly) using the Scheduled Trigger node, or manually via the Manual Trigger. This flexibility allows you to choose between regular automated updates or on-demand inventory refreshes.

Step 2: Fetch All Workflows

The "Get All Workflows" node connects to your n8n instance using API credentials and retrieves a complete list of all workflows along with their metadata, including names, IDs, tags, activation status, and node configurations.

Step 3: Process Each Workflow Individually

A "Split in Batches" or loop node iterates through each retrieved workflow one by one. This ensures proper handling even with large numbers of workflows and allows for individual processing and error handling per workflow.

Step 4: Extract Key Information

A custom JavaScript node processes each workflow's data, extracting essential details: workflow name, unique ID, tags, activation status, creation date, last update timestamp, and a deduplicated list of all node types used within the workflow.

Step 5: Update Google Sheet

The extracted data is formatted and sent to Google Sheets via the Google Sheets node. The workflow ID serves as a unique key to prevent duplicate entries, ensuring your inventory sheet maintains data integrity over time.

Step 6: Manage Rate Limits

A "Wait" node introduces a configurable delay between processing each workflow, preventing API rate limit issues and ensuring smooth operation even with large n8n instances.

Quick Setup Guide

  1. Download the Template using the button above and import it into your n8n instance.
  2. Configure n8n API Credentials in the "Get All Workflows" node with your instance URL and authentication token.
  3. Create Your Google Sheet with these column headers: id, title, link, tags, nodes, CreatedAt, UpdatedAt, Active, Archived.
  4. Set Up Google Sheets Connection by authenticating the Google Sheets node and entering your spreadsheet ID.
  5. Test the Workflow manually first to ensure data flows correctly from n8n to your sheet.
  6. Schedule Regular Updates by enabling the Schedule Trigger node for daily or weekly automatic refreshes.

Pro tip: Add a "Last Synced" timestamp column to your Google Sheet and configure the workflow to update it each run. This creates an audit trail of when your inventory was last refreshed.

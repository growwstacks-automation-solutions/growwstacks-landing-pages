What This Workflow Does

This workflow demonstrates how to build a custom AI agent using LangChain and JavaScript within n8n. It solves the problem of automating intelligent conversations and information retrieval without requiring extensive coding expertise. Businesses often need AI systems that can understand queries, retrieve relevant information, and generate appropriate responses—but building such systems traditionally requires deep technical knowledge.

The template provides a ready-to-use structure that handles two distinct AI tasks simultaneously: generating conversational responses (like jokes) and retrieving factual information (like historical data). By combining LangChain's framework with n8n's visual automation, you can create AI agents that interact with external APIs, process data, and deliver intelligent outputs automatically.

This automation reduces the time spent on manual research, improves response accuracy, and enables scalable AI-powered customer interactions. Whether you need an internal knowledge assistant or a customer-facing AI chatbot, this workflow provides the foundation.

How It Works

Step 1: Manual Trigger Activation

The workflow starts with a manual trigger that activates two parallel paths when you click "Execute Workflow." This allows you to test different AI functionalities simultaneously or trigger the automation based on specific events in your business processes.

Step 2: Conversational AI Path

The first path takes a preset string like "Tell me a joke" and processes it through a custom LangChain LLM chain node. This node interacts with an OpenAI node to generate context-aware conversational responses. The system understands the query intent and produces appropriate AI-generated content.

Step 3: Information Retrieval Path

The second path handles factual queries like "What year was Einstein born?" by passing them to an Agent node. This agent interacts with a Chat OpenAI node and a custom Wikipedia node (or other data sources) to retrieve, verify, and present accurate information. The agent decides which tools to use and how to process the data based on the query.

Step 4: Output Integration

Both paths produce structured outputs that can be sent to other systems—like your CRM, knowledge base, or customer communication channels. The workflow demonstrates how AI-generated content can be integrated into your existing business processes automatically.

Quick Setup Guide

  1. Download and import – Click the download button above and import the JSON file into your n8n instance.
  2. Configure OpenAI node – Add your OpenAI API key in the Chat OpenAI and OpenAI nodes.
  3. Test with preset queries – Execute the workflow to see both conversational and factual AI responses.
  4. Customize queries – Replace "Tell me a joke" and "What year was Einstein born?" with your own business questions.
  5. Connect outputs – Add nodes to send AI responses to your desired destinations (email, Slack, database, etc.).
  6. Add triggers – Replace the manual trigger with schedule-based, webhook, or event-based triggers for automation.
  7. Expand data sources – Modify the Wikipedia node to connect to your internal knowledge bases or other APIs.

What This Workflow Does

This automation solves a common problem for growing businesses: customer support requests scattered across Slack channels with no formal tracking system. Without structure, issues get missed, priorities aren't clear, and response times suffer.

The workflow creates a simple yet effective support pipeline by connecting Slack (where customers naturally communicate) with Linear (for organized ticket management). When support team members react to a Slack message with a ✅ emoji, the automation captures it, uses AI to analyze and prioritize the issue, then creates a properly formatted ticket in Linear with all relevant context.

This approach maintains the conversational ease of Slack while adding the accountability and organization of a proper ticketing system—without requiring customers to learn new tools or processes.

How It Works

1. Slack Message Capture

The workflow monitors a designated Slack channel for messages that receive a specific reaction (✅). This manual trigger ensures only genuine support issues get processed, preventing automation from creating tickets for every message.

2. AI Analysis & Classification

Each captured message is sent to an AI model that analyzes the content to determine priority (urgent/high/medium/low), categorize the issue type (bug/feature request/question), and rewrite it into a clear, actionable ticket description.

3. Linear Ticket Creation

The processed information automatically creates a ticket in your Linear project with proper labels, assignees (based on rules), and all original context including user information and message history.

4. Notification & Follow-up

The support team receives notifications in their preferred channels, and the workflow can be extended to provide status updates back to customers in Slack when issues are resolved.

What You'll Need

  1. A Slack workspace with a dedicated support channel (or channels)
  2. A Linear account with a project set up for support tickets
  3. n8n instance (cloud or self-hosted) with API access to both services
  4. AI model access (OpenAI, Anthropic, or similar) for ticket classification
  5. Basic understanding of how to set up webhooks and API connections

Pro tip: Start with a private test channel before rolling out to your main support channel. This lets you refine the AI prompts and ticket formatting without affecting real customer interactions.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure Slack connection by adding your bot token and specifying which channel to monitor.
  3. Set up Linear credentials and map fields to your project structure (labels, statuses, teams).
  4. Connect your AI provider and customize the classification prompts to match your support categories.
  5. Test with sample messages to ensure tickets are created correctly before going live.
  6. Train your team on the ✅ reaction trigger and new workflow process.

What This Workflow Does

Manually checking your Ethereum wallet balance and token holdings is a repetitive, time-consuming task that's easy to forget. This automation solves that by creating a hands-off monitoring system. It connects directly to the blockchain via Etherscan to pull your latest balance, fetches the current ETH price from CoinGecko, calculates the total USD value, and delivers a clean, formatted summary to your preferred channel—like Discord, Slack, or email.

Beyond simple balance checks, this workflow acts as your personal crypto portfolio assistant. It runs silently on a schedule you set (daily, hourly, etc.), ensuring you have a consistent record of your holdings. This is invaluable for tracking portfolio performance, preparing for taxes, or monitoring company or client wallets for unexpected activity.

The real power lies in its adaptability. While the template provides a solid foundation for a single wallet, it's built on n8n's flexible canvas. You can easily extend it to track multiple wallets, add alerts for specific price thresholds, include ERC-20 token breakdowns, or export data to a spreadsheet for long-term analysis.

How It Works

The workflow follows a logical, step-by-step process to transform raw blockchain data into an actionable alert.

1. Schedule Trigger Initiates the Check

The automation starts with a Schedule Trigger node. You configure it to run at your desired frequency—perhaps every morning at 9 AM for a daily snapshot, or every hour for active trading oversight. This node kicks off the entire process automatically, requiring zero manual intervention.

2. Fetch Wallet Data from Etherscan

An HTTP Request node calls the Etherscan API using your wallet address and an API key. It retrieves the current ETH balance and can be extended to list ERC-20 token holdings. This step gathers the foundational on-chain data.

3. Get Live Market Prices from CoinGecko

Concurrently, another node queries the CoinGecko API to fetch the latest Ethereum price in USD (or other fiat currencies). Combining this price with the balance from Etherscan gives you the real-time fiat value of your holdings.

4. Process and Format the Data

A Function or Code node takes the raw numbers—balance and price—and performs the calculation (balance * price = total value). It then structures this data into a readable message, often including emojis, markdown formatting, and percentage changes from the previous check for context.

5. Deliver the Alert via Discord

The final formatted message is sent to a Discord channel via a webhook. The Discord node posts the summary directly into your chosen server, providing instant visibility to you or your team. This step can be swapped for Slack, Telegram, Email, or any other notification channel n8n supports.

What You'll Need

  1. An n8n instance: Either n8n.cloud (cloud) or a self-hosted setup.
  2. Etherscan API Key: A free API key from etherscan.io/apis to query wallet balances.
  3. CoinGecko API Key (Optional but recommended): A free tier key from coingecko.com for reliable price data with higher rate limits.
  4. Discord Webhook URL: Create one in your Discord server under Server Settings > Integrations > Webhooks. This is where alerts will be posted.
  5. The Ethereum wallet address you want to monitor.

Quick Setup Guide

Import and configure this workflow in under 10 minutes.

  1. Download the Template: Click the "Download Template" button above to get the JSON file.
  2. Import into n8n: In your n8n workspace, go to "Workflows" > "Import from file" and select the downloaded JSON.
  3. Configure Credentials: In the workflow canvas, click the Etherscan and CoinGecko nodes to add your respective API keys as new credentials.
  4. Set Your Wallet Address: In the HTTP Request (Etherscan) node, replace the placeholder wallet address with your own.
  5. Set Your Discord Webhook: In the Discord node, paste the webhook URL from your Discord server.
  6. Adjust the Schedule: Click the Schedule Trigger node to set your preferred frequency (e.g., daily at 9 AM).
  7. Test & Activate: Click "Execute Workflow" once to test. If successful, toggle the workflow to "Active". You're done!

Pro tip: For testing, you can use the "Manual Trigger" node instead of the Schedule Trigger. This lets you run the workflow on-demand to verify everything works before letting it run automatically.

Daily Workflow Backups to GitHub with Slack Notifications

Automatically back up all your n8n workflows to a GitHub repository daily. Get Slack alerts when backups complete or fail.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves a critical problem for n8n self-hosted users: manual workflow backups are tedious, easily forgotten, and put your business automation at risk. Every day, this workflow automatically exports all your n8n workflows as JSON files and commits them to a designated GitHub repository.

It creates an organized backup structure using each workflow's unique ID as the filename, ensuring you can quickly identify and restore any automation. The workflow is designed to handle memory efficiently by processing workflows in batches, and it optionally sends a Slack notification to confirm successful completion or alert you of any failures.

This transforms a manual, error-prone task into a reliable, hands-off process that provides version control, disaster recovery, and peace of mind for your entire automation infrastructure.

How It Works

1. Trigger & Initial Setup

The workflow runs on a daily schedule (cron trigger) or can be manually executed. It begins by reading configuration settings from a dedicated Config node, which stores your GitHub repository details and target folder path.

2. Workflow Retrieval & Processing

Using n8n's native nodes, the automation fetches all workflows from your instance. It then processes them in manageable batches to prevent memory issues, extracting each workflow's JSON definition and metadata.

3. GitHub Commit & Version Control

For each workflow, the automation creates or updates a JSON file in your GitHub repository using the GitHub API. Files are organized by your specified path structure, and each commit includes descriptive messages with timestamps and workflow names.

4. Notification & Error Handling

Upon successful completion, the workflow sends a summary notification to your Slack channel. If any step fails, it immediately alerts your team with error details, ensuring you never miss a failed backup.

What You'll Need

  1. A self-hosted n8n instance (version 0.195+ recommended)
  2. GitHub account with a repository for storing backups
  3. GitHub Personal Access Token with repo permissions
  4. Slack workspace (optional, for notifications)
  5. Slack app with incoming webhook URL (if using notifications)
  6. Basic understanding of n8n workflow import/export

Quick Setup Guide

  1. Download the template JSON file from this page.
  2. Import it into your n8n instance via the workflow import feature.
  3. Configure the Config node with your GitHub repository owner, repo name, and desired folder path.
  4. Set up credentials for GitHub (using your Personal Access Token) and optionally for Slack.
  5. Adjust the schedule trigger to your preferred backup frequency (default is daily).
  6. Test the workflow manually first to verify GitHub integration works correctly.
  7. Activate the workflow and monitor the first few runs via Slack notifications or n8n execution logs.

Pro tip: Create a dedicated GitHub repository solely for n8n backups. Use branch protection rules on your main branch to prevent accidental deletions. Consider implementing a retention policy by periodically archiving old backups to reduce repository size.

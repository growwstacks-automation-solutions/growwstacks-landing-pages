How to Use workflowStaticData() in n8n

Persist variables across workflow executions for handling access tokens and temporary data

Download Template JSON · n8n compatible · Free

What This Workflow Does

This workflow demonstrates how to use the workflowStaticData() function to set any type of variable that will persist within workflow executions. This is particularly useful when working with access tokens that expire after a certain time period or any temporary data that needs to be reused between executions.

By storing this data persistently, you can significantly reduce redundant API calls for token generation and maintain state between workflow executions, making your automations more efficient and cost-effective.

How It Works

1. Initialization

The workflow checks if static data exists and is still valid (for tokens with expiration). If not, it generates new data through API calls or other methods.

2. Data Storage

Using workflowStaticData(), the workflow stores the generated data (like tokens and expiration timestamps) in a persistent storage that survives between executions.

3. Subsequent Executions

Future executions first check the static data before making new API calls, using the persisted data if it's still valid.

What This Workflow Does

This n8n workflow solves a critical problem for hosting providers and managed service businesses: manually deploying Docker containers for each client is time-consuming, error-prone, and unscalable. The template provides an API backend that connects your WHMCS or WISECP billing system directly to your Docker servers.

When a client purchases a Docker Immich photo management service, the workflow automatically provisions the container, configures NGINX proxies, sets up storage volumes, and delivers a ready-to-use service within minutes. It also handles suspension when payments fail, upgrades when plans change, and monitoring for all deployed instances.

This turns Docker hosting from a technical manual task into a fully automated product that generates recurring revenue with minimal ongoing effort.

How It Works

Step 1: API Webhook Receives WHMCS/WISECP Commands

The workflow starts with a secure webhook endpoint that accepts authenticated commands from your billing system. When a client action occurs (purchase, suspension, upgrade), WHMCS/WISECP sends a structured request to this API.

Step 2: Command Processing and Parameter Mapping

n8n processes the incoming command, extracts client details, service parameters, and action type. It maps these to template variables like server domain, client directory paths, and Docker Compose configurations.

Step 3: SSH Connection to Docker Server

The workflow establishes a secure SSH connection to your Docker server using stored credentials. It executes bash scripts that check server status, validate resources, and prepare the environment for the new container.

Step 4: Docker Container Deployment

Based on the parameters, n8n generates and executes a Docker Compose file specifically tailored for the client. It pulls the Immich image, configures volumes, sets environment variables, and starts the container with proper networking.

Step 5: NGINX Proxy Configuration

The workflow automatically configures an NGINX reverse proxy to expose the Immich web interface securely. It sets up SSL, custom headers, and access rules based on the client's domain and security requirements.

Step 6: Response Delivery to Billing System

Finally, n8n returns a JSON response to WHMCS/WISECP confirming successful deployment, providing access details, and updating the service status in the client's account. This completes the automated provisioning loop.

Quick Setup Guide

Follow these steps to implement the template in your environment:

  1. Import the Template: Download the JSON file and import it into your n8n instance via the workflow import feature.
  2. Configure Credentials: Create n8n credentials for Basic Auth (webhook security) and SSH (server access).
  3. Adjust Parameters: Update the workflow parameters: server_domain, clients_dir, mount_dir to match your server setup.
  4. Test Webhook: Send a test request from WHMCS/WISECP or manually to verify the API endpoint responds correctly.
  5. Connect Billing System: Configure WHMCS/WISECP module to call the n8n webhook URL for provisioning actions.
  6. Monitor First Deployment: Watch the workflow execute for a test client purchase and verify container creation.

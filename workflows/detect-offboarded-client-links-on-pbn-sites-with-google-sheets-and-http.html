What This Workflow Does

Managing a Private Blog Network (PBN) comes with significant risk management responsibilities. When a client project ends and their domain is removed from your services, any remaining links to that domain across your PBN can trigger Google penalties, waste valuable crawl budget, and point visitors to dead destinations. Manual checking of dozens or hundreds of PBN sites is impractical, error-prone, and doesn't scale.

This n8n workflow automates the entire audit process. It reads your PBN site list from Google Sheets, fetches the live HTML of each site via HTTP requests, compares it against a master list of offboarded client domains, and automatically logs any matches back into your tracking sheet. The system intelligently skips already-processed rows and includes built-in pauses to prevent rate limiting.

For SEO agencies and link builders, this represents a fundamental shift from reactive manual checking to proactive, automated compliance monitoring. You can run audits weekly, monthly, or whenever a client offboards—ensuring your network remains clean, compliant, and focused only on active, valuable links.

How It Works

The workflow follows a logical, step-by-step process that mimics what an SEO specialist would do manually—but at machine speed and with perfect accuracy.

1. Trigger & Data Collection

The workflow starts with a manual trigger (which can be replaced with a schedule). It first reads all PBN sites from your designated Google Sheet, including URLs and row identifiers. A filter node immediately excludes any rows that have already been processed, marked by a filled "Offboarded Links" column.

2. Site Processing Loop

Each remaining PBN URL enters an iterative loop. For every site, the workflow sends an HTTP GET request to fetch the live HTML content. Retry logic handles temporary network issues, ensuring reliable data collection even with intermittent connectivity.

3. Domain Matching Logic

Simultaneously, the workflow reads your "offboard projects" sheet containing all removed client domains. A custom code node performs intelligent text matching against the fetched HTML, searching for any occurrence of these domains—whether in links, text content, or meta tags.

4. Results Logging & Continuation

When a match is found, the workflow structures the data (PBN URL, matched domain, timestamp) and writes it directly to the corresponding row in your PBN tracking sheet. A brief pause between iterations prevents API rate limits and allows for smooth, observable progress through large site lists.

What You'll Need

  1. Active n8n Instance: Either self-hosted n8n or n8n.cloud account with workflow execution capabilities.
  2. Google Sheets Access: OAuth credentials configured in n8n for reading from and writing to Google Sheets.
  3. PBN Tracking Sheet: A Google Sheet with columns for Site URL, Offboarded Links (initially empty), and a unique row_number identifier.
  4. Offboard Projects Sheet: A separate sheet or tab listing all removed client domains in Column A (one per row).
  5. Network Access: Your n8n instance needs internet access to fetch HTML from your PBN sites.

Pro tip: Use a dedicated Google Service Account for automation rather than personal credentials. This ensures uninterrupted access even if team members leave, and provides better audit trails for compliance purposes.

Quick Setup Guide

You can have this automation running in under 15 minutes with these simple steps:

  1. Download & Import: Download the JSON template and import it into your n8n instance via the workflow import function.
  2. Configure Google Sheets Credentials: Set up OAuth authentication for Google Sheets in n8n's credentials section if you haven't already.
  3. Update Sheet URLs: In the "Read PBN Sites from Sheet" node, paste the URL of your PBN tracking sheet. In the "Read Offboarded Project Domains" node, update to your offboard projects sheet.
  4. Verify Column Mapping: Ensure the sheet column names in the workflow nodes match your actual sheet headers (typically "Site URL", "Offboarded Links", "row_number").
  5. Test with One Site: Run the workflow manually with a single PBN URL first to verify the connection and data flow works correctly.
  6. Schedule Regular Runs: Replace the manual trigger with a Schedule Trigger node to run weekly or monthly automatically.

What This Workflow Does

Manual company research on Dun & Bradstreet (DNB) is a tedious, time-consuming process that involves searching, copying details, and organizing data into spreadsheets. This workflow automates the entire pipeline, transforming hours of work into minutes.

It performs automated search, scraping, and intelligent extraction of DNB company profiles. Starting with a search query, it uses Bright Data's MCP agents to find the relevant DNB page via Google, scrapes the content, and then employs OpenAI's 4o mini model to parse the unstructured markdown into clean, structured business data—like company name, size, address, and industry—ready for analysis, CRM import, or reporting.

How It Works

1. Input & Search Initiation

You provide a company search query (e.g., "software startups Berlin"). The workflow triggers a Bright Data MCP Client to perform a targeted Google search to find the corresponding DNB company profile page.

2. Data Acquisition

Using Bright Data's Web Unlocker, the workflow navigates to and scrapes the target DNB page, returning the full content as clean markdown text, bypassing anti-bot measures and ensuring reliable data access.

3. AI-Powered Structuring

The raw markdown is sent to an OpenAI LLM (like GPT-4o mini) with a tailored prompt. The AI identifies and extracts key business fields—company name, employee size, physical address, industry classification, and other relevant details—outputting a consistent JSON structure.

4. Output & Delivery

The structured data is sent to a webhook of your choice for immediate integration (e.g., into a CRM like HubSpot or a database) and is also saved locally to disk for logging and backup purposes, completing the automated intelligence cycle.

What You'll Need

  1. A Self-Hosted n8n Instance: This workflow uses community nodes (MCP Client) which require a self-hosted n8n setup.
  2. Bright Data Account: For reliable web scraping and search capabilities via their MCP server and Web Unlocker proxy.
  3. OpenAI API Key: To power the AI extraction model (GPT-4o mini or similar).
  4. Technical Setup: Installation of the Bright Data MCP server (@brightdata/mcp) and the n8n MCP nodes on your local machine.

Pro tip: Start with a small batch of test companies to fine-tune the AI extraction prompt for your specific needs—like prioritizing certain fields (e.g., "estimated revenue" or "year founded")—before scaling to hundreds of searches.

Quick Setup Guide

  1. Import the Template: Download the JSON file and import it into your self-hosted n8n instance.
  2. Configure Credentials: Set up your OpenAI account credentials within n8n. Create and configure the "MCP Client (STDIO)" credentials to connect to your locally running Bright Data MCP server, ensuring your Bright Data API token is correctly placed in the environment variables.
  3. Set Your Bright Data Zone: In your Bright Data control panel, create a "Web Unlocker" proxy zone (e.g., mcp_unlocker) as outlined in their documentation.
  4. Customize Inputs: Update the "Set" node with your target search query and the webhook URL where you want to receive the structured data.
  5. Define Output Path: Adjust the "Save to Disk" node to specify your desired local file path and naming convention for the output JSON files.
  6. Test & Run: Execute the workflow once to verify all connections work and the output matches your expectations, then schedule it or trigger it via webhook for ongoing automation.

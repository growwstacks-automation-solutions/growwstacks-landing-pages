What This Workflow Does

Manual web research is time-consuming and inconsistent. Whether you're building a dataset for AI training, conducting competitive analysis, or performing SEO audits, you need to visit multiple pages, follow links, and extract clean text—a process that can take hours for even modest websites.

This n8n automation solves that problem by implementing a recursive web crawler that starts from a single URL and systematically explores linked pages within the same domain. It controls how deep it goes (default: 3 levels), extracts meaningful text content while filtering out navigation and boilerplate, and returns structured data ready for analysis or integration with other systems.

The workflow handles the complexities of web crawling: avoiding infinite loops, managing visited URLs, filtering out irrelevant links (PDFs, anchors, external sites), and structuring the output. What would take a human researcher hours to accomplish manually happens automatically in minutes with perfect consistency.

How It Works

1. Webhook Trigger & Initialization

The workflow begins with a webhook that accepts a JSON payload containing the starting URL. It extracts the domain, sets crawl parameters (including maximum depth), and initializes data structures to track visited pages, pending URLs, and collected content.

2. Recursive Page Fetching & Link Extraction

For each URL in the queue, the workflow fetches the page using an HTTP Request node. It then uses an HTML node to extract both the main text content and all hyperlinks on the page. The text is cleaned of excessive whitespace and irrelevant markup.

3. Intelligent Link Filtering & Queue Management

Extracted links undergo sophisticated filtering: external domains are excluded, anchors (#), mailto:, tel:, and javascript: links are removed, and document files (.pdf, .docx, etc.) are filtered out. Valid internal links are added to the queue if they haven't been visited and if the current depth hasn't exceeded the maximum.

4. Depth Control & Loop Management

The workflow uses n8n's SplitInBatches node to process URLs efficiently while maintaining depth tracking. Each "level" of links represents one depth increment. The automation stops when either all reachable pages have been visited or the maximum depth has been reached.

5. Data Compilation & Output

As pages are crawled, their URL, depth, and extracted content are stored. When crawling completes, all content is combined into a structured JSON response containing both individual page results and a combined text corpus, ready for downstream processing.

Quick Setup Guide

  1. Import the Template: Download the JSON file and import it into your n8n instance via the "Workflows" section.
  2. Configure the Webhook: Note the webhook URL generated by n8n—this is your endpoint for triggering crawls.
  3. Test with a Simple URL: Send a POST request to your webhook with JSON like {"url": "https://example.com"} using curl, Postman, or another HTTP client.
  4. Adjust Parameters if Needed: Modify the "maxDepth" value in the "Init Crawl Params" Set node if you need shallower or deeper crawling.
  5. Connect to Destinations: Add nodes after the final webhook response to send extracted content to databases, file storage, or other applications.
  6. Schedule or Trigger: Set up the workflow to run on a schedule or trigger it via external events as needed.

Pro tip: Start with a depth of 2-3 for most websites. Very large sites or those with complex navigation might require careful depth limiting to avoid excessive crawling. Always test with a small site first to understand the output structure.

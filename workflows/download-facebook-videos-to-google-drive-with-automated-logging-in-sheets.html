Download Facebook Videos to Google Drive with Automated Logging

Automatically fetch Facebook videos, convert to MP4, upload to Drive, and log results in Sheets—all without manual work.

What This Workflow Does

Manual Facebook video downloading is a tedious, time-consuming process for marketing teams, content creators, and social media managers. You have to find the video, download it locally, manually upload to cloud storage, then update spreadsheets to track what was downloaded—often repeating this dozens of times per week.

This automation workflow solves that entirely. It automatically fetches Facebook videos via API, converts them to downloadable MP4 files, uploads them directly to your Google Drive folder with proper sharing permissions, and logs every successful and failed download in a Google Sheets spreadsheet. The entire process runs automatically whenever you submit a video URL.

The result is a streamlined content pipeline where videos move from Facebook to organized cloud storage with complete tracking—eliminating hours of manual work each week and ensuring your video library is consistently managed.

How It Works

Step 1: Trigger with Video URL

The workflow starts when you submit a Facebook video URL through a simple form interface. This trigger can be customized to accept multiple URLs, include metadata like campaign names, or even run on a schedule for batch processing.

Step 2: Fetch Video via API

The system sends the URL to a Facebook Video Downloader API which returns downloadable MP4 links. This step handles the conversion from Facebook's native format to standard MP4 files ready for storage.

Step 3: Validate and Download

An "If" node checks the API response for errors before proceeding. If successful, the MP4 Downloader node retrieves the video file from the provided media URL. Failed downloads are routed to a separate logging path.

Step 4: Upload to Google Drive

The downloaded file is automatically uploaded to your designated Google Drive folder. You can configure folder organization by date, campaign, or content type.

Step 5: Set Sharing Permissions

The workflow sets appropriate permissions on the uploaded file—typically making it shareable with your team or clients while maintaining security controls.

Step 6: Log Success in Sheets

Successful downloads are logged in Google Sheets with the original Facebook URL, Drive link, timestamp, and any custom metadata. This creates your content tracking database.

Step 7: Handle Failed Downloads

Failed conversions are logged after a brief wait period (to avoid rapid writes) with "N/A" as the Drive URL and error details for troubleshooting.

What You'll Need

  1. A Facebook Video Downloader API key (available through RapidAPI or similar services)
  2. Google Drive account with a designated folder for video storage
  3. Google Sheets spreadsheet prepared for logging (columns for URL, Drive link, timestamp)
  4. n8n instance (cloud or self-hosted) to run the workflow
  5. Basic understanding of n8n node configuration (credentials setup)

Pro tip: Before running the workflow at scale, test with a few videos to ensure your API returns consistent MP4 links and your Drive folder permissions are correctly set.

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n instance.
  2. Configure the Facebook Video Downloader API node with your RapidAPI credentials.
  3. Set up Google Drive credentials and specify your target folder ID.
  4. Configure Google Sheets credentials and specify your logging spreadsheet ID.
  5. Test the workflow by submitting a Facebook video URL through the form trigger.
  6. Monitor the Sheets log to confirm successful downloads and troubleshoot any errors.
  7. Customize nodes as needed—add email notifications, change storage destinations, or integrate additional quality options.

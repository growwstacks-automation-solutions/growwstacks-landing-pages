What This Workflow Does

This automation solves a common business problem: critical emails get buried in crowded inboxes, causing delays and missed opportunities. The workflow monitors your email account, detects new messages, and instantly sends notifications to your Telegram channel or group. But it goes further—it also converts the email content into a temporary HTML page hosted on GitHub Gist, creating a secure, shareable preview that automatically expires after a set time.

This approach is particularly valuable for teams sharing subscription access, distributing one-time codes, or reviewing purchase confirmations without exposing sensitive email credentials. Instead of forwarding entire emails (which can contain metadata and thread history), you get clean, temporary web pages that team members can view on any device, with automatic cleanup built in.

How It Works

The automation follows a logical sequence that ensures reliability and security while minimizing manual intervention.

Step 1: Email Detection

The workflow uses the IMAP email trigger to continuously monitor your specified email account. When a new email arrives, it captures the sender, subject, body content, and any relevant metadata. This trigger can be configured to watch specific folders, filter by sender, or match certain subject patterns to reduce noise.

Step 2: Content Processing

The email body is processed and converted into clean HTML format. This step handles character encoding issues, removes problematic special characters that Telegram might reject, and structures the content for web display. The system can extract key information like verification codes, order numbers, or customer details for inclusion in the Telegram alert.

Step 3: Temporary Hosting

Using GitHub Gist's API, the workflow creates a secret Gist containing the HTML-formatted email content. This generates a unique, unguessable URL that serves as a temporary web page. The Gist is configured as "secret" (not public) and includes appropriate metadata for tracking. This approach eliminates the need for web hosting infrastructure while providing reliable, fast content delivery.

Step 4: Telegram Notification

A formatted message is sent to your designated Telegram chat or channel. This includes the email subject, sender information, a brief excerpt, and the secure GitHub Gist URL for full content viewing. The notification can include custom formatting, emojis, or buttons depending on your Telegram bot configuration.

Step 5: Automatic Cleanup

After a configurable delay (typically 1-24 hours), the workflow automatically deletes the GitHub Gist, removing the temporary web page. This ensures sensitive information isn't permanently accessible and helps maintain data hygiene. The delay period can be adjusted based on your security requirements and how long team members typically need to review the content.

Pro tip: Configure the workflow to watch specific email senders or subjects to reduce notification noise. For example, only trigger on emails from your payment processor or support system rather than all incoming messages.

What You'll Need

  1. Email account with IMAP access: Most modern email providers (Gmail, Outlook, Yahoo, etc.) support IMAP. You'll need the server address, port, and app-specific password if using two-factor authentication.
  2. Telegram bot token: Create a bot via @BotFather on Telegram and save the API token. You'll also need the chat ID of the channel or group where notifications should appear.
  3. GitHub personal access token: Generate a token with "gist" permissions from your GitHub account settings. This allows the workflow to create and delete Gists programmatically.
  4. n8n instance: Either self-hosted n8n or n8n.cloud account with webhook capability for continuous operation.
  5. Basic understanding of web URLs: No coding required, but you should be comfortable copying API credentials and testing web links.

Quick Setup Guide

Follow these steps to implement this automation in your environment:

  1. Download and import: Click the download button above to get the JSON file. In your n8n instance, go to Workflows → Import from file and select the downloaded template.
  2. Configure email trigger: Open the Email Trigger (IMAP) node and enter your email server details, username, and password. Test the connection to ensure emails can be detected.
  3. Set up Telegram: In both Telegram nodes, enter your bot token and chat ID. Send a test message to verify the connection works correctly.
  4. Connect GitHub: In the HTTP Request nodes for GitHub Gist, add your personal access token in the authentication section. The workflow includes the proper API endpoints for creating and deleting Gists.
  5. Adjust timing: Modify the Wait node to set how long temporary links should remain active before automatic deletion. Start with 2-4 hours and adjust based on your team's review patterns.
  6. Test thoroughly: Send a test email to your monitored account and verify that the Telegram notification arrives with a working link. Check that the link expires after the configured time.

Pro tip: Before going live with important emails, test with a dedicated test email account for a day or two. This helps you fine-tune filtering, formatting, and timing without affecting real business communications.

What This Workflow Does

If you're using Glasp to highlight articles, research papers, or online content, you know the frustration of having valuable insights trapped in yet another app. This workflow solves that by automatically exporting your Glasp highlights to wherever you need them—Notion for personal knowledge bases, Slack for team sharing, Google Sheets for analysis, or any webhook for custom integrations.

The automation runs on a schedule (default every 6 hours), fetches new highlights via the Glasp API, filters out already-exported content to avoid duplicates, formats the data, and sends it to your chosen destination. It turns your scattered highlights into organized, actionable knowledge without any manual work.

For researchers, content creators, and teams building knowledge bases, this represents hours saved each week. Instead of copying and pasting between apps, you get a seamless pipeline that ensures every insight you capture is immediately available where you actually use it.

How It Works

The workflow follows a logical sequence to ensure reliable, duplicate-free exports.

1. Schedule Trigger

The workflow starts with a schedule trigger that runs automatically at your chosen interval (default: every 6 hours). This ensures your highlights are regularly synced without manual intervention.

2. Prepare Parameters & Deduplication

A Code node calculates the time range for fetching new highlights and manages deduplication tracking. It checks what's already been exported and prepares the API query parameters to only fetch new content.

3. Fetch Highlights via Glasp API

An HTTP Request node with Bearer authentication calls the Glasp API to retrieve your highlights. It handles pagination to ensure all your data is captured, not just the first page.

4. Filter & Format Data

Another Code node processes the raw API response, extracting and formatting the key information: article titles, URLs, highlights in plain text and markdown, tags, authors, and timestamps. This creates clean, structured data ready for any destination.

5. Send to Your Chosen Destination

The formatted data flows to your connected destination node—whether that's creating Notion pages, posting to Slack channels, appending to Google Sheets, or triggering webhooks for custom integrations.

Pro tip: You can easily add multiple destination branches to send the same highlights to Notion, Slack, and Google Sheets simultaneously—creating both personal archives and team-shared knowledge in one workflow.

What You'll Need

  1. A Glasp account with saved highlights
  2. Your Glasp access token (available at glasp.co/settings/access_token)
  3. Access to n8n (cloud or self-hosted)
  4. Credentials for your destination(s): Notion integration token, Slack webhook, Google Sheets connection, or webhook URL
  5. Basic familiarity with connecting API services in n8n (the template handles the complex parts)

Quick Setup Guide

You can have this automation running in under 5 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Get your Glasp token by visiting glasp.co/settings/access_token and copying your personal access token.
  3. Configure the Glasp API node by creating a new Bearer Auth credential with your token.
  4. Connect your destination by setting up the appropriate node (Notion, Slack, Google Sheets, or Webhook) after the "Filter & Format" node.
  5. Test and activate by running the workflow once manually to verify everything works, then click "Publish" to let it run automatically on schedule.

Pro tip: Start with one destination (like Notion) to verify the flow works perfectly, then duplicate the branch to add additional destinations like Slack or Google Sheets.

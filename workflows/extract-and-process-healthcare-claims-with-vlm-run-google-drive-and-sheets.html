Healthcare Claims Automation with VLM Run & Google Sheets

Automatically extract and process healthcare claims from Google Drive into structured Google Sheets using AI-powered VLM Run extraction.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Healthcare claims processing is a time-consuming administrative task that involves manually reviewing documents, extracting data, and entering it into systems for billing and tracking. This workflow automates that entire process.

It monitors a designated Google Drive folder for new healthcare claim documents—whether they're PDFs, scanned images, or digital forms. When a new file appears, it downloads it, sends it to VLM Run for AI-powered extraction of key information, fetches additional structured data from healthcare processing domains, and then appends all normalized fields as a new row in a Google Sheet.

The result is a fully automated pipeline that turns unstructured claim documents into structured, searchable data without any manual intervention.

How It Works

Step 1: Monitor Google Drive

The workflow starts with a Google Drive Trigger node that watches a specific folder for new files. Whenever a healthcare claim document is uploaded—by a patient, provider, or staff member—the workflow immediately detects it and begins processing.

Step 2: Download and Prepare

The file is downloaded inside n8n for local processing. This ensures the document is available for analysis while maintaining security and control over the data flow.

Step 3: AI Extraction with VLM Run

The document is sent to VLM Run, an AI service that performs transcription, analysis, and data extraction. It identifies key fields like patient name, insurance details, charges, dates, physician information, and other claim-specific data regardless of the document's format or layout.

Step 4: Data Enhancement

Additional structured data from healthcare claims processing domains is fetched as JSON to enrich the extracted information. This ensures comprehensive field coverage and standardized formatting.

Step 5: Append to Google Sheets

All normalized fields are appended as a new row in a predefined Google Sheet. The sheet serves as a centralized, searchable database of all processed claims with consistent columns for easy analysis and reporting.

What You'll Need

  1. A Google account with access to Google Drive and Google Sheets
  2. VLM Run API credentials (available from their dashboard)
  3. An n8n instance (cloud or self-hosted)
  4. A Google Drive folder designated for claim document intake
  5. A Google Sheet with predefined columns for claim data

Pro tip: Create your Google Sheet columns before setting up the workflow. Include fields like timestamp, file_name, patient_info, insurance_details, charges, dates, and processing_notes to ensure comprehensive data capture.

Quick Setup Guide

  1. Create or identify a Google Drive folder for claim documents and note its Folder ID.
  2. Create a Google Sheet with appropriate column headers for your claim data structure.
  3. Configure Google Drive OAuth2 authentication in n8n for the trigger and download nodes.
  4. Add your VLM Run API credentials to the VLM Run node in n8n.
  5. Configure Google Sheets OAuth2 and specify your Spreadsheet ID and target sheet.
  6. Upload a sample claim document to your Drive folder and activate the workflow to test.

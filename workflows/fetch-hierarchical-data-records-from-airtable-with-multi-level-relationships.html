What This Workflow Does

Many businesses use Airtable to manage complex relationships: projects with tasks, customers with orders, products with variants, or organizations with departments. While Airtable excels at storing these relationships, extracting the complete hierarchy manually is time-consuming and error-prone.

This automation solves that problem by fetching a parent record along with all its linked child records (and optionally grandchild records) in a single operation. It assembles everything into a clean, nested JSON structure that's ready for reporting, integration with other systems, or triggering downstream business processes.

Instead of manually clicking through links or running multiple queries, you get the entire data tree automatically—saving hours of administrative work while ensuring data consistency and completeness.

How It Works

1. Input Configuration

The workflow accepts parameters specifying which Airtable base, table, and record to fetch. You can optionally specify which linked fields should fetch third-level records, optimizing performance by only going deeper where needed.

2. Parent Record Retrieval

It first fetches the main record you're interested in—whether that's a project, customer, product, or any other entity that serves as the root of your hierarchy.

3. Child Record Aggregation

Using Airtable's filterByFormula capability, it efficiently fetches all linked records in batch operations rather than making individual API calls for each relationship. This minimizes API usage and maximizes speed.

4. Optional Grandchild Fetching

For specified relationships, it can fetch a third level—like tasks under projects that have subtasks. This selective depth control prevents unnecessary data retrieval while maintaining flexibility.

5. Structured JSON Assembly

All records are organized into a hierarchical JSON object that mirrors your business relationships, ready for immediate use in applications, dashboards, or further processing.

What You'll Need

  1. An Airtable account with bases containing linked records
  2. n8n installed (cloud or self-hosted)
  3. Airtable API credentials with access to your bases
  4. Basic understanding of your data structure (which tables link to which)
  5. Optional: The "marked" npm package if you want rich text conversion to HTML

Quick Setup Guide

1. Download the template and import it into your n8n instance.

2. Configure your Airtable credentials in the Airtable nodes.

3. Modify the input parameters to match your base ID, table ID, and record ID.

4. Test with a single record to verify the hierarchy extraction works correctly.

5. Connect the output to your desired destination—whether that's another application, a webhook, or a data storage system.

6. Schedule the workflow or trigger it via webhook based on your business needs.

Pro tip: Start by testing with a simple 2-level hierarchy before adding third-level fetches. This helps you verify the basic structure works before optimizing performance with selective depth control.

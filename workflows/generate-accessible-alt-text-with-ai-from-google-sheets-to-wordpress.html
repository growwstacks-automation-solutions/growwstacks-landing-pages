Generate Accessible Alt Text with AI from Google Sheets to WordPress

Automate WCAG-compliant alt text generation for your entire WordPress media library using AI and a simple spreadsheet.

What This Workflow Does

Manually writing alt text for hundreds of WordPress images is a tedious, error-prone task that often gets neglected. This creates accessibility barriers, hurts your SEO, and exposes your business to compliance risks. This automation solves that by connecting Google Sheets, AI (Claude), and WordPress to generate descriptive, compliant alt text at scale.

The workflow reads a list of image URLs and WordPress media IDs from a Google Sheet, sends each image to an AI model for analysis, generates concise alt text (under 125 characters), and then updates the corresponding media file in your WordPress library via its REST API. It includes smart error handling to skip unsupported formats and continue processing, making it robust for large media libraries.

Beyond saving dozens of hours, this ensures every image on your site meets WCAG (Web Content Accessibility Guidelines) standards, improves your site's SEO through better image context, and provides a consistent, professional experience for all users.

How It Works

The automation follows a clear, sequential process to transform raw image data into accessible content.

Step 1: Data Extraction from Google Sheets

The workflow starts by reading a specially formatted Google Sheet. One tab contains your exported WordPress media list with columns for Media ID and Image URL. A second tab securely stores your WordPress admin credentials. It authenticates with Google Sheets and retrieves all necessary data to begin processing.

Step 2: AI-Powered Image Analysis

For each image URL in the sheet, the workflow sends the image to the Claude AI API. A custom prompt instructs the AI to generate a concise, descriptive alt text suitable for screen readers and SEO, strictly under 125 characters. The AI analyzes visual content, context, and any provided metadata.

Step 3: WordPress Media Library Update

Using the retrieved WordPress credentials, the workflow constructs a Base64 authentication token. It then calls the WordPress REST API for each media ID, posting the newly generated alt text directly to the `alt_text` field of the media attachment. This update happens in real-time.

Step 4: Logging & Error Handling

The workflow writes the generated alt text back to the Google Sheet for a clear audit trail. If an image format is unsupported or an API call fails, the process is designed to skip that item gracefully and continue with the next row, ensuring batch processing completes.

What You'll Need

  1. A WordPress website with the REST API enabled and Application Passwords configured (under User Profile).
  2. A Google Sheets document formatted with your media export data (Media IDs and URLs).
  3. An Anthropic Claude API key (or another AI model API key you can adapt the workflow to use).
  4. The "Export Media URLs" WordPress plugin to generate the initial spreadsheet of your media library.
  5. An n8n instance (cloud or self-hosted) to run the workflow.

Pro tip: Use the duplicate link for the pre-formatted Google Sheets template included in the workflow notes. This ensures your sheet has the correct column structure (ID, URL, Alt Text) and a separate tab for secure credential storage.

Quick Setup Guide

Follow these steps to get your automated alt text pipeline running in under 30 minutes.

  1. Export Your WordPress Media: Install the "Export Media URLs" plugin. Run an export, ensuring the ID and URL columns are selected, and paste the data into the "Export media" tab of the provided Google Sheets template.
  2. Configure WordPress Application Passwords: In your WordPress admin, go to Users → Your Profile. In the "Application Passwords" section, create a new password (e.g., "n8n API") and copy it immediately into the "Infos client" tab of your sheet.
  3. Duplicate the Template Sheet: Use the link in the workflow description to create your own copy of the pre-structured Google Sheet. Fill in your WordPress site URL (without https://), username, and the application password.
  4. Configure n8n Credentials: In your n8n dashboard, add credentials for Google Sheets and the Anthropic Claude (or chosen AI) API.
  5. Import & Run: Import the downloaded JSON template into n8n. Update the Google Sheets node with your new sheet's URL. Execute the workflow once to test, then schedule it for regular runs.

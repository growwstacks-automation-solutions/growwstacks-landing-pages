What This Workflow Does

When teams collaborate in Notion, comment sections can become lengthy discussion threads with important decisions, feedback, and action items scattered throughout. Manually reviewing these conversations is time-consuming and critical points often get overlooked. This automation solves that problem by using AI to regularly scan your Notion pages, analyze new comments, and generate intelligent summaries that capture the essence of the discussion.

The workflow automatically detects when new comments are added to specified Notion databases or pages, processes the conversation through an AI model of your choice (Gemini, ChatGPT, Claude, etc.), and adds the summary back to the page as a property. It also tracks when summaries were last generated to avoid redundant processing, ensuring your team always has an up-to-date understanding of ongoing discussions without manual effort.

How It Works

1. Scheduled Monitoring

The workflow runs on a schedule you define (hourly, daily, or weekly) to check your specified Notion databases for pages with new comments since the last execution. This trigger ensures the automation only processes active discussions without overwhelming your systems.

2. Comment Extraction & Processing

For each page with new activity, the workflow retrieves all comments, organizes them chronologically, and prepares them for AI analysis. It filters out system notifications and focuses on human-generated content to ensure the summary reflects actual team discussion.

3. AI-Powered Summarization

The collected comments are sent to your preferred AI model with a customizable prompt that instructs it to identify key decisions, action items, unresolved questions, and consensus points. You can tailor this prompt to match your team's specific reporting needs.

4. Summary Integration

The generated summary is added to a designated property on the Notion page, creating a living document that evolves with the discussion. The workflow also updates a "Last Summary" timestamp to prevent duplicate processing of the same comments.

What You'll Need

  1. Notion Account & Database: A Notion workspace with API access and a database containing pages you want to monitor for comments.
  2. Notion Integration: API credentials from Notion with permissions to read comments and update page properties.
  3. AI Model Access: API key for your preferred AI service (Google Gemini, OpenAI ChatGPT, Anthropic Claude, etc.).
  4. n8n Instance: Either n8n.cloud account or self-hosted n8n installation to run the workflow.
  5. Defined Properties: Two properties in your Notion database: one for the AI summary text and one for the last execution timestamp.

Pro tip: Start with a single Notion database for testing before expanding to multiple databases. This lets you refine your AI prompt and summary format based on actual team feedback before scaling the automation.

Quick Setup Guide

Follow these steps to implement this automation in your workflow:

  1. Download & Import: Download the template file and import it into your n8n instance using the workflow import function.
  2. Configure Notion Connection: Set up your Notion API credentials in the Notion nodes, specifying the database ID you want to monitor.
  3. Set Up AI Integration: Add your AI service API key and configure the prompt to match your preferred summary style and length.
  4. Define Property Names: Update the workflow variables to match your Notion database property names for the summary and timestamp fields.
  5. Adjust Schedule: Modify the schedule trigger to run at intervals appropriate for your team's discussion pace (daily is recommended for most teams).
  6. Test & Refine: Run the workflow manually on a test page, review the generated summary, and tweak the AI prompt if needed before enabling automatic execution.

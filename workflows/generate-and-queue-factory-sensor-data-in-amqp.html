What This Workflow Does

This automation solves a critical testing challenge in industrial IoT: validating incident reporting systems without physical hardware. It generates realistic factory sensor data—temperature, pressure, vibration readings—and queues them in AMQP (Advanced Message Queuing Protocol) for downstream processing.

Instead of waiting for expensive sensor installations, you can simulate thousands of data points to test alert thresholds, workflow reliability, and team response procedures. The queued messages mimic real IoT ecosystems, allowing you to build robust monitoring automation before capital investment.

This template specifically supports a companion incident reporting workflow, creating a complete factory monitoring simulation. You can adjust data frequency, value ranges, and message formats to match your actual equipment specifications.

How It Works

Step 1: Interval Trigger

The workflow starts with an Interval node configured to trigger every second (adjustable). This simulates continuous sensor data streams as you'd receive from real IoT devices in a production environment.

Step 2: Data Generation & Formatting

A Set node creates structured sensor payloads containing timestamp, sensor ID, measurement type, value, and location metadata. You can customize these fields to match your specific monitoring requirements.

Step 3: AMQP Queuing

The formatted data is sent to an AMQP Sender node, which publishes messages to your message queue (ActiveMQ, RabbitMQ, etc.). The queue acts as a buffer, ensuring data persists even if downstream systems are temporarily unavailable.

Pro tip: Configure your AMQP queue with appropriate durability settings so sensor data survives system restarts—critical for factory environments where uptime is paramount.

Quick Setup Guide

  1. Download the template JSON file and import into your n8n workspace.
  2. Configure the Interval node to your desired trigger frequency (1–60 seconds).
  3. Adjust the Set node fields to match your sensor data schema.
  4. Connect the AMQP Sender node to your message queue using your credentials.
  5. Activate the workflow and monitor the queue for incoming messages.
  6. Test downstream systems by consuming the queued sensor data.

AMQP (Advanced Message Queuing Protocol) is a messaging standard used for reliable data exchange between systems. It's ideal for IoT sensor data because it ensures messages are delivered even if systems are temporarily offline, making it perfect for factory monitoring where data continuity is critical.

Unlike simple HTTP calls that fail when networks drop, AMQP queues persist messages until consumers are ready. This guarantees sensor readings aren't lost during maintenance windows or system updates.

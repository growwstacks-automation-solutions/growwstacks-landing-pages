What This Workflow Does

Manual badge creation for events is time-consuming, error-prone, and scales poorly. This automated solution transforms event registration into a seamless, professional experience. When attendees register, the system instantly generates personalized badges with unique QR codes, converts them to high-quality images, logs everything to Google Sheets, and emails badges directly to attendees.

The workflow handles validation, error recovery, and admin notifications automatically. It's designed for conferences, corporate events, universities, training programs, and trade shows—anywhere you need professional identification with quick check-in capabilities. The system processes registrations individually or in batches, ensuring scalability from small workshops to large conferences with thousands of attendees.

How It Works

1. Registration Data Reception

The workflow starts by receiving attendee information via webhook from your registration form or platform. It validates email formats, sanitizes names, and extracts event details. Each attendee receives a unique identifier that will be encoded in their QR code for secure check-in and tracking.

2. QR Code Generation

Using the unique attendee ID, the system generates a scannable QR code. This code can contain check-in information, session preferences, or networking profiles. The QR code is sized appropriately for badge placement and tested for readability across different scanning devices.

3. HTML Badge Creation

A professional HTML template creates visually appealing badges with gradient designs, attendee information, and embedded QR codes. The template is fully customizable for different event types, roles (VIP, speaker, staff), and branding requirements. CSS ensures consistent appearance across all generated badges.

4. Image Conversion via HTMLCSStoImage

The HTML badge is converted to a high-quality PNG image (400x680px) using the HTMLCSStoImage API. This ensures professional print-ready output that maintains design integrity across different devices and printing methods. The conversion process handles fonts, gradients, and layout precisely.

5. Google Sheets Logging

Every badge generation is logged to Google Sheets with timestamps, attendee details, badge image URLs, and status information. This creates a complete audit trail for event management, reprint requests, and attendance analytics. The sheet serves as a central database for all event operations.

6. Email Delivery

Personalized emails are sent to each attendee with their badge attached as a downloadable PNG file. The email includes event instructions, check-in details, and any relevant pre-event information. The system uses your branded Gmail account for consistent communication.

7. Error Handling & Notifications

If any step fails, the workflow continues processing other registrations while sending admin notifications about the specific failure. This ensures partial system failures don't stop the entire event registration process, and issues can be addressed promptly.

What You'll Need

  1. HTMLCSStoImage API Key: Sign up at htmlcsstoimg.com for image conversion capabilities
  2. Gmail Account: With OAuth2 permissions enabled for sending automated emails
  3. Google Sheets: A spreadsheet with headers for Name, Email, Event, Role, Attendee ID, Badge URL, and Timestamp
  4. Registration System: Any form or platform that can send data via webhook (POST request)
  5. n8n Instance: Self-hosted or cloud version of n8n to run the workflow

Pro tip: Create a dedicated Gmail account for event automation to maintain clean sender reputation and separate from personal communications. Monitor your HTMLCSStoImage API limits if processing large events.

Quick Setup Guide

  1. Import the downloaded JSON template into your n8n instance
  2. Add credentials for HTMLCSStoImage API, Gmail OAuth2, and Google Sheets OAuth2
  3. Update the Google Sheet ID in the workflow with your tracking spreadsheet
  4. Replace the admin email address with your notification email
  5. Test with sample registration data using the webhook trigger
  6. Connect your registration form to the webhook URL provided by n8n
  7. Activate the workflow and monitor initial badge generations

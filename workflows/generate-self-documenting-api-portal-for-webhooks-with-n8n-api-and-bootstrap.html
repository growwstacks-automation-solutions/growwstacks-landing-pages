Generate Self-Documenting API Portal for Webhooks

Automatically create a live, interactive documentation portal for all your n8n webhook endpoints—no manual updates required.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Keeping API documentation updated is a constant challenge for development teams. When your endpoints are powerful n8n webhooks that evolve frequently, manual documentation becomes a bottleneck that leads to outdated specs, frustrated developers, and integration errors.

This workflow transforms your n8n instance into a self-documenting API platform. It automatically scans all your active workflows, identifies webhooks designated for documentation, and generates a single, beautiful HTML portal with interactive endpoint details. The documentation updates in real-time whenever you modify your workflows—no manual intervention needed.

The result is a professional, dark-themed documentation page accessible via a simple URL. Developers get instant access to accurate endpoint specifications, request/response examples, and ready-to-use cURL commands, dramatically reducing integration time and support overhead.

How It Works

1. Automatic Discovery

The workflow uses n8n's API to scan all active workflows on your instance. It looks for specific metadata markers (a Set node named API_DOCS) that indicate which webhooks should be included in the documentation.

2. Metadata Processing

Each documented webhook contains structured JSON metadata describing the endpoint: path, HTTP method, summary, description, tags, request/response schemas, and success/error codes. This metadata is extracted and standardized.

3. HTML Generation

The workflow dynamically builds a complete HTML page using Bootstrap components. It creates an accordion-style interface where developers can expand each endpoint to see detailed documentation, including automatically generated cURL commands for testing.

4. Live Delivery

A webhook trigger serves the generated documentation page at a configurable URL (default: /api-doc). Anyone with access to your n8n instance can view the live documentation portal in their browser.

Pro tip: Use consistent tagging in your endpoint metadata (like "Sales", "Automation", "Notifications") to create logical groupings in your documentation portal. This makes it easier for developers to find relevant endpoints.

Quick Setup Guide

Follow these steps to implement self-documenting APIs in your n8n environment:

  1. Import the template: Download and import the JSON workflow file into your n8n instance.
  2. Configure n8n API credentials: Add your n8n API credentials to the "Get Workflows" node with read permissions.
  3. Customize portal settings: Update the "Configs" node with your documentation title, version, and description.
  4. Document your first webhook: Add a Set node named API_DOCS to any webhook workflow with the JSON metadata structure.
  5. Access your documentation: Trigger the workflow's webhook (default path /api-doc) to view your live API portal.

Pro tip: Start by documenting just one or two critical webhooks to validate the system works. Once confirmed, gradually add documentation to all your webhook workflows using the same pattern.

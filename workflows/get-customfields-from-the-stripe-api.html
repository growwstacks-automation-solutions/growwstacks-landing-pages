What This Workflow Does

If you use Stripe for payments, you've likely added custom fields to your checkout to capture vital customer information—project details, company size, referral source, or specific requirements. The frustrating reality is that this valuable data disappears from view after payment. Stripe's standard Invoice and Charge APIs don't include these custom fields, leaving you with incomplete customer records and manual data entry.

This n8n workflow solves that exact problem. It automatically queries Stripe's Checkout Sessions endpoint—the only place where custom field data is stored—retrieves the complete session details, transforms the complex API response into usable data, and lets you filter and route that information to any other system in your stack. Instead of manually digging through Stripe logs, you get clean, structured data flowing automatically to your CRM, project management tools, or internal databases.

The business value is immediate: eliminate hours of manual data lookup, ensure no customer context is lost between sales and fulfillment, and create automated triggers based on what customers actually told you during checkout. Whether you're a SaaS company, agency, or e-commerce store, this workflow turns Stripe custom fields from hidden metadata into actionable business intelligence.

How It Works

This workflow acts as a bridge between Stripe's raw API data and your business applications. It follows a logical sequence to extract, transform, and deliver the custom field information you need.

Step 1: Fetch Recent Checkout Sessions

The workflow starts by making an authenticated API call to Stripe's Checkout Sessions endpoint. It's configured to retrieve sessions from the last 7 days by default, ensuring you capture all recent transactions. The HTTP Request node handles pagination automatically, so you get complete results even with high-volume sales.

Step 2: Transform Complex API Response

Stripe's API returns nested JSON objects that can be difficult to work with directly. This workflow includes a Function or Set node that flattens the response, extracting the custom_fields array from each session and converting it into a clean, tabular format. This transformation is crucial for making the data usable in subsequent steps.

Step 3: Filter and Route Data

Once the data is structured, you can apply filters based on the custom field values. For example, you might route all purchases with "Company Size: Enterprise" to your sales team's Slack channel, or send "Product Type: Subscription" orders to your billing system. The Filter node gives you complete control over which data goes where.

Step 4: Deliver to Destination Systems

The final step sends the processed custom field data to your chosen destinations. This could be creating enriched customer records in HubSpot or Salesforce, generating tailored onboarding emails in Mailchimp, creating project tickets in Asana, or simply logging everything to a Google Sheet for reporting.

Pro tip: Set this workflow to run on a schedule (every hour or daily) to continuously sync new Stripe data. This creates a real-time bridge between your payment processing and customer management systems.

Quick Setup Guide

Getting this workflow running takes about 15 minutes if you have your Stripe API keys ready.

Step 1: Download the template JSON file using the button above and import it into your n8n instance.

Step 2: Create a Stripe credential in n8n using your Stripe secret key. This allows the workflow to authenticate with Stripe's API.

Step 3: Review the HTTP Request node configuration. By default, it fetches sessions from the last 7 days—adjust the timestamp if you need historical data.

Step 4: Examine the data transformation step. The template includes a sample transformation, but you may want to modify it based on your specific custom field structure.

Step 5: Configure the Filter node to route data based on your business rules. Set up conditions that match your custom field values.

Step 6: Connect your destination systems. Replace the sample webhook or email nodes with connections to your actual CRM, project management, or notification systems.

Step 7: Test with a recent transaction. Run the workflow once manually to verify it captures and processes your custom field data correctly.

Step 8: Schedule it. Set the workflow to run automatically every hour, day, or week depending on your transaction volume and process urgency.

Important: Always use a restricted Stripe API key with only the permissions needed (read access to checkout sessions). Never use full-access keys in automation.

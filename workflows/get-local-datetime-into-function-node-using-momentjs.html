What This Workflow Does

This n8n workflow template solves a common challenge in business automation: accurately handling dates and times across different timezones. Many businesses struggle with scheduling errors, missed deadlines, and incorrect timestamps when their automation systems don't properly account for local time.

The template demonstrates how to use the powerful moment.js library within n8n's Function node to get precise local datetime information. It shows you how to extract year, month, day, hour, minute, second, and millisecond components while properly formatting the output for your business needs.

This is particularly valuable for companies operating internationally, scheduling time-sensitive communications, or processing data from multiple regions where consistent time handling is critical for compliance and customer experience.

How It Works

The workflow follows a straightforward but powerful approach to datetime management:

1. Environment Configuration

The workflow relies on the GENERIC_TIMEZONE environment variable being properly set in your n8n instance. This centralizes timezone configuration and makes your workflows portable across different deployments (development, staging, production).

2. Moment.js Integration

By requiring the moment.js library in the Function node, you gain access to robust datetime manipulation capabilities that go beyond n8n's built-in Date & Time node. The template includes the necessary environment variable configuration (NODE_FUNCTION_ALLOW_EXTERNAL=moment) to enable this integration.

3. Timezone-Aware Processing

The core function uses moment().tz() to create a datetime object aware of your configured timezone. This ensures all subsequent operations—extracting components, formatting, or calculations—respect the correct local time rather than defaulting to UTC or server time.

4. Structured Output

The workflow returns a clean JSON object containing all datetime components separately (year, month, day, etc.) plus a formatted string. This structure makes it easy to use these values in downstream nodes for conditional logic, database storage, or human-readable outputs.

What This Workflow Does

This automation solves the problem of manually checking weather conditions for business decisions. Whether you're planning outdoor events, optimizing delivery routes, managing agricultural operations, or adjusting retail strategies based on weather, this workflow provides real-time, accurate weather data automatically.

The template connects to OpenWeatherMap's API to retrieve comprehensive weather metrics for any specified city: temperature, humidity, precipitation, wind speed and direction, atmospheric pressure, and visibility. It transforms raw weather data into structured information ready for integration with your CRM, logistics software, dashboards, or alert systems.

Instead of employees spending time checking multiple weather sources or missing critical weather changes, this automation delivers consistent, reliable data on schedule or triggered by specific events.

How It Works

Step 1: Trigger Configuration

The workflow starts with a manual trigger or scheduled execution. You can set it to run daily, hourly, or on-demand when weather data is needed for a decision.

Step 2: City Input

The workflow accepts a city name as input. This can be hardcoded for regular monitoring of specific locations or dynamically provided from another system (like a list of delivery destinations).

Step 3: API Connection

The n8n OpenWeatherMap node connects to the weather API using your API key (free tier available). It requests current weather data for the specified city.

Step 4: Data Processing

The raw API response is parsed to extract key metrics: current temperature, conditions (rain, snow, clear), humidity percentage, wind details, and forecast indicators.

Step 5: Output Formatting

The data is formatted into a clean structure suitable for further use: JSON for integration with other apps, human-readable text for alerts, or specific values for threshold checking.

Step 6: Delivery

The weather data is delivered to your chosen destination: emailed to a team, posted to Slack, saved to a database, or fed into a decision-making workflow.

What You'll Need

  1. n8n instance – either self-hosted or n8n.cloud account
  2. OpenWeatherMap API key – free tier available (sign up at openweathermap.org)
  3. City names or coordinates – the locations you want to monitor
  4. Destination system – where the weather data should go (email, Slack, database, etc.)

Quick Setup Guide

  1. Download the template JSON file using the button above.
  2. Import it into your n8n instance via the workflow import function.
  3. Add your OpenWeatherMap API key in the OpenWeatherMap node credentials.
  4. Configure the city input – either hardcode your target city or connect to a dynamic source.
  5. Set up the output destination: configure the email, Slack, or database node as needed.
  6. Test the workflow with a manual execution to verify data retrieval.
  7. Schedule regular runs if needed, or integrate with triggers from other systems.

Pro tip: Use the free OpenWeatherMap tier for up to 1,000 calls/day. For multiple locations or frequent updates, consider their paid plans. Combine this weather data with other workflows – for example, trigger delivery route changes when precipitation exceeds a threshold.

What This Workflow Does

This template solves a common challenge: there's no native Home Assistant trigger node in n8n. Instead of inefficiently polling the Home Assistant API on a schedule, this workflow uses the AppDaemon add‑on to create a listener app inside Home Assistant. When the listener detects a subscribed event, it calls an n8n webhook, passing the event data directly to your workflow.

The result is real‑time, event‑driven automation that connects your physical environment to digital business processes. Whether it's a motion sensor triggering a task creation, a temperature reading logging to a spreadsheet, or a door sensor sending an alert, this bridge enables sophisticated conditional logic based on real‑world events.

This approach is more efficient, scalable, and responsive than API polling, reducing latency from minutes to milliseconds while minimizing API load on your Home Assistant instance.

How It Works

1. Event Detection in Home Assistant

AppDaemon runs a Python app that listens for specific Home Assistant events—state changes, device triggers, automation activations, or custom events. The listener is configured via a simple YAML file specifying which events to monitor.

2. Webhook Trigger

When the target event occurs, the AppDaemon app packages the event data (event name, entity data, timestamps) into a JSON payload and sends it via HTTP POST to your n8n webhook URL. Authentication is handled via headers for security.

3. n8n Workflow Processing

The n8n webhook node receives the payload, triggering your workflow. You can then route, transform, and act on this data—sending notifications, updating databases, calling APIs, or triggering further automations across hundreds of connected apps.

4. Business System Integration

The final steps connect the Home Assistant event to your business tools: creating CRM records, logging to analytics platforms, updating project management boards, or alerting teams via communication apps.

Quick Setup Guide

  1. Install AppDaemon: In Home Assistant, go to Settings → Add‑ons, search for "AppDaemon 4", install, and start it with watchdog enabled.
  2. Create Python App: In /config/appdaemon/apps/, create n8n_webhook.py with the template code (provided in the downloaded JSON's sticky note).
  3. Configure Event & URL: Edit the Python file to specify your target event (e.g., state_changed) and your n8n webhook URL.
  4. Register App: Add an entry in apps.yaml and restart AppDaemon.
  5. Import n8n Template: Import the downloaded JSON into n8n, activate the webhook, and test by triggering an event in Home Assistant.
  6. Connect Business Tools: Add nodes after the webhook to send data to your CRM, database, or notification systems.

AWS IAM Compliance Automation

Automatically enforce MFA and deactivate access keys for non-compliant AWS IAM users

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solution addresses critical AWS security gaps by systematically enforcing Multi-Factor Authentication (MFA) and cleaning up inactive access keys across your IAM user base. It helps organizations maintain compliance with security best practices and regulatory requirements.

The workflow performs daily checks to identify non-compliant IAM users, sends notifications via Slack, and automatically deactivates access keys for accounts without MFA enabled. This reduces the attack surface and ensures only properly secured accounts maintain access to your AWS environment.

AWS IAM dashboard showing user MFA status before automation

How It Works

1. Scheduled Trigger

The workflow runs on a daily schedule to check all IAM users in your AWS account. The frequency can be adjusted based on your compliance requirements.

2. User Inventory

Retrieves a complete list of IAM users from your AWS account using the ListUsers API call, gathering essential user metadata.

3. MFA Verification

For each user, the workflow checks MFA device registration status using the ListMFADevices API. Users without registered MFA devices are flagged for action.

4. Access Key Audit

The workflow identifies all active access keys for non-compliant users and prepares them for deactivation while preserving key metadata for audit purposes.

What You'll Need

  1. AWS account with IAM access
  2. IAM credentials with ListUsers, ListMFADevices, ListAccessKeys, and UpdateAccessKey permissions
  3. Slack workspace for notifications (optional but recommended)
  4. n8n instance (cloud or self-hosted)

Quick Setup Guide

  1. Download the template JSON file
  2. Import into your n8n instance
  3. Configure AWS credentials in n8n
  4. Set up Slack webhook if using notifications
  5. Adjust schedule frequency as needed
  6. Test with a small group of users first

Pro tip: Before full deployment, create a test IAM user without MFA to verify the workflow identifies and handles non-compliant accounts correctly.

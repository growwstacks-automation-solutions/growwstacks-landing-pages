Import CSV Contacts to Notion Database from Google Drive

Automatically transfer contact lists from CSV files in Google Drive to your Notion database. Eliminate manual data entry and keep your contacts synchronized.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manually copying contact information from CSV files into Notion is tedious, error-prone, and wastes valuable time that could be spent on actual relationship building. This automation solves that problem by creating a seamless bridge between Google Drive and Notion.

Whenever a new CSV file containing contacts is added to a specific Google Drive folder, this workflow automatically detects it, extracts the contact data, and creates corresponding records in your Notion database. It handles mapping CSV columns to Notion properties, ensuring data consistency and eliminating the risk of manual entry mistakes.

This is particularly valuable for sales teams importing lead lists, marketing teams managing campaign contacts, or any business that regularly receives contact information in spreadsheet format but wants to leverage Notion's powerful database capabilities for relationship management.

How It Works

The automation follows a logical sequence to ensure reliable data transfer between the two platforms.

1. Monitor Google Drive for New CSV Files

The workflow watches a designated Google Drive folder for new CSV files. You can configure it to trigger on file creation or modification, giving you flexibility in how you deliver contact data to the system.

2. Extract and Parse CSV Data

Once a file is detected, the automation reads its contents and parses the CSV structure. It identifies column headers and maps them to corresponding fields in your Notion database schema.

3. Transform Data for Notion Compatibility

The workflow processes each contact record, formatting data types appropriately for Notion (dates, numbers, select options) and handling any necessary data cleansing or validation before import.

4. Create Notion Database Entries

Each contact from the CSV is created as a new page in your Notion database. The automation maintains the relationships between fields and ensures all contact information is preserved accurately.

5. Handle Errors and Notifications

If any records fail to import due to data issues or API limits, the workflow captures these errors and can notify your team via email or Slack, ensuring no contact falls through the cracks.

What You'll Need

  1. n8n instance (cloud or self-hosted) with access to install and run workflows
  2. Google Drive account with API access enabled and credentials configured in n8n
  3. Notion account with an existing database for contacts and integration connection set up
  4. CSV files with consistent column structure (name, email, company, etc.)
  5. Basic understanding of your Notion database property types and field mappings

Pro tip: Before running the automation with live data, test it with a small sample CSV file containing 5-10 contacts. This helps verify your field mappings work correctly and prevents accidental duplication or data issues in your main Notion database.

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment:

  1. Download and import the workflow JSON file into your n8n instance using the import function.
  2. Configure Google Drive node by authenticating with your Google account and specifying the folder to monitor for CSV files.
  3. Set up Notion connection by creating an internal integration in Notion, sharing your database with it, and adding the API key to n8n.
  4. Map CSV columns to your Notion database properties in the workflow settings, ensuring data types match correctly.
  5. Test the workflow by adding a sample CSV to your Google Drive folder and verifying contacts appear correctly in Notion.
  6. Activate the workflow and set it to run automatically on schedule or trigger based on file changes.

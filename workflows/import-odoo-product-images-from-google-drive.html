Import Odoo Product Images from Google Drive

Automatically sync product images from Google Drive to your Odoo inventory. Eliminate manual uploads and keep your catalog visually updated.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Managing product images manually is a tedious, error-prone task for e-commerce businesses. Designers upload images to Google Drive, but then someone must manually download, rename, and upload each file into Odoo—a process that wastes hours and delays product launches.

This n8n workflow solves that by automating the entire image import pipeline. It monitors a designated Google Drive folder, fetches new or updated image files, matches them to corresponding Odoo product templates or variants using filename patterns or metadata, and attaches them automatically. The result is a synchronized visual catalog that updates in real-time, eliminating manual steps and ensuring consistency.

Beyond basic import, this automation can be configured to handle bulk updates, apply image transformations (like resizing or format conversion), and send notifications if any images fail to match. It turns a fragmented, manual process into a seamless, reliable integration between your design repository and your ERP.

How It Works

Step 1: Monitor Google Drive Folder

The workflow starts by polling a specific Google Drive folder (or subfolders) for new image files. You can set it to trigger on schedule (e.g., every hour) or when a file is added. It filters for image file types (JPG, PNG, WEBP) and reads metadata like filename, SKU, or product ID embedded in the file name.

Step 2: Extract & Map Product Data

Each image file is analyzed. The system extracts identifiers from the filename (like "PROD-1234-main.jpg") or from a companion CSV/JSON file in the same folder. This mapping determines which Odoo product or template the image belongs to.

Step 3: Fetch Odoo Product Record

Using the extracted identifier, the workflow queries Odoo's API to find the matching product or product template. If a match is found, it proceeds to attach the image. If not, it can log the error and optionally notify your team via Slack or email.

Step 4: Upload & Attach Image to Odoo

The image file is downloaded from Drive, optionally processed (resized, converted), and uploaded to Odoo via its API. The image is attached as a product image or variant image, updating the visual representation instantly.

Step 5: Logging & Notification

Every import action is logged. Successes and failures are recorded, and you can set up alerts to notify your team if any images couldn't be matched or uploaded. This ensures visibility and allows for quick manual correction if needed.

What You'll Need

  1. Odoo instance with API access enabled.
  2. Google Drive folder where product images are stored (shared with the automation service account).
  3. n8n instance (cloud or self-hosted) with the Odoo and Google Drive nodes installed.
  4. API credentials for Odoo and Google Drive (service account or OAuth).
  5. Clear naming convention for image files that maps to Odoo product SKUs or IDs.
  6. Optional: A mapping file (CSV/JSON) if filenames don't directly match Odoo identifiers.

Pro tip: Use a consistent folder structure in Google Drive—for example, organize by product category or season. This makes the automation easier to manage and allows you to run separate workflows for different product lines.

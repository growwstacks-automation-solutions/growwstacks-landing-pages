What This Workflow Does

This beginner-friendly tutorial demonstrates how to use the Evaluation tool to automatically score AI outputs against known correct answers ("ground truth") stored in Google Sheets. Many businesses struggle with manually testing AI implementations, leading to inconsistent quality and potential errors in automated processes.

The workflow solves this by creating a systematic testing framework that compares AI-generated responses with expected outcomes, providing quantitative scores for factual correctness. This eliminates guesswork from AI deployment and gives businesses confidence in their automation quality before going live with customers or internal processes.

By integrating Google Sheets as your ground truth database, you create a collaborative testing environment where non-technical team members can contribute expected answers, while the automation handles the repetitive evaluation work.

How It Works

The workflow operates in two distinct modes depending on your testing needs:

1. Manual Test Mode

Triggered by clicking "Execute workflow" for quick ad-hoc testing. It sends a fixed question to the AI agent and returns the answer directly to you, perfect for immediate verification of AI responses.

2. Evaluation Mode

Triggered automatically when fetching dataset rows from Google Sheets. This mode reads questions and expected answers from your spreadsheet, processes them through the AI agent with calculator tool integration, then uses a separate AI "judge" to evaluate factual correctness on a 1-5 scale.

3. Results Tracking

Both the AI's actual output and the automated correctness score are written back to your Google Sheet, creating a complete audit trail of AI performance over time. This historical data becomes invaluable for identifying patterns and improvement areas.

Quick Setup Guide

Follow these steps to implement this evaluation system in your business:

  1. Download and import the template JSON file into your n8n instance
  2. Configure credentials for Google Sheets and Google Gemini API in n8n
  3. Clone the template Google Sheet to your own Drive and update document IDs in all three evaluation nodes
  4. Customize column names to match your dataset structure if different from default
  5. Add your test cases with questions and expected answers to the Google Sheet
  6. Activate the workflow and run initial tests to verify configuration
  7. Review scores and adjust your ground truth data or AI prompts as needed

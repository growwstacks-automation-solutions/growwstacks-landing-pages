Linear to Productboard Feature Sync

Automatically sync project status, due dates, and progress between Linear and Productboard to keep product and engineering teams perfectly aligned.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves a critical alignment problem between engineering and product teams. When developers update project status or due dates in Linear, product managers often don't see these changes in Productboard until days later—leading to misaligned roadmaps, missed deadlines, and unnecessary status meetings.

The workflow automatically monitors Linear for changes to projects or issues, maps those statuses to Productboard's feature tracking system, updates feature details including timelines, and sends Slack notifications to keep everyone informed. It transforms what was a manual, error-prone process into a seamless, real-time sync that ensures both tools always reflect the same reality.

By eliminating manual data entry, teams save hours each week while gaining confidence that their product roadmap accurately reflects engineering progress. This alignment enables better decision-making, more accurate forecasting, and smoother collaboration between technical and product teams.

How It Works

1. Monitor Linear for Changes

The workflow starts by monitoring Linear projects or issues for updates. It can be configured to watch specific teams, projects, or issue types, triggering whenever status changes, due dates are updated, or assignees are modified.

2. Map Status Between Platforms

Linear statuses (like "In Progress," "Done," "Canceled") are mapped to corresponding Productboard feature statuses ("Developing," "Launched," "Archived"). This mapping ensures terminology consistency across teams while respecting each platform's workflow structure.

3. Update Productboard Features

The automation identifies the corresponding feature in Productboard using custom field matching, then updates its status, timeframe, and any other mapped fields. It handles API rate limits and error conditions gracefully to ensure reliable updates.

4. Notify Teams via Slack

A summary of the update is sent to designated Slack channels, informing both product and engineering teams of the change. Notifications include links to both the Linear item and Productboard feature for easy reference.

What You'll Need

  1. Linear API credentials with appropriate permissions to read projects/issues
  2. Productboard API access with feature update permissions
  3. A custom field in Productboard named "Linear" to store Linear item URLs
  4. Slack workspace access (optional, for notifications)
  5. n8n instance (cloud or self-hosted) to run the workflow
  6. Basic understanding of your team's status mapping between Linear and Productboard

Quick Setup Guide

Follow these steps to implement this sync in your organization:

  1. Download and import the template JSON file into your n8n instance
  2. Configure credentials for Linear, Productboard, and Slack in n8n's credential management
  3. Set up the Productboard custom field as described in the workflow documentation
  4. Adjust status mappings to match your team's terminology and workflow stages
  5. Test with a single project to verify the sync works correctly before scaling
  6. Activate the workflow and monitor initial runs for any adjustments needed

Pro tip: Start with a pilot project involving one engineering team and one product manager. This allows you to refine the status mappings and notification preferences before rolling out to the entire organization.

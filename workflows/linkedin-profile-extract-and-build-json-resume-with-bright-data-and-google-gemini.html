LinkedIn Profile Extract & JSON Resume Builder

Automate candidate sourcing by scraping LinkedIn profiles, extracting key data with AI, and generating structured JSON resumes—ready for your ATS.

What This Workflow Does

Manual candidate sourcing from LinkedIn is a massive time sink for recruiters and HR teams. This workflow automates the entire process: it takes a LinkedIn profile URL, uses Bright Data's Web Unlocker to reliably scrape the public profile data (bypassing CAPTCHAs and anti-bot measures), then sends the raw HTML to Google Gemini. The AI extracts clean, structured information—name, headline, experience, skills, education—and formats it into a standards-compliant JSON resume schema.

The final output is a machine-readable JSON file that can be instantly pushed to your Applicant Tracking System (ATS), used for candidate matching, or stored for analytics. It turns hours of manual profile review into a 60-second automated task, ensuring consistency and eliminating human error in data entry.

How It Works

Step 1: Input & Configuration

The workflow is triggered manually or via a webhook. You provide the LinkedIn profile URL and configure your Bright Data zone credentials. A webhook URL is also set for receiving the final JSON output.

Step 2: LinkedIn Profile Scraping

Using the HTTP Request node with Bright Data's Web Unlocker, the workflow fetches the full HTML of the LinkedIn profile page. This service handles proxies, rotation, and CAPTCHA solving, ensuring a high success rate for data extraction.

Step 3: AI-Powered Data Extraction

The raw HTML is passed to a Google Gemini node. A carefully crafted prompt instructs the AI to identify and extract key professional details: name, current role, company, past positions, skills, education, and location. The AI returns this as clean, structured text.

Step 4: JSON Resume Building

A Function node or a series of Set nodes takes the extracted data and maps it into the official JSON Resume schema format. This creates a standardized object with sections for basics, work, education, skills, and awards.

Step 5: Output & Delivery

The completed JSON resume is sent to the configured webhook (e.g., your ATS or internal database) and simultaneously saved as a file on your n8n server for backup. The process is complete, and the structured candidate data is ready for use.

Pro tip: Use this workflow as the first step in a larger pipeline. Add a node to compare the extracted skills against a job description and auto-score candidates, or trigger a personalized email to the candidate informing them they've been added to your talent pool.

What You'll Need

  1. A Bright Data account with a Web Unlocker zone set up (for reliable LinkedIn scraping).
  2. A Google Gemini API key (or access via Vertex AI) for the AI processing step.
  3. An n8n instance (cloud or self-hosted) where you can import and run the workflow.
  4. A webhook endpoint (like webhook.site for testing, or your ATS API endpoint for production) to receive the JSON output.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n workspace.
  2. Configure credentials: Create a "Header Auth" credential in n8n for Bright Data (Value: Bearer YOUR_BRIGHT_DATA_TOKEN). Create a "Google Gemini" credential with your API key.
  3. Update the trigger node: In the "Set URL and Bright Data Zone" node, input a sample LinkedIn profile URL, your Bright Data zone ID, and your target webhook URL.
  4. Test the workflow: Execute the workflow once manually. Check the webhook destination to see the JSON resume output.
  5. Go live: Once testing is successful, you can set the workflow to trigger on a schedule, from a form submission, or via an API call.

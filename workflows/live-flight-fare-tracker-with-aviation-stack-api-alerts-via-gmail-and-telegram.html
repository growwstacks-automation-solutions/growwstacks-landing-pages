What This Workflow Does

This automated system solves the tedious problem of manually checking flight prices by continuously monitoring airline APIs like Amadeus and Skyscanner. It compares new fares against historical prices and instantly notifies you via Gmail and Telegram when significant price drops occur - perfect for travelers waiting for the best deal.

The workflow structures all tracked data for analysis, allowing you to identify pricing trends and optimal booking times. It works post-booking to track price fluctuations for already booked routes, potentially identifying refund opportunities when fares drop substantially.

How It Works

1. Scheduled API Queries

The workflow runs on a set schedule (typically every 6-12 hours) to query airline APIs for current flight fares on your specified routes.

2. Price Comparison

New prices are compared against historical fare data stored in your database. The system calculates both absolute dollar differences and percentage changes.

3. Alert Triggers

When prices drop below your configured thresholds, the workflow triggers notifications through both Gmail (for formal records) and Telegram (for instant mobile alerts).

4. Data Logging

All fare data is structured and logged with timestamps, creating an audit trail for analysis of pricing trends over time.

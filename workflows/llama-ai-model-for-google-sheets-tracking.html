What This Workflow Does

Businesses today are drowning in documents—competitor reports, regulatory updates, market research PDFs, and internal documents that contain critical intelligence. Manually checking websites for new documents, downloading them, reading through hundreds of pages, and extracting key insights is a time-consuming process prone to human error and oversight.

This automation solves that problem by creating a 24/7 document intelligence system. It automatically monitors target websites for new PDFs, downloads them as they appear, extracts the content using advanced AI models like Llama, summarizes the key findings into actionable insights, and logs everything in Google Sheets for easy team access and analysis.

The workflow ensures you never miss important updates, transforms lengthy documents into concise intelligence, and creates a living knowledge base that grows smarter with every document processed.

How It Works

Step 1: Website Monitoring & PDF Detection

The workflow begins by accessing a configured target website using HTTP Request nodes. It scans the site structure to identify new PDF documents that haven't been processed before. Advanced filtering ensures only relevant documents are captured based on file patterns, dates, or specific directories.

Step 2: Automated Download & Content Extraction

Once new PDFs are identified, they're automatically downloaded to a temporary storage location. The workflow then extracts raw text content from each PDF, handling various formats and layouts to ensure complete content capture.

Step 3: AI-Powered Summarization & Analysis

The extracted content is sent to an AI model (like Llama via OpenRouter) with specific prompts designed for business intelligence. The AI identifies key trends, risks, opportunities, and actionable insights, generating concise summaries that decision-makers can quickly digest.

Step 4: Google Sheets Logging & Notification

The AI-generated summaries, along with metadata like source URLs, download dates, and confidence scores, are automatically appended to a Google Sheets spreadsheet. The workflow also marks processed URLs to prevent duplicate analysis and can send notifications to Discord or other channels when significant findings are detected.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow execution capabilities
  2. Google Cloud API access with Drive and Sheets enabled, and service account credentials
  3. Target website URL that hosts PDF documents you want to monitor
  4. OpenRouter API key or access to another AI model service for summarization
  5. Google Sheets spreadsheet prepared with appropriate columns for logging results
  6. Discord webhook URL (optional) for receiving notifications about new findings

Quick Setup Guide

  1. Import the template: Download the JSON file and import it into your n8n instance through the workflow import interface.
  2. Configure website monitoring: Update the HTTP Request node with your target website URL and adjust the filtering logic if needed.
  3. Connect Google services: Add your Google Cloud service account credentials to the Google Sheets and Drive nodes, and specify your spreadsheet ID.
  4. Set up AI summarization: Enter your OpenRouter API key in the AI node and select your preferred model (Llama, GPT, Claude, etc.).
  5. Test and activate: Run the workflow once manually to verify all connections work, then schedule it for regular execution (daily, hourly, etc.).

Pro tip: Start with one high-value website before scaling to multiple sources. This allows you to refine your AI prompts and output format based on real results before expanding the system.

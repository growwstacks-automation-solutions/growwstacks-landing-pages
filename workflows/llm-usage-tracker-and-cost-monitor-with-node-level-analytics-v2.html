What This Workflow Does

This workflow solves a critical problem for businesses using AI in their automation: visibility into LLM costs and usage. When you run multiple AI agents, language models, and automated workflows, token consumption can quickly become a significant, unpredictable expense. This template provides comprehensive monitoring that extracts detailed token usage data from any n8n workflow execution and calculates precise costs based on current model pricing.

The system bridges the gap between workflow execution and cost analytics by retrieving execution data using the workflow ID, extracting all LLM usage from nested structures, and providing detailed breakdowns per node, model, and team. It transforms AI from a black-box expense into a manageable resource with clear accountability and optimization opportunities.

How It Works

1. Execution Data Retrieval

The workflow starts by fetching execution data from your n8n instance using the execution ID. It connects to the n8n API (available in both free and paid versions) to pull complete execution details, including all intermediate steps and agent interactions.

2. LLM Usage Extraction

Using a dynamic parsing approach, the system identifies all LLM calls regardless of their nesting level within the workflow. It captures prompt tokens, completion tokens, model parameters, execution times, and contextual information about each AI interaction.

3. Cost Calculation & Standardization

The workflow standardizes model names (e.g., "gpt-4-0613" → "gpt-4") and applies current pricing per million tokens. It calculates costs for prompt usage, completion usage, and total expenses while handling multiple pricing models and custom rate configurations.

4. Analytics & Reporting

Finally, the system generates comprehensive analytics including cost breakdowns by model type, usage per node, total execution statistics, and performance metrics. It provides both detailed per-call data and summary statistics for decision-making.

Pro tip: Enable "Return Intermediate Steps" in your AI agent settings to capture complete token usage data. This ensures the monitoring workflow can extract all LLM interactions, not just final outputs.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflows containing LLM nodes
  2. Basic familiarity with n8n's AI agent and LLM node configurations
  3. n8n API access configured in your instance settings (available in free version)
  4. Agent settings configured with "Return Intermediate Steps" enabled
  5. Model pricing information for the LLMs you're using (defaults included)

Quick Setup Guide

  1. Import this monitoring workflow into your n8n instance
  2. Navigate to Settings → n8n API → create an API key for execution data access
  3. Configure the "Get an Execution" node with your API credentials
  4. Update the "Standardize Names" node with your model naming conventions
  5. Add current pricing to the "Model Prices" node (prices per 1M tokens)
  6. To monitor any workflow, add an "Execute Workflow" node at the end, select this monitoring workflow, turn OFF "Wait For Sub-Workflow Completion," and pass the execution ID

Important: This monitoring works after full workflow execution. You cannot get token data before all tasks complete. Plan your cost tracking accordingly, using webhooks or scheduled triggers for regular reporting.

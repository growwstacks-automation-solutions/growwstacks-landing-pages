What This Workflow Does

This automation solution helps you maintain optimal workflow performance by identifying outdated nodes across your n8n instance. It serves as an add-on to the parent template that checks for built-in nodes not running the latest version.

The workflow highlights outdated nodes directly on your canvas, places updated versions alongside them for easy replacement, and sends comprehensive email summaries via Gmail. This eliminates manual version checking and reduces the risk of running deprecated node versions.

Visual indicators show outdated nodes with new versions ready for implementation

How It Works

Node Version Detection

The parent workflow scans your entire n8n instance, identifying nodes that aren't running the latest available version. This data gets passed to our add-on for processing.

Canvas Marking

Outdated nodes are renamed with a warning emoji (default: ⚠️) for visual identification. The system prevents duplicate processing by checking for these markers.

New Node Placement

Current versions of each outdated node are added to the workflow canvas in an offset position, ready for testing and implementation without disrupting existing connections.

Email Notification

A detailed Gmail summary lists all modified workflows, helping teams track updates and maintain version control across the organization.

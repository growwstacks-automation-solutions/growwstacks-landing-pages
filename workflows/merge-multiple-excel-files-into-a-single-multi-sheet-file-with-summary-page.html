What This Workflow Does

If your team deals with multiple Excel files from different departments, regions, or systems, you know the pain of manual consolidation. This workflow automates the entire process of merging numerous .xlsx files from a designated folder into a single, well-organized Excel workbook.

Each input file becomes its own sheet within the output file, preserving the original data structure. Additionally, a summary sheet is automatically generated at the beginning, providing a convenient overview of all merged files—including their original names and record counts. This eliminates hours of copy-paste work, reduces human error, and creates standardized reports ready for analysis or distribution.

This solution is particularly valuable for financial reporting, sales data consolidation, inventory management, customer data aggregation, and any scenario where multiple spreadsheets need to be combined into one manageable file.

How It Works

1. File Collection

The workflow monitors a designated folder for all files ending with .xlsx extension. It reads each file systematically, preparing them for processing regardless of quantity.

2. Sequential Processing

Using batch processing, the workflow iterates through each file one by one. For each file, it extracts data from the first sheet while preserving the original structure and content.

3. Data Cleaning & Preparation

A custom code node gathers data from all files, removing completely empty rows and preparing it for final Excel generation. The original filename becomes the name of the new sheet for easy identification.

4. Multi-Sheet Generation

The core logic uses the xlsx library to create a new Excel workbook in memory. It adds a dedicated sheet for each processed file, populating it with corresponding data while maintaining formatting consistency.

5. Summary Creation & Output

A "Summary" sheet is created listing all source files and their record counts. The final workbook is saved with a timestamped filename in an output directory, ready for use.

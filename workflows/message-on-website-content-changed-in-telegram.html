What This Workflow Does

Manually checking websites for updates is a tedious, time-consuming task that's prone to human error. Whether you're tracking competitor blogs for new content, monitoring news sites for breaking updates, or watching pricing pages for changes, doing it manually wastes valuable hours each week.

This automated workflow solves that problem completely. It periodically fetches any website's content, compares it against previous versions, and instantly sends you a Telegram notification the moment something changes. You get real-time intelligence without lifting a finger.

The system is particularly valuable for businesses conducting competitive analysis, marketers tracking industry trends, or anyone who needs to stay informed about specific web content changes. It transforms passive waiting into proactive notification.

How It Works

The automation follows a simple but powerful logic flow that runs continuously in the background.

1. Initial Website Fetch

The workflow starts by using an HTTP Request node to fetch the complete HTML content of your target webpage. This could be a competitor's blog homepage, a product pricing page, a news article, or any publicly accessible URL.

2. Scheduled Waiting Period

A Wait node pauses the workflow for your configured interval—whether that's 30 minutes, 4 hours, or 24 hours. This determines how frequently you want to check for changes.

3. Second Content Fetch & Comparison

After the waiting period, another HTTP Request node fetches the same webpage again. An If node then compares the new content with the previously stored version.

4. Conditional Notification

If the content is identical (no changes), the workflow simply loops back and waits again. If any difference is detected, it triggers the Telegram node to send you an immediate alert with details about what changed.

Pro tip: For monitoring competitor sitemaps or RSS feeds instead of full pages, you can modify the HTTP Request to fetch XML content and parse specific elements, making the detection even more precise.

What You'll Need

  1. A running n8n instance (cloud or self-hosted)
  2. Telegram account and a bot token (free to create via @BotFather)
  3. Your Telegram chat ID (where notifications will be sent)
  4. The URL(s) you want to monitor (competitor sites, news pages, etc.)
  5. Basic understanding of web page structure (optional, for advanced filtering)

Quick Setup Guide

Getting this automation running takes about 10 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure the HTTP Request nodes with the URL you want to monitor.
  3. Set up your Telegram bot if you haven't already (instructions in the workflow notes).
  4. Add your bot token and chat ID to the Telegram node credentials.
  5. Adjust the Wait node timing to your preferred checking frequency.
  6. Activate the workflow and test it with a known-changing webpage.

Pro tip: Start with a longer interval (like 24 hours) when first testing to avoid hitting rate limits. Once confirmed working, adjust to your desired frequency.

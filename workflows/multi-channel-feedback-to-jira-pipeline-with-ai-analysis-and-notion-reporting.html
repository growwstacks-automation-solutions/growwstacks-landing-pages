What This Workflow Does

Product teams drown in feedback scattered across emails, forms, chat messages, and support tickets. Manually collecting, categorizing, and prioritizing these requests consumes hours each week, leads to important insights being lost, and creates a backlog that's based on gut feeling rather than data.

This workflow solves that by creating a fully automated product feedback pipeline. It listens to three channels—Telegram bot messages, Google Form/Sheet submissions, and Gmail emails—normalizes all inputs, sends them through AI analysis for categorization and scoring, calculates a RICE-style priority, creates a Jira ticket with full context, documents everything in Notion, and sends a monthly product intelligence report to stakeholders.

The result is a repeatable, objective process that turns chaotic user input into a structured, actionable product backlog with zero manual triage.

How It Works

1. Multi-channel Intake

Three triggers listen for new feedback: a Telegram bot receives direct messages, a Google Form/Sheet captures structured submissions, and a Gmail filter catches emails with specific subjects. Each path normalizes the data into a unified JSON structure containing the user, message, source, and timestamp.

2. AI-Powered Analysis & Enrichment

The normalized feedback is sent to an AI model (OpenAI or Anthropic) that performs several tasks: classifies the type (bug, feature request, question), extracts sentiment and pain level, estimates business impact and implementation effort, generates relevant tags, and creates a concise summary with acceptance criteria.

3. Priority Calculation & Ticket Creation

A custom calculator applies a RICE-style formula using the AI-derived pain level, impact, effort, and user tier weight to produce an objective priority score (P0-P3). This maps to Jira priority levels. A Jira issue is automatically created with the AI summary, full analysis, and RICE breakdown as the description.

4. Documentation & Notification

A Notion page is generated linking the feedback, Jira ticket, AI analysis, and acceptance criteria for permanent documentation. If the source was Telegram, a rich confirmation message is sent back to the user with the ticket details. All data is logged to a Google Sheet for analytics and dashboarding.

5. Monthly Reporting

Once a month, the workflow aggregates all feedback from the analytics sheet, computes volume trends, category breakdowns, average scores, and top issues. Another AI model generates a written "Product Intelligence Report" with executive summary, key insights, and strategic recommendations, which is saved to Notion and emailed to stakeholders.

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance.
  2. Configure credentials for all required services (Google Sheets, Gmail, Telegram, Jira, Notion, AI provider).
  3. Set up your triggers: create a Telegram bot, connect your Google Form, and define a Gmail filter (e.g., subject contains "[Feedback]").
  4. Map your user tiers in the "User Enrichment" node to assign weights (e.g., enterprise = 1.5, pro = 1.2, free = 1.0).
  5. Adjust priority thresholds in the "Priority Calculator" node to match your team's definitions of P0-P3.
  6. Test with sample feedback from each channel to ensure tickets create correctly and notifications send.
  7. Activate the workflow and let it run continuously, checking the analytics sheet and monthly reports for insights.

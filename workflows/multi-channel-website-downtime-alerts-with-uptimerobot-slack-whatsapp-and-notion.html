Multi-channel Website Downtime Alerts

Automatically detect website outages and notify your team via Slack, WhatsApp, Email, and create a Notion task for immediate resolution.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Website downtime is a silent killer for online businesses. Lost sales, frustrated customers, and damaged reputation happen within minutes. This automation solves that by turning passive monitoring into proactive action.

When UptimeRobot detects your website is down, this workflow instantly triggers alerts across multiple channels: Slack for team chat, WhatsApp for urgent mobile notifications, Email for detailed logs, and creates a Notion task assigned to the responsible engineer. It ensures everyone knows, and someone acts—immediately.

How It Works

Step 1: Downtime Detection

UptimeRobot continuously monitors your website's status. When it changes to "down," it sends a webhook payload to this n8n workflow, containing details like the website URL, downtime timestamp, and monitor name.

Step 2: Alert Generation & Distribution

The workflow parses the webhook data and simultaneously sends formatted alerts to Slack (team channel), WhatsApp (engineer's phone), and Email (management inbox). Each message includes critical details: which site is down, when it happened, and a direct link to the monitor.

Step 3: Task Creation & Assignment

Finally, the workflow creates a new task in a designated Notion database. It tags the responsible engineer based on pre-configured rules (e.g., shift schedule, expertise), sets a high priority, and includes all incident details for tracking resolution.

Pro tip: Configure UptimeRobot to only trigger on "down" status to avoid unnecessary alerts for temporary "paused" or "maintenance" states.

  6. n8n instance: Self-hosted or cloud version to run the workflow.

Quick Setup Guide

  1. Download and import: Click the download button above and import the JSON file into your n8n workspace.
  2. Configure UptimeRobot: In your UptimeRobot monitor settings, add the webhook URL provided by n8n's Webhook node, triggering only on "down" status.
  3. Set up credentials: In n8n, create credentials for Slack, WhatsApp, Email, and Notion using your accounts' API keys or access tokens.
  4. Customize messages: Adjust the alert text in each node to match your team's terminology and include specific instructions.
  5. Test the flow: Temporarily pause your website or use UptimeRobot's test feature to simulate downtime and verify alerts arrive correctly.
  6. Activate: Turn the workflow on in n8n and let it run continuously.

Pro tip: Use n8n's "Function" node to add logic that routes alerts based on time of day or engineer availability, making your response smarter.

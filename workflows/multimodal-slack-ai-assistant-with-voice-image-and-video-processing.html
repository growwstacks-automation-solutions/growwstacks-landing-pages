What This Workflow Does

This template solves a common modern workplace problem: information trapped in different formats. Teams share voice notes in Slack for quick updates, post screenshots and diagrams, share video clips, and of course, send text messages. Manually processing all these different media types is time-consuming and inconsistent.

This multimodal Slack AI assistant automatically detects what type of content was shared—voice message, image, video, or text—processes it through the appropriate AI model, and generates intelligent responses. It turns your Slack channel into a powerful AI-powered collaboration hub where team members can interact with AI using whatever format is most convenient for them.

The business value is substantial: reduced meeting time (voice notes get transcribed and summarized), faster problem-solving (screenshots get analyzed instantly), and consistent information extraction from various media formats. What used to require switching between multiple tools and manual work now happens automatically within your existing communication platform.

How It Works

The workflow acts as an intelligent middleware between Slack and various AI services, routing different content types to specialized processors before generating a unified response.

Step 1: Slack Trigger & Content Detection

When someone mentions your Slack bot in a channel, the workflow triggers. It first examines the message to determine what type of content was shared—whether it contains audio files, images, video attachments, or plain text. This detection happens automatically based on Slack's message metadata and file attachments.

Step 2: Specialized AI Processing

Based on the content type, the workflow routes the media to the appropriate AI service. Voice messages go to OpenAI's Whisper for transcription. Images and videos are sent to Google Gemini for visual analysis and description. Text messages proceed directly to the main AI agent for processing.

Step 3: Context Assembly & AI Response

The extracted information from all media types is combined with the original message context. This enriched context is then sent to your chosen LLM (like GPT-4, Claude, or others) which generates a relevant, helpful response based on your custom instructions and system prompt.

Step 4: Delivery Back to Slack

The AI's response is formatted and posted back to the same Slack channel, creating a seamless conversation. The entire process—from receiving multimodal input to delivering an intelligent response—happens automatically within seconds.

Pro tip: You can customize the system prompt to make your AI assistant behave according to your team's needs—whether it's a customer support bot, technical troubleshooting assistant, or creative brainstorming partner.

What You'll Need

  1. Slack Workspace with Admin Permissions: You need permission to create and install Slack apps in your workspace.
  2. Slack Bot Token: Generated through Slack's API portal with appropriate scopes (app_mentions:read, channels:history, chat:write, files:read).
  3. OpenAI API Key: For processing voice messages through Whisper speech-to-text.
  4. Google Gemini API Key: For analyzing images and video content (alternatives available).
  5. Primary LLM Access: API key for your chosen AI model (OpenAI GPT, Anthropic Claude, etc.) to power the main assistant.
  6. n8n Instance: Either self-hosted or n8n.cloud account to run the workflow.

Quick Setup Guide

Follow these steps to deploy your multimodal Slack AI assistant:

  1. Download and Import: Download the JSON template file and import it into your n8n instance.
  2. Create Slack App: Go to api.slack.com → "Create New App" → "From Scratch." Name your bot and select your workspace.
  3. Configure OAuth Scopes: Under "OAuth & Permissions," add these Bot Token Scopes: app_mentions:read, channels:history, channels:join, channels:read, chat:write, files:read, links:read, links:write.
  4. Install to Workspace: Click "Install to Workspace" and copy the "Bot User OAuth Token" that appears.
  5. Configure n8n Slack Credentials: In the workflow, open the Slack trigger node, create new credentials, and paste your bot token.
  6. Add AI API Keys: Configure the OpenAI, Gemini, and your chosen LLM nodes with their respective API keys.
  7. Test Your Bot: Invite the bot to a channel using @your-bot-name and send a test message with different media types.

Important: Remember to replace the placeholder "[Your bot access token goes here]" in the HTTP Request nodes with your actual token, keeping the "Bearer " prefix intact.

What This Workflow Does

This automation solves a critical problem for teams using n8n: workflow protection. Your n8n workflows represent hours of development and core business processes. Without backups, a server failure, accidental deletion, or corruption could wipe out your automation infrastructure.

The workflow automatically exports all your n8n workflows as JSON files to a designated folder in Nextcloud. It runs daily (or on any schedule you choose), creates the backup directory if needed, converts each workflow to a clean JSON format, uploads them securely, and maintains a 7-day retention policy—automatically deleting older backups to conserve storage.

This transforms backup from a manual, unreliable task into a systematic, error-proof process. You get offsite storage within your own infrastructure (Nextcloud), versioned backups for recovery, and peace of mind that your automation investments are protected.

How It Works

Step 1: Scheduled Trigger

The workflow activates automatically via a schedule trigger (default: daily). You can also run it manually for immediate backup. This ensures backups happen consistently without human intervention.

Step 2: Directory Preparation

The workflow checks if the /N8N-Backup directory exists in your Nextcloud. If not, it creates it. This eliminates pre-configuration requirements and makes the workflow self-sufficient.

Step 3: Workflow Collection

It fetches all workflows from your n8n instance using the "Get many workflows" node with filters to include nodes, connections, settings, pinData, and tags—capturing the complete workflow structure.

Step 4: JSON Conversion & File Preparation

Each workflow is converted to a clean JSON object (no arrays or meta fields) using a Set node that builds JSON in binary data. Filenames are sanitized to avoid special-character issues.

Step 5: Upload to Nextcloud

The JSON files are uploaded to Nextcloud with binary property mapping configured for proper file handling. Each backup is stored with timestamp information for organization.

Step 6: Retention Management

The workflow lists existing backups, identifies those older than 7 days, and deletes them from Nextcloud. This maintains storage efficiency while keeping recent backups available.

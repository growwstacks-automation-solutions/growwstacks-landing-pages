What This Workflow Does

This automation solves the challenge of creating visual content from diverse customer inputs without manual intervention. Businesses often receive requests for images through various channels—text descriptions, reference photos, voice instructions, or document attachments—and traditionally require human designers to interpret and create these visuals.

The workflow connects Telegram as a universal interface with Google's cutting-edge Nano Banana (Gemini 2.5 Flash Image) AI model. It accepts any input format, intelligently processes it using OpenAI GPT models for understanding and prompt optimization, then generates high-quality, tailored images automatically. This eliminates the design bottleneck, reduces turnaround time from hours to seconds, and enables scalable visual content production.

Beyond simple generation, the bot maintains conversation memory, allowing iterative refinement ("make it darker", "add more contrast", "change the background"). This creates a collaborative design experience through chat, perfect for businesses offering personalized visual services or internal teams needing rapid prototyping.

How It Works

Step 1: Multi-modal Input Reception

The Telegram bot listens for messages, images, voice recordings, and PDF files. Each input type triggers specialized processing: text goes directly to analysis, images are analyzed for content, voice is transcribed, PDF text is extracted.

Step 2: Intelligent Prompt Construction

OpenAI GPT models analyze the input context, combine it with conversation history if applicable, and construct optimized prompts specifically tuned for Nano Banana's capabilities. This ensures the AI receives clear, effective instructions for image generation.

Step 3: Nano Banana Image Generation

Google's Nano Banana (Gemini 2.5 Flash Image) model generates the visual content based on the refined prompt. This state-of-the-art model produces superior quality images with commercial-grade consistency at approximately $0.04 per image.

Step 4: Delivery and Conversation Continuation

The generated image is sent back through Telegram, and the conversation memory is updated. Users can immediately request modifications, creating an iterative design loop without restarting the process.

What You'll Need

  1. A Telegram bot token created via @BotFather
  2. A Google Gemini API key from Google AI Studio for Nano Banana image generation
  3. An OpenAI API key for GPT models (conversation, analysis, transcription)
  4. An n8n instance (cloud or self-hosted) to run the workflow
  5. Basic understanding of API credential management

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n workspace.
  2. Configure the three API credentials: Telegram bot token, Google Gemini key, OpenAI key.
  3. Update the HTTP request nodes with your bot token for file download functionality.
  4. Test the workflow by sending a text prompt, image, voice message, or PDF to your Telegram bot.
  5. Monitor the generated images and conversation memory to ensure iterative modification works.
  6. Deploy the workflow as a webhook or schedule it for continuous operation.

Pro tip: Start with text-only prompts to verify basic functionality before testing multi-modal inputs. This isolates any issues with the core image generation pipeline before adding complexity.

What This Workflow Does

Security Operations Center (SOC) teams are often overwhelmed with alerts scattered across different tools. Manually checking TheHive for new cases and then notifying the team via Slack or email creates delays and increases the risk of missing critical incidents.

This n8n automation solves that by creating a real-time bridge between TheHive and Slack. Whenever a new security case is created in TheHive, it automatically triggers a detailed Slack notification in your designated channel. More than just an alert, this workflow allows analysts to take immediate action—like updating the assignee, severity, or status—directly from Slack using interactive buttons or forms, without ever leaving their collaboration hub.

It turns Slack into a command center for your security operations, reducing mean time to detect (MTTD) and mean time to respond (MTTR) by eliminating manual steps and context switching.

How It Works

1. TheHive Triggers the Workflow

The workflow starts when a new case is created in TheHive. The n8n webhook or TheHive trigger node listens for this event, capturing all case details like title, description, severity, and tags.

2. Data Processing & Enrichment

The workflow processes the incoming case data. It can format the description, map severity levels to specific Slack channels or mentions (e.g., @security-team for 'Critical' cases), and add a direct link back to the case in TheHive for quick access.

3. Smart Slack Notification

A rich, interactive message is posted to your chosen Slack channel. This isn't just text—it can include buttons for common actions ("Assign to Me", "Mark as False Positive") or a form to update case attributes on the spot.

4. Two-Way Interaction (Optional)

When an analyst interacts with the Slack message (clicks a button, submits a form), the workflow receives this input and uses TheHive's API to update the corresponding case—closing the loop without manual data entry.

What You'll Need

  1. A running n8n instance (cloud or self-hosted).
  2. TheHive instance with API access credentials.
  3. Slack workspace with permissions to create a Slack app and post to channels.
  4. Slack App configuration with a Bot User OAuth Token and the necessary scopes (chat:write, commands, interactive components).
  5. Basic understanding of webhook URLs for setting up the trigger.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure TheHive credentials in the TheHive node. Input your TheHive URL and API key.
  3. Set up the Slack bot by creating a Slack app in your workspace, installing it, and adding the Bot Token to the Slack node in the workflow.
  4. Define your Slack channel where notifications should be sent (e.g., #security-alerts).
  5. Test the workflow by creating a test case in TheHive and verifying the Slack notification appears correctly.
  6. Activate the workflow and let it run. Monitor the first few alerts to ensure everything is working as expected.

Pro tip: Start by sending notifications to a small, dedicated test channel with your core SOC team. Fine-tune the message format and actions based on their feedback before rolling it out to broader channels.

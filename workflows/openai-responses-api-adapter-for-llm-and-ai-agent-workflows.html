What This Workflow Does

This template solves a specific but critical problem in AI automation: API compatibility. When OpenAI released their new Responses API with enhanced features for AI agents and structured outputs, it wasn't directly compatible with existing LLM frameworks and automation tools built for the older Chat Completions API.

The adapter acts as a middleware layer that intercepts requests from your existing AI workflows (using n8n's LLM or AI Agent nodes) and transforms them to work with the new Responses API. This allows you to access improved features like more reliable tool calling and structured outputs without rewriting your entire automation stack.

For businesses, this means you can upgrade to OpenAI's latest technology while maintaining your investment in existing AI automations. The adapter handles the technical translation transparently, so your customer support bots, content generators, or data analysis agents continue working while gaining access to better capabilities.

How It Works

The adapter uses n8n's webhook capabilities to create a proxy between your existing workflows and OpenAI's new API endpoint.

Step 1: Request Interception

Your existing AI agent or LLM node is configured with a custom OpenAI credential that points to the adapter's webhook URL instead of directly to OpenAI. When your workflow runs, the request first goes to the adapter.

Step 2: Format Transformation

The adapter parses the incoming request (in the old Chat Completions format) and maps all parameters to the new Responses API format. This includes converting messages, tools, and other settings to their new equivalents.

Step 3: API Communication

The transformed request is sent to OpenAI's actual Responses API endpoint using n8n's HTTP Request node. The adapter handles authentication and all required headers.

Step 4: Response Translation

When OpenAI responds, the adapter converts the Responses API format back to the Chat Completions format that your existing LLM nodes expect. This ensures complete compatibility with your workflow.

Step 5: Delivery Back to Workflow

The translated response is returned through the webhook to your original AI node, which processes it as if it came directly from the old API. Your workflow continues normally with no changes required.

Pro tip: This adapter pattern isn't just for OpenAI. You can use the same approach to bridge compatibility between any old and new APIs in your automation stack, saving countless hours of migration work.

  1. An active OpenAI account with access to the Responses API (available to most API users)
  2. An n8n instance (cloud or self-hosted) where you can import and run workflows
  3. Existing AI workflows using n8n's LLM or AI Agent nodes (or you can create new ones)
  4. Basic understanding of webhooks and API authentication (the template handles the complexity)
  5. Your OpenAI API key to configure the HTTP Request node in the adapter

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance
  2. Activate the workflow to enable the webhook endpoints (they must be active to receive requests)
  3. Configure the HTTP Request node with your OpenAI API key and ensure it points to the correct Responses API endpoint
  4. Create a custom OpenAI credential in n8n that uses the adapter's webhook URL as the base URL
  5. Update your existing AI workflows to use this custom credential instead of the standard OpenAI credential
  6. Test with a simple query to verify the adapter is translating requests and responses correctly
  7. Monitor performance for the first few runs to ensure stability before scaling up

Important: Keep the adapter workflow active whenever your AI automations are running. The webhooks need to be live to intercept and process requests.

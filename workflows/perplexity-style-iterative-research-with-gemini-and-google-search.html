What This Workflow Does

Manual market research is slow, inconsistent, and hard to scale. This automation solves that by replicating the sophisticated, iterative research process of tools like Perplexity, but within your own controlled environment. It transforms a simple business question into a deep, multi-source investigation.

The workflow takes a user's query—like "What are the emerging trends in sustainable packaging for e-commerce?"—and doesn't just perform a single search. It uses Google's Gemini AI to brainstorm the best possible search terms, executes those searches via Google's search API, critically analyzes the results to find contradictions or missing information, and then loops back to ask better follow-up questions. The final output is a consolidated, well-cited report that answers the original question comprehensively, saving you hours of browsing and synthesis.

How It Works

The automation is built as a stateful loop, managed efficiently using Redis for persistence. This allows it to "remember" what it has learned and decide what to do next.

Step 1: Intelligent Query Generation

The Google Gemini model first deconstructs the user's broad question. It doesn't just rephrase it; it breaks it down into 3-5 distinct, complementary search queries designed to cover the topic from different angles, gather quantitative data, find expert opinions, and uncover recent news.

Step 2: Parallel Web Research

The workflow sends all generated queries to the Google Search API simultaneously. It fetches snippets, titles, and URLs from the top results. This parallel execution mimics a researcher opening multiple browser tabs at once, drastically speeding up the data collection phase.

Step 3: AI-Powered Reflection & Gap Analysis

This is the core intelligence. Another Gemini instance reviews all the gathered information. It asks itself: "Is this answer complete? Are sources conflicting? What important subtopic wasn't covered?" Based on this analysis, it generates a new set of refined, gap-filling search queries.

Step 4: Iterative Looping Until Satisfaction

The workflow returns to Step 2 with the new queries. This loop continues for a predefined number of cycles or until the AI's reflection indicates sufficient coverage has been achieved. Redis keeps track of the loop count and aggregates all findings.

Step 5: Synthesis & Delivery

Finally, all collected data from multiple search iterations is merged. Gemini synthesizes it into a coherent, structured answer complete with citations (source URLs). This final report can be delivered via email, saved to a database, or posted to a Slack channel.

What You'll Need

  1. A Google Gemini API Key: This provides access to the Gemini AI models for both query generation and reflection analysis. You can obtain one from Google AI Studio.
  2. A Redis Database Instance: Essential for managing the workflow's state across loop iterations. You can use a cloud service like Redis Cloud, Aiven, or a self-hosted Redis server.
  3. An n8n Instance: You can use n8n.cloud (hosted), self-host n8n, or run it locally to import and execute this workflow.
  4. Basic Understanding of API Credentials: You'll need to create credentials in n8n for the Gemini API and your Redis connection.

Pro tip: Start with a simple, focused question for your first test. "What are the best project management tools for remote software teams in 2024?" is better than a vague "Tell me about project management." This helps you calibrate the workflow's depth and output quality.

Quick Setup Guide

  1. Download & Import: Click the "Download Template" button above. In your n8n instance, go to Workflows > Import from File and select the downloaded JSON file.
  2. Configure Credentials: In the imported workflow, locate the three nodes requiring the "Google Gemini" credential (Chat Model, Search, and Reflection). Create a new credential using your Gemini API key and connect it to all three.
  3. Set Up Redis: Find all nodes labeled "Redis". Create a Redis credential with your database host, port, and password (if required), and connect it to each Redis node.
  4. Adjust Parameters (Optional): Open the "Set" node at the beginning. You can modify variables like `number_of_initial_queries` (default 3) and `max_research_loops` (default 3) to control the research depth.
  5. Test & Execute: Use the "Chat Trigger" node to input your first research question. Activate the workflow and watch as it automatically generates queries, searches, reflects, and delivers a final answer.

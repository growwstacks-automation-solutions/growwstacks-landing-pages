What This Workflow Does

This automation solves the morning scramble of checking multiple apps to plan your day. It seamlessly connects your Google Calendar with real-time weather data, uses AI to craft a thoughtful summary, and delivers it directly to your Telegram every morning.

Instead of manually cross-referencing your schedule with weather forecasts, this workflow automatically fetches today's and tomorrow's events, retrieves location-specific weather conditions (temperature, humidity, wind, UV index), and generates a human-friendly briefing that helps you prepare effectively. Whether you have outdoor meetings, travel plans, or just want to know if you need an umbrella, this assistant ensures you're informed and ready.

The system intelligently handles missing location data—if an event has no location, it still includes it in your agenda without weather details. This makes it robust enough for real-world use where not every calendar entry includes a physical address.

How It Works

The workflow runs on a scheduled trigger (typically each morning) and follows a logical sequence to compile your personalized briefing.

1. Schedule Trigger & Timezone Setup

A Schedule Trigger node initiates the workflow at your specified time (e.g., 7:00 AM daily). A Set node ensures all timestamps are adjusted to your local timezone, preventing confusion with events pulled from Google Calendar.

2. Fetch Calendar Events

The Google Calendar node retrieves events for the current day and the next day. It connects via OAuth2 to your calendar, filtering for events within your specified time range and primary calendar ID.

3. Extract & Validate Locations

An If node checks each event for location data. Events with valid locations proceed to weather lookup; events without locations bypass this step and move directly to the agenda compilation stage.

4. Retrieve Weather Data

For events with locations, an HTTP Request node queries the WeatherAPI with your API key. It fetches current conditions and forecast data including temperature, conditions, humidity, wind speed, and UV index—all crucial for planning your day.

5. AI-Powered Summarization

An OpenRouter Chat Model node (using Gemini or similar AI) receives the structured event and weather data. It generates a concise, friendly message that highlights key appointments, suggests preparations based on weather, and maintains a motivational tone.

6. Telegram Delivery

The final Telegram node sends the AI-crafted message to your specified chat ID. You receive a clean, readable briefing in your Telegram app each morning, combining everything you need to know about your day.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow execution capabilities
  2. Google Calendar account with OAuth2 credentials configured in n8n
  3. WeatherAPI key (free tier available from weatherapi.com)
  4. Telegram Bot Token and your personal/group chat_id
  5. OpenRouter API key (or alternative AI model access) for summarization
  6. Basic understanding of n8n node configuration (credentials, field mapping)

Pro tip: Use the free tier of WeatherAPI for personal use—it provides ample calls for daily briefings. For business use with multiple locations, consider their affordable paid plans.

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure the Schedule Trigger to run at your preferred morning time (e.g., 7:00 AM daily).
  3. Set up Google Calendar OAuth2 credentials in n8n and connect them to the "Get Many Events" node.
  4. Enter your WeatherAPI key in the HTTP Request node's authentication settings.
  5. Add your OpenRouter API credentials to the AI Chat Model nodes (you can use Gemini, Claude, or other supported models).
  6. Configure Telegram nodes with your Bot Token and chat_id (find your chat_id via @userinfobot on Telegram).
  7. Test the workflow with a manual trigger to ensure all connections work, then activate it.

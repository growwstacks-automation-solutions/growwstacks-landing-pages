What This Workflow Does

This workflow creates a complete MCP (Model Context Protocol) server that exposes all ProfitWell Tool operations to AI agents. It solves the problem of AI assistants being disconnected from critical subscription and revenue data. Instead of manually checking ProfitWell dashboards or building custom integrations, your AI agents can now directly query company settings, retrieve metrics, and analyze subscription performance through natural language commands.

The template provides immediate access to both ProfitWell operations with zero configuration needed. AI agents automatically receive properly formatted API responses, enabling them to answer complex questions about MRR, churn rates, customer lifetime value, and other key subscription metrics without any manual intervention.

How It Works

The workflow follows a clean MCP server pattern that's optimized for AI agent interaction:

1. MCP Trigger Endpoint

The workflow starts with an MCP Server Trigger node that creates a webhook endpoint. This serves as the connection point for AI agents like Claude Desktop, custom AI applications, or other n8n workflows. When activated, it generates a unique URL that AI agents can call to access ProfitWell operations.

2. Request Parsing & Validation

Incoming AI agent requests are automatically parsed to extract parameters. The workflow uses $fromAI() expressions that dynamically populate required fields based on what the AI agent provides. This eliminates manual parameter mapping and ensures AI agents can use natural language to specify what data they need.

3. ProfitWell API Integration

The core of the workflow consists of pre-configured ProfitWell Tool nodes for both available operations: Get Company Settings and Get Metric. These nodes use n8n's native ProfitWell integration with built-in error handling, retry logic, and proper credential management.

4. Response Formatting

ProfitWell API responses are formatted into a structure that AI agents can easily understand and process. The workflow ensures consistent output formatting regardless of which operation was called, making it simple for AI agents to parse and utilize the data.

  1. n8n instance (cloud or self-hosted) with workflow execution capabilities
  2. ProfitWell account with API access enabled
  3. ProfitWell API credentials (Public Key and Secret Key) configured in n8n
  4. AI agent platform that supports MCP servers (Claude Desktop, custom AI apps, etc.)
  5. Basic understanding of webhook URLs and API authentication

Pro tip: Before activating the workflow, test your ProfitWell credentials in n8n's credential manager to ensure they work correctly. This prevents activation failures and makes troubleshooting easier.

Quick Setup Guide

  1. Import the template into your n8n instance using the JSON file
  2. Configure ProfitWell credentials in n8n's credential manager with your Public and Secret Keys
  3. Activate the workflow to start the MCP server and generate your webhook URL
  4. Copy the webhook URL from the MCP Server Trigger node configuration
  5. Connect your AI agent by adding the MCP server URL to its configuration (Claude Desktop: Settings → Developer → MCP Servers)
  6. Test the connection by asking your AI agent a ProfitWell-related question like "What's our current MRR?"

What This Workflow Does

This automation solves a common pain point for content teams: the tedious, manual process of moving content from Google Docs to WordPress while maintaining SEO quality. Instead of copying, pasting, and manually optimizing each post, this workflow automates the entire publishing pipeline.

It transforms "Document Ops" from a time-consuming chore into a seamless, automated process. The workflow monitors your Google Drive for new content, cleans the formatting, checks for duplicates, uses AI to generate SEO metadata, and publishes directly to WordPress with RankMath fields pre-populated. The result is consistent, high-SEO content published in minutes instead of hours.

How It Works

Step 1: Monitor Google Drive for New Content

The workflow watches a designated "Drafts" folder in Google Drive. When a new HTML or Google Doc file appears, it triggers the automation process. This eliminates manual file management and ensures no content gets overlooked.

Step 2: Clean and Prepare Content

Raw HTML from Google Docs often contains messy styles and unnecessary tags. The workflow sanitizes this content, removing formatting inconsistencies while preserving your intended structure and readability.

Step 3: Smart Duplicate Detection

Before publishing, the workflow checks if a similar post already exists on your WordPress site by comparing slugs. This prevents content duplication and decides whether to create a new draft or update an existing post.

Step 4: AI-Powered SEO Analysis

Using Gemini AI, the workflow analyzes your content to extract the best focus keyword, generate compelling SEO titles, and craft meta descriptions that follow SEO best practices while accurately representing your content.

Step 5: WordPress Publishing with RankMath Integration

The workflow publishes the content directly to WordPress while populating RankMath's custom meta fields with the AI-generated SEO data. This ensures immediate SEO optimization without manual intervention.

Step 6: Automated Archiving

Once published, the original file moves to a "Published" folder in Google Drive, keeping your workspace organized and providing a clear audit trail of processed content.

Pro tip: Configure the workflow to send a notification to your team when content is published, creating a seamless review and approval process without manual follow-ups.

What You'll Need

  1. Google Drive access with OAuth2 credentials and appropriate folder permissions
  2. Google Gemini API key for AI-powered SEO analysis
  3. WordPress installation with the REST API enabled and Application Passwords configured
  4. RankMath SEO plugin installed on your WordPress site
  5. Custom WordPress helper snippet to expose RankMath meta fields to the REST API
  6. n8n instance (cloud or self-hosted) to run the workflow

Important: You must add a custom PHP snippet to your WordPress site's mu-plugins folder to expose RankMath's meta fields to the REST API and fix authentication headers. This is a one-time setup that enables the automation to work properly.

Quick Setup Guide

1. Download the template and import it into your n8n instance.

2. Configure credentials for Google Drive, Gemini AI, and WordPress Application Passwords.

3. Add the WordPress helper snippet to your site's mu-plugins folder.

4. Set your WordPress base URL and Google Drive folder IDs in the configuration node.

5. Test with a sample Google Doc to verify the publishing flow works correctly.

6. Deploy the workflow and start automating your content publishing process.

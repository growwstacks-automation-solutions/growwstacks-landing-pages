What This Workflow Does

Managing a consistent social media presence across multiple platforms is a massive time sink. Manually uploading the same post to X, Instagram, Facebook, LinkedIn, TikTok, and Threads can eat up hours each week, leading to inconsistencies, missed platforms, and scheduling headaches.

This n8n automation solves that by creating a single, unified submission form. Marketing teams, content creators, or business owners can fill out one form with their caption, select target platforms, and upload an image or video. The workflow then automatically routes the content to the correct APIs (via a service like Upload-Post) and publishes it simultaneously across all selected networks.

It handles the complexity of different API requirements, file formats, and authentication, turning a multi-step, error-prone process into a one-click operation. This ensures your brand message is delivered consistently everywhere, on time, every time.

How It Works

The workflow acts as a central dispatcher for your social content, intelligently processing and routing each submission.

1. Form Submission & Data Collection

The process begins with a user-friendly n8n Form Trigger node. Team members or clients submit a post through a simple web form. The form captures the essential details: the post caption, the specific social media platforms to publish to (selected via checkboxes or a dropdown), the account/profile name, and the image or video file itself.

2. Content Type Detection & Routing

A "Switch" node examines the uploaded file's MIME type. If it's an image (like JPEG or PNG), the workflow branches to the image-publishing path. If it's a video (like MP4 or MOV), it takes the video-publishing path. This is crucial because different platforms and APIs often have separate endpoints or parameters for photos versus videos.

3. API Integration & Platform Publishing

Each branch contains an HTTP Request node configured to communicate with the Upload-Post.com API (or a similar social media posting service). The node packages the caption, selected platform identifiers, account credentials, and the file binary into a properly formatted API request. It sends this request to publish the content.

4. Success/Error Handling & Notification

After the API call, the workflow checks the response. A successful post returns a confirmation, which can be logged to a database or spreadsheet for record-keeping. If an error occurs (like an invalid token or file size limit), the workflow captures the error message and can trigger an alert via email, Slack, or another channel, allowing for immediate manual intervention.

Pro tip: Use the "Split In Batches" node if you need to add a delay between posts to avoid triggering spam filters on the social platforms' APIs.

What You'll Need

  1. A running n8n instance (cloud or self-hosted).
  2. An API key from Upload-Post.com or a similar multi-platform social media posting service. The free tier of Upload-Post offers 10 uploads per month to get started.
  3. Social media accounts set up and connected to your chosen API service. You'll need to have your profiles (e.g., Facebook Page, Instagram Business Account, X developer app) authorized within the service's dashboard.
  4. Basic familiarity with n8n to import the JSON template and paste in your API credentials.

Quick Setup Guide

You can have this automation running in under 15 minutes.

  1. Download the template using the button above and import it into your n8n canvas.
  2. Get your API key from your Upload-Post.com account dashboard.
  3. Configure the HTTP Request nodes ("Post photo" and "Post video"). In each node's "Authentication" section, add a Header with the name Authorization and the value Apikey YOUR_API_KEY_HERE.
  4. Test the form trigger by clicking "Test workflow" in n8n and submitting a sample post with an image.
  5. Activate the workflow and share the form URL with your team or embed it in your internal tools to start publishing.

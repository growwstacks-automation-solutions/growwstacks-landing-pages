Read RSS Feed from Two Different Sources

Automate content monitoring from multiple RSS feeds. Aggregate news, blog updates, and industry content without manual checking.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manually checking multiple websites, blogs, or news sources for updates is a repetitive task that consumes valuable time. This automation solves that by connecting to two different RSS feeds simultaneously, fetching new content items, and processing them according to your business rules.

The workflow acts as your 24/7 content monitoring assistant. It can filter items by keywords, exclude irrelevant topics, format the information, and deliver it to your preferred destination—whether that's a Slack channel, email digest, database, or internal dashboard. Instead of scattered manual checks, you get consolidated, processed intelligence delivered automatically.

How It Works

Step 1: Fetch from Multiple Sources

The workflow connects to two separate RSS feed URLs simultaneously. It checks each feed for new items since the last execution, ensuring you capture updates from both sources in a single run.

Step 2: Process and Filter Content

Each retrieved item passes through filtering logic. You can configure keyword matching, date ranges, or content type filters to ensure only relevant items proceed further in the workflow.

Step 3: Transform and Structure Data

The workflow extracts key elements like titles, descriptions, publication dates, and links. It structures this data consistently, making it ready for integration with other systems or for human review.

Step 4: Deliver to Destination

Processed items route to your chosen output—posting to Slack channels, adding rows to Google Sheets, sending formatted emails, or triggering subsequent automations based on the content.

What You'll Need

  1. n8n instance (cloud or self-hosted)
  2. RSS feed URLs you want to monitor
  3. Destination credentials (Slack webhook, Google Sheets access, email SMTP, etc.)
  4. Filtering criteria (keywords, topics to include/exclude)

Pro tip: Start with just two critical feeds to refine your filtering logic before adding more sources. Quality monitoring beats quantity—better to get precise alerts from a few key sources than overwhelming noise from many.

Quick Setup Guide

  1. Import the template: Download the JSON file and import it into your n8n instance.
  2. Configure RSS sources: Replace the placeholder feed URLs with your actual target RSS feeds.
  3. Set up filtering: Adjust the filter nodes to match your keyword criteria and content preferences.
  4. Connect your destination: Configure the output node (Slack, email, spreadsheet, etc.) with your credentials.
  5. Test and schedule: Run a manual test to verify the flow, then set an appropriate schedule (hourly, daily, etc.).

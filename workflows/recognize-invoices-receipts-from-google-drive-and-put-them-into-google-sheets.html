What This Workflow Does

Manual invoice processing is a tedious, error-prone task that consumes valuable hours each month. This automation workflow solves that problem by automatically scanning a designated Google Drive folder for new invoice files (PDF, PNG, or JPG format), extracting key data using OCR technology, and populating a structured Google Sheets spreadsheet with the results.

The workflow operates on two triggers: you can manually run it via a test button to process existing invoices, or it can automatically monitor the folder for new files, processing them as soon as they arrive. This creates a seamless, real-time pipeline from document storage to organized financial data.

For businesses, this translates to eliminating manual data entry, reducing processing time from hours to minutes, ensuring accuracy in financial records, and freeing up staff for higher-value tasks. The extracted data—vendor names, dates, amounts, line items—is ready for accounting, reporting, or further analysis.

How It Works

Step 1: Folder Monitoring & File Detection

The workflow continuously monitors a specified Google Drive folder. When a new invoice file (PDF, PNG, JPG) is uploaded, it automatically detects the file and prepares it for processing. This trigger ensures real-time automation without manual intervention.

Step 2: OCR Data Extraction

Each detected file is sent to an OCR (Optical Character Recognition) API via an HTTP request. The API scans the document image, identifies text patterns, and extracts structured data such as vendor information, invoice date, total amount, item descriptions, and quantities.

Step 3: Data Transformation & Cleaning

The raw extracted data is processed through transformation nodes to format dates consistently, convert amounts to numerical values, clean text fields, and structure the information into standardized columns. This ensures the output is clean and ready for spreadsheet import.

Step 4: Google Sheets Population

The formatted data is appended as new rows to a predefined Google Sheets spreadsheet. Each invoice becomes a row with columns for vendor, date, amount, description, and status. The sheet serves as a centralized, automatically updated invoice register.

What You'll Need

  1. n8n Instance: A running n8n environment (cloud or self-hosted) to import and execute the workflow.
  2. Google Account: With access to Google Drive and Google Sheets, and configured OAuth credentials in n8n.
  3. OCR API Key: Access to an OCR service (like the referenced RapidAPI) with an active subscription/key.
  4. Google Sheets Template: A preconfigured Google Sheets spreadsheet with columns matching the extracted data structure.
  5. Invoice Files: Digital invoices in PDF, PNG, or JPG format stored in a dedicated Google Drive folder.

Quick Setup Guide

Follow these steps to get the automation running in your environment:

  1. Import the Workflow: Download the JSON template and import it into your n8n instance via the workflow import function.
  2. Configure Google Credentials: Set up Google OAuth credentials in n8n to allow access to your Drive and Sheets.
  3. Set OCR API Key: Create an HTTP header credential in n8n and paste your OCR API key (from RapidAPI or similar service).
  4. Prepare Google Sheets: Duplicate the provided Sheets template or create your own with columns: Vendor, Date, Amount, Description, Status.
  5. Specify Drive Folder: Update the Google Drive node to point to your actual invoice storage folder ID.
  6. Test Execution: Trigger the workflow manually to process existing files, then activate the folder monitor for automatic future processing.

Pro tip: Create a dedicated "Invoices" folder in Google Drive with subfolders by year or month. The workflow can monitor the main folder, keeping your digital filing system organized while automating extraction.

Release New Versions via Telegram Bot Command

Automate your DevOps release pipeline with a simple Telegram command. Deploy new versions securely from anywhere.

What This Workflow Does

Manual software deployments are time-consuming, error-prone, and require developers to be at their computers. This automation solves that by letting you trigger new version releases through a simple Telegram bot command like /deploy v2.1.0.

The workflow integrates Telegram with GitHub (or other version control systems) to create a secure, auditable release pipeline. It's perfect for continuous delivery environments where you need quick, reliable deployments without complex infrastructure.

Beyond basic deployment, this template includes validation logic to ensure only authorized users can trigger releases, version format checking, and proper logging. It transforms a previously manual 10-15 minute process into a 10-second command.

How It Works

1. Telegram Command Trigger

The workflow listens for specific commands in your Telegram bot. When an authorized user sends /deploy [version], the bot captures the message and validates the user's permissions.

2. Command Validation & Parsing

An IF node checks if the command matches the expected pattern. If valid, a Set node extracts the version number from the message for use in subsequent steps. Invalid commands are rejected with an error message.

3. GitHub Release Creation

The workflow connects to your GitHub repository using the extracted version to create a new release tag. It can also generate release notes, attach binaries, or trigger additional CI/CD pipelines.

4. Notification & Logging

After successful deployment, the bot sends confirmation messages back to Telegram. All deployment activities are logged for audit purposes, creating a complete history of who deployed what and when.

What You'll Need

  1. A Telegram bot token (create via @BotFather)
  2. GitHub personal access token with repo permissions
  3. n8n instance (cloud or self-hosted)
  4. Basic understanding of your project's versioning scheme
  5. List of authorized Telegram user IDs for security

Pro tip: Start by testing this workflow in a staging environment before moving to production. Use a separate Telegram bot and GitHub test repository to validate everything works without risking your live systems.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance
  2. Configure the Telegram trigger node with your bot token and set up webhooks
  3. Add your GitHub credentials to the GitHub node and specify your repository
  4. Set up user validation by adding authorized Telegram user IDs to the IF node conditions
  5. Test with a dry run using a test repository before connecting to production
  6. Activate the workflow and try deploying with /deploy test-1.0.0

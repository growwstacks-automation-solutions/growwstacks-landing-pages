What This Workflow Does

This workflow solves a critical challenge in AI automation: reliably getting structured JSON output from AI agents without depending on often-unreliable structured output parsers. It implements a robust validation system that ensures your AI returns data in exactly the format your systems require.

The template demonstrates this with a nutritional information use case, but the pattern applies to any scenario where you need predictable JSON output - from customer support responses to data extraction results. The workflow handles up to 4 retries with progressively clearer instructions when the initial output doesn't match your schema.

How It Works

1. Initial AI Request

The workflow starts by sending your prompt to the AI (OpenAI by default) with clear instructions about the required JSON format. The system prompt is carefully crafted to maximize first-attempt success.

2. Schema Validation

A Switch node checks if the response matches your expected JSON structure. This validation can be customized for any schema requirements.

3. Success Path

When valid JSON is received, the workflow proceeds to process the data through a Set node that serves as your integration point with other systems.

4. Retry Logic

If the response fails validation, the workflow automatically retries with adjusted prompts that provide more explicit formatting guidance. This happens up to 4 times before failing gracefully.

Pro tip: For best results with OpenAI models, use gpt-4.1-nano or similar versions which tend to follow JSON formatting instructions particularly well.

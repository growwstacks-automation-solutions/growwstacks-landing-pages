Restaurant Daily Insights with Gemini AI

Automatically fetch sales, waste, and feedback data, generate AI-powered insights, and send daily email summaries.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Running a restaurant involves juggling sales data, inventory waste, and customer feedback—often manually. This automation solves that by creating a self-running system that delivers daily insights without human intervention.

The workflow automatically retrieves daily sales figures, food waste records, and customer feedback from Google Sheets. It then uses Gemini AI to analyze this data, identifying top-performing dishes, waste reduction opportunities, and common feedback themes. Finally, it merges these insights into a structured report and sends it as an email draft for review or distribution.

Think of it as your tireless data analyst: every day, it processes raw numbers into actionable recommendations, saving managers hours of manual work and providing consistent, error-free insights.

How It Works

Step 1: Scheduled Trigger

The workflow initiates daily at a set time (e.g., 9 AM) using a scheduler. This ensures consistent reporting without manual activation.

Step 2: Data Fetching

Three separate nodes pull data from Google Sheets: sales records, food waste logs, and customer feedback entries. Each dataset is structured with date, item details, quantities, and reasons.

Step 3: Data Normalization

The raw data is standardized for AI analysis—formatting dates, cleaning text, and organizing fields. This prepares it for accurate insight generation.

Step 4: AI Analysis

Gemini AI processes each dataset: sales data to highlight top performers and profit trends; waste data to suggest reduction strategies; feedback to summarize themes and sentiment.

Step 5: Insight Formatting

The AI outputs are structured into readable sections—bullet points for sales insights, actionable recommendations for waste, and summarized feedback highlights.

Step 6: Report Merging & Email Creation

All formatted insights are combined into a cohesive daily report. The content is then prepared as an email draft with a clear subject line and body.

Step 7: Email Delivery

The final email is sent via Gmail to designated recipients (manager, owner, team). The report is ready for immediate review or further distribution.

Pro tip: Schedule this workflow to run after closing hours. This ensures all daily data is captured and the report arrives fresh each morning.

What You'll Need

  1. Google Sheets Setup: Three separate sheets for sales, waste, and feedback data with consistent columns (Date, Item, Quantity, etc.).
  2. Google & Gemini API Credentials: OAuth2 access for Sheets and Gmail, plus API keys for Gemini AI model access.
  3. n8n Instance: A running n8n environment (cloud or self-hosted) with the necessary nodes installed.
  4. Email Recipients: Defined Gmail addresses for report delivery (manager, owner, relevant staff).
  5. Daily Data Entry Discipline: Ensure sales, waste, and feedback data is logged consistently in the Sheets—the automation depends on updated inputs.

Quick Setup Guide

Import and configure this workflow in under 30 minutes:

  1. Download & Import: Click the download button above to get the JSON file. In your n8n editor, use "Import from Clipboard" or "Import from File" to load the workflow.
  2. Configure Google Sheets: Update each fetch node with your specific Sheet IDs and column ranges. Set OAuth2 credentials for access.
  3. Set Up Gemini AI: Add your Gemini API key in the AI nodes. Adjust prompt templates if you want customized insight formats.
  4. Connect Gmail: Authorize the Gmail node with your account credentials. Define the recipient email addresses.
  5. Adjust Schedule: Modify the scheduler node to your preferred daily time (e.g., 8 AM for morning reports).
  6. Test Run: Execute the workflow once manually to verify data fetching, AI analysis, and email delivery. Check the output email for completeness.
  7. Activate Automation: Enable the workflow in n8n. It will now run automatically each day, delivering insights without further intervention.

Pro tip: Create a test sheet with sample data first. Run the workflow against it to confirm all nodes work before connecting to your live data.

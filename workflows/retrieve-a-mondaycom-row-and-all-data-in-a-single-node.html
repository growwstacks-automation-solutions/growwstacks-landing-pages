What This Workflow Does

This workflow solves the common bottleneck of manually fetching detailed data from Monday.com. When managing projects, you often need not just a task name, but all its column values, subitems, linked board items, and metadata. Doing this manually is time-consuming and error-prone.

This template acts as a reusable building block. When called from your main workflow (using an Execute Workflow node), it takes a Monday.com item ID (pulse) and returns a complete structured dataset. You get the item's name and ID, all column values indexed by both column name and ID, all board relation columns with their data, and all subitems with their column values. This enables downstream automations like generating reports, updating dashboards, triggering notifications, or syncing with other systems.

Pro tip: Use this block whenever you need Monday.com data within a larger automation. It ensures consistency and reduces the risk of API errors or data mapping mistakes.

How It Works

Step 1: Receive the Pulse ID

Your main workflow passes the Monday.com item ID (the "pulse") to this sub-workflow via an Execute Workflow node. This ID tells the block exactly which row to retrieve.

Step 2: Fetch the Item Details

The workflow uses Monday.com's API to get the core item information—name, creation date, and basic metadata.

Step 3: Retrieve Column Values

It then pulls all column values for that item, organizing them into two accessible formats: by column name (for readability) and by column ID (for reliability).

Step 4: Gather Board Relations and Subitems

The workflow fetches any linked items from other boards (relations) and all nested subitems, including their own column data. This gives you a complete hierarchical view.

Step 5: Return Structured Data

All retrieved data is packaged into a clean JSON output and sent back to your main workflow, ready for use in subsequent steps.

Quick Setup Guide

Follow these steps to integrate this block into your automation:

  1. Import the Template: Download the JSON file and import it into your n8n instance.
  2. Configure Monday.com Nodes: Update all Monday.com nodes with your API credentials and save the workflow.
  3. Copy the Workflow ID: Note the ID from this workflow's URL (or n8n interface).
  4. Prepare Your Main Workflow: Add an Edit Fields node to output a field named "pulse" containing the Monday.com item ID you want to fetch.
  5. Call the Block: Feed the Edit Fields node into an Execute Workflow node, paste the copied workflow ID, and connect the output to your downstream steps.

Pro tip: Use an expression in your Edit Fields node to dynamically determine the pulse ID based on triggers—like a new form submission or an email alert.

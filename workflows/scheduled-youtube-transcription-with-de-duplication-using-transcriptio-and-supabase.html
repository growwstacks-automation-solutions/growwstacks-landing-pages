What This Workflow Does

This automation solves a common problem for content teams, marketers, and researchers who need to systematically capture YouTube video transcripts without manual effort. Instead of manually downloading and transcribing videos, this workflow runs on a daily schedule, monitors your specified YouTube channels, and automatically processes new content.

The system intelligently prevents duplicate processing by checking your Supabase database before transcribing. Only brand-new videos get sent to the transcription API, saving costs and ensuring clean data. Each transcript is stored with metadata including video title, URL, publication date, and channel information, creating a searchable archive of your video content.

How It Works

1. Daily Schedule Trigger

The workflow activates automatically each day (configurable to any frequency). It begins by fetching your list of YouTube channel IDs to monitor.

2. Video Discovery & Filtering

For each channel, the system retrieves recent videos and applies your configured time filter (e.g., "last 7 days"). This ensures you only process relevant new content without revisiting old videos.

3. Duplicate Prevention Check

Each video URL is checked against your Supabase database. If the video already exists in your system, it's skipped entirely—preventing wasted API calls and maintaining data integrity.

4. Automated Transcription

New videos are sent to the YouTube Transcript API which returns accurate, timestamped transcripts. The system handles API authentication and error recovery automatically.

5. Database Storage

Complete transcript data—including raw text, video metadata, and processing timestamp—is saved to your Supabase table. This creates a permanent, queryable archive of all transcribed content.

Pro tip: Configure the time filter based on your channel's publishing frequency. Daily channels might use 1-2 days, while weekly channels could use 7-14 days to catch all new content.

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Import the Template: Download the JSON file and import it into your n8n instance using the workflow import feature.
  2. Configure Channel IDs: Replace the example channel IDs in the "Channels To Track" node with your actual YouTube channel identifiers.
  3. Set Time Filter: Adjust the "Max Days" value based on how frequently your channels publish new content.
  4. Add API Credentials: Create Header Auth credentials for the YouTube Transcript API with your x-api-key.
  5. Connect Supabase: Configure both Supabase nodes with your project credentials and table details.
  6. Test & Activate: Run the workflow once manually to verify everything works, then activate the schedule trigger.

Pro tip: Start with one channel for testing. Once confirmed, add additional channels. Monitor the first few runs to ensure duplicate prevention is working correctly before scaling.

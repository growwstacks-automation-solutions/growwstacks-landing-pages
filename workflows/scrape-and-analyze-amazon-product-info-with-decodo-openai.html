What This Workflow Does

Manual Amazon product research is time-consuming and inconsistent. E-commerce sellers, product managers, and competitive intelligence teams spend hours each week copying prices, features, and reviews into spreadsheets—only to struggle with analysis and trend spotting.

This automation solves that by creating a complete product intelligence pipeline. It scrapes Amazon product pages using Decodo's web scraping capabilities, extracts key information like pricing, features, and advertising data, then sends this data to OpenAI for intelligent analysis. The AI generates descriptive summaries, competitive positioning insights, and structured analytical output that would take hours to produce manually.

The final results are automatically organized and exported to Google Sheets, creating a living database of product intelligence that updates automatically. This transforms sporadic manual research into consistent, scalable competitive intelligence.

Pro tip: This workflow is particularly valuable for Amazon sellers monitoring 5+ competing products, product managers tracking feature evolution, or brands analyzing market positioning across different regions.

How It Works

Step 1: Product Page Scraping

The workflow starts by scraping the target Amazon product page using the Decodo community node. It extracts product specifications, key feature descriptions, current pricing, advertisement placements, and supplemental metadata directly from the live product page.

Step 2: Data Processing & Organization

Raw scraped data is cleaned and organized using n8n's data transformation nodes. The workflow separates product specs from advertising data, normalizes pricing information, and structures the data for AI analysis.

Step 3: AI-Powered Analysis

Structured product data is sent to OpenAI where three parallel analyses occur: a descriptive product summary is generated, competitive positioning insights are identified, and a structured product insight schema is created with standardized fields for comparison.

Step 4: Results Consolidation

All AI-generated outputs are merged with the original scraped data, creating a comprehensive product intelligence record that includes both raw data and analytical insights.

Step 5: Automated Export

The complete dataset is written to Google Sheets, organized by product, with timestamps for tracking changes over time. This creates a searchable, sortable database of product intelligence.

What You'll Need

  1. n8n instance (self-hosted or cloud) with access to community nodes
  2. Decodo API credentials for web scraping Amazon product pages
  3. OpenAI API key with access to GPT-4 or similar models for analysis
  4. Google Sheets account with OAuth setup for exporting results
  5. Amazon product URLs you want to analyze (ASIN-based links work best)

Note: This workflow requires n8n self-hosted installation since it uses the Decodo community node. Cloud n8n users may need to adapt the scraping approach using alternative methods.

Quick Setup Guide

  1. Install the Decodo community node in your n8n instance through the community nodes panel.
  2. Add credentials for Decodo API, OpenAI, and Google Sheets OAuth in n8n's credentials management.
  3. Import the template using the downloaded JSON file through n8n's workflow import feature.
  4. Configure the input by modifying the "Set the Input Fields" node with your target Amazon product URL.
  5. Test the workflow with a single product URL to verify scraping, AI analysis, and Google Sheets export work correctly.
  6. Schedule automation to run daily or weekly for continuous product intelligence tracking.

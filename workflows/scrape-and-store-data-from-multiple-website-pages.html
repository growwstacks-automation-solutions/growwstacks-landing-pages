What This Workflow Does

This automation solves the time-consuming problem of manually collecting data from websites with multiple pages and pagination. Businesses often need to gather information like product listings, directory entries, or market data spread across hundreds of pages, which can take days of manual work.

The workflow intelligently navigates through website pagination, extracts structured data from each page, implements local caching to avoid redundant requests, and stores everything in MongoDB for easy analysis. It handles complex scenarios like country-specific pages, multi-level pagination, and duplicate prevention.

By automating this process, companies can collect market intelligence, monitor competitors, generate leads, or build databases without manual data entry. The workflow includes smart error handling, rate limiting considerations, and can be scheduled to run periodically for ongoing data collection.

How It Works

1. Country List Initialization

The workflow starts by fetching a master list of countries from a directory website. It extracts individual country URLs that will be processed sequentially, creating the foundation for multi-country data collection.

2. Intelligent Pagination Handling

For each country page, the automation detects and follows pagination links, processing all available pages automatically. It uses workflow static data to remember progress between runs, ensuring no pages are missed even if the process is interrupted.

3. Local Caching System

Before making new web requests, the workflow checks if a local cached version of the page exists. This dramatically improves performance on subsequent runs and respects website servers by minimizing redundant requests.

4. Data Extraction & Transformation

HTML extraction nodes parse page content to isolate specific data points like names, codes, addresses, or other structured information. The data is cleaned, formatted, and prepared for database storage.

5. MongoDB Storage with Duplicate Prevention

Extracted records are saved to MongoDB with duplicate checking based on unique identifiers. This ensures data integrity and prevents the same information from being stored multiple times.

What You'll Need

  1. n8n instance (cloud or self-hosted) with internet access to target websites
  2. MongoDB database (local or cloud-based like MongoDB Atlas) for data storage
  3. Target website URLs with consistent HTML structure across pages
  4. Basic understanding of CSS selectors or XPath for data extraction
  5. Optional: Proxy service for large-scale scraping to avoid IP blocks

Pro tip: For production use with high-volume scraping, consider implementing proxy rotation using services like Scrapoxy or rotating residential proxies. This prevents IP blocking and ensures reliable data collection.

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure MongoDB credentials in the MongoDB node with your connection string
  3. Update the starting URL in the HTTP Request node to your target website
  4. Adjust data extraction selectors to match your target website's HTML structure
  5. Test with a single country/page before running the full workflow
  6. Schedule execution using n8n's scheduler or external cron jobs for regular updates

What This Workflow Does

Building a targeted database of healthcare providers is a time-consuming, manual process that can take medical recruiters, pharmaceutical sales teams, and healthcare marketers weeks of research. This workflow automates that entire process by scraping physician profiles from healthcare directories using BrowserAct, organizing the data in Google Sheets, and notifying your team via Slack when new profiles are added.

The automation specifically targets physicians by location and specialty, extracts key details like name, specialty, and address, and intelligently prevents duplicate entries in your database. What used to require hours of manual searching and data entry now happens automatically on a schedule you define, ensuring your outreach lists are always current and comprehensive.

How It Works

1. Define Target Location & Criteria

The workflow starts by setting your target location parameters—city and state—in a configuration node. This determines which geographic area to search for healthcare providers.

2. Execute BrowserAct Scraping Task

BrowserAct executes a pre-configured "Physician Profile Enricher" template that searches healthcare directories (like Healow) for doctors matching your criteria, extracting profile details through automated browser interaction.

3. Parse & Structure Extracted Data

A code node processes the raw output from BrowserAct, converting it into clean, structured JSON objects with standardized fields for name, specialty, address, and other relevant information.

4. Update Google Sheets Database

The structured data flows into Google Sheets, where it's appended as new rows. The workflow includes duplicate detection based on physician names to maintain database integrity.

5. Send Team Notification via Slack

Finally, a Slack message is sent to your designated channel confirming the batch job completion and summarizing how many new physician profiles were added to your database.

What You'll Need

  1. A BrowserAct account with the "Physician Profile Enricher" template saved
  2. A Google Sheets account with a spreadsheet created (headers: Name, Specialty, Address)
  3. A Slack workspace with appropriate channel access for notifications
  4. An n8n instance (cloud or self-hosted) to run the workflow
  5. API credentials for BrowserAct, Google Sheets, and Slack configured in n8n

Pro tip: Before running the workflow at scale, test with a small geographic area to ensure your BrowserAct template is correctly configured and extracting the desired data fields from your target directory.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance
  2. Configure credentials for BrowserAct, Google Sheets, and Slack in n8n's credentials management
  3. Update the "Define Location" node with your target city and state parameters
  4. Connect to your Google Sheet by selecting the correct spreadsheet and worksheet in the Google Sheets node
  5. Set your Slack channel in the notification node where completion alerts should be sent
  6. Test the workflow with manual trigger to verify data flows correctly through all steps
  7. Schedule execution (daily, weekly) based on how frequently you want to update your database

Automate Torrent Downloads with Google Assistant & n8n

What This Workflow Does

This n8n automation transforms how you manage your media center by eliminating manual torrent searches and downloads. Instead of browsing websites, checking seed counts, and manually adding torrents to your download client, you can simply say "Ok Google, download [movie name]" and let the automation handle everything.

The workflow connects Google Assistant (via IFTTT) to a torrent search API and your Transmission daemon. When you voice-command a download, it automatically searches multiple torrent indexes, selects the best match based on seeders and quality, authenticates with your Transmission server, and starts the download—all without touching a keyboard.

This is particularly valuable for home media servers running Plex, Jellyfin, or similar software where you want to build your library effortlessly. It turns hours of manual work into seconds of voice interaction while ensuring your downloads are organized and managed properly.

How It Works

Step 1: Voice Command Trigger

You use Google Assistant with a custom IFTTT applet that captures your voice command (like "download The Matrix") and sends it as a webhook to your n8n instance. This creates a hands-free initiation point that works from anywhere in your home.

Step 2: Torrent Search & Selection

The workflow uses the torrent-search-api Node.js package to query multiple torrent trackers simultaneously. It filters results based on seed count, file size, and naming patterns to identify the most reliable and appropriate download for your request.

Step 3: Transmission Integration

Once the optimal torrent is identified, the workflow authenticates with your Transmission daemon using Basic Auth, obtains a session token for security, and submits the magnet link or .torrent file. It handles Transmission's CSRF protection by making two sequential authenticated calls.

Step 4: Notification & Completion

Optionally, the workflow can send Telegram notifications about download status—whether successful or if no torrent was found. This creates a closed-loop system where you're informed about the outcome without checking manually.

Quick Setup Guide

Follow these steps to implement this automation in your environment:

  1. Install Dependencies: Navigate to your n8n directory and install the torrent-search-api package: npm i torrent-search-api
  2. Configure Docker: Update your docker-compose.yaml to mount the node_modules folder, ensuring n8n can access the torrent search library.
  3. Import Workflow: Download the JSON template from this page and import it into your n8n instance.
  4. Configure Credentials: Set up Transmission credentials in n8n's credential store and configure the Basic Auth for both Transmission nodes.
  5. Set Up IFTTT: Create an IFTTT applet that triggers on Google Assistant voice commands and sends a webhook to your n8n webhook URL.
  6. Test & Deploy: Test with simple searches, then activate the workflow for regular use.

Pro tip: For enhanced security, add a shared secret token between IFTTT and n8n to validate webhook requests. This prevents unauthorized triggering of downloads.

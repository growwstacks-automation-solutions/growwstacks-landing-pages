Automate Security Reconnaissance with Google Dorks & Parsera

Proactively find exposed data and vulnerabilities. This n8n workflow scrapes Google Dorks, filters results, and delivers detailed security reports via email.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Security teams spend countless hours manually searching for exposed data and vulnerabilities across their digital footprint. This automated reconnaissance workflow eliminates that manual effort by systematically scanning for security exposures using Google Dorks—specialized search queries that find sensitive information.

The workflow accepts a target domain, generates relevant security-focused Google Dorks, scrapes the search results using Parsera's AI-powered extraction, filters out irrelevant links, formats the findings into a clear markdown report, and emails it directly to your security team. It transforms what was traditionally a days-long manual process into a fully automated, scheduled security check that runs in minutes.

How It Works

1. Domain Input & Dork Generation

The workflow starts by accepting a target domain from a web form trigger. It then generates a comprehensive list of security-focused Google Dorks specifically tailored to that domain. These dorks are designed to find exposed configuration files, database dumps, login portals, and other sensitive information that shouldn't be publicly accessible.

2. Automated Search & Scraping

Each Google Dork is executed through Parsera's AI scraping agent, which navigates Google search results just like a human would. The agent extracts all relevant links from the search results pages, handling pagination and CAPTCHAs automatically. This step replaces manual browsing through dozens of search result pages.

3. Result Filtering & Validation

Raw scraping results often contain irrelevant links, Google internal pages, and non-HTTP resources. The workflow applies intelligent filtering to remove this noise, keeping only valid, security-relevant URLs that point to the target domain or its subdomains. This ensures the final report contains actionable intelligence, not just raw data.

4. Report Generation & Delivery

Valid findings are organized into a structured markdown report with clear categorization, severity indicators, and direct links. This report is then sent via Gmail to designated security personnel, complete with a timestamp and summary statistics. The email delivery creates an audit trail and ensures immediate visibility of any exposures.

It's particularly valuable for businesses handling sensitive data, companies in regulated industries, technology startups with expanding digital footprints, and security consultants serving multiple clients. What You'll Need

  1. Self-hosted n8n instance (required for community nodes)
  2. Parsera.io account with an API key and a configured "Google" agent
  3. Google Cloud Platform account with Gmail API enabled and OAuth2 credentials
  4. Target domains you want to monitor for security exposures
  5. Email address to receive security reports

Pro tip: Start with a limited set of non-critical domains to tune your Google Dorks and filtering rules before scaling to your entire digital footprint. This helps minimize false positives and ensures your team doesn't get overwhelmed with irrelevant findings.

Quick Setup Guide

  1. Import the template into your self-hosted n8n instance using the downloaded JSON file.
  2. Configure Parsera credentials by adding your API key to the 'Scrape with agent' node and ensuring your "Google" agent is properly set up.
  3. Set up Gmail OAuth2 by creating credentials in Google Cloud Console, enabling the Gmail API, and connecting them to the Gmail node.
  4. Customize the Google Dorks list in the workflow to match your specific security concerns and target technologies.
  5. Test with a single domain to verify everything works, then schedule the workflow for regular execution (weekly recommended).

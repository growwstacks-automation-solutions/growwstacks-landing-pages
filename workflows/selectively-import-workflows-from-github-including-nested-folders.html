Selectively Import Workflows from GitHub

Free n8n template to import specific workflows from GitHub repositories, even with deeply nested folder structures

Download Template JSON · n8n compatible · Free

What This Workflow Does

Managing n8n workflows across multiple environments—development, staging, production, or client instances—creates a significant operational challenge. When you store workflows in GitHub for version control, you need a way to selectively restore specific workflows without importing everything. This becomes especially complex when your repository uses deeply nested folder structures organized by date, project, or function.

This n8n template solves that problem by providing intelligent, selective workflow imports. Unlike basic import tools that load everything automatically, this template traverses all nested folders in your GitHub repository, collects every workflow JSON file found, and dynamically generates a selection form. You choose exactly which workflows to load, and the system cleans incompatible fields for n8n API compatibility before creating workflows directly in your instance.

The automation is designed for teams that maintain structured GitHub directories (like YYYY/MM/ folders) and need precise control over workflow restoration. Whether you're managing staging vs. production environments, deploying client-specific workflows, or recovering from incidents, this template gives you surgical precision over what gets imported where.

How It Works

Step 1: Repository Scanning

The workflow starts by connecting to your GitHub repository using authenticated credentials. It recursively scans through all folders and subfolders, identifying every JSON file that contains n8n workflow definitions. This deep scanning ensures no workflow is missed, regardless of how nested your organizational structure might be.

Step 2: Workflow Collection & Presentation

All discovered workflows are collected with their full paths and metadata. The system then generates a dynamic form interface where you can see each workflow's location, name, and last modified date. You can select individual workflows, groups by folder, or use pattern matching to choose subsets.

Step 3: Selection & Validation

Once you've made your selections, the template validates each workflow file for basic integrity and n8n compatibility. It checks for required fields, version compatibility, and potential issues that might cause import failures. This validation step prevents problematic workflows from reaching your n8n instance.

Step 4: Field Cleaning & Preparation

Before import, the template cleans each workflow JSON of fields that aren't compatible with the n8n API or that might cause conflicts in your target environment. This includes removing instance-specific IDs, cleaning up credential references, and standardizing format for smooth import.

Step 5: Targeted Import

The final step uses n8n's API to create each selected workflow in your target instance. The process includes error handling, progress tracking, and confirmation of successful imports. You receive a detailed report showing which workflows were imported successfully and any that encountered issues.

What You'll Need

  1. n8n instance with API access enabled and appropriate permissions to create workflows
  2. GitHub repository containing your n8n workflow JSON files (can be private or public)
  3. GitHub personal access token with repo scope permissions for authentication
  4. Basic understanding of your repository's folder structure and organization
  5. Clear strategy for which workflows belong in which environments

Pro tip: Before using this template in production, test it with a small subset of workflows in a development environment. This helps you understand the selection interface and ensures your GitHub token has correct permissions.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance using the import function.
  2. Configure GitHub credentials in the GitHub node with your personal access token and repository details.
  3. Set up n8n API credentials in the n8n nodes with your instance URL and API key.
  4. Test the scanning by running the workflow once to see if it correctly identifies your repository structure.
  5. Make your first selective import by choosing a few test workflows and importing them to a test environment.
  6. Review the import report to ensure everything worked correctly before scaling to production use.

What This Workflow Does

This workflow solves a common limitation in email automation: the inability to embed images directly within the body of Gmail messages using standard automation tools. While most platforms only support image attachments (which appear as separate files), this template enables true inline embedding where images display seamlessly within your email content.

By leveraging the Gmail API through the HTTP Request node, the workflow downloads images, converts them to base64 format, constructs proper multipart MIME messages, and sends professionally formatted emails with embedded visuals. This is essential for marketing newsletters, product announcements, tutorial emails, and any communication where visual presentation matters.

How It Works

1. Image Preparation

The workflow begins by fetching your image from a URL or local source. It converts the image to base64 encoding, which is the format required by the Gmail API for embedded content. This encoding process ensures the image data can be transmitted securely within the email structure.

2. MIME Message Construction

A multipart MIME message is built with proper boundaries and headers. The workflow creates an HTML email body that references the embedded images using Content-ID tags, allowing the images to display inline rather than as attachments.

3. Gmail API Integration

Using the HTTP Request node with OAuth2 authentication, the workflow sends the properly formatted message to Gmail's API endpoint. The node handles all the technical complexity of API communication while you focus on your email content and recipient list.

4. Email Delivery

The completed email is delivered through your Gmail account with images embedded directly in the body. Recipients see a polished, professional email with visuals that load automatically and integrate seamlessly with your message.

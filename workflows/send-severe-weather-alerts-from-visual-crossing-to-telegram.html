What This Workflow Does

This automation solves the problem of unreliable, manual weather monitoring. Businesses and individuals often miss critical severe weather warnings because they're not constantly checking forecasts. This workflow automatically fetches weather data from Visual Crossing's professional API, analyzes it for severe conditions, and sends instant alerts to Telegram—ensuring you're notified proactively, not reactively.

It transforms weather data into actionable safety notifications. Instead of generic forecasts, you receive specific alerts about thunderstorms, heavy rain, high winds, or other hazardous conditions relevant to your location. This enables timely decisions like securing outdoor assets, rescheduling deliveries, or evacuating teams.

How It Works

Step 1: Scheduled Trigger

The workflow activates automatically at regular intervals (typically hourly). This scheduled trigger ensures continuous monitoring without manual intervention, providing round-the-clock vigilance.

Step 2: Fetch Weather Data

An HTTP request node calls the Visual Crossing API with your location coordinates and API key. It retrieves detailed forecast data including temperature, precipitation, wind speed, and severe weather indicators.

Step 3: Condition Analysis

A function node processes the JSON response, checking for predefined severe weather thresholds. It filters out normal conditions and isolates only hazardous scenarios requiring alerts.

Step 4: Alert Distribution

If severe conditions are detected, the workflow formats a clear notification message and sends it via Telegram's bot API. The alert includes specific details like severity level, expected timing, and recommended actions.

What You'll Need

  1. A Visual Crossing API key (free tier available)
  2. A Telegram bot token and chat ID
  3. An n8n instance (cloud or self-hosted)
  4. Your location coordinates (latitude/longitude)
  5. Defined severe weather thresholds for your specific risks

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n workspace.
  2. Configure the Visual Crossing node with your API key and location.
  3. Set up the Telegram node with your bot token and target chat ID.
  4. Adjust the severe weather thresholds in the function node to match your risk profile.
  5. Activate the workflow and test with current weather data.

Pro tip: Create multiple alert tiers—critical alerts for immediate threats (like tornado warnings) and advisory alerts for less severe conditions (like heavy rain). This prevents alert fatigue while maintaining safety.

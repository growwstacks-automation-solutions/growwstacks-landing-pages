What This Workflow Does

This automation solves the tedious, manual process of weekly WooCommerce finance reporting. Instead of spending hours exporting data, combining spreadsheets, and calculating metrics, this workflow automatically gathers completed order and refund data directly from your WooCommerce store via HTTP APIs.

It processes this data to calculate essential financial KPIs—total sales, order count, refund amounts, and critical ratios—then formats a clear, executive-ready summary and posts it directly to a designated Slack channel. The result is consistent, accurate weekly insights delivered to your team without any manual intervention, freeing finance personnel to focus on analysis and strategy.

How It Works

Step 1: Weekly Schedule Trigger

The workflow activates automatically every week, ensuring regular reporting without manual initiation. You can configure the specific day and time to align with your review cycles.

Step 2: Fetch WooCommerce Order Data

Using WooCommerce's REST API with secure authentication, the workflow retrieves all completed orders for the relevant period. It filters out pending or cancelled orders to focus only on successful sales.

Step 3: Fetch WooCommerce Refund Data

Simultaneously, the workflow pulls refund records from the WooCommerce refunds API. This ensures refund analysis is included alongside sales data for a complete financial picture.

Step 4: Data Normalization & Combination

Sales and refund datasets are cleaned, standardized, and merged into a single structured input. Essential fields like order IDs, dates, totals, and refund amounts are extracted for calculation.

Step 5: KPI Calculation

The workflow calculates total sales amount, number of orders, total refund amount, refund count, refund-to-sales ratio, and refund-to-order ratio. It also flags potential risks if thresholds are exceeded.

Step 6: Slack Delivery

A formatted message containing all KPIs and insights is posted to your chosen Slack channel. The message is designed for quick review by finance leaders and operations teams.

What You'll Need

  1. A WooCommerce store with REST API access enabled.
  2. WooCommerce Consumer Key and Consumer Secret for API authentication.
  3. Access to a Slack workspace and a channel for receiving reports.
  4. An n8n instance or compatible automation platform to run the workflow.
  5. Permission to configure API credentials and Slack integrations.

Quick Setup Guide

Follow these steps to implement this automation in your environment:

  1. Download the template file and import it into your n8n workspace.
  2. Update the WooCommerce store domain in the configuration node.
  3. Add your WooCommerce Consumer Key and Consumer Secret to the HTTP Request nodes.
  4. Connect your Slack account and select the destination channel for reports.
  5. Activate the workflow schedule and test with a manual run.
  6. Monitor the Slack channel for your first automated weekly summary.

Pro tip: Start with a manual test run to verify data accuracy before enabling the weekly schedule. This ensures your API credentials are working and the Slack message format meets your team's needs.

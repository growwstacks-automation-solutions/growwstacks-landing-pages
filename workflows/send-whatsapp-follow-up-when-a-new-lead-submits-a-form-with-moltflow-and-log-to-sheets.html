What This Workflow Does

This automation solves the critical problem of slow lead response times. When potential customers submit a contact form, inquiry form, or application, they expect quick acknowledgment. Manual follow-ups often take hours or days, causing lost opportunities.

The workflow instantly captures form submissions from tools like Typeform, JotForm, Google Forms, or any webhook-enabled form. It extracts the lead's contact information, sends a personalized WhatsApp message within seconds via MoltFlow, and simultaneously logs all lead details to a Google Sheets spreadsheet for your sales team to track and follow up.

This creates a seamless bridge between lead capture and immediate engagement, ensuring no potential customer falls through the cracks while maintaining organized records for your sales pipeline.

How It Works

1. Form Submission Trigger

When a visitor submits your form, the form tool sends the data via webhook to this n8n workflow. The webhook node receives the raw submission data containing the lead's name, phone number, email, and any other form fields you've configured.

2. Data Parsing & Validation

A code node processes the incoming webhook payload, extracting and structuring the relevant information. It validates phone numbers for WhatsApp compatibility, formats names properly, and prepares the data for the next steps. Conditional logic can filter out incomplete submissions or spam entries.

3. WhatsApp Message Delivery

The workflow sends the lead's phone number and personalized message content to MoltFlow via API. MoltFlow delivers the WhatsApp message instantly using your connected WhatsApp number. The message can include the lead's name, reference their specific inquiry, and provide next steps or resources.

4. Google Sheets Logging

Simultaneously, all lead data is appended to a Google Sheets spreadsheet. This includes timestamp, source form, contact details, and the message sent. This creates a searchable database for your sales team and enables further automation or reporting.

5. Error Handling & Notifications

The workflow includes error handling for failed WhatsApp deliveries or Sheets updates. You can configure notifications to your team if any step fails, ensuring system reliability and immediate troubleshooting.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance.
  2. Create a MoltFlow API key in your MoltFlow account (Sessions page → API Keys tab).
  3. Add the API key as an HTTP Header Auth credential in n8n (Header Name: X-API-Key).
  4. Activate the workflow and copy the webhook URL generated by n8n.
  5. Configure your form tool to send POST submissions to this webhook URL.
  6. Map your form fields in the "Parse Form Data" code node (update field names for name, phone, email, interest).
  7. Set your MoltFlow session ID in the same code node.
  8. Connect Google Sheets (optional) by authenticating and selecting your spreadsheet.
  9. Test the workflow with a real form submission and verify WhatsApp delivery and Sheets logging.

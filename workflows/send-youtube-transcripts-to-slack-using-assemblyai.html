What This Workflow Does

This automation solves a common productivity bottleneck: teams wasting time manually extracting and sharing content from YouTube videos. Whether it's training sessions, competitor analysis, customer testimonials, or internal meetings recorded on video, valuable information gets trapped in audio format that's difficult to search, reference, or repurpose.

The workflow automatically converts YouTube video links into clean, formatted transcripts delivered directly to your Slack channels. When someone shares a YouTube link via Slack command, the system validates the video length, extracts the audio, sends it to AssemblyAI for high-accuracy transcription, then posts the complete transcript back to Slack—all without any manual intervention.

This creates an instant knowledge base from video content, making spoken information searchable, shareable, and actionable for your entire team. It eliminates the 15-45 minutes typically spent per video on manual transcription work while ensuring consistency and completeness.

How It Works

1. Slack Command Trigger

The workflow activates when a team member uses a custom Slack slash command (like /transcribe) with a YouTube URL. This creates an immediate, familiar interface that requires no training—users simply share videos as they normally would, but with a special command.

2. Video Validation & Processing

The system checks the video duration to ensure it's within processing limits (typically 10-60 minutes depending on your plan), then extracts the audio track. This step prevents timeouts on excessively long content and optimizes for transcription accuracy by isolating speech from visual elements.

3. AI Transcription via AssemblyAI

The audio file is securely sent to AssemblyAI's speech-to-text API, which uses advanced machine learning models to convert speech into text with high accuracy. The system polls for completion status, handling the asynchronous nature of transcription processing automatically.

4. Transcript Cleaning & Formatting

Raw transcription output is processed to remove filler words, add paragraph breaks at natural pauses, timestamp key sections, and format for readability. This transforms technical transcription output into team-friendly documentation ready for immediate use.

5. Delivery to Slack

The formatted transcript is posted to the originating Slack channel or a designated documentation channel. The message includes the video title, original link, speaker identification if available, and the complete searchable transcript—creating a permanent, accessible record.

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Import the template: Download the JSON file above and import it into your n8n instance via the "Import from File" option.
  2. Configure Slack: Create a new Slack app in your workspace, enable slash commands, and set the request URL to your n8n webhook endpoint.
  3. Connect AssemblyAI: Add your AssemblyAI API key to the HTTP Request node credentials in n8n.
  4. Test with a short video: Use the /transcribe command in Slack with a 2-3 minute YouTube video to verify the complete flow works.
  5. Adjust formatting: Modify the transcript formatting in the "Edit Fields (Set)" node to match your team's preferred documentation style.
  6. Set up monitoring: Configure error notifications to a dedicated Slack channel for any failed transcriptions.

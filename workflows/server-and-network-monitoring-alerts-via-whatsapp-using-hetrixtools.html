Server & Network Monitoring Alerts via WhatsApp

Get instant notifications on your phone when servers go down or hit critical resource limits. Automate HetrixTools alerts to WhatsApp for faster response times.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves the critical problem of delayed incident response in IT operations. When servers fail or hit resource limits, traditional email alerts often get missed in crowded inboxes, leading to extended downtime that costs businesses revenue and reputation.

The workflow connects HetrixTools monitoring with WhatsApp messaging through the GOWA API, creating a direct communication channel to your IT team's phones. It intelligently distinguishes between uptime events (service down/up) and resource usage alerts (CPU, memory, disk thresholds), formatting appropriate messages for each scenario.

By delivering alerts to the platform where teams already communicate daily, response times can be reduced from hours to minutes. This is particularly valuable for businesses running e-commerce platforms, SaaS applications, or any service where uptime directly impacts customer experience and revenue.

How It Works

1. Alert Detection & Classification

HetrixTools sends monitoring data to a webhook endpoint when thresholds are breached. The workflow first analyzes the incoming data to determine whether it's an uptime event or resource usage alert, routing it through different processing paths.

2. Message Formatting

For uptime events, the workflow extracts server name, status change (up/down), duration, and check time. For resource alerts, it includes specific metrics, current values, thresholds, and affected services. All data is formatted into clear, actionable WhatsApp messages.

3. WhatsApp Delivery

Using the GOWA WhatsApp REST API, the workflow sends the formatted alert to predefined phone numbers or groups. It can include emojis for visual severity indicators (⚠️ for warnings, 🔴 for critical) and structured information for quick assessment.

4. Response Tracking

The workflow can be extended to log all sent alerts, track acknowledgment from team members, and even escalate unacknowledged critical alerts to secondary contacts or alternative channels after defined timeouts.

What You'll Need

  1. HetrixTools Account: With active uptime monitors and/or resource monitoring configured for your servers.
  2. n8n Instance: Self-hosted or cloud version with webhook capability accessible via public URL.
  3. GOWA WhatsApp API: Running instance with API credentials (key, base URL, phone number configuration).
  4. WhatsApp Business Account: Or personal number configured in GOWA for sending messages.
  5. Team Phone Numbers: Recipient numbers added to your WhatsApp contacts.

Pro tip: Start with non-critical servers for testing. Configure different message templates for development vs production environments, and use WhatsApp groups to alert entire teams simultaneously during major incidents.

Quick Setup Guide

  1. Import the Template: Download and import the JSON file into your n8n instance.
  2. Configure Webhook: Note your n8n webhook URL and add it to HetrixTools as a contact method in alert settings.
  3. Set Up GOWA Credentials: Add your GOWA API credentials to n8n's credential manager.
  4. Customize Messages: Edit the text nodes to match your alerting preferences and include relevant server naming conventions.
  5. Test Thoroughly: Trigger test alerts from HetrixTools to verify WhatsApp delivery and message formatting.
  6. Activate & Monitor: Enable the workflow and monitor initial alerts to ensure proper routing and team response.

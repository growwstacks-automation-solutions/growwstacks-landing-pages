Singapore University Scholarship Eligibility Analyzer

Automate scholarship tracking by scraping university sites, assessing eligibility via AI, and publishing results to WordPress or Slack. Eliminates manual searches for students, counselors, and education platforms.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves the tedious, time-consuming process of manually tracking scholarship opportunities across multiple universities. For students, counselors, and educational platforms, staying updated on eligibility criteria, deadlines, and application requirements requires constant monitoring of disparate websites.

The workflow automatically scrapes scholarship pages from Singapore universities (NUS, NTU, SIT, SUTD), uses AI to assess eligibility against student profiles, aggregates qualified opportunities, and publishes results via WordPress for students or Slack for counselors. It transforms hours of weekly research into instant, personalized recommendations.

How It Works

Step 1: Parallel University Scraping

HTTP Request nodes simultaneously fetch scholarship data from four university websites. This parallel approach ensures complete coverage without sequential delays.

Step 2: Data Merging and Preparation

Scraped data is merged into a unified dataset, cleaning inconsistencies and structuring information for AI processing.

Step 3: AI Eligibility Assessment

OpenAI GPT-4o-Mini analyzes each scholarship's criteria against provided student profiles (GPA, citizenship, major, etc.), calculating match scores and identifying qualifying opportunities.

Step 4: Aggregation and Summary Generation

Qualified scholarships are aggregated, prioritized by match score, and summarized into actionable recommendations.

Step 5: Multi-Channel Publishing

Results are published to WordPress as a public resource page, sent via Slack notifications to counselors, and emailed to students with personalized details.

Step 6: Appeal Handling

A separate webhook handles rejection appeals, allowing AI re-evaluation with additional context or counselor override.

What You'll Need

  1. OpenAI API key for GPT-4o-Mini access
  2. WordPress site with REST API enabled
  3. Slack workspace and webhook URL for notifications
  4. Gmail/SMTP credentials for email delivery
  5. Student profile data (GPA, citizenship status, major, etc.)
  6. n8n instance (cloud or self-hosted)

Pro tip: Start with a single university source to validate scraping selectors before scaling to multiple institutions. This prevents configuration errors across parallel requests.

Quick Setup Guide

  1. Import the downloaded JSON template into your n8n workspace.
  2. Configure OpenAI credentials in the AI node with your API key.
  3. Update HTTP Request nodes with current university URLs and test selectors.
  4. Add your WordPress site URL and API credentials in the WordPress node.
  5. Create a Slack webhook and paste the URL into the Slack notification node.
  6. Configure Gmail/SMTP settings for email notifications.
  7. Test with a sample student profile to verify scraping, AI assessment, and publishing.

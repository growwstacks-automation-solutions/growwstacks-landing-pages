AI-Powered SEO Insights: Summarize Umami Data & Save to Baserow

Transform raw website analytics into actionable SEO recommendations with AI. Automatically analyze Umami data, generate insights via Openrouter, and store structured reports in Baserow—saving hours of manual analysis each week.

What This Workflow Does

Website analytics are rich with insights, but manually sifting through Umami data to find actionable SEO improvements is time-consuming and often inconsistent. This automation solves that by connecting your Umami analytics directly to AI analysis and structured storage.

The workflow automatically fetches your latest website performance data—pageviews, bounce rates, referral sources, and user behavior—then sends it to an AI model via Openrouter. The AI generates plain-English summaries highlighting top-performing content, traffic trends, and specific recommendations. Finally, these insights are saved to a Baserow database, creating a searchable history of SEO performance that your entire team can access.

How It Works

1. Data Collection from Umami

The workflow starts by calling the Umami API with your website property ID and date range. It retrieves metrics like pageviews, unique visitors, average session duration, and bounce rates for your specified period.

2. AI Analysis & Insight Generation

Raw metrics are formatted and sent to Openrouter (connecting to models like GPT-4 or Claude). A carefully crafted prompt instructs the AI to identify patterns, explain traffic changes, suggest content improvements, and highlight opportunities.

3. Structured Storage in Baserow

The AI-generated insights—including date, summary, top pages, and specific recommendations—are mapped to columns in your Baserow table. This creates a timestamped record that builds over time, enabling trend analysis and performance tracking.

Pro tip: Configure the workflow to run weekly. This gives you consistent insights without manual effort, and the historical data in Baserow will reveal long-term trends that one-time reports miss.

What You'll Need

  1. Umami account with API access and your website property ID
  2. Openrouter API key (free tier available) for AI analysis
  3. Baserow account with a table containing columns: Date, Summary, Top Pages, and Blog/Website name
  4. n8n instance (cloud or self-hosted) to run the workflow
  5. Basic understanding of your website's key performance indicators

Quick Setup Guide

  1. Download and import the template JSON into your n8n instance.
  2. Configure credentials for Umami, Openrouter, and Baserow in n8n's credentials management.
  3. Update node parameters with your specific Umami website property ID and Baserow table ID.
  4. Test the workflow manually with a small date range to verify data flow and formatting.
  5. Schedule execution (weekly recommended) using n8n's schedule trigger.
  6. Review initial insights in Baserow and refine AI prompts if needed for your specific context.

What This Workflow Does

This automation solves a critical problem for sales and marketing teams: disconnected customer data between your CRM (Pipedrive) and email marketing platform (KlickTipp). When contact information, consent status, or segmentation changes in one system, it often doesn't update in the other, leading to missed opportunities, compliance risks, and wasted marketing efforts.

The workflow creates a complete two-way synchronization that keeps both platforms perfectly aligned. It automatically syncs contact profiles (name, email, phone, birthday), marketing consent status (subscribed/no consent/unsubscribed), and segmentation (KlickTipp Tags ↔ Pipedrive Labels) in near real-time. This eliminates manual CSV imports, prevents data discrepancies, and ensures your sales and marketing teams are always working with the same accurate customer information.

Beyond basic contact syncing, this template includes GDPR-compliant deletion handling, timezone-aware birthday synchronization, and intelligent routing based on consent changes. Whether a contact subscribes via a form, unsubscribes from emails, or gets tagged in a campaign, both systems stay updated automatically.

How It Works

The automation runs on two parallel tracks, each triggered by changes in their respective platforms.

1. KlickTipp → Pipedrive Sync

Triggered by KlickTipp webhooks or activation tag events. When a contact is tagged or updated in KlickTipp, the workflow fetches their complete profile, checks if they exist in Pipedrive, and either creates a new person record or updates an existing one. It maps consent status appropriately and applies corresponding Pipedrive labels based on KlickTipp tags.

2. Pipedrive → KlickTipp Sync

Triggered by Pipedrive webhooks on person changes. The workflow intelligently routes each update: new contacts with consent get created via Single Opt-In, form submissions trigger Double Opt-In, existing contacts get updated, and unsubscribes are properly handled. It also syncs birthday information without timezone issues and applies KlickTipp tags based on Pipedrive labels.

3. GDPR Deletion Handling

A separate trigger watches for GDPR deletion requests in KlickTipp. When a contact requests data deletion via email links, the workflow immediately removes them from KlickTipp and, if a linked Pipedrive record exists, deletes that person from your CRM too, ensuring full compliance across both systems.

Pro tip: This workflow uses KlickTipp community nodes, which are only available for self-hosted n8n instances. If you're using n8n Cloud, you'll need to adapt it using HTTP Request nodes and the KlickTipp API.

What You'll Need

  1. A self-hosted n8n instance (community nodes required)
  2. Active KlickTipp account with API access
  3. Pipedrive account with API access (token or OAuth)
  4. Webhook configuration in both platforms
  5. Custom fields created in both systems for cross-referencing IDs
  6. Segmentation mapping planned (which KlickTipp tags correspond to which Pipedrive labels)

Quick Setup Guide

  1. Install the KlickTipp community node in your self-hosted n8n instance.
  2. Create API credentials in n8n for both KlickTipp and Pipedrive.
  3. Set up custom fields in both platforms to store cross-references (like "Pipedrive Person ID" in KlickTipp).
  4. Configure your segmentation mapping by noting the ID numbers of relevant tags and labels.
  5. Import this template into your n8n instance and update the field mappings to match your setup.
  6. Test with a single contact first, then activate the workflow for all contacts.

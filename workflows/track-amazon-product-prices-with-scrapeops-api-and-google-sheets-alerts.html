Track Amazon Product Prices Automatically

Monitor price changes, get instant alerts, and maintain historical data—all without manual checking. This free n8n workflow integrates ScrapeOps API with Google Sheets for reliable, maintenance-free price tracking.

What This Workflow Does

Manual Amazon price checking is tedious, inconsistent, and misses opportunities. This automation solves that by running 24/7, capturing every price change instantly, and alerting you immediately when significant fluctuations occur. It transforms reactive price watching into proactive market intelligence.

The workflow monitors multiple Amazon products using their ASINs, fetches current pricing via ScrapeOps' reliable API (avoiding anti-bot blocks), calculates both absolute and percentage changes, updates a Google Sheets spreadsheet with historical data, and sends customizable email alerts when prices cross your defined thresholds. Unlike DIY scrapers that break frequently, this API-based solution delivers consistent data without maintenance headaches.

How It Works

Step 1: Read Product List from Google Sheets

The workflow starts by reading your list of Amazon product ASINs from a Google Sheets spreadsheet. You maintain a simple sheet with product identifiers and optional alert thresholds.

Step 2: Fetch Current Prices via ScrapeOps API

For each ASIN, the workflow calls ScrapeOps' Amazon Product API to retrieve current pricing data. The API handles all the complexity—proxy rotation, anti-bot measures, parsing—ensuring reliable results.

Step 3: Calculate Price Changes & Update Spreadsheet

The workflow compares new prices against previous values stored in your spreadsheet, calculates both absolute ($) and percentage (%) changes, and updates the sheet with fresh data and historical records.

Step 4: Trigger Alerts Based on Thresholds

If price movements exceed your defined thresholds (e.g., 10% drop or $50 increase), the workflow sends customizable email alerts with details about the change, product information, and links.

Step 5: Schedule Regular Execution

Configure the workflow to run hourly, daily, or at any frequency you choose. Scheduled execution ensures continuous monitoring without manual intervention.

Pro tip: Start with conservative alert thresholds (e.g., 15% change) to avoid notification overload. Adjust based on the frequency of actionable opportunities you discover.

What You'll Need

  1. ScrapeOps API key – Register at scrapeops.io for access to their structured Amazon data API.
  2. Google account – For Google Sheets integration (free with any Gmail account).
  3. SMTP email configuration – For sending alerts (can use Gmail, Outlook, or any email service).
  4. n8n instance – Self-hosted n8n or n8n.cloud account to run the workflow.
  5. Amazon product ASINs – The unique identifiers for products you want to monitor.

Quick Setup Guide

  1. Copy the template spreadsheet: Make a copy of the provided Google Sheets template and add your product ASINs.
  2. Configure the workflow: Import the JSON template into n8n, add your ScrapeOps API key to the Setup node, and update the spreadsheet URL.
  3. Set email alert settings: Configure your SMTP credentials in the Email node for notifications.
  4. Adjust schedule frequency: Set the Schedule Trigger node to run hourly, daily, or as needed.
  5. Test and deploy: Run the workflow once to verify data collection and alerting, then activate the schedule.

Pro tip: Use the historical data in your spreadsheet to analyze price trends over time. Identify patterns like seasonal drops, competitor pricing cycles, and optimal buying windows.

Track and Sync Workflow Status in Notion from n8n

Automatically maintain a centralized inventory of all your n8n workflows in Notion. Get instant visibility into active vs. deactivated automations, creation dates, and metadata—all in one searchable dashboard.

What This Workflow Does

As automation scales across your organization, losing track of what workflows exist, their status, and purpose becomes a real problem. Teams duplicate efforts, waste time searching for existing automations, and lack visibility into operational health.

This n8n workflow solves that by automatically syncing your entire workflow inventory from n8n to a Notion database. It creates a living documentation system that updates itself, giving you a centralized dashboard showing all automations, their current status (active/deactivated), creation dates, last updates, and key metadata.

The automation runs on a schedule, fetching workflow data from the n8n API, normalizing it into readable formats, and creating or updating corresponding records in your Notion database. It's a read-only sync—your workflows remain untouched in n8n while gaining comprehensive documentation in Notion.

How It Works

The workflow follows a straightforward but powerful sequence to maintain perfect synchronization between your n8n instance and Notion documentation.

1. Scheduled Trigger & API Fetch

A schedule node triggers the workflow daily (configurable). It authenticates with your n8n instance via API credentials and fetches all workflows, including their IDs, names, descriptions, active status, and timestamps.

2. Data Normalization & Processing

The raw API data is transformed into a clean, consistent format. Boolean active status becomes "Active" or "Deactivated," timestamps convert to readable dates, and workflow descriptions are extracted for documentation purposes.

3. Notion Database Query & Matching

The workflow queries your existing Notion database to find records matching n8n workflow IDs. This matching ensures existing entries are updated rather than duplicated, maintaining data integrity over time.

4. Create or Update Records

For new workflows, the automation creates complete Notion database entries with all metadata. For existing workflows, it updates any changed information—particularly useful when you activate/deactivate workflows or update their names/descriptions.

5. Error Handling & Logging

Built-in error handling manages API rate limits, connection issues, or schema mismatches. Failed operations are logged for troubleshooting while successful syncs proceed uninterrupted.

What You'll Need

  1. n8n Instance: Either n8n Cloud or self-hosted with API access enabled.
  2. n8n API Credentials: User API key with read access to workflows (created in User Settings → API).
  3. Notion Account: Any Notion plan with database creation permissions.
  4. Notion Integration: Internal integration created at notion.so/my-integrations with "Read content" and "Update content" capabilities.
  5. Notion Database: A database (new or existing) with properties for Workflow ID, Name, Status, Created Date, and Last Updated.

Pro tip: Create your Notion database first with the exact property names used in the template. This prevents mapping errors during initial setup.

Quick Setup Guide

Follow these steps to implement this workflow in under 15 minutes:

  1. Download & Import: Download the template JSON file and import it into your n8n instance via the "Import from File" option.
  2. Configure n8n Credentials: Add your n8n API credentials to the HTTP Request node. Use your instance URL and personal API key.
  3. Set Up Notion Connection: Create a Notion integration, share your database with it, and add the integration token to the Notion nodes.
  4. Map Database Properties: Update the Notion node properties to match your database schema (property names and types).
  5. Test & Schedule: Run the workflow once manually to verify synchronization, then set your preferred schedule (daily recommended).
  6. Customize (Optional): Add additional fields from the n8n API response or create linked databases in Notion for enhanced reporting.

What This Workflow Does

This automation solves the manual effort of tracking trending GitHub repositories across different time periods (daily, weekly, monthly). It reliably collects repository data including rank, stars, language, and other metrics, then stores it in Google Sheets with automatic deduplication.

Teams use this to identify emerging open source projects, track competitor activity, and maintain a historical record of technology trends. The structured data enables analysis that would be impossible with manual tracking methods.

How It Works

1. Scheduled Trigger

The workflow runs on your preferred schedule (daily recommended) to capture trending repository snapshots.

2. ScrapeOps API Call

ScrapeOps fetches the GitHub trending page HTML reliably, handling proxies and retries automatically.

3. Data Parsing

The workflow extracts structured data from the HTML including repository name, stars, language, and ranking position.

4. Google Sheets Update

Cleaned data is written to Google Sheets using a dedupe_key to prevent duplicates while updating changed information.

What This Workflow Does

Manually exporting data from PostgreSQL to Excel is a repetitive, time-consuming task that plagues many businesses. Analysts spend hours running queries, copying results, and formatting spreadsheets—only to repeat the process tomorrow. This workflow automates that entire pipeline.

It connects directly to your Postgres database, executes your SQL queries, transforms the results into a properly formatted Excel file, and delivers it wherever needed—whether that's to a shared drive, email inbox, or cloud storage. The automation ensures consistency, eliminates human error, and frees your team to focus on analysis rather than data wrangling.

Whether you need daily sales reports, weekly inventory summaries, or monthly customer analytics, this template provides the foundation. You can schedule it to run automatically or trigger it on-demand, turning database queries into business-ready spreadsheets with zero manual intervention.

How It Works

Step 1: Connect to PostgreSQL

The workflow begins by establishing a secure connection to your PostgreSQL database. You configure the credentials once, and the automation handles all subsequent connections. It can execute parameterized queries, allowing you to filter data by date ranges, specific categories, or other business logic.

Step 2: Execute Query & Retrieve Data

Your SQL query runs against the database, pulling exactly the records you need. The workflow can handle complex joins, aggregations, and calculations directly in SQL, returning clean, structured data ready for export. Error handling ensures failed connections or malformed queries don't break the process.

Step 3: Transform Data for Excel

Raw database results are processed into Excel-friendly format. This includes setting proper column headers, formatting dates and numbers, applying currency symbols, and even adding basic styling. You can also filter out sensitive columns or add calculated fields before the Excel file is created.

Step 4: Generate & Distribute Excel File

The transformed data is written to an .xlsx file with proper worksheets and formatting. The completed file can then be saved to a local directory, uploaded to Google Drive or Dropbox, attached to an email, or posted to a Slack channel—all automatically.

What You'll Need

  1. PostgreSQL Database Access: Connection credentials (host, database name, username, password) and a SQL query you want to automate.
  2. n8n Instance: Either self-hosted n8n or n8n.cloud account.
  3. Destination for Excel Files: A folder path, cloud storage destination, or email/Slack channel where reports should be delivered.
  4. Basic Understanding: Familiarity with your database schema and the specific data you need to export.

Quick Setup Guide

Import and configure this workflow in under 10 minutes:

  1. Download the Template: Click the download button above to get the JSON file.
  2. Import to n8n: In your n8n dashboard, create a new workflow and import the JSON file.
  3. Configure PostgreSQL Credentials: Update the Postgres node with your database connection details.
  4. Set Your SQL Query: Replace the sample query with your own business logic.
  5. Choose Output Destination: Configure where the Excel file should be saved or sent.
  6. Test & Schedule: Run the workflow once to verify, then set up a schedule or trigger.

Pro tip: Start with a simple query that returns a small dataset for testing. Once confirmed, expand to your full production query. Use n8n's built-in scheduler to run exports daily, weekly, or monthly automatically.

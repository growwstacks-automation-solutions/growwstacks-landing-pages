What This Workflow Does

This automation solves the common problem of team members missing important task updates in Trello. When someone moves a card from "To Do" to "In Progress" or marks something as "Done," the entire team remains unaware until they manually check the board. This creates communication gaps, slows down projects, and leads to unnecessary status update meetings.

The workflow automatically monitors your Trello board for card movements and sends instant, formatted notifications to your Slack channel. It captures essential details like the task name, who moved it, the previous and new status, and includes a direct link to the Trello card. This creates real-time visibility without requiring anyone to leave their primary communication tool.

How It Works

1. Trello Trigger Monitoring

The workflow continuously watches your specified Trello board for any card movements between lists. It uses Trello's webhook or polling system to detect changes in real-time, capturing the card details, assignee information, and movement context.

2. Data Processing & Formatting

Once a movement is detected, the automation extracts relevant information and structures it into a clear message format. It identifies the card name, previous list, new list, due dates, labels, and any comments added during the move.

3. Slack Notification Delivery

The formatted message is sent to your designated Slack channel or direct message. The notification includes actionable information with proper formatting (bold headers, clear status indicators) and a direct Trello link for quick access.

What You'll Need

  1. Trello account with a board you want to monitor
  2. Trello API key and token (free from Trello Developer portal)
  3. Slack workspace with appropriate permissions
  4. Slack app with Bot Token (created via Slack API dashboard)
  5. n8n instance (self-hosted or cloud version)
  6. Basic understanding of webhook concepts (helpful but not required)

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Download the template using the button above and import it into your n8n instance
  2. Create Trello credentials in n8n using your API key and token from the Trello Developer portal
  3. Create Slack credentials in n8n using your Slack Bot Token from your Slack app configuration
  4. Configure the Trello trigger node by selecting your board ID (found in your Trello board URL)
  5. Update the Slack node with your target channel name or user ID
  6. Test the workflow by moving a card in Trello and verifying the Slack notification arrives correctly
  7. Activate the workflow and enjoy automated team updates

Pro tip: Start by monitoring only critical boards or lists to avoid notification overload. You can expand coverage once your team adapts to the automated updates.

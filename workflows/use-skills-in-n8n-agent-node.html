What This Workflow Does

This template transforms how you build AI agents in n8n by implementing the "skills" pattern popularized by Anthropic. Instead of creating monolithic, hard-to-maintain system prompts, you can now build agents that dynamically load and execute specific skill files from GitHub repositories.

The workflow creates an intelligent agent that responds to chat messages, scans configured GitHub repos for skill files (structured markdown instructions), and uses those skills to guide task execution. When a user asks the agent to perform a task, it first searches for relevant skills, reads the detailed instructions, and then follows them step-by-step—just like a human consulting a procedure manual.

This approach solves the common problem of "prompt overload" where agents become confused by overly complex instructions. By breaking capabilities into modular skills, you create more reliable, maintainable, and explainable AI automations that can handle sophisticated business processes without constant manual intervention.

How It Works

1. Chat Trigger & Skill Discovery

The workflow starts when a message arrives via n8n Chat or Chat Hub. It immediately scans all configured GitHub repositories (both root directories and /skills folders) to build a comprehensive index of available skills. This happens dynamically, so new skills added to your repos become immediately available.

2. Skill Index Injection

The directory map of available skills gets injected into the agent's system prompt. This gives the AI context about what capabilities are available and how to access them. The agent learns what skills exist, their purposes, and where to find them—all without hardcoding this information into the prompt.

3. Dynamic Skill Selection & Execution

When the agent determines it needs a specific skill to complete a task, it uses built-in tools to fetch the relevant skill file from GitHub. The skill contains step-by-step instructions in markdown format that the agent follows precisely. This could include data processing rules, API call sequences, validation steps, or formatting requirements.

4. Result Delivery & Context Management

After executing the skill, the agent returns the results through the chat interface. The workflow maintains conversation context across interactions, allowing for follow-up questions and multi-step processes. All skill executions are logged for auditing and improvement.

What You'll Need

  1. n8n instance (cloud or self-hosted) with AI nodes enabled
  2. GitHub account with repository access for skill storage
  3. OpenRouter API key (or alternative LLM provider credentials)
  4. Basic understanding of markdown for writing skill files
  5. GitHub Personal Access Token with repo read permissions

Pro tip: Start with public GitHub repositories containing existing skill libraries (like Anthropic's knowledge-work-plugins) to see examples before creating your own. This gives you immediate value while you learn the pattern.

Quick Setup Guide

  1. Import the template into your n8n workspace using the download button above
  2. Configure GitHub credentials in the "List Files" and "Get File" nodes with your Personal Access Token
  3. Add your OpenRouter API key to the AI Chat Model node (or switch to your preferred provider)
  4. Update the SKILLS_REPOS variable to include your GitHub repositories (format: "owner/repo-name")
  5. Test with a simple query like "What skills are available?" to verify the connection works
  6. Create your first skill file in markdown format and commit it to your configured repository

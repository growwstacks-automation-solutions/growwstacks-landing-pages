What This Workflow Does

Invalid email addresses waste marketing budgets, hurt sender reputations, and cause missed opportunities. Manually verifying lists is tedious and error-prone. This automation solves that by instantly checking if an email address is valid, deliverable, and safe to contact using the Icypeas verification service.

The workflow connects your data sources (like forms, CRMs, or spreadsheets) to Icypeas, performs the validation, and routes emails based on the result. Valid addresses proceed to your mailing list or CRM; invalid ones are flagged for review or removal. It ensures every email in your system is real before you send a single message.

Beyond simple validation, this template can be extended to trigger follow-up actions—like notifying sales reps about high-quality leads or automatically cleaning up your database on a schedule. It’s a foundational automation for any business that relies on email communication.

How It Works

The workflow follows a clear, three-step process to validate each email address efficiently.

Step 1: Trigger the Verification

The automation can be triggered manually for one-off checks, or automatically when a new email enters your system—like after a website sign-up, lead import, or CRM update. The trigger passes the email address to the next step.

Step 2: Connect to Icypeas

The workflow authenticates with your Icypeas account using your API credentials. It securely sends the email address to Icypeas’s verification API for analysis.

Step 3: Analyze and Route the Result

Icypeas returns a detailed status: valid, invalid, risky, or unknown. The workflow interprets this result and decides what to do next. For example, valid emails might be added to a “Clean List” in your CRM, while invalid ones are logged in a separate sheet for review.

What You'll Need

  1. An active Icypeas account with API credentials (Key, Secret, and User ID).
  2. A running n8n instance (cloud or self-hosted).
  3. The email address(es) you want to verify.
  4. Optional: Access to your CRM, database, or spreadsheet where emails are stored.

Pro tip: Start by testing the workflow with a small batch of known good and bad emails to ensure it’s configured correctly before processing large lists.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n workspace.
  2. Configure the Icypeas node by entering your API Key, Secret, and User ID from your Icypeas account.
  3. Set your email input—modify the manual trigger node to accept the email you want to verify, or connect it to an automated trigger from your app.
  4. Adjust the output routing based on your needs. Decide where valid and invalid emails should go (e.g., different CRM lists, alerts, logs).
  5. Test and activate the workflow with a few sample addresses, then set it live.

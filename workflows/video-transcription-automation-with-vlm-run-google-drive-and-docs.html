Video Transcription Automation with VLM Run, Google Drive & Docs

Automatically transcribe videos uploaded to Google Drive using AI. Generate transcripts with timestamps, save to Google Docs, and make video content accessible.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manual video transcription is time-consuming, error-prone, and expensive. This automation solves that by creating a seamless pipeline from video upload to formatted transcript. Whenever a video file is added to a specific Google Drive folder, the workflow automatically processes it using VLM Run's AI transcription service, extracts both audio content and visual descriptions, and saves a structured report to Google Docs.

The result is instant accessibility: meeting recordings become searchable minutes, training videos get accurate subtitles, and customer interviews are documented without human effort. This transforms passive video libraries into active knowledge bases that teams can actually use.

How It Works

Step 1: Monitor Google Drive for New Videos

The workflow watches a designated Google Drive folder for new video file uploads. It triggers automatically when MP4, MOV, AVI, or other common video formats appear.

Step 2: Download and Send to VLM Run AI

The video file is downloaded temporarily and sent to the VLM Run API for processing. The AI model analyzes both audio tracks and visual content, generating a detailed transcript with timestamps.

Step 3: Format Transcript with Timestamps & Descriptions

The raw transcript is cleaned and formatted. Timestamps are added for easy reference, speaker segments are identified where possible, and key visual descriptions are included.

Step 4: Save to Google Docs for Team Access

The final transcript is saved as a new Google Docs document in a shared folder. Team members can immediately access, comment, and search the content without needing the original video.

What You'll Need

  1. A Google Drive account with a designated folder for video uploads
  2. VLM Run API credentials (available from their dashboard)
  3. Google Docs access for transcript storage
  4. A self-hosted n8n instance (community nodes require self-hosting)
  5. The VLM Run community node installed in your n8n environment

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n workspace.
  2. Configure the Google Drive node with OAuth2 credentials and specify your video upload folder ID.
  3. Add your VLM Run API key to the corresponding node.
  4. Set up the Google Docs node to create documents in your preferred folder.
  5. Test the workflow by uploading a sample video to your Google Drive folder.
  6. Activate the workflow and let it run automatically.

Pro tip: Create separate Google Drive folders for different types of content (meetings, training, interviews) and duplicate this workflow for each. You can customize the transcript formatting per content type.

What This Workflow Does

Managing a fantasy football team involves constant checking—player statuses, roster moves, waiver wires. This manual process eats into your time and can cause you to miss crucial updates. This automation solves that by creating a conversational interface to your team data.

The workflow builds a Telegram chatbot that connects directly to the Sleeper fantasy football API. When you send your Sleeper username to the bot, it automatically retrieves your user ID, finds your leagues, pulls your full roster, matches player IDs to detailed information, and returns a clean, formatted list of your players with their positions and NFL teams—all within seconds.

This transforms hours of weekly manual roster checking into a simple text command, giving you more time for strategy and less time on administrative tasks. It's perfect for commissioners managing multiple leagues or serious players who want instant access to their team composition.

How It Works

Step 1: Telegram Trigger

The workflow starts when a user sends a message to your Telegram bot containing their Sleeper username. The Telegram trigger node captures this input and passes it to the next stage of the automation.

Step 2: Sleeper User ID Lookup

Using the HTTP Request node, the workflow calls the Sleeper API's user endpoint with the provided username. It retrieves the unique Sleeper user ID, which is required for all subsequent API calls to fetch league and roster data.

Step 3: League Retrieval

The workflow then fetches all leagues associated with that user ID. By default, it selects the first league found, but this logic can be modified to handle multiple leagues or prompt the user for selection.

Step 4: Roster Data Extraction

With the league ID obtained, the automation pulls the complete roster data for that league. This includes all teams and their player IDs, which are then filtered to find the specific roster owned by the requesting user.

Step 5: Player Information Enrichment

The player IDs from the roster are matched against a database (like Airtable or Google Sheets) that contains full player details—name, position, NFL team, and other relevant information. This step transforms anonymous IDs into actionable player data.

Step 6: Formatted Response Delivery

Finally, the Telegram node sends back a clean, readable message to the user listing all players on their roster with names, positions, and teams. The entire process happens in under 10 seconds, providing instant team insight.

What You'll Need

  1. A Telegram Bot created via BotFather with its API token.
  2. A Sleeper Fantasy Football account with at least one active league.
  3. A player database (Airtable, Google Sheets, or similar) containing player ID mappings to names, positions, and teams. The companion "Sleeper NFL Players Daily Sync" template can help build this.
  4. An n8n instance (cloud or self-hosted) to import and run the workflow.
  5. Basic understanding of how to set credentials in n8n for Telegram and your chosen database.

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure credentials for Telegram (your bot token) and your database (Airtable/Sheets API keys).
  3. Update the database reference to point to your player information table with the correct field names.
  4. Test the workflow by sending your exact Sleeper username to your Telegram bot.
  5. Customize as needed—adjust the league selection logic, add more player stats, or change the response format.
  6. Activate the workflow and start using your fantasy football chatbot immediately.

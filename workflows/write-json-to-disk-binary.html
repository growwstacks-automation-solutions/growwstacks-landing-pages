Write JSON to Disk (Binary)

Automatically convert JSON data to binary format and save it to your local file system or network storage.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Many systems and applications require data in binary format rather than human-readable JSON. This creates a common challenge for businesses that need to exchange data between modern APIs (which typically use JSON) and legacy systems, embedded devices, or specific storage solutions that expect binary input.

This n8n workflow solves this interoperability problem by automatically converting JSON data into binary files and saving them to disk. It acts as a bridge between JSON-based systems and binary-dependent applications, enabling seamless data flow without manual conversion. The workflow is particularly valuable for data backup scenarios, system migrations, and preparing datasets for specialized processing tools.

By automating this conversion, you eliminate the risk of human error in manual file conversion, ensure consistent formatting, and free up technical staff for higher-value tasks. The workflow can be triggered on a schedule, by webhook, or when new data arrives, making it a flexible component in your data pipeline.

How It Works

The workflow uses n8n's powerful node-based architecture to transform JSON data into binary format through a custom JavaScript function, then writes the result to your specified file location.

Step 1: JSON Data Input

The workflow begins by receiving JSON data from your chosen source. This could be from an HTTP request, a database query, another application via webhook, or even a scheduled data export. The JSON structure is preserved throughout the initial processing stages.

Step 2: Binary Conversion Function

A custom Function node contains JavaScript code that converts the JSON object to a string, then encodes it as binary data. This conversion ensures the data is in the exact format required by systems expecting binary input, maintaining all structural information while changing the encoding.

Step 3: File Write Operation

The Write Binary File node takes the converted binary data and saves it to your specified directory with your chosen filename. You can configure the path, filename pattern, and whether to overwrite existing files or append to them.

Step 4: Success Notification & Logging

Once the file is successfully written, the workflow can trigger notifications (email, Slack, etc.) and log the operation details for audit purposes. This ensures you always know when conversions happen and can verify successful execution.

  1. A running n8n instance (self-hosted or cloud)
  2. File system access with write permissions for the target directory

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment:

  1. Download the template using the button above and import it into your n8n instance
  2. Configure your JSON source - Replace the example data node with your actual data source (HTTP Request, Database, Webhook, etc.)
  3. Adjust the Function node if needed - The default conversion handles standard JSON; modify the JavaScript if your target system requires specific binary formatting
  4. Set your file path - Update the Write Binary File node with your desired directory and filename pattern
  5. Test the workflow with sample data to ensure files are created correctly
  6. Add error handling (optional) - Extend the workflow with error notification nodes if conversion fails
  7. Activate the workflow and set your preferred trigger (manual, schedule, or event-based)

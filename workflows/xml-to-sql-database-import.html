What This Workflow Does

Manually importing XML data into SQL databases is a tedious, error-prone task that wastes valuable developer time. This automation workflow eliminates the manual copy-paste process by automatically reading XML files, parsing their structure, and inserting clean records directly into your MySQL, PostgreSQL, or other SQL database.

Whether you're syncing product catalogs, customer data, financial reports, or API responses, this template handles the entire pipeline. It reads binary XML files, extracts content, converts XML nodes into JSON objects, and maps them to database columns—all with configurable validation and error handling.

Businesses use this to automate daily inventory updates, nightly sales data imports, customer record synchronization, or regulatory reporting. Instead of spending hours on data entry, your team gets accurate, query-ready data automatically.

How It Works

The workflow follows a logical four‑step process to transform raw XML into structured database entries.

1. Load XML File

The Read Binary Files node retrieves the XML file from your server, cloud storage (like AWS S3, Google Cloud Storage), or FTP location. It handles file encoding and prepares the binary data for parsing.

2. Extract & Decode Content

A Code node takes the binary buffer and converts it into a readable XML string. This step ensures proper character encoding and prepares the data for XML parsing.

3. Parse XML Structure

The XML node transforms the XML string into a structured JSON object. It can handle nested elements, attributes, and arrays, mapping XML nodes to fields that match your database schema.

4. Insert into SQL Database

The final MySQL (or other SQL) node inserts each parsed record as a row in your specified table. You can configure field mapping, add conditional logic, and handle duplicates or errors.

Pro tip: The template includes a disabled MySQL node that can create the target table automatically based on your XML structure, perfect for prototyping or one‑off imports.

Quick Setup Guide

Get this automation running in under 10 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure the Read Binary Files node to point to your XML file location (server path, S3 bucket, FTP).
  3. Adjust the XML node settings to match your XML structure (tag names, nesting).
  4. Set up the MySQL node with your database connection and table name.
  5. Map fields from the parsed XML to your database columns.
  6. Test with a sample file, then activate the workflow for automated imports.

Pro tip: Use the sample MySQL node (currently disabled) to auto‑create a table schema based on the classicmodels sample database. Enable it once to generate the table, then disable it and switch to your own database connection.
